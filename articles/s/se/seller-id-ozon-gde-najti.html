Где найти Seller ID на Ozon: подробное руководство для продавцов

Каждый продавец на маркетплейсе Ozon рано или поздно сталкивается с необходимостью узнать свой уникальный идентификатор — Seller ID. Этот числовой код присваивается автоматически при регистрации и используется для интеграции с API, настройки рекламных кампаний, подключения аналитических сервисов и даже для решения спорных ситуаций с поддержкой. Однако найти его в лабиринте настроек личного кабинета не так просто, как кажется.

В этой статье мы не только покажем все возможные способы получения Seller ID (включая скрытые), но и объясним, почему этот идентификатор важен для бизнеса. Вы узнаете, как он отличается от Client-ID и Shop-ID, где его применять в Ozon Seller и Ozon Global, а также что делать, если идентификатор suddenly исчез из интерфейса после обновления платформы. Особое внимание уделим типичным ошибкам, которые приводят к блокировке аккаунта при неправильном использовании ID.

Что такое Seller ID и зачем он нужен продавцу

Seller ID — это уникальный числовой код, который Ozon присваивает каждому зарегистрированному продавцу. Он служит основным идентификатором вашего аккаунта во всех внутренних системах маркетплейса, включая:

  • 🔄 API-интеграции с 1С, MoySklad, Bitrix24 и другими CRM;
  • 📊 Аналитические сервисы (Ozon Statistics, Retail Rocket, Google Analytics);
  • 💰 Рекламные кампании в Ozon Advertising и внешних площадках;
  • 📦 Логистические схемы (FBS/FBO) и работу с Ozon РЭД;
  • 🔧 Техническую поддержку при разблокировке аккаунта или спорных ситуациях.

Без этого идентификатора невозможно настроить автоматическую выгрузку товаров, отслеживать продажи через сторонние инструменты или подтвердить права на аккаунт при обращении в службу поддержки. Например, при подключении Ozon API первым делом потребуется указать sellerId в запросах авторизации. А в Ozon Seller этот ID используется для разделения прав доступа между несколькими магазинами одного продавца.

⚠️ Внимание: Не путайте Seller ID с Client-ID (используется для авторизации в API) или Shop-ID (идентификатор витрины). Эти коды выполняют разные функции, и подмена одного другим приведёт к ошибкам интеграции.

Где посмотреть Seller ID в личном кабинете Ozon Seller

Самый надёжный способ найти идентификатор — через веб-версию Ozon Seller. Алгоритм зависит от версии интерфейса (обновлённый или классический), но мы опишем оба варианта.

Способ 1: Через раздел «Профиль компании» (новый интерфейс)

Если ваш кабинет обновлён (обычно это продавцы, зарегистрированные после 2023 года), следуйте инструкции:

  1. Авторизуйтесь на сайте seller.ozon.ru;
  2. В правом верхнем углу кликните на иконку профиля (аватар или первую букву названия компании);
  3. В выпадающем меню выберите Профиль компании;
  4. Прокрутите страницу до блока Основная информация — там будет строка ID продавца.

Внимание: в некоторых аккаунтах ID может скрываться под кнопкой «Показать» рядом с названием компании — нажмите на неё, если поля нет в открытом виде.

Способ 2: Через URL адреса (старый интерфейс)

Для продавцов с классической версией кабинета (до 2023 года) алгоритм другой:

  1. Перейдите в раздел Товары → Список товаров;
  2. Скопируйте URL страницы — он будет выглядеть так:
    https://seller.ozon.ru/app/products/list/12345678/...
  3. Число после /list/ и до следующего слеша — это и есть ваш Seller ID.

Способ 3: Через API-запрос (для технических специалистов)

Если у вас уже настроен доступ к Ozon API, можно получить ID программно. Отправьте GET-запрос:

https://api-seller.ozon.ru/v1/analytics/data

Headers:


Client-Id: [ваш_client_id]


Api-Key: [ваш_api_key]

В ответе будет поле "seller_id" с нужным значением. Этот метод актуален, если IDSuddenly исчез из интерфейса после обновлений.

Как найти Seller ID в мобильном приложении Ozon Seller

Мобильное приложение Ozon Seller (доступно для iOS и Android) имеет урезанный функционал, но отображать ID продавца всё же умеет. Инструкция:

  1. Откройте приложение и авторизуйтесь;
  2. Тапните на иконку профиля в правом нижнем углу;
  3. Выберите Настройки аккаунта;
  4. Прокрутите до блока Информация о компании — там будет строка ID продавца.
⚠️ Внимание: В приложении ID может отображаться не полностью (например, только последние 4 цифры). Чтобы увидеть полный код, используйте веб-версию кабинета.

Если в мобильной версии ID отсутствует вовсе, это означает, что:

  • 🔹 Ваш аккаунт ещё не прошёл верификацию (нужно завершить регистрацию в веб-версии);
  • 🔹 Вы используете устаревшую версию приложения (обновите её в App Store или Google Play);
  • 🔹 У вас ограниченные права доступа (обратитесь к администратору аккаунта).

Отличия Seller ID от Client-ID и Shop-ID

Новички часто путают эти три идентификатора, что приводит к ошибкам при интеграциях. Разберёмся в различиях:

Тип ID Назначение Где используется Пример
Seller ID Уникальный код продавца на Ozon API, поддержка, аналитика, реклама 12345678
Client-ID Идентификатор для авторизации в API Запросы к Ozon API, вебхуки 987654321098
Shop-ID Идентификатор витрины (может быть несколько) Настройки магазина, промоакции 12345

Client-ID выдаётся отдельно при подключении к API в разделе Настройки → Интеграции → API-ключи. Его нельзя использовать вместо Seller ID — это приведёт к ошибке 403 Forbidden при запросах. Например, при настройке выгрузки товаров через 1С потребуются оба идентификатора:

  • sellerId — для привязки к аккаунту;
  • clientId + apiKey — для авторизации запросов.

Shop-ID актуален для продавцов с несколькими витринами (например, для разных брендов). Его можно найти в разделе Магазины → Настройки витрины. В большинстве случаев для API нужен именно Seller ID, а не Shop-ID.

💡

Если вы видите ошибку "Invalid seller_id" при работе с API, проверьте, что передаёте именно 8-значный код без пробелов и тире. Часто проблема кроется в лишних символах, скопированных вместе с ID.

Что делать, если Seller ID исчез из личного кабинета

После обновлений интерфейса Ozon Seller некоторые продавцы сталкиваются с тем, что ID продавцаSuddenly пропадает из привычных мест. Причины и решения:

Причина 1: Обновление интерфейса

С 2026 года Ozon постепенно переводит продавцов на новую версию кабинета, где расположение элементов меняется. В этом случае:

  1. Попробуйте найти ID через URL (способ 2 из предыдущего раздела);
  2. Используйте API-запрос для получения идентификатора программно;
  3. Обратитесь в поддержку с просьбой указать актуальное местоположение ID.

Причина 2: Ограничение прав доступа

Если вы работаете в команде и ваша роль в аккаунте — «Менеджер» или «Аналитик», доступ к профилю компании может быть ограничен. Решение:

  • Попросите администратора аккаунта предоставить вам Seller ID;
  • Или запросите повышение прав до «Владелец»/«Администратор».

Причина 3: Блокировка аккаунта

Если ваш аккаунт заблокирован за нарушения (например, за продажу запрещённых товаров), доступ к профилю и ID может быть ограничен. В этом случае:

  1. Проверьте письма от Ozon на предмет блокировки;
  2. Свяжитесь с поддержкой через форму Помощь → Написать в поддержку;
  3. Укажите в обращении данные паспорта/ИНН для подтверждения личности — поддержка предоставит ID после разблокировки.
⚠️ Внимание: Никогда не делитесь своим Seller ID с посторонними лицами, особенно если они просят его для «проверки аккаунта» или «настройки рекламы». Мошенники могут использовать его для кражи данных или блокировки вашего магазина.

Как использовать Seller ID для интеграций и аналитики

Знание своего идентификатора открывает дополнительные возможности для автоматизации и анализа продаж. Рассмотрим основные сценарии применения:

1. Подключение к Ozon API

Для работы с API Ozon (выгрузка товаров, управление заказами, получение статистики) Seller ID обязателен. Пример запроса для получения списка заказов:

GET https://api-seller.ozon.ru/v2/posting/fbs/list

Headers:


Client-Id: [ваш_client_id]


Api-Key: [ваш_api_key]


Body:


{


"seller_id": 12345678,


"since": "2026-01-01T00:00:00Z"


}

2. Настройка Google Analytics 4

Чтобы отслеживать трафик с Ozon в GA4, добавьте параметр seller_id в настройки потока данных. Это поможет сегментировать пользователей по источникам переходов.

3. Интеграция с CRM-системами

В Bitrix24, АмоCRM или MoySklad при настройке синхронизации с Ozon потребуется указать:

  • 🔑 Seller ID — для идентификации аккаунта;
  • 🔑 Client-ID и API-Key — для авторизации;
  • 🔑 Shop-ID — если у вас несколько витрин.

4. Настройка рекламных кампаний

В Ozon Advertising и внешних системах (например, Retail Rocket) Seller ID используется для:

  • 📈 Привязки рекламного аккаунта к магазину;
  • 📈 Сегментации аудитории по покупателям вашего бренда;
  • 📈 Аналитики эффективности кампаний.
Пример ошибки при неверном Seller ID

Если передать в API запрос с несуществующим seller_id (например, 00000000), система вернёт ответ: {

"error": {

"code": "VALIDATION_ERROR",

"message": "Seller not found",

"details": [...]

}

}

Это означает, что идентификатор указан неверно или аккаунт заблокирован.

Типичные ошибки при работе с Seller ID и как их избежать

Даже опытные продавцы иногда допускают ошибки, связанные с использованием идентификатора. Разберём самые распространённые:

Ошибка 1: Передача ID с пробелами или тире

При копировании из интерфейса в ID могут попадать лишние символы. Например, вместо 12345678 вы копируете 12 345 678 или 1234-5678. Это приведёт к ошибке валидации.

Решение: Всегда проверяйте формат ID перед использованием. Правильный вариант — 8 цифр подряд без разделителей.

Ошибка 2: Использование Shop-ID вместо Seller ID

В некоторых CRM-системах есть поля для обоих идентификаторов. Если перепутать их местами, интеграция не заработает.

Решение: Уточните в документации сервиса, какой именно ID требуется. Например, для Ozon API нужен Seller ID, а для настройки промоакций — Shop-ID.

Ошибка 3: Разглашение ID третьим лицам

Мошенники могут запросить ваш Seller ID под предлогом «помощи с настройками» или «проверки аккаунта». С этим идентификатором они смогут:

  • 🚨 Получить доступ к вашей статистике продаж;
  • 🚨 Блокировать товары через поддержку;
  • 🚨 Подключить ваш аккаунт к своим сервисам.

Решение: Никогда не отправляйте ID посторонним. Для официальных интеграций используйте только проверенные сервисы (например, Ozon Partner).

Ошибка 4: Попытка изменить Seller ID

Некоторые продавцы пытаются «обнулить» или сменить ID через поддержку, чтобы «начать с чистого листа». Это невозможно — идентификатор присваивается один раз при регистрации и не изменяется.

Решение: Если вам нужен новый ID, зарегистрируйте отдельный аккаунт. Но помните, что многократная регистрация может привести к блокировке всех профилей.

💡

Seller ID — это как паспорт вашего магазина на Ozon. Его нельзя изменить, но можно использовать для расширения возможностей бизнеса: от автоматизации до глубокой аналитики.

FAQ: Частые вопросы о Seller ID на Ozon

Можно ли узнать Seller ID другого продавца?

Нет, Seller ID — это конфиденциальная информация. Увидеть его можно только в своём личном кабинете. Исключение: если продавец сам опубликовал свой ID (например, для партнёрских программ).

Что делать, если Seller ID начинается с нуля?

Это нормально — ID может начинаться с любой цифры, включая ноль. Главное, чтобы его длина была 8 символов. Например, 01234567 — валидный идентификатор.

Меняется ли Seller ID при смене названия магазина?

Нет, идентификатор остаётся неизменным независимо от редактирования профиля. Он привязан к аккаунту, а не к названию или юридическим данным.

Можно ли использовать один Seller ID для нескольких магазинов?

Да, если магазины зарегистрированы на одного продавца (одно юридическое лицо). В этом случае у них будет общий Seller ID, но разные Shop-ID.

Где взять Seller ID для Ozon Global?

Для продавцов на Ozon Global (международная версия) алгоритм получения ID аналогичный. Ищите его в профиле компании или через API. Отличие только в префиксе URL: seller-global.ozon.com.
