SKU ID на Ozon: полное руководство для продавцов в 2026 году

Вы когда-нибудь сталкивались с ситуацией, когда менеджер Озона просит предоставить SKU ID товара, а вы теряетесь в догадках, где его искать? Или замечали, что в отчётах по продажам вместо привычных артикулов отображаются странные цифробуквенные комбинации вроде FB-12345678-9? Это и есть SKU ID — уникальный идентификатор продукта на платформе Озон, который играет ключевую роль в логистике, аналитике и взаимодействии с маркетплейсом.

Для новичков на Озоне SKU ID часто становится источником путаницы. Многие продавцы ошибочно путают его с артикулом производителя или штрихкодом, но на самом деле это отдельная сущность, которая генерируется автоматически при загрузке товара. В этой статье мы разберём, почему SKU ID важен для работы на маркетплейсе, как он формируется, где его найти в личном кабинете и API, а также как правильно работать с этим идентификатором, чтобы избежать ошибок при управлении ассортиментом.

Что такое SKU ID на Озоне и зачем он нужен

SKU ID (от англ. Stock Keeping Unit Identifier) — это уникальный код, который Озон присваивает каждому товару в вашем ассортименте. Он состоит из префикса (например, FB- для схемы FBS или FBO- для FBO) и цифробуквенной комбинации, сгенерированной системой. В отличие от артикула производителя, который вы указываете сами, SKU ID создаётся автоматически и не может быть изменён вручную.

Основные функции SKU ID на Озоне:

  • 🔄 Связь с логистикой: по этому идентификатору склады Озона отслеживают движение вашего товара — от приёмки до отгрузки покупателю.
  • 📊 Аналитика продаж: все отчёты (по обороту, остаткам, возвратам) привязаны именно к SKU ID, а не к названию товара.
  • 🔗 Интеграция с API: при работе через программный интерфейс Озона SKU ID используется для обновления цен, остатков и другой информации.
  • 🛠️ Техническая поддержка: при обращении в саппорт Озона вам потребуется указать SKU ID для быстрого поиска проблемы.

Важно понимать, что один и тот же товар может иметь разные SKU ID в зависимости от:

  • 📦 Схемы работы: FBS и FBO генерируют разные идентификаторы для одного артикула.
  • 🏷️ Вариаций товара: если у вас есть один продукт в разных цветах или размерах, каждому варианту будет присвоен свой SKU ID.
  • 🔄 Повторной загрузки: если вы удалили и заново добавили товар, он получит новый SKU ID.
⚠️ Внимание: Никогда не используйте SKU ID от одного товара для другого! Это приведёт к ошибкам в логистике, неверным отчётам и возможным штрафам от Озона за несоответствие данных.

Как формируется SKU ID на Озоне: структура и примеры

SKU ID на Озоне имеет чёткую структуру, которая помогает быстро определить ключевые параметры товара. Рассмотрим на примере:

Типичный SKU ID выглядит так: FB-12345678-9 или FBO-98765432-1. Разберём его по частям:

Часть SKU ID Пример Что означает
Префикс схемы FB- или FBO- Указывает на модель работы: FBS (Fulfillment by Ozon) или FBO (Fulfillment by Operator).
Основной блок 12345678 Уникальный цифровой код товара, сгенерированный системой Озона. Длина варьируется от 6 до 10 символов.
Суффикс вариации -9 или -1 Используется для обозначения вариаций одного товара (например, разных размеров или цветов). Если суффикса нет, это базовая версия.

Примеры реальных SKU ID с Озона:

  • FB-55673241 — товар на FBS без вариаций.
  • FBO-89456123-2 — вторая вариация товара на FBO (например, синий цвет, если базовая была чёрная).
  • RTS-33456789 — товар, участвующий в программе Ozon RTS (доставка за 2 часа).

SKU ID с префиксом "FB-" всегда относится к схеме FBS, даже если выlater перевели товар на FBO. Новый идентификатор будет сгенерирован только при повторной загрузке товара в личном кабинете.

Где найти SKU ID товара в личном кабинете Озона

Найти SKU ID в интерфейсе Озона можно несколькими способами. Рассмотрим самые быстрые и удобные методы.

Способ 1: Карточка товара

  1. Перейдите в раздел Товары → Мой ассортимент.
  2. Найдите нужный товар в списке и кликните на его название.
  3. В открывшейся карточке товара SKU ID будет отображаться в блоке Основная информация под названием или артикулом.

Способ 2: Экспорт списка товаров

Если вам нужны SKU ID для всех товаров сразу:

  1. В разделе Товары → Мой ассортимент нажмите кнопку Экспорт.
  2. Выберите формат XLSX и дождитесь формирования файла.
  3. В экспортированной таблице SKU ID будет в одном из первых столбцов (обычно называется Идентификатор товара или SKU).

Способ 3: Отчёты по продажам

SKU ID также указывается во всех отчётах, связанных с продажами и остатками:

  • 📄 Отчёт Продажи (раздел Аналитика).
  • 📦 Отчёт Остатки на складах (раздел Логистика).
  • 🔄 Отчёт Возвраты (раздел Заказы).

💡

Если вы не можете найти SKU ID в карточке товара, попробуйте обновить страницу (клавиша F5) или очистить кэш браузера. Иногда интерфейс Озона подгружает данные с задержкой.

SKU ID в API Озона: как работать с идентификаторами программно

Если вы используете API Озона для автоматизации процессов (обновление цен, остатков, загрузка товаров), SKU ID становится критически важным параметром. В большинстве запросов к API вам потребуется указывать именно этот идентификатор, а не артикул или название товара.

Примеры использования SKU ID в API:

  • 📈 Обновление цен: в запросе POST /v2/prices SKU ID передаётся в массиве items.
  • 📦 Управление остатками: в методе POST /v2/warehouse/stocks SKU ID указывается вместе с количеством товара.
  • 🔍 Получение информации о товаре: запрос POST /v1/product/info требует передачи SKU ID в теле запроса.

Пример тела запроса для обновления цены через API:

{

"items": [


{


"offer_id": "FB-12345678",  // Здесь указывается SKU ID


"price": "1999.00"


}


]


}

💡

При работе с API всегда проверяйте, что вы используете актуальный SKU ID. Если товар был удалён и заново загружен, его идентификатор изменится, и старые запросы перестанут работать.

Чтобы получить SKU ID для API:

  1. Используйте метод POST /v1/product/info/list для получения списка всех ваших товаров с их идентификаторами.
  2. Экспортируйте список товаров в личном кабинете (как описано выше) и берите SKU ID оттуда.
  3. Используйте отчёты по продажам — в них всегда есть колонка с SKU ID.
⚠️ Внимание: В API Озона термин offer_id чаще всего означает именно SKU ID, а не артикул продавца. Не путайте эти понятия при формировании запросов!

Частые ошибки с SKU ID и как их избежать

Даже опытные продавцы иногда допускают ошибки при работе с SKU ID, что приводит к проблемам с логистикой, штрафам или потерям продаж. Рассмотрим самые распространённые случаи.

Ошибка 1: Использование старого SKU ID после повторной загрузки товара

Если вы удалили товар и заново его добавили (например, для исправления ошибок в карточке), он получит новый SKU ID. продолжение использования старого идентификатора приведёт к:

  • ❌ Ошибкам при обновлении цен/остатков через API.
  • ❌ Неправильным данным в отчётах.
  • ❌ Проблемам с логистикой (товар не будет найден на складе).

Ошибка 2: Путаница между SKU ID и артикулом продавца

Многие продавцы пытаются использовать свой внутренний артикул (vendor_code) вместо SKU ID в API или при обращении в поддержку. Это приводит к:

  • 🚫 Задержкам в обработке запросов (менеджеры Озона не работают с артикулами продавцов).
  • 🚫 Ошибкам в логистике (склады Озона оперируют только SKU ID).

Ошибка 3: Неучёт вариаций товара

Если у вашего товара есть вариации (например, разные цвета или размеры), каждая из них имеет свой SKU ID с суффиксом (-1, -2 и т. д.). Игнорирование этого приводит к:

  • 🔄 Неправильному учёту остатков (вы обновляете остатки для одной вариации, а система применяет изменения ко всем).
  • 📦 Ошибкам при сборке заказов на складе (сотрудники Озона не смогут идентифицировать нужный вариант).

Что делать, если SKU ID suddenly изменился?

Это может произойти при массовом обновлении карточек товаров со стороны Озона или при смене категории. В таком случае:

1. Проверьте актуальный SKU ID в карточке товара.

2. Обновите его во всех интеграциях (API, 1C, CRM).

3. Свяжитесь с поддержкой Озона, если изменение произошло без ваших действий.

SKU ID и аналитика: как использовать идентификаторы для роста продаж

SKU ID — это не только технический идентификатор, но и мощный инструмент для анализа продаж. Зная, как работать с этими данными, вы можете:

  • 📈 Отслеживать динамику продаж по каждому товару отдельно.
  • 📊 Анализировать остатки и предотвращать дефицит или переизбыток товара.
  • 🔍 Выявлять проблемные товары (низкий спрос, высокий уровень возвратов).
  • 💰 Оптимизировать ценовую политику на основе данных по конкретным SKU.

Пример использования SKU ID для анализа:

  1. Экспортируйте отчёт Продажи за последний месяц.
  2. Отсортируйте данные по колонке SKU ID.
  3. Проанализируйте:
    • 📉 Товары с падающим спросом (может потребоваться акция или снятие с продажи).
    • 📈 Товары с ростом продаж (стоит увеличить запасы).
    • 🔄 Товары с высоким процентом возвратов (проверьте описание, фото, качество).

Также SKU ID можно использовать для:

  • 🎯 Сегментации ассортимента: группируйте товары по категориям, брендам или ценам на основе их SKU.
  • 📦 Управления логистикой: отслеживайте, какие товары чаще задерживаются на складе или имеют проблемы с доставкой.
  • 💡 Тестирования гипотез: например, проанализируйте, как изменение цены для конкретного SKU повлияло на конверсию.

SKU ID и возвраты: как идентификатор помогает разобраться с проблемными заказами

При работе с возвратами на Озоне SKU ID играет ключевую роль. В отчёте по возвратам (Заказы → Возвраты) каждый возвращённый товар привязан к своему идентификатору. Это позволяет:

  • 🔍 Быстро находить проблемные товары (например, с высоким процентом возвратов).
  • 📦 Отслеживать статус возврата на складе Озона.
  • 💰 Контролировать возврат денежных средств или товаров на ваш склад.

Что делать, если вы видите в отчёте по возвратам незнакомый SKU ID?

  1. Проверьте, не удаляли ли вы этот товар ранее (возможно, это старый SKU).
  2. Используйте поиск по SKU ID в разделе Товары → Мой ассортимент.
  3. Если товар не найден, свяжитесь с поддержкой Озона и уточните, к какому продукту относится этот идентификатор.

Регулярно анализируйте отчёты по возвратам с группировкой по SKU ID. Это поможет выявить системные проблемы с конкретными товарами и снизить уровень возвратов.

FAQ: Частые вопросы о SKU ID на Озоне

Можно ли изменить SKU ID товара вручную?

Нет, SKU ID генерируется автоматически системой Озона и не подлежит ручному редактированию. Если вам нужно "сбросить" идентификатор, удалите товар и заново загрузите его — он получит новый SKU ID.

Чем отличается SKU ID от артикула продавца (vendor_code)?

SKU ID — это внутренний идентификатор Озона, который присваивается автоматически. Артикул продавца (vendor_code) — это ваш собственный код товара, который вы указываете при загрузке. В отчётах Озона обычно присутствуют оба идентификатора.

Почему у одного товара несколько SKU ID?

Это происходит, если:

  • У товара есть вариации (разные цвета, размеры и т. д.). Каждая вариация получает свой SKU ID с суффиксом (-1, -2).
  • Товар был удалён и заново загружен (новый SKU ID сгенерируется автоматически).
  • Товар присутствует в разных схемы (FBS и FBO), каждая из которых имеет свой идентификатор.
Как найти SKU ID для товара, который уже удалён из ассортимента?

Если товар удалён, его SKU ID можно найти:

  • В старых отчётах по продажам или возвратам (они хранятся в личном кабинете до 2 лет).
  • В архивных данных вашей CRM или 1C (если вы вели учёт).
  • Через поддержку Озона (придётся указать артикул продавца или название товара).

Если товар был удалён давно и данных нет, восстановить его SKU ID невозможно — при повторной загрузке будет сгенерирован новый идентификатор.

Нужно ли указывать SKU ID при обращении в поддержку Озона?

Да, в большинстве случаев поддержка Озона требует указать SKU ID для быстрого поиска товара в системе. Без этого идентификатора обработка запроса может занять значительно больше времени. Исключение — общие вопросы, не привязанные к конкретному товару.
