Почему продавцы Ozon теряют часы на поиск SKU ID и как этого избежать

Вы когда-нибудь тратили 20 минут на то, чтобы найти SKU ID одного товара в личном кабинете Ozon Seller? Если да — вы не одиноки. Этот уникальный идентификатор, который маркетплейс присваивает каждому товару, становится головной болью для новичков и иногда даже для опытных продавцов. Проблема в том, что Ozon регулярно обновляет интерфейс, а старые инструкции перестают работать. Сегодня мы разберём актуальные способы поиска SKU ID — от очевидных до скрытых.

Зачем вообще нужен этот код? Без него невозможно:

  • 🔄 обновить информацию о товаре через API;
  • 📦 оформить возврат или обмен;
  • 📊 отследить продажи конкретного артикула в аналитике;
  • 🛠️ решить проблему с модерацией через поддержку.

В этой статье вы найдёте не только пошаговые инструкции с картинками (их нет, но описания максимально детализированы), но и скрытые методы получения SKU ID для товаров, которые ещё не прошли модерацию или были удалены. А в конце — ответы на частые вопросы, которые продавцы стесняются задавать в чате поддержки.

Что такое SKU ID на Ozon и чем он отличается от артикула продавца

Многие путают SKU ID (идентификатор товара на Ozon) с артикулом продавца (вашим внутренним кодом). Разница принципиальна:

Где используется API, поддержка, аналитика Ozon Ваш склад, 1С, внутренние документы
Можно ли изменить Нет Да (но нежелательно)

Важно: один и тот же товар может иметь разные SKU ID в разных категориях Ozon. Например, если вы продаёте кроссовки в разделах "Мужская обувь" и "Спортивная обувь", система присвоит им уникальные идентификаторы. Это часто становится причиной путаницы при работе с Excel-выгрузками.

Способ 1: Поиск SKU ID через карточку товара в личном кабинете

Самый очевидный метод — зайти в карточку товара. Но даже здесь есть подводные камни. Следуйте алгоритму:

  1. Авторизуйтесь в личном кабинете Ozon Seller.
  2. Перейдите в раздел Товары → Мои товары.
  3. Найдите нужный товар через поиск или фильтры. Если товаров много, используйте фильтр по артикулу продавца.
  4. Кликните по названию товара, чтобы открыть его карточку.

Теперь внимательно смотрите на URL в адресной строке браузера. Он будет выглядеть так:

https://seller.ozon.ru/app/products/{SKU_ID}/edit

Где {SKU_ID} — это и есть нужный вам идентификатор. Например, в URL https://seller.ozon.ru/app/products/123456789/edit SKU ID равен 123456789.

💡

Если вы не видите цифры в URL, попробуйте обновить страницу (F5) или открыть карточку товара в режиме инкогнито. Иногда Ozon подгружает данные с задержкой.

Способ 2: Экспорт списка товаров в Excel — массовое получение SKU ID

Если вам нужны SKU ID для сотен товаров, ручной поиск через карточки займёт вечность. В этом случае поможет выгрузка каталога:

  1. В личном кабинете перейдите в Товары → Мои товары.
  2. Нажмите кнопку Экспортировать (в правом верхнем углу).
  3. Выберите формат Excel и дождитесь формирования файла (может занять до 10 минут).
  4. В полученном файле ищите колонку sku — это и есть SKU ID.

Обратите внимание: в выгрузке могут быть не все товары. Например, если товар на модерации или заблокирован, его SKU ID может отсутствовать. В этом случае используйте способ №1 или обратитесь в поддержку.

Способ 3: Получение SKU ID через API Ozon (для продвинутых пользователей)

Если вы работаете с Ozon через API, то SKU ID можно получить программно. Для этого используйте метод ProductInfo из документации API. Пример запроса:

GET https://api-seller.ozon.ru/v2/product/info

Headers:


- Client-Id: [ваш_client_id]


- Api-Key: [ваш_api_key]


Body:


{


"product_id": [ваш_артикул_продавца],


"offer_id": ""


}

В ответе вы получите JSON с полем "sku", где и будет нужный идентификатор. Этот метод удобен, если вам нужно автоматизировать работу с большим каталогом. Однако для его использования требуются:

  • 🔑 доступ к API (нужно запросить в личном кабинете);
  • 💻 базовые знания работы с Postman или cURL;
  • 📡 стабильное интернет-соединение (API Ozon иногда "подвисает").
Что делать, если API возвращает ошибку 404?

Ошибка 404 при запросе ProductInfo обычно означает, что товар не найден. Проверьте:

1. Правильность указанного product_id (артикула продавца).

2. Статус товара — он должен быть опубликован или на модерации.

3. Права вашего API-ключа (возможно, отсутствует доступ к методу).

Если проблема остаётся, обратитесь в поддержку с логами запроса.

Скрытые способы: как найти SKU ID для удалённых или неопубликованных товаров

Ситуация: товар удалён, на модерации или заблокирован, а SKU ID нужен срочно. Официальные методы не работают. Что делать?

Есть два неочевидных варианта:

  1. Через историю изменений:
    1. Перейдите в Товары → История изменений.
    2. Найдите запись о последнем редактировании нужного товара.
    3. В деталях изменения будет ссылка на карточку товара со SKU ID в URL.
  2. Через поддержку:
    1. Напишите в чат поддержки с просьбой предоставить SKU ID.
    2. Укажите артикул продавца и название товара.
    3. Приложите скриншот из личного кабинета (если товар виден в списке).
    ⚠️ Внимание: поддержка может отказать, если товар был удалён более 30 дней назад. В этом случае восстановите товар временно, скопируйте SKU ID и удалите снова.

Если товар ещё не прошёл модерацию, SKU ID ему ещё не присвоен. В этом случае используйте артикул продавца для взаимодействия с поддержкой.

Типичные ошибки при работе с SKU ID и как их избежать

Даже опытные продавцы иногда допускают ошибки, которые ведут к проблемам с товарами. Вот самые распространённые:

  • 🔢 Путаница между SKU ID и артикулом продавца. Например, при обращении в поддержку указывают свой внутренний артикул вместо SKU ID, из-за чего запрос обрабатывается дольше.
  • 📋 Копирование SKU ID с лишними символами. Часто в Excel или URL попадают пробелы, кавычки или знаки =, что приводит к ошибкам в API.
  • 🔄 Использование старого SKU ID после редизайна карточки. Ozon иногда меняет идентификаторы при значительных изменениях товара (например, смене категории).
  • 🚫 Попытка изменить SKU ID вручную. Это технически невозможно, но некоторые продавцы тратят время на поиск такого функционала.

Чтобы избежать проблем:

  • 📌 всегда проверяйте SKU ID на наличие лишних символов;
  • 🔍 используйте поиск по Ctrl+F в Excel-выгрузках;
  • 📑 ведите внутреннюю таблицу с соответствием ваших артикулов и SKU ID от Ozon.
💡

SKU ID — это не просто номер, а ключ к управлению товаром на Ozon. Ошибка в одном символе может привести к сбою в API или неправильной обработке возврата.

FAQ: Ответы на частые вопросы о SKU ID на Ozon

Можно ли использовать один SKU ID для разных товаров?

Нет, каждый SKU ID уникален и присваивается конкретному товару в конкретной категории. Если вы продаёте один и тот же товар в разных категориях (например, футболку в "Мужская одежда" и "Спортивная одежда"), у неё будут разные SKU ID.

Что делать, если SKU ID изменился после редактирования товара?

Это нормальная ситуация, если вы значительно изменили товар (например, поменяли категорию или бренд). В этом случае:

  1. Используйте новый SKU ID для всех операций.
  2. Обновите его в своих внутренних системах (1С, Excel).
  3. Если товар был в заказах, старый SKU ID останется в истории заказов, но для новых операций он недействителен.
Как найти SKU ID для товара, который ещё не прошёл модерацию?

SKU ID присваивается только после первой успешной модерации. До этого момента у товара есть только ваш внутренний артикул. Если нужен идентификатор для взаимодействия с поддержкой, используйте:

  • артикул продавца;
  • название товара;
  • ссылку на черновик из личного кабинета.
Можно ли восстановить SKU ID для удалённого товара?

Да, но с оговорками:

  • Если товар удалён менее 30 дней назад, его можно восстановить через Товары → Архив и увидеть SKU ID в карточке.
  • Если прошло больше 30 дней, обратитесь в поддержку с указанием артикула продавца и приблизительной даты удаления.
  • Если товар удалён навсегда (например, по требованию Ozon), SKU ID восстановить нельзя.
Зачем поддержке Ozon нужен SKU ID, если у меня есть артикул продавца?

SKU ID — это внутренний идентификатор Ozon, по которому система быстро находит товар в базе. Ваш артикул может:

  • совпадать с артикулами других продавцов;
  • быть изменён вами со времени создания товара;
  • не соответствовать текущей версии товара (например, после редизайна).

Поэтому поддержка всегда просит SKU ID — это гарантия, что речь идёт о конкретном товаре в их системе.
