Когда речь заходит о маркетплейсе Озон, большинство пользователей думают о покупках, доставке и продавцах. Но за внешним интерфейсом с карточками товаров и кнопкой «Купить» скрывается сложная техническая инфраструктура, способная обрабатывать миллионы запросов в секунду. Эта система работает круглосуточно, обеспечивая стабильность даже в пиковые периоды — например, во время «Чёрной пятницы» или новогодних распродаж.

В отличие от многих российских IT-проектов, Ozon Tech (техническое подразделение компании) не афиширует все детали своей архитектуры. Тем не менее, анализ вакансий, патентов, публичных выступлений инженеров и косвенных данных позволяет реконструировать основные компоненты платформы. Эта статья поможет продавцам понять, почему иногда возникают задержки в работе личного кабинета, а разработчикам — оценить масштаб технических решений, лежащих в основе маркетплейса.

Ядро платформы: микросервисы и облачная инфраструктура

Озон построил свою архитектуру на принципах микросервисов, что позволяет независимо масштабировать разные части системы — от поиска товаров до обработки платежей. Каждый микросервис отвечает за свою бизнес-функцию:

  • 🛒 Каталог товаров — хранение и индексация миллиардов SKU с атрибутами
  • 💳 Платежная система — интеграция с банками и Ozon Банком
  • 🚚 Логистика — расчёт маршрутов, управление складами и курьерами
  • 📊 Аналитика — сбор данных о поведении пользователей и генерация отчётов для продавцов

Для оркестрации микросервисов Озон использует Kubernetes (собственная модификация на базе открытого исходного кода), что подтверждается упоминаниями в вакансиях компании. Это решение позволяет автоматически балансировать нагрузку и перезапускать упавшие контейнеры без простоя. Интересно, что в 2020–2022 годах Озон активно нанимал специалистов по Istio — инструменту для управления сервисным мешем, что говорит о переходе на более зрелую архитектуру микросервисов.

Все сервисы развернуты в гибридном облаке:

  • 🌍 Собственные дата-центры — для критически важных систем (например, обработки платежей)
  • ☁️ Облачные провайдеры — AWS, Google Cloud и частично Yandex Cloud для менее чувствительных к задержкам сервисов (например, рекомендательных систем)
⚠️ Внимание: В 2022 году Озон начал активный перенос части инфраструктуры из зарубежных облаков в российские дата-центры из-за геополитических рисков. Это могло временно влиять на стабильность работы некоторых сервисов для продавцов.

Бэкенд: языки программирования и фреймворки

Основной стек технологий бэкенда Озон включает:

Компонент Технологии Примеры использования
Основные сервисы Java (Spring Boot), Kotlin Обработка заказов, управление складами, API для продавцов
Высоконагруженные сервисы Go (Golang), Rust Поисковый движок, система рекомендаций, обработка платежей
Скрипты и автоматизация Python (Django, FastAPI) Интеграции с партнёрами, ETL-процессы, внутренние инструменты
Фронтенд сервисов TypeScript (Node.js, NestJS) API-шлюзы, серверный рендеринг для мобильных приложений

Особенно стоит отметить использование Golang для критически важных сервисов. Этот язык выбран за счёт своей производительности и низкого потребления памяти, что критично для систем, обрабатывающих тысячи запросов в секунду. Например, поисковый движок Озон построен на базе Elasticsearch с кастомизированными плагинами на Go для оптимизации скорости выдачи результатов.

Для обработки платежей применяется Rust — язык, гарантирующий памятьную безопасность и высокую скорость выполнения. Это особенно важно для предотвращения утечек данных и мошенничества. В вакансиях Озон неоднократно упоминает опыт работы с Tokio (асинхронный runtime для Rust), что подтверждает использование этого стека.

Озон разработал собственную систему мониторинга на базе Prometheus + Grafana с кастомизированными дашбордами для отслеживания метрик в реальном времени. Это позволяет инженерам оперативно реагировать на сбои, например, при падении доступности API для продавцов.

Базы данных: как Озон хранит миллиарды записей

Для хранения данных Озон использует комбинацию из SQL и NoSQL решений в зависимости от задачи:

  • 🗃️ PostgreSQL — основная реляционная база для транзакционных данных (заказы, платежи, профили пользователей). Озон активно использует TimescaleDB (расширение для PostgreSQL) для работы с временными рядами — например, для анализа продаж по часам.
  • 🔍 Elasticsearch — поисковый движок для каталога товаров. Индексы обновляются в реальном времени с использованием Kafka для стриминга изменений.
  • 📦 ScyllaDB (совместимая с Cassandra) — для хранения данных о логистике и истории заказов. Эта база выбрана за счёт высокой скорости записи и горизонтальной масштабируемости.
  • 📈 ClickHouse — аналитическая база для генерации отчётов и дашбордов в личном кабинете продавца.

Интересный факт: Озон разработал собственную систему шардинга для PostgreSQL, которая позволяет автоматически распределять нагрузку между серверами в зависимости от географического расположения пользователя. Например, запросы продавцов из Сибири могут обрабатываться на серверах в Новосибирске, а москвичей — в подмосковных дата-центрах.

Для кэширования часто запрашиваемых данных (например, карточек товаров или промо-баннеров) используется Redis с кластеризацией. Это снижает нагрузку на основные базы данных и ускоряет отдачу контента пользователям. В пиковые нагрузки (например, во время распродаж) Озон временно увеличивает размер кэша, чтобы избежать падения производительности.

⚠️ Внимание: Если вы как продавец заметили, что изменения в карточке товара (например, цена или описание) обновляются с задержкой, это может быть связано с работой кэша. В среднем актуализация занимает от 5 до 30 минут, но в редких случаях может растянуться до часа.

Фронтенд: как устроены веб и мобильные приложения

Пользовательский интерфейс Озон — это не только сайт, но и мобильные приложения для iOS/Android, а также внутренние инструменты для сотрудников и продавцов. Все они построены на современных фреймворках:

  • 🌐 Веб-версия — React (с использованием Next.js для серверного рендеринга). Для стилей применяется CSS-in-JS (конкретно — styled-components).
  • 📱 Мобильные приложения — кросс-платформенная разработка на React Native с нативными модулями для критически важных функций (например, сканирование штрихкодов на складах).
  • 🛠️ Внутренние инструменты — Vue.js для админ-панелей и дашбордов.

Озон активно использует feature flags (флаги функций), что позволяет постепенно внедрять новые возможности для отдельных групп пользователей. Например, новый дизайн карточки товара может сначала показаться только 10% посетителей, а затем — после анализа метрик — раскатывается на всех.

Для оптимизации скорости загрузки страниц Озон применяет:

  • 🚀 Lazy loading — изображения и некритичные скрипты грузятся только при прокрутке.
  • 🗜️ Сжатие ресурсов — используются Brotli и Gzip для уменьшения размера CSS/JS-файлов.
  • 🔄 Service Worker — кэширование статических ресурсов в браузере для офлайн-работы (например, история заказов доступна без интернета).

Если вам как продавцу нужно протестировать, как ваша карточка товара отображается на мобильных устройствах, используйте режим Device Toolbar в Chrome (F12 → Ctrl+Shift+M). Это поможет выявить проблемы с вёрсткой до публикации.

Логистическая платформа: как Озон управляет доставкой

Одно из ключевых конкурентных преимуществ Озон — собственная логистическая сеть, которая включает склады, сортировочные центры и службу доставки. За её работу отвечает отдельный технический комплекс, построенный на следующих решениях:

  • 🗺️ Геоинформационная система — на базе PostGIS (расширение PostgreSQL для работы с геоданными) с интеграцией с Yandex Maps API и 2GIS для расчёта маршрутов.
  • 🤖 Автоматизация складов — система управления роботами-сортировщиками (в партнёрстве с Geek+) на базе ROS (Robot Operating System).
  • 📦 WMS (Warehouse Management System) — собственная разработка на Java с использованием Apache Kafka для обработки событий в реальном времени (например, приёмка товара от продавца).

Для оптимизации маршрутов курьеров Озон использует алгоритмы машинного обучения, обученные на исторических данных о времени доставки, пробках и погодных условиях. Модели построены на фреймворке PyTorch и обучаются на кластере NVIDIA GPU в облаке.

Интересный технический нюанс: Озон разработал систему динамического перераспределения заказов между курьерами. Если курьер задерживается (например, из-за пробки), алгоритм автоматически переназначает часть его заказов другим свободным курьерам в этом же районе. Это снижает количество просроченных доставок на 15–20%.

Как работает система возвратов?

При возврате товара продавцу Озон использует обратную логистику: курьер забирает посылку и доставляет её на ближайший сортировочный центр. Там товар сканируется, проверяется на соответствие описанию (например, наличие пломб) и отправляется обратно продавцу или на утилизацию (если товар испорчен). Весь процесс отслеживается в системе на базе Apache Kafka, где каждое действие (сканирование, проверка, отправка) фиксируется как событие.

Безопасность: как Озон защищает данные пользователей и продавцов

Безопасность в Озон построена по принципу «защиты в глубину» (Defense in Depth), где несколько слоёв защиты дублируют друг друга. Основные меры:

  • 🔐 Шифрование:
    • Данные пользователей (пароли, платежная информация) хранятся в зашифрованном виде с использованием AES-256.
    • Для передачи данных по сети используется TLS 1.3 с совершенной прямой секретностью (Perfect Forward Secrecy).
  • 🛡️ Защита от DDoS — Озон использует собственное решение на базе NGINX с кастомизированными правилами фильтрации трафика, а также услуги Cloudflare для поглощения атак.
  • 🤖 Борьба с ботами — система Ozon Shield анализирует поведение пользователей в реальном времени, блокируя подозрительные действия (например, массовый парсинг цен или создание фейковых аккаунтов).

Для аутентификации продавцов и пользователей Озон внедрил многофакторную авторизацию (MFA) на базе TOTP (Time-based One-Time Password). При этом для внутренних систем (например, доступа сотрудников к базам данных) используется YubiKey — аппаратные ключи безопасности.

Особое внимание уделяется защите API для продавцов. Все запросы к API подписываются с использованием JWT-токенов с ограниченным временем жизни. Кроме того, Озон внедрил систему rate limiting, которая блокирует IP-адреса, отправляющие слишком много запросов в секунду (это защищает от перегрузки серверов и злоумышленников).

⚠️ Внимание: Если вы как продавец используете сторонние сервисы для автоматизации работы с Озон (например, для выгрузки товаров), убедитесь, что они не нарушают правила использования API. Превышение лимитов запросов может привести к временной блокировке вашего аккаунта.

Интеграции с партнёрами: как Озон взаимодействует с внешними системами

Озон не существует в вакууме — платформа интегрирована с десятками внешних сервисов: банками, логистическими партнёрами, системами аналитики и т. д. Для этого используются:

  • 🏦 Банковские интеграции — подключение к ЦБ РФ, СБП (Система быстрых платежей), а также к процессинговым центрам Сбербанка, Тинькофф и других. Для обмена данными используется протокол ISO 8583 с дополнительным шифрованием.
  • 📦 Логистические партнёры — интеграция с СДЭК, Boxberry, Почтой России через EDI (Electronic Data Interchange) на базе AS2 или SFTP.
  • 📈 Аналитические системы — передача данных в Google Analytics, AppMetrica и внутренние BI-инструменты через Apache Kafka Connect.

Для продавцов Озон предоставляет несколько способов интеграции:

  1. REST API — для автоматизации выгрузки товаров, управления заказами и получения отчётов. Документация доступна в официальном портале.
  2. FTP/SFTP — для массовой загрузки каталога товаров в формате CSV или XLSX.
  3. 1C:Предприятие — готовые модули для синхронизации номенклатуры и остатков.

FAQ: ответы на частые вопросы о технической платформе Озон

На каком языке написан поисковый движок Озон?

Основной поисковый движок Озон работает на базе Elasticsearch с кастомизированными плагинами, написанными на Go (Golang). Для обработки запросов также используются алгоритмы машинного обучения на Python (фреймворк PyTorch), которые ранжируют результаты поиска с учётом персональных предпочтений пользователя.

Почему личный кабинет продавца иногда тормозит?

Замедления в работе личного кабинета могут быть связаны с несколькими факторами:

  • 🔄 Обновление системы — Озон регулярно выпускает апдейты, которые могут временно нагружать серверы.
  • 📊 Пиковая нагрузка — в периоды распродаж (например, Ozon Sale) количество запросов от продавцов резко возрастает.
  • 🌍 Географическое расположение — если вы находитесь далеко от ближайшего дата-центра Озон, задержки могут увеличиваться.

Если тормоза повторяются регулярно, попробуйте:

  1. Очистить кэш браузера или использовать режим инкогнито.
  2. Попробовать другой браузер (например, Chrome вместо Safari).
  3. Проверьте скорость интернета — иногда проблема на стороне провайдера.
Можно ли использовать API Озон для парсинга цен конкурентов?

Нет, массовый парсинг цен с Озон запрещён правилами использования API. Система Ozon Shield автоматически блокирует аккаунты, которые отправляют подозрительно большое количество запросов к каталогу товаров. Для легального мониторинга цен вы можете:

  • Использовать официальные отчёты в личном кабинете продавца (раздел «Аналитика»).
  • Подключить сервисы-партнёры Озон, которые имеют доступ к данным в рамках соглашения (например, Retail Rocket или eLama).

Нарушение правил может привести к блокировке API-ключа или даже аккаунта продавца.

Как Озон обеспечивает высокую доступность (99.9% uptime)?

Озон достигает высокой доступности за счёт:

  • 🔄 Многозональной архитектуры — сервисы дублируются в нескольких дата-центрах (например, в Москве, Санкт-Петербурге и Новосибирске).
  • 🛡️ Автоматического переключения — при падении основного сервера трафик перенаправляется на резервный (используется Consul для сервис-дискавери).
  • 📡 CDN — статические ресурсы (изображения, JS/CSS-файлы) раздаются через Cloudflare и Akamai, что снижает нагрузку на основные серверы.

Критические сервисы (например, обработка платежей) имеют трехкратное резервирование — даже если два дата-центра выйдут из строя, система продолжит работать.

Какие технологии используются для рекомендательной системы Озон?

Рекомендательная система Озон построена на базе:

  • 🤖 Машинного обучения — модели обучаются на PyTorch и TensorFlow с использованием данных о поведении пользователей (просмотры, покупки, время на странице).
  • 🗃️ Графовых баз данных — для анализа связей между товарами (например, «покупатели этого товара также берут…») используется Neo4j.
  • ⚡ Стриминговой обработки — события (клики, добавления в корзину) обрабатываются в реальном времени с помощью Apache Kafka и Flink.

Интересно, что Озон патентовал алгоритм, который учитывает геолокацию пользователя при формировании рекомендаций. Например, в Москве и Сочи могут показываться разные товары даже при одинаковом поисковом запросе.
