Как найти список заказов Озон, подлежащих оплате за прошедший период: все способы для продавцов

Работаете на маркетплейсе Ozon и нужно срочно сверить заказы, которые должны были оплатить за прошлый месяц или другой период? Поиск информации о выплатах — одна из самых частых задач для продавцов, особенно когда речь идёт о больших объёмах продаж или спорных ситуациях. Платформа предоставляет несколько инструментов для получения этих данных, но не все знают, где именно искать и как правильно интерпретировать отчёты.

В этой статье мы разберём все официальные и альтернативные способы получения списка заказов, подлежащих оплате за прошедший период: от личного кабинета и экспорта отчётов до работы с Ozon API и сторонними сервисами. Особое внимание уделим нюансам выплат (например, почему сумма в отчёте может не совпадать с фактической выплатой) и типичным ошибкам, которые допускают продавцы при анализе данных.

Если вы ищете конкретный отчёт или не понимаете, почему какие-то заказы не попали в выплату, здесь вы найдёте пошаговые инструкции с скриншотами (для актуальной версии личного кабинета Ozon Seller на 2026 год) и практические советы по работе с финансовой аналитикой.

1. Личный кабинет Ozon Seller: где искать заказы к оплате

Основной источник информации — раздел "Финансы" в личном кабинете продавца. Здесь собраны все данные о выплатах, удержаниях и транзакциях. Чтобы найти заказы, подлежащие оплате за прошлый период, выполните следующие шаги:

1. Авторизуйтесь на портале seller.ozon.ru и перейдите в меню Финансы → Выплаты.

2. В верхней части экрана выберите период, за который нужно получить данные (например, "Прошлый месяц" или укажите даты вручную).

3. Нажмите кнопку Сформировать отчёт — система покажет таблицу с выплатами, где каждый строка соответствует пакету заказов, отправленному на оплату.

Обратите внимание: в этом разделе отображаются только те заказы, которые уже прошли все проверки (включая контроль качества, возвраты и удержания). Если заказ ещё не попал в выплату, его нужно искать в других разделах (об этом ниже).

  • 💰 Выплата перенесена на следующий период из-за технических задержек (например, при смене реквизитов).

2. Отчёт "Детализация выплат": как экспортировать данные по заказам

Для детального анализа заказов, подлежащих оплате, используйте отчёт "Детализация выплат". Он содержит расширенную информацию, включая:

  • 📌 Номер и дату заказа,
  • 💵 Сумму к выплате (с учётом комиссий и удержаний),
  • 📦 Статус логистики (доставлен, возвращён, утерян),
  • ⚠️ Причины удержаний (если есть).

Как экспортировать отчёт:

1. В личном кабинете перейдите в Финансы → Отчёты → Детализация выплат.

2. Укажите диапазон дат (например, с 1 по 31 число прошлого месяца).

3. Нажмите Сформировать и дождитесь генерации файла (обычно занимает 1–5 минут).

4. Скачайте отчёт в формате .xlsx или .csv для дальнейшего анализа.

⚠️ Внимание: В экспортированном файле обратите внимание на колонку "Статус выплаты". Если там указано "Не выплачен", проверьте:

  • 🔄 Срок возврата (заказы FBS выплачиваются только после истечения 14 дней с момента доставки).
  • 🚫 Штрафы или блокировки (в колонке "Примечание" может быть указана причина).
  • 📄 Ошибки в реквизитах (если вы недавно их меняли, выплата могла "зависнуть").

3. Раздел "Аналитика": как сверить заказы с выручкой

Если вам нужно сопоставить заказы с фактической выручкой (до удержаний комиссий), используйте отчёт "Продажи" в разделе Аналитика. Здесь данные представлены в разрезе:

  • 📊 Выручка брутто (сумма всех заказов без вычета комиссий),
  • 💸 Выручка нетто (за минусом комиссий Ozon),
  • 📦 Количество заказов (включая отменённые и возвращённые).

Как пользоваться отчётом:

1. Перейдите в Аналитика → Продажи.

2. Выберите период и нажмите Применить.

3. В таблице найдите колонку "Выручка нетто" — это сумма, которая должна была попасть в выплату (за вычетом комиссий).

4. Сравните её с данными из раздела "Финансы → Выплаты". Расхождения могут указывать на:

- Удержанные штрафы,

- Невыплаченные заказы (например, из-за возвратов),

- Технические задержки.

4. API Ozon: автоматизированное получение данных о выплатах

Если вы работаете с большими объёмами заказов или ведёте учёт в 1С/Excel, ручной экспорт отчётов может быть неэффективным. В этом случае используйте Ozon Seller API, который позволяет автоматически получать данные о заказах и выплатах.

Основные методы API для работы с финансами:

  • 📤 /v3/finance/transaction/totals — суммарная информация по транзакциям.
  • 📊 /v2/finance/transaction/list — детализация выплат по заказам.
  • 🔍 /v1/finance/withdrawal/list — история выплат на ваш счёт.

Пример запроса для получения списка заказов к оплате (на языке Python): 

import requests


url = "https://api-seller.ozon.ru/v2/finance/transaction/list"


headers = {


"Client-Id": "Ваш_Client_ID",


"Api-Key": "Ваш_API_ключ",


"Content-Type": "application/json"


}


payload = {


"date_from": "2026-05-01T00:00:00Z",


"date_to": "2026-05-31T23:59:59Z",


"page": 1,


"page_size": 1000


}



response = requests.post(url, headers=headers, json=payload)


print(response.json())

⚠️ Внимание: Для работы с API необходимы:

  • 🔑 Client-ID и API-ключ (генерируются в личном кабинете в разделе Настройки → API).
  • 📖 Знание основ JSON и HTTP-запросов (или готовое решение для интеграции, например, Ozon Integration для 1С).
  • 🛡️ Соблюдение лимитов запросов (обычно 100 запросов в минуту).

Что делать, если API возвращает ошибку 403 Forbidden?

Ошибка 403 при работе с Ozon API обычно означает, что:

1. Неверный Client-ID или API-ключ (проверьте их в личном кабинете).

2. Истёк срок действия ключа (обновите его в разделе Настройки → API).

3. Ваш IP-адрес не внесён в белый список (актуально для некоторых типов ключей).

4. У вас недостаточно прав доступа (например, если вы не главный администратор аккаунта).

Для устранения ошибки:

- Сгенерируйте новый API-ключ.

- Проверьте, что запрос отправляется с правильными заголовками (Client-Id и Api-Key).

- Обратитесь в поддержку Ozon, если проблема сохраняется.

5. Сторонние сервисы для анализа выплат Ozon

Если стандартные отчёты Ozon кажутся неудобными, можно использовать специализированные сервисы для продавцов маркетплейсов. Они автоматизируют сбор данных, визуализируют аналитику и помогают выявлять расхождения.

Популярные инструменты:

  • 📈 Sellerboard — аналитика выручки, расчёт рентабельности, интеграция с Ozon API.
  • 💰 Lingonberry — мониторинг выплат, штрафов и удержаний.
  • 📊 Peak — дашборды с визуализацией финансовых показателей.
  • 🔗 МойСклад + Ozon — синхронизация заказов и выплат с учётной системой.

Плюсы сторонних сервисов:

- Автоматическое сопоставление заказов с выплатами (выявляют "потерянные" заказы).

- Уведомления о задержках выплат (например, если заказ не попал в выплату из-за возврата).

- Расчёт реальной прибыли с учётом всех комиссий, логистики и штрафов.

⚠️ Внимание: При выборе сервиса проверьте:

  • 🔒 Безопасность (сервис должен использовать OAuth 2.0 для доступа к вашему аккаунту).
  • 💳 Стоимость (некоторые тарифы берут процент от оборота).
  • 📅 Актуальность данных (некоторые сервисы обновляют информацию с задержкой).

💡

Сторонние сервисы удобны для глубокого анализа, но не заменяют официальные отчёты Ozon. Всегда сверяйте данные из личного кабинета с данными в стороннем инструменте, особенно при спорных ситуациях.

6. Типичные ошибки при поиске заказов к оплате

Многие продавцы сталкиваются с проблемами при анализе выплат из-за непонимания механики расчётов или технических нюансов. Вот самые распространённые ошибки:

1. Игнорирование периода возврата

Заказы по модели FBS (хранение на складе Ozon) выплачиваются только после истечения 14 дней с момента доставки. Если вы ищете заказы за май, но некоторые из них были доставлены 20–31 мая, они попадут в выплату только в июне.

2. Путаница между "выручкой" и "выплатой"

- Выручка — это сумма всех заказов (до вычета комиссий).

- Выплата — это сумма, которая фактически поступила на ваш счёт (за минусом комиссий, штрафов и удержаний).

Сравнивать нужно нетто-выручку (из отчёта "Продажи") с суммой к выплате (из отчёта "Детализация").

3. Неучёт удержаний за логистику

При модели FBO (самостоятельная доставка) Ozon удерживает комиссию за логистику (если вы использовали услуги партнёров Ozon, например, Ozon РЭД). Эти суммы не всегда очевидны в отчётах.

4. Забывают про "замороженные" заказы

Если покупатель инициировал возврат или претензию, сумма по такому заказу замораживается до разрешения спора. Эти заказы не попадут в выплату, пока проблема не будет устранена.

💡

Чтобы найти "зависшие" заказы, экспортируйте отчёт "Детализация выплат" и отфильтруйте строки со статусом "Не выплачен" или "В обработке". Затем проверьте каждый такой заказ в разделе Заказы → Претензии.

7. Что делать, если заказы не попали в выплату

Если вы обнаружили, что какие-то заказы отсутствуют в выплате, действуйте по алгоритму:

Шаг 1. Проверьте статус заказа

Перейдите в Заказы → История заказов, найдите нужный заказ и посмотрите его статус:

  • 🟢 "Доставлен" — проверьте дату доставки (возможно, не прошло 14 дней для FBS).
  • 🟡 "Возвращён" или "Отменён" — такие заказы не оплачиваются.
  • ❌ "Претензия" — разберитесь с проблемой (обратитесь в поддержку, если претензия необоснованна).

Шаг 2. Сверьтесь с отчётом "Штрафы"

Иногда заказы не выплачиваются из-за скрытых штрафов (например, за нарушение упаковки или просрочку отгрузки). Проверьте раздел Финансы → Штрафы.

Шаг 3. Обратитесь в поддержку

Если заказ доставлен более 14 дней назад, нет претензий и штрафов, но он не попал в выплату, создайте обращение в поддержку:

1. Перейдите в Помощь → Написать в поддержку.

2. Выберите тему "Выплаты" и укажите:

- Номер заказа,

- Дату доставки,

- Сумму, которая не поступила.

3. Прикрепите скриншоты из отчётов (например, из "Детализации выплат").

⚠️ Внимание: В поддержке Ozon не рассматривают вопросы по выплатам, если с момента доставки заказа прошло менее 14 дней (для FBS). Не создавайте лишние обращения — это замедлит обработку реальных проблем.

FAQ: Частые вопросы о заказах к оплате на Ozon

❓ Почему сумма в выплате меньше, чем в отчёте "Продажи"?

Это нормально: в отчёте "Продажи" показана выручка нетто (за минусом комиссий Ozon), но без учёта:

- Штрафов (проверьте раздел Финансы → Штрафы),

- Удержаний за логистику (если использовали Ozon РЭД или другие услуги),

- Замороженных сумм (из-за возвратов или претензий).

Сверяйте данные с отчётом "Детализация выплат" — там указана точная сумма к перечислению.

❓ Как узнать, какие заказы попали в конкретную выплату?

1. В разделе Финансы → Выплаты найдите нужную выплату и нажмите на её номер.

2. Система покажет список заказов, включённых в этот платеж.

3. Для детального анализа экспортируйте отчёт "Детализация выплат" за соответствующий период.

❓ Можно ли получить данные о выплатах за год?

Да, но с ограничениями:

- В личном кабинете доступна история выплат за последние 12 месяцев.

- Для более ранних периодов нужно обращаться в поддержку с обоснованием (например, для бухгалтерского учёта).

- Альтернатива: используйте Ozon API или сторонние сервисы (например, Sellerboard), которые архивируют данные.

❓ Почему выплата задерживается?

Частые причины:

- 🔄 Смена реквизитов (выплата приостанавливается на 1–3 дня для проверки).

- 📝 Ошибка в платежных данных (например, неверный расчётный счёт).

- 🚫 Блокировка аккаунта (из-за нарушений или подозрительной активности).

- 📅 Технические работы на стороне Ozon (реже, но бывает).

Проверьте статус выплаты в разделе Финансы → Выплаты. Если там указано "В обработке" дольше 3 дней, обратитесь в поддержку.

❓ Как экспортировать данные о выплатах для бухгалтерии?

1. Скачайте отчёт "Детализация выплат" в формате .xlsx за нужный период.

2. Добавьте в файл колонки для бухгалтерских проводок (например, "Счёт дебет", "Счёт кредит").

3. Если используете 1С, настройте автоматический импорт через Ozon API или модуль Ozon Integration.

4. Для упрощения работы используйте сервисы вроде МойСклад или Peak, которые автоматически формируют отчёты для ФНС.
