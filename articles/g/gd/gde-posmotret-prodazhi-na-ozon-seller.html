Где продавцу на Озоне посмотреть свои продажи: все способы с пошаговыми инструкциями

Продажи на Ozon — это не просто цифры в отчётах, а ключевой показатель успеха вашего бизнеса. Без точного мониторинга продаж невозможно оценить эффективность маркетинговых кампаний, спрогнозировать спрос или оперативно реагировать на изменения рынка. Однако многие новички сталкиваются с проблемой: где именно в личном кабинете Ozon Seller искать актуальные данные? А что делать, если стандартных отчётов недостаточно?

В этой статье мы разберём все доступные способы отслеживания продаж — от базовых инструментов в веб-версии до продвинутых методов через API и сторонние сервисы. Вы узнаете, как настроить автоматические уведомления о новых заказах, где скачать детализированные отчёты по периодам, и почему данные в мобильном приложении могут отличаться от веб-версии. А ещё — как увидеть продажи в реальном времени с разбивкой по товарам, регионам и каналам трафика.

1. Личный кабинет Ozon Seller: базовые отчёты о продажах

Основной источник информации для продавца — раздел Аналитика → Продажи в личном кабинете. Здесь собраны все ключевые метрики: количество заказов, выручка, средний чек и динамика по дням. Но как не потеряться в обилии цифр и найти именно те данные, которые нужны?

Чтобы открыть отчёт:

  • 📊 Перейдите в Личный кабинет → Аналитика → Продажи.
  • 📅 Выберите период (по умолчанию — последние 7 дней).
  • 🔍 Используйте фильтры: по статусам заказов (Новый, В обработке, Доставлен), типам товаров или регионам.
  • 📥 Нажмите Экспорт, чтобы скачать данные в Excel или CSV.

Обратите внимание: в этом разделе отображаются только завершённые продажи (с учётом возвратов и отмен). Если вам нужны данные по отменённым или возвращённым заказам, переходите в Аналитика → Возвраты.

⚠️ Внимание: Данные в разделе Продажи обновляются с задержкой до 24 часов. Для оперативного мониторинга используйте Дашборд или мобильное приложение.

2. Дашборд продавца: продажи в реальном времени

Если вам нужны актуальные данные без задержки, переходите в Дашборд (главная страница личного кабинета). Здесь отображаются:

  • 📈 Текущая выручка за сегодня (сравнение с прошлым днём).
  • 📦 Количество новых заказов за последние 24 часа.
  • 🔄 Динамика продаж за неделю/месяц в виде графика.
  • 🚀 Топ-5 товаров по количеству продаж.

Преимущество дашборда — онлайн-обновление (данные появляются почти мгновенно после оформления заказа покупателем). Однако здесь нет детализации по регионам или каналам трафика. Для этого придётся использовать Аналитику или API.

💡

Добавьте дашборд в закладки браузера — так вы сможете быстро проверять продажи, не авторизуясь заново.

3. Мобильное приложение Ozon Seller: удобство на ходу

Для тех, кто ведёт бизнес в дороге, незаменимо официальное приложение Ozon Seller (доступно для iOS и Android). В нём дублируются основные функции веб-версии, но с удобным мобильным интерфейсом:

  • 📱 Пуш-уведомления о новых заказах и возвратах.
  • 📊 Быстрый доступ к графику продаж за неделю/месяц.
  • 🔍 Поиск заказов по номеру или статусу.
  • 📌 Отметки о обработке заказов прямо в приложении.

Важный нюанс: в мобильной версии нет некоторых фильтров, доступных в веб-кабинете. Например, нельзя отсортировать продажи по типу доставки (FBS/FBO) или категориям товаров. Зато здесь есть удобный виджет с итогами за день, который можно вывести на главный экран смартфона.

⚠️ Внимание: В приложении иногда возникают расхождения с веб-версией из-за кэширования данных. Чтобы обновить информацию, потяните экран вниз (жест pull-to-refresh).

4. Экспорт данных через API: для продвинутых продавцов

Если вам нужны сырые данные для собственной аналитики или интеграции с 1С, Excel или Power BI, используйте Ozon Seller API. Через него можно получить:

  • 📂 Полный список заказов с детализацией по товарам, ценам и скидкам.
  • 📈 Статистику продаж по любым временным интервалам (вплоть до часов).
  • 🔗 Данные о трафике (откуда пришли покупатели: поиск, рекомендации, реклама).
  • 📦 Информацию о возвратах и отменах с причинами.

Чтобы начать работать с API:

  1. Сгенерируйте Client-ID и API-Key в разделе Настройки → Интеграции → API.
  2. Изучите официальную документацию (там есть примеры запросов на Python, PHP и JavaScript).
  3. Используйте эндпоинты: 
    GET /v2/analytics/sales

    для получения данных о продажах.

Пример запроса для получения продаж за последний месяц:

curl -X GET'https://api-seller.ozon.ru/v2/analytics/sales' \

-H'Client-Id: {ваш_client_id}' \


-H'Api-Key: {ваш_api_key}' \


-H'Content-Type: application/json' \


-d'{


"date_from":"2026-06-01T00:00:00Z",


"date_to":"2026-06-30T23:59:59Z"


}'

⚠️ Внимание: API имеет лимиты на количество запросов (обычно 1000 в минуту). Превышение лимита приводит к блокировке на 5 минут. Для массовой выгрузки данных используйте пагинацию (limit и offset).
Как автоматизировать выгрузку данных через API?

Для автоматизации можно написать скрипт на Python с использованием библиотеки requests и планировщика cron (на Linux) или Задач Windows. Пример скрипта:

import requests

import json


from datetime import datetime, timedelta



Получаем вчерашнюю дату


yesterday = (datetime.now - timedelta(days=1)).strftime("%Y-%m-%d")



url ="https://api-seller.ozon.ru/v2/analytics/sales"


headers = {


"Client-Id":"YOUR_CLIENT_ID",


"Api-Key":"YOUR_API_KEY",


"Content-Type":"application/json"


}


payload = {


"date_from": f"{yesterday}T00:00:00Z",


"date_to": f"{yesterday}T23:59:59Z"


}



response = requests.post(url, headers=headers, json=payload)


data = response.json



Сохраняем в файл


with open(f"sales_{yesterday}.json","w") as f:


json.dump(data, f, indent=4)

Этот скрипт ежедневно будет сохранять данные о продажах в JSON-файл. Для автоматического запуска настройте cron командой:

0 9   * /usr/bin/python3 /path/to/your/script.py

При выборе сервиса обращайте внимание на:

  • 🔄 Частоту обновления данных (некоторые сервисы обновляют статистику раз в сутки).
  • 🔒 Безопасность (насколько надёжно хранятся ваши API-ключи).
  • 📊 Гибкость настройки (можно ли добавить свои метрики или фильтры).

Сторонние сервисы полезны для мультимаркетплейсовых продавцов, но требуют дополнительных затрат на интеграцию и обучение.

6. Как анализировать продажи: ключевые метрики

Просто смотреть на цифры недостаточно — нужно уметь их интерпретировать. Вот 5 ключевых метрик, на которые стоит обращать внимание:

  • 💰 Выручка (нетто) — сумма продаж за вычетом скидок, возвратов и комиссий Ozon.
  • 📦 Конверсия в заказ — процент посетителей карточки товара, оформивших покупку.
  • 🔄 Уровень возвратов — сколько заказов было возвращено (норма — до 5% для большинства категорий).
  • 🛒 Средний чек — показывает, насколько эффективно работают ваши апселлы (дополнительные продажи).
  • 📈 Динамика по дням недели — помогает спланировать рекламные кампании.

Пример анализа:

Если вы видите, что конверсия падает по определённому товару, проверьте:

  • 📸 Качество фотографий (возможно, они не соответствуют стандартам Ozon).
  • 📝 Описание товара (достаточно ли информации для принятия решения?).
  • ⭐ Отзывы (есть ли негативные комментарии, которые отпугивают покупателей?).

Если средний чек снижается, попробуйте:

  • 🎁 Добавить комплекты товаров (например,"купи 2 по цене 1,5").
  • 🔥 Запустить акцию"бесплатная доставка от X рублей".
  • 📢 Рекламировать более дорогие модели в карточке товара.

7. Частые ошибки при анализе продаж

Даже опытные продавцы иногда допускают ошибки, которые искажают реальную картину. Вот что нельзя делать:

  • 🚫 Игнорировать возвраты. Выручка"грязная" (без учёта возвратов) может вводить в заблуждение. Всегда смотрите на чистые продажи.
  • 🚫 Сравнивать разные периоды без учёта сезонности. Продажи в декабре (перед Новым годом) всегда выше, чем в феврале.
  • 🚫 Не учитывать комиссии Ozon. Ваша"выручка" и реальная прибыль — разные вещи.
  • 🚫 Полагаться только на автоматические отчёты. Иногда данные в них обновляются с задержкой или содержат ошибки.

Ещё одна распространённая проблема — неверная интерпретация графиков. Например, резкий скачок продаж может быть связан не с успехом вашей рекламной кампании, а с:

  • 🎉 Акцией на площадке (например, Ozon Sale).
  • 📢 Рекламой товара в рассылке Ozon.
  • 🔍 Изменением алгоритмов выдачи (ваш товар вдруг попал в топ по запросу).

Чтобы избежать ошибок:

  1. Всегда проверяйте данные в нескольких источниках (например, сверяйте API с личным кабинетом).
  2. Используйте сегментацию (разбивайте продажи по регионам, категориям, типам доставки).
  3. Ведите собственные таблицы в Excel или Google Sheets для сравнения периодов.

FAQ: Ответы на частые вопросы

Можно ли увидеть продажи по конкретному товару?

Да. В разделе Аналитика → Продажи используйте фильтр По товарам и выберите нужный SKU или артикул. Также эту информацию можно получить через API с параметром product_id.

Почему данные в мобильном приложении и веб-версии разные?

Это связано с кэшированием данных в приложении. Чтобы обновить информацию, потяните экран вниз (жест pull-to-refresh) или перезапустите приложение. В веб-версии данные обновляются чаще.

Как экспортировать продажи за год?

В личном кабинете можно выгрузить данные максимум за 90 дней. Для более периодов используйте API или сторонние сервисы (например, Peak или RetailCRM).

Можно ли получить данные о продажах по регионам?

Да, в разделе Аналитика → Продажи есть фильтр По регионам. Также эту информацию предоставляет API в ответе на запрос /v2/analytics/sales (поле region).

Как отследить продажи по рекламным кампаниям?

Для этого перейдите в Реклама → Статистика. Здесь можно увидеть, сколько заказов и выручки принесла каждая кампания. Также эти данные доступны в API через эндпоинт /v1/performance/statistics.
