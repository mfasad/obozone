Где в Озоне смотреть и скачивать отчёты о реализации товара: инструкция для продавцов

Продажи на Ozon требуют постоянного анализа: без актуальных данных о реализации товара невозможно оценить эффективность кампаний, спрогнозировать спрос или оперативно реагировать на изменения. Однако многие продавцы теряют часы, пытаясь найти нужные отчёты в лабиринте меню личного кабинета. Где именно в Ozon Seller хранятся данные о продажах? Как их экспортировать в удобном формате? И почему иногда цифры в отчётах не сходятся с реальными поступлениями?

В этой статье — пошаговый разбор всех способов получения отчётности о реализации, включая скрытые нюансы работы с FBS и FBO, различия между отчётами «По заказам» и «По товарам», а также уникальные лайфхаки по автоматизации выгрузки данных через API. Информация актуализирована на июнь 2026 года с учётом последних обновлений интерфейса Ozon.

1. Где в личном кабинете Озон искать отчёты о продажах

Основной раздел для работы с отчётностью — Аналитика → Отчёты. Однако путь к нему зависит от типа вашего аккаунта (индивидуальный продавец или компания) и модели работы (FBS или FBO). Вот точные маршруты:

  • 📊 Для FBS: Аналитика → Отчёты → Продажи → Отчёт по реализации (FBS)
  • 📦 Для FBO: Аналитика → Отчёты → Продажи → Отчёт по реализации (FBO)
  • 🔍 Универсальный путь: используйте поиск по ключевому слову «реализация» в верхней панели личного кабинета

Если у вас мультиаккаунт (несколько магазинов на одной платформе), перед загрузкой отчёта обязательно проверьте выбранный магазин в верхнем правом углу — данные будут сформированы только по текущему профилю.

💡

Сохраните в закладки браузера прямую ссылку на раздел отчётов: https://seller.ozon.ru/analytics/reports/sales. Это сэкономит время при ежедневном мониторинге.

2. Виды отчётов о реализации: какой выбрать

Ozon предлагает 4 типа отчётов о продажах, каждый из которых решает свои задачи. Их легко спутать, но различия критичны для анализа:

Тип отчёта Что показывает Когда использовать Формат
Отчёт по реализации (детализированный) Все продажи по каждому SKU с разбивкой по заказам, датам, статусам Для глубокого анализа ассортимента, выявления лидеров/аутсайдеров Excel, CSV
Сводный отчёт по продажам Агрегированные данные по товарам (количество, сумма) без привязки к заказам Для быстрой оценки динамики продаж по категориям Excel, PDF
Отчёт по заказам Информация о статусах заказов (отмены, возвраты, доставки) Для контроля логистики и выявления проблемных заказов Excel, CSV
Финансовый отчёт Поступления денег на счёт с учётом комиссий и удержаний Для сверки с банковскими выписками Excel

Наиболее востребован среди продавцов детализированный отчёт по реализации — он содержит SKU, артикулы, цены продажи, даты заказов и даже данные о промокодах, если они применялись. Однако у него есть ограничение: максимальный период выгрузки — 3 месяца. Для анализа более длительных периодов придётся скачивать данные частями.

3. Пошаговая инструкция: как скачать отчёт о реализации

Разберём процесс на примере детализированного отчёта по реализации (актуально для FBS и FBO):

  1. Перейдите в Аналитика → Отчёты → Продажи → Отчёт по реализации.

  2. Выберите период (максимум 90 дней). Для анализа сезонности рекомендуем сравнивать одинаковые периоды разных лет (например, май 2023 vs май 2026).

  3. Укажите статусы заказов:

    • 🟢 Доставлен — для анализа успешных продаж
    • 🟡 В обработке — для контроля текущих заказов
    • 🔴 Отменён — для выявления проблем (недостаток товара, ошибки цены)

  • Нажмите Сформировать отчёт и дождитесь уведомления о готовности (приходит на email и в раздел Уведомления личного кабинета).

  • Скачайте файл в формате Excel или CSV (последний подходит для автоматизированной обработки).

    • Указан верный магазин (если несколько)|Период не превышает 90 дней|Выбраны нужные статусы заказов|Формат файла соответствует задачам (Excel для анализа, CSV для интеграций)|-->

    Время формирования отчёта зависит от объёма данных:

    • 🕒 До 1000 заказов — 1–2 минуты
    • 🕓 1000–10 000 заказов — 5–15 минут
    • 🕛 Более 10 000 заказов — до 1 часа (рекомендуем формировать в непиковые часы: с 22:00 до 8:00 МСК)

    Почему отчёт долго формируется?

    Ozon ограничивает серверные мощности для генерации отчётов в пиковые часы (с 10:00 до 20:00 МСК). Кроме того, задержки возникают при:

    - Большом количестве SKU в отчёте (более 5000 позиций)

    - Одновременных запросах от других продавцов

    - Технических работах на платформе (обычно по вторникам)

    Если отчёт «завис» дольше часа, попробуйте:

    1. Обновить страницу.

    2. Сузить период или разбить запрос на части.

    3. Скачать данные через API (требует навыков программирования).

    4. Расшифровка столбцов в отчёте: на что обратить внимание

    Детализированный отчёт содержит более 30 столбцов, но для анализа критичны лишь некоторые. Вот ключевые поля и их значение:

    • 📌 order_id — уникальный номер заказа (используйте для поиска в разделе Заказы)
    • 📌 sku — ваш внутренний артикул товара
    • 📌 product_id — идентификатор товара в Ozon (нужен для API-запросов)
    • 💰 price — цена продажи с учётом скидок (не путать с рекомендованной ценой!)
    • 📦 quantity — количество единиц товара в заказе
    • 📅 created_at — дата создания заказа (а не доставки!)
    • 🔄 status — текущий статус (delivered, cancelled и т.д.)
    • 🏷️ promo_code — применённый промокод (если есть)

    Ловушка для новичков: столбец revenue показывает выручку без учёта комиссий Ozon. Чтобы узнать реальный доход, вычтите:

    • Комиссию маркетплейса (от 5% до 15% в зависимости от категории)
    • Логистические расходы (для FBS)
    • Стоимость возвратов (если были)

    💡

    Сверяйте данные из отчёта по реализации с финансовым отчётом раз в неделю. Расхождения более 3% — повод обратиться в поддержку с запросом о пересчёте комиссий.

    5. Типичные ошибки при работе с отчётами и как их избежать

    Даже опытные продавцы допускают ошибки, которые искажают аналитику. Вот TOP-5 промахов и способы их предотвратить:

    ⚠️ Внимание: Если в отчёте отсутствуют данные за последние 2–3 дня, это не сбой системы. Ozon обновляет информацию о реализации с задержкой до 48 часов для учёта возможных отмен и возвратов.
    • ❌ Игнорирование статусов заказов

      Многие скачивают отчёт только по статусу delivered, забывая про shipped (отгруженные, но не доставленные) и returned (возвраты). Это приводит к занижению реальных продаж на 5–15%.

      Решение: всегда выгружайте данные по всем статусам и фильтруйте их уже в Excel.
    • ❌ Сравнение «яблок с апельсинами»

      Сравнивать продажи по created_at (дате создания заказа) и delivered_at (дате доставки) бессмысленно — разница может достигать 2 недель.

      Решение: используйте только created_at для анализа динамики спроса.
    • ❌ Неучёт мультизаказов

      Если один заказ содержит несколько SKU, в отчёте они разбиваются на отдельные строки с одинаковым order_id. При суммировании по заказам это приводит к двойному счёту.

      Решение: группируйте данные по order_id перед анализом.

    Ещё одна распространённая проблема — расхождения между отчётом по реализации и финансовым отчётом. Причины:

    • 💸 Удержание комиссий за прошлые периоды (например, за возвраты месячной давности)
    • 📦 Логистические штрафы (за нарушение сроков отгрузки в FBS)
    • 🎁 Скидки по акциям, которые не отразились в данных о реализации
    =Выручка - (Выручка * Комиссия%) - Логистика - Возвраты

    Обновляйте коэффициенты комиссий раз в квартал (их Ozon меняет без предупреждения).-->

    6. Автоматизация выгрузки отчётов: API и сторонние сервисы

    Ручная выгрузка отчётов отнимает время, особенно если у вас тысячи заказов в день. Решения для автоматизации:

    • 🤖 Ozon API

      Позволяет получать данные о реализации в реальном времени через запрос GET /v2/analytics/sales. Требует знания Python или JavaScript для интеграции.

      Плюсы: актуальные данные, гибкая фильтрация. Минусы: ограничение 1000 запросов в час, сложность настройки.
    • 📊 Сервисы-коннекторы (МойСклад, 1С:УНФ, RetailCRM)

      Синхронизируют данные с Ozon и визуализируют их в дашбордах.

      Плюсы: нет нужды в программировании, удобные графики. Минусы: платная подписка (от 1500 ₽/мес), задержка данных до 24 часов.
    • 📈 Google Sheets + Apps Script

      Бесплатное решение для автоматизации выгрузки через API. Подходит для небольших магазинов.

      Пример кода: 
      function getOzonSales() {

      const token = 'Ваш_API_ключ';
      

      const url = 'https://api-seller.ozon.ru/v2/analytics/sales';
      

      const response = UrlFetchApp.fetch(url, {headers: { 'Client-Id': 'Ваш_Client_ID', 'Api-Key': token }});
      

      const data = JSON.parse(response.getContentText());
      

      // Дальше парсинг данных
      

      }

    ⚠️ Внимание: При использовании API Ozon блокирует аккаунты за превышение лимитов запросов. Всегда проверяйте ответ сервера на наличие ошибки 429 Too Many Requests и добавляйте задержки между запросами (рекомендуем 1 запрос в 2 секунды).

    7. Как анализировать отчёты: практические кейсы

    Данные о реализации бесполезны без правильной интерпретации. Рассмотрим 3 реальных сценария, как их использовать:

    • 📉 Падение продаж конкретного SKU
      Действия:
      1. Проверьте динамику по датам в отчёте — падение началось резко или постепенно?
      2. Сравните с конкурентами: не появились ли у них более выгодные предложения?
      3. Изучите отзывы — возможно, проблема с качеством товара.
      Пример: Если продажи упали на 30% за неделю, а у конкурентов — нет, причина скорее в вашем товаре (описание, цена, репутация).
    • 🔄 Высокий процент отмен заказов
      Действия:
      1. Отфильтруйте отчёт по статусу cancelled.
      2. Проверьте причины отмен (столбец cancellation_reason).
      3. Если преобладает out_of_stock — срочно настройте автоматическое обновление остатков.
    Критический порог: более 5% отмен от общего числа заказов.
  • 🎯 Оценка эффективности акций
    Действия:
    1. Выгрузите отчёт за период акции и за аналогичный период без неё.
    2. Сравните quantity (количество продаж) и revenue (выручку).
    3. Посчитайте ROMI (возврат на инвестиции в акцию):
      (Дополнительная выручка - Затраты на акцию) / Затраты на акцию
    Норма: ROMI > 200% — акция окупилась.
    • 💡

    Самый простой способ выявить проблемные товары — отсортировать отчёт по столбцу quantity по убыванию. SKU в конце списка (с нулевыми продажами за месяц) — кандидаты на удаление или ребрендинг.

    FAQ: Частые вопросы о отчётах в Озоне

    Можно ли получить отчёт о реализации за период больше 3 месяцев?

    Нет, в личном кабинете максимальный период — 90 дней. Для анализа более длительных периодов:

    1. Разбейте запрос на части (например, по кварталам).
    2. Скачайте данные через API — там ограничение мягче (до 1 года).
    3. Используйте сторонние сервисы (например, RetailCRM), которые хранят историю дольше.

    Важно: Ozon хранит данные о заказах не более 2 лет, после чего они безвозвратно удаляются.

    Почему в отчёте нет данных по некоторым заказам?

    Причины могут быть следующими:

    • Заказы ещё не обработаны системой (задержка до 48 часов).
    • Заказы были созданы в другом магазине вашего мультиаккаунта.
    • Заказы попали под техническую ошибку (редко, но бывает при сбоях API).
    • Вы указали неверный статус (например, искали только delivered, а заказ ещё shipped).

    Что делать: проверьте раздел Заказы в личном кабинете — если заказы там есть, но их нет в отчёте, обратитесь в поддержку с указанием order_id.

    Как экспортировать отчёт в Google Таблицы автоматически?

    Способ 1: Через Apps Script (для технически подкованных):

    1. Создайте новую таблицу в Google Sheets.
    2. Откройте Расширения → Apps Script.
    3. Вставьте код для запроса к API Ozon (пример см. в разделе 6).
    4. Настройте триггер для автоматического обновления (например, раз в день в 9:00).

    Способ 2: Через Zapier или Make (ex-Integromat):

    1. Создайте новый сценарий с триггером «Новый отчёт в Ozon».
    2. Добавьте действие «Добавить строку в Google Sheets».
    3. Настройте mapping полей (например, order_id → Колонка A).

    Ограничение: бесплатные тарифы Zapier позволяют обрабатывать до 100 задач в месяц.

    Что делать, если цифры в отчёте не сходятся с финансовым отчётом?

    Расхождения возникают из-за:

    1. Комиссий и удержаний: финансовый отчёт показывает чистую сумму после вычета комиссий Ozon (от 5% до 20%), а отчёт по реализации — валовую выручку.
    2. Возвратов: деньги за возвращённые товары списываются из финансового отчёта, но остаются в данных о реализации (со статусом returned).
    3. Логистических расходов: для FBS стоимость доставки вычитается из финансового отчёта, но не отражается в реализации.

    Как сверить:

    1. Возьмите сумму из столбца revenue (отчёт по реализации).
    

    2. Вычтите комиссии (средний % для вашей категории).
    

    3. Вычтите стоимость возвратов (данные из отчёта по статусу returned).
    

    4. Для FBS вычтите логистические расходы (из финансового отчёта).
    

    5. Сравните результат с поступлениями на счёт.

    Если разница превышает 3%, запросите у поддержки Ozon детализацию удержаний за спорный период.

    Можно ли получить отчёт о реализации по конкретному ПВЗ?

    Прямой фильтрации по ПВЗ в стандартных отчётах нет, но есть обходные пути:

    1. Через API: используйте параметр warehouse_id в запросе /v2/analytics/sales.
    2. Через Excel:
      1. Скачайте полный отчёт по реализации.
      2. Добавьте столбец с данными о ПВЗ (их можно выгрузить отдельно через Аналитика → Логистика → Отчёт по ПВЗ).
      3. Сделайте сводную таблицу по полю warehouse_id.
    3. Через поддержку: напишите запрос в службу помощи с просьбой предоставить данные по конкретному ПВЗ (указывайте его ID).

    Важно: данные о ПВЗ в отчётах появляются только для модели FBO. В FBS информация о пунктах выдачи не раскрывается.
