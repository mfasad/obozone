Где посмотреть SKU на Озон: все способы для продавцов и покупателей в 2026 году

Артикул SKU (Stock Keeping Unit) на Ozon — это уникальный идентификатор товара, который используется для отслеживания позиций в каталоге, управления остатками и обработки заказов. Без этого кода невозможно корректно загрузить товар в личный кабинет продавца, синхронизировать данные с 1С или другими системами учета, а иногда он требуется даже покупателям для возвратов или уточнения деталей заказа. Проблема в том, что интерфейс маркетплейса регулярно обновляется, и старые инструкции по поиску SKU перестают работать.

В этой статье мы разберем все актуальные способы, как узнать SKU на Ozon в 2026 году — от стандартных методов для продавцов (через личный кабинет, API, экспорт данных) до лайфхаков для покупателей, которые хотят уточнить артикул по карточке товара. Особое внимание уделим нюансам для схем FBS и FBO, а также типичным ошибкам, из-за которых SKU может не отображаться или дублироваться. Если вы продавец, который только начинает работать с маркетплейсом, или покупатель, столкнувшийся с необходимостью указать артикул при возврате — здесь вы найдете пошаговые инструкции с скриншотами и пояснениями.

1. Что такое SKU на Ozon и зачем он нужен

SKU (произносится как «скью») — это внутренний артикул товара, который присваивается каждой позиции в каталоге Ozon. В отличие от штрихкода или артикула производителя (vendor code), SKU генерируется автоматически при создании карточки товара и используется исключительно внутри экосистемы маркетплейса. Его основные функции:

  • 🔄 Синхронизация остатков между вашим складом и Ozon (актуально для FBS).
  • 📦 Идентификация товара при сборке заказов на складах Ozon (FBO).
  • 🔍 Поиск и редактирование карточек в личном кабинете продавца.
  • 📑 Формирование отчетов по продажам, возвратам и штрафам.

Для покупателей SKU может потребоваться в следующих случаях:

  • 🔄 При оформлении возврата, если нужно указать точный артикул товара.
  • 📞 Для уточнения деталей заказа в поддержке Ozon.
  • 🔍 При поиске аналогичного товара (например, если вы хотите заказать ту же модель, но в другом цвете).
⚠️ Внимание: SKU на Ozon и артикул производителя — это разные вещи. Например, у смартфона Samsung Galaxy S23 может быть артикул производителя SM-S911BZKD, а SKU на маркетплейсе — 2134567890. Не путайте их при загрузке товаров!

Если вы продавец, SKU потребуется:

  • 📤 При загрузке товаров через Excel или API.
  • 📊 Для анализа продаж в отчетах (например, в Ozon Statistics).
  • 🔄 При обновлении цен или остатков.
  • ⚠️ Для оспаривания штрафов (например, если Ozon списал деньги за «несоответствие товара»).

2. Где продавцу найти SKU в личном кабинете Ozon

Самый простой способ узнать SKU — через личный кабинет продавца (Seller Cabinet). Рассмотрим два варианта: для уже загруженных товаров и для новых позиций, которые вы только планируете добавить.

2.1. SKU для существующих товаров

Если товар уже есть в вашем каталоге на Ozon, выполните следующие шаги:

  1. Авторизуйтесь в личном кабинете продавца.
  2. Перейдите в раздел Товары → Мой каталог.
  3. Найдите нужный товар через поиск или фильтры. SKU отобразится в колонке Артикул (иногда она скрыта — включите отображение через настройки таблицы).

Если колонка Артикул отсутствует:

  • 🔧 Нажмите на иконку шестеренки (Настройки таблицы) в правом верхнем углу.
  • 📋 В списке доступных колонок найдите Артикул (SKU) и перетащите её в активные.
  • 🔍 Сохраните изменения — теперь SKU будет виден для всех товаров.

Также SKU можно увидеть в карточке товара:

  1. Кликните по названию товара в списке.
  2. В открывшейся карточке SKU будет указан в блоке Основная информация (рядом с названием и штрихкодом).

2.2. SKU для новых товаров

Если вы только загружаете товар на Ozon, SKU можно:

  • 📝 Указать вручную при создании карточки (если у вас уже есть своя система артикулов).
  • 🤖 Сгенерировать автоматически — Ozon присвоит уникальный код после сохранения.

Рекомендации по формированию SKU:

  • ⚠️ Не используйте пробелы, буквы или специальные символы (только цифры).
  • 🔢 Длина SKU — от 6 до 20 символов.
  • 📌 Если у товара есть варианты (цвет, размер), добавьте суффиксы (например, 123456_black, 123456_white).
⚠️ Внимание: Если вы загружаете товары через Excel, SKU должен быть указан в колонке offer_id. Без этого файла не пройдет валидацию!

3. Как узнать SKU через API Ozon

Для продавцов, которые работают с большим количеством товаров, ручной поиск SKU неэффективен. В этом случае поможет API Ozon — интерфейс программирования, который позволяет автоматизировать получение данных. Чтобы узнать SKU через API, выполните следующие шаги:

3.1. Получение токена API

Сначала нужно получить API-key:

  1. Перейдите в Личный кабинет → Настройки → API-ключи.
  2. Нажмите Создать ключ и выберите тип доступа (например, Чтение данных о товарах).
  3. Скопируйте сгенерированный ключ — он потребуется для запросов.

3.2. Запрос списка товаров с SKU

Используйте метод POST /v2/product/info/list, чтобы получить данные о товарах, включая SKU. Пример запроса на Python:

import requests


url = "https://api-seller.ozon.ru/v2/product/info/list"


headers = {


"Client-Id": "Ваш_Client_ID",


"Api-Key": "Ваш_API_Key",


"Content-Type": "application/json"


}


payload = {


"limit": 100,


"offset": 0


}



response = requests.post(url, headers=headers, json=payload)


print(response.json())

В ответе вы получите массив товаров с полем offer_id — это и есть SKU. Пример ответа:

{

"result": {


"items": [


{


"offer_id": "1234567890",  // <-- Это SKU


"name": "Смартфон Samsung Galaxy S23",


"price": "79990"


}


]


}


}

3.3. Альтернативные методы API

Если вам нужно получить SKU для конкретного товара по его product_id (идентификатор карточки), используйте метод:

GET /v1/product/info?product_id={product_id}

Для работы с остатками (актуально для FBS) используйте метод:

POST /v1/product/info/stocks
💡

Если вы не знакомы с API, используйте готовые интеграции, например, плагин для 1С или сервис MoySklad. Они автоматически синхронизируют SKU между вашей системой учета и Ozon.

4. Как покупателю найти SKU в карточке товара

Покупателям SKU может понадобиться для возврата, уточнения деталей заказа или поиска аналогичного товара. К сожалению, Ozon не отображает SKU напрямую в публичной карточке, но есть несколько способов его узнать:

4.1. Через URL страницы товара

В некоторых случаях SKU можно извлечь из адресной строки браузера:

  1. Откройте карточку товара на Ozon.
  2. Посмотрите на URL — он может содержать параметр ?asb=... или ?sku=... (например, https://www.ozon.ru/product/...?sku=1234567890).
  3. Скопируйте цифры после sku= — это и есть артикул.

Пример URL с SKU:

https://www.ozon.ru/product/smartfon-samsung-galaxy-s23-128gb-black-160456789/?sku=1234567890
⚠️ Внимание: Этот метод работает не для всех товаров! Если в URL нет параметра sku, попробуйте другие способы.

4.2. Через источник страницы (HTML-код)

Если SKU не виден в URL, его можно найти в HTML-коде страницы:

  1. Откройте карточку товара в браузере (Chrome, Firefox и др.).
  2. Нажмите Ctrl + U (или правая кнопка мыши → Просмотр кода страницы).
  3. В открывшемся коде нажмите Ctrl + F и введите "sku" или "offerId".
  4. Найдите строку вида "offerId": "1234567890" — это и есть SKU.

Пример фрагмента кода с SKU:

<script type="application/ld+json">

{


"@context": "https://schema.org",


"@type": "Product",


"sku": "1234567890",


"name": "Смартфон Samsung Galaxy S23"


}


</script>

4.3. Через поддержку Ozon

Если самостоятельно найти SKU не удалось, обратитесь в поддержку:

  1. Перейдите в раздел «Мои заказы».
  2. Выберите нужный заказ и нажмите Нужна помощь.
  3. В чате укажите, что вам требуется SKU товара, и прикрепите скриншот карточки.

Обычно поддержка предоставляет SKU в течение 1–2 часов.

Почему Ozon скрывает SKU от покупателей?

Маркетплейс не заинтересован в том, чтобы покупатели использовали SKU для сравнения цен или поиска товаров на других площадках. Кроме того, SKU может меняться при обновлении карточки, что приведет к путанице. Однако при возврате или спорных ситуациях артикул все же может потребоваться — поэтому его можно узнать через поддержку или HTML-код.

5. Где искать SKU в отчетах Ozon

Продавцы могут получить SKU не только в личном кабинете, но и в автоматически формируемых отчетах. Это удобно, если нужно быстро найти артикулы для большого количества товаров.

5.1. Отчет «Список товаров»

Чтобы скачать отчет со всеми SKU:

  1. Перейдите в Личный кабинет → Отчеты → Товары → Список товаров.
  2. Выберите период (например, За все время).
  3. Нажмите Сформировать отчет и дождитесь письма на email.

В полученном файле Excel или CSV будет колонка offer_id — это и есть SKU.

5.2. Отчет по остаткам (для FBS)

Если вы работаете по модели FBS, SKU можно найти в отчете по остаткам:

  1. Перейдите в Личный кабинет → Отчеты → Остатки.
  2. Выберите склад (если их несколько).
  3. Скачайте отчет в формате CSV.

В файле будет колонка sku или offer_id.

5.3. Отчет по продажам

SKU также присутствует в отчетах по продажам:

  1. Перейдите в Личный кабинет → Отчеты → Продажи → Детализация по товарам.
  2. Укажите период и скачайте отчет.

В результате вы получите таблицу, где каждому товару будет соответствовать его SKU.

Отчет «Список товаров» — самый универсальный способ получить все SKU сразу. Он подходит и для FBS, и для FBO, а данные в нем обновляются ежедневно.

6. Типичные ошибки при работе с SKU на Ozon

Даже опытные продавцы иногда сталкиваются с проблемами, связанными с SKU. Рассмотрим самые распространенные ошибки и способы их избежать.

6.1. Дублирование SKU

Если вы случайно присвоите один и тот же SKU двум разным товарам, Ozon не сможет их различить. Это приведет к:

  • ❌ Ошибкам при загрузке остатков.
  • ❌ Путанице в отчетах по продажам.
  • ❌ Штрафам за несоответствие товара.

Как избежать:

  • 🔢 Используйте уникальные числовые коды (например, на основе артикула производителя + префикс).
  • 📊 Перед загрузкой проверяйте SKU на дубликаты в Excel (функция Условное форматирование → Повторяющиеся значения).

6.2. Изменение SKU существующего товара

Если вы измените SKU товара, который уже продается на Ozon, все его статистика (отзывы, продажи, остатки) будет утеряна! Маркетплейс воспримет это как новый товар, и:

  • ⭐ Рейтинг и отзывы обнулятся.
  • 📉 Товар потеряет позиции в поиске.
  • 📦 Остатки на складах Ozon (FBO) будут сброшены.

Что делать, если SKU нужно изменить:

  1. Создайте новую карточку с правильным SKU.
  2. Свяжите старую и новую карточки через поддержку (укажите, что это один и тот же товар).
  3. Дождитесь переноса отзывов и статистики (может занять до 7 дней).

6.3. SKU с буквами или символами

Ozon требует, чтобы SKU состоял только из цифр. Если вы укажете буквы или символы (например, ABC-123), возникнут ошибки:

  • ❌ Файл не загрузится через Excel.
  • ❌ API вернет ошибку валидации.

Исправить просто:

  • 🔢 Замените буквы на цифры (например, A → 1, B → 2).
  • 📌 Используйте подчеркивание (_) для разделения вариантов (например, 123456_black).

6.4. SKU не совпадает с артикулом производителя

Многие продавцы путают SKU на Ozon с артикулом бренда (например, для iPhone 15 это может быть MQ2J3). Это приводит к:

  • ❌ Ошибкам при загрузке товаров.
  • ❌ Проблемам с возвратами (если покупатель укажет артикул производителя вместо SKU).

Как различать:

  • 🏷️ Артикул производителя — указывается на упаковке товара (например, MQ2J3 для iPhone).
  • 📦 SKU Ozon — внутренний код маркетплейса (например, 1234567890).
💡

Если вы работаете с большим ассортиментом, ведите отдельную таблицу, где будут сопоставлены SKU Ozon, артикулы производителя и названия товаров. Это сэкономит время при обновлении данных.

7. SKU для FBS и FBO: в чем разница

Модель работы (FBS или FBO) влияет на то, как используется SKU. Разберем ключевые отличия.

7.1. SKU в модели FBS (продавец хранит товар)

Если вы работаете по схеме FBS (Fulfillment by Seller), SKU нужен для:

  • 📦 Синхронизации остатков между вашим складом и Ozon.
  • 🚚 Формирования этикеток для отправки заказов.
  • 📊 Отчетности по продажам и возвратам.

Особенности:

  • 🔢 Вы можете использовать собственные SKU (главное, чтобы они были уникальными).
  • 📌 При загрузке остатков через Excel SKU указывается в колонке offer_id.

7.2. SKU в модели FBO (хранение на складе Ozon)

В схеме FBO (Fulfillment by Ozon) SKU используется для:

  • 🏭 Идентификации товара на складах Ozon.
  • 📦 Сборки заказов (складские работники сканируют SKU при упаковке).
  • 🔄 Контроля остатков в реальном времени.

Особенности:

  • ⚠️ Ozon может автоматически изменить ваш SKU при приемке товара на склад.
  • 📋 SKU должен совпадать с тем, что указан на складской этикетке.
  • 🔍 При расхождениях SKU товар может быть отклонен при приемке.

8. Альтернативные способы получить SKU

Если стандартные методы не помогли, попробуйте эти варианты.

8.1. Через мобильное приложение Ozon Seller

В приложении для продавцов SKU можно найти так:

  1. Откройте приложение Ozon Seller.
  2. Перейдите в раздел Товары.
  3. Выберите нужный товар — SKU будет указан под названием.

8.2. Через партнерские сервисы

Некоторые сервисы для работы с маркетплейсами (например, MoySklad, Битрикс24, RetailCRM) автоматически синхронизируют SKU между вашей системой учета и Ozon. Чтобы найти SKU:

  1. Откройте карточку товара в вашем сервисе.
  2. Найдите поле Артикул Ozon или SKU.

8.3. Через экспорт данных из 1С

Если вы ведете учет в 1С, SKU можно экспортировать:

  1. Откройте справочник Номенклатура.
  2. Найдите колонку Артикул Ozon (если интеграция настроена).
  3. Экспортируйте данные в Excel.

Если интеграция не настроена, свяжите 1С и Ozon через:

  • 🔌 Готовые обработки (например, 1С-Ozon).
  • 🤖 API (требуются навыки программирования).
  • 📊 Партнерские сервисы (например, Атомик, Пайп).
💡

Если вы используете 1С, настройте автоматическую синхронизацию SKU с Ozon. Это избавит от ручного ввода и уменьшит риск ошибок.

FAQ: Частые вопросы о SKU на Ozon

❓ Можно ли использовать один SKU для разных товаров?

Нет! Каждый товар (и даже его варианты, например, разные цвета) должен иметь уникальный SKU. Если вы укажете одинаковый SKU для нескольких позиций, Ozon не сможет их различить, что приведет к ошибкам в остатках и заказах.

❓ Что делать, если SKU потерялся или не отображается?

Если SKU не виден в личном кабинете:

  1. Проверьте настройки таблицы (включите колонку Артикул).
  2. Скачайте отчет Список товаров — там будет колонка с SKU.
  3. Обратитесь в поддержку Ozon с указанием названия товара.
❓ Можно ли изменить SKU после загрузки товара?

Технически можно, но это приведет к потере всей статистики товара (отзывы, продажи, позиции в поиске). Если изменение необходимо:

  1. Создайте новую карточку с правильным SKU.
  2. Свяжите старую и новую карточки через поддержку.
  3. Дождитесь переноса данных (до 7 дней).
❓ Где покупателю взять SKU для возврата?

Покупатель может узнать SKU:

  • 🔍 Через HTML-код страницы (искать "offerId").
  • 📧 В письме с чеком (иногда SKU указывается в деталях заказа).
