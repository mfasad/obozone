Как посмотреть ID товара на Ozon: все способы для продавцов и покупателей в 2026 году

  • Способ 4: Получение ID через API Ozon Seller (для автоматизации)
  • Способ 5: ID в мобильном приложении Ozon (скрытые функции)
  • Типичные ошибки при работе с ID и как их избежать
  • FAQ: Частые вопросы о ID товаров на Ozon

Вы ищете уникальный идентификатор товара (ID) на маркетплейсе Ozon, но не знаете, где его найти? Этот номер — ключевой элемент для продавцов (при загрузке каталога, работе с API, оформлении возвратов) и иногда нужен покупателям (например, для уточнения характеристик или жалоб на дубликаты). В отличие от артикула производителя, ID на Ozon генерируется автоматически платформой и не дублируется даже для идентичных товаров от разных продавцов.

В этой статье мы разберём все актуальные способы получения ID в 2026 году, включая скрытые методы для мобильного приложения и массового экспорта через Excel. Особое внимание уделим нюансам для схем FBS и FBO, а также типичным ошибкам, из-за которых продавцы теряют доступ к идентификаторам после обновления каталога. Если вы работаете с дропшиппингом или интеграцией через 1С, здесь найдёте решения для автоматизации.

Что такое ID товара на Ozon и зачем он нужен

Идентификатор товара (или Product ID) — это уникальный числовой код, который Ozon присваивает каждой позиции в каталоге. Он отличается от:

  • 🔖 Артикула продавца (SKU) — внутреннего кода, который вы задаёте сами (например, IPHONE15-128-BLACK).
  • 🏷️ Штрихкода (EAN/UPC) — международного стандарта для физических товаров.
  • 🔗 Ссылки на товар — URL-адреса, который содержит ID, но не является им (https://www.ozon.ru/product/{здесь_ID}).

Без знания ID невозможно:

  • 📦 Оформить возврат или обмен через поддержку Ozon (требуют указать именно Product ID).
  • 📊 Обновить информацию о товаре через Личный кабинет продавца или API.
  • 🔄 Синхронизировать остатки и цены в системах типа МойСклад, 1С или Bitrix24.
  • 🛒 Настроить дропшиппинг с автоматическим импортом заказов.
💡

Если вы продавец и используете Ozon Seller API, ID товара нужен для всех запросов к методам /v2/product/info, /v1/stock/update и другим. Без него сервер вернёт ошибку 400 Bad Request

Важно понимать, что ID присваивается не продавцу, а конкретной карточке товара. Например, если вы продаёте один и тот же смартфон Samsung Galaxy S23 в чёрном и белом цветах, у каждой модификации будет свой уникальный идентификатор. Это касается и товаров с разными комплектациями (например, с чехлом и без).

Способ 1: Как узнать ID через карточку товара (для покупателей и продавцов)

Самый простой метод — извлечь идентификатор прямо из адресной строки браузера. Он работает и для мобильной, и для десктопной версии сайта.

Пошаговая инструкция:

  1. Откройте карточку нужного товара на Ozon.ru.
  2. Посмотрите на адресную строку браузера. Ссылка будет выглядеть так: 
    https://www.ozon.ru/product/{здесь_ваш_ID}-{название-товара}/
  3. Скопируйте числовой код перед первым дефисом. Например, в ссылке https://www.ozon.ru/product/iphone-15-pro-max-256gb-natural-titanium-123456789/ ID равен 123456789.

Если ссылка выглядит иначе (например, содержит параметры типа ?asb=... или &sh=...), очистите её от всего после знака ? — ID всегда находится сразу после /product/.

Убедитесь, что скопированный код состоит только из цифр|

Проверьте, что в ID нет дефисов или букв|

Сравните длину кода: обычно 7–9 цифр|

⚠️ Внимание: В мобильном приложении Ozon ссылки на товары не отображаются в адресной строке. Чтобы увидеть ID, нажмите «Поделиться» и выберите «Скопировать ссылку», затем вставьте её в любой текстовый редактор.

Способ 2: Поиск ID в Личном кабинете продавца (для FBS/FBO)

Если вы продавец, самый надёжный способ — использовать Личный кабинет Ozon Seller. Здесь ID отображается в нескольких разделах, включая Товары, Заказы и Отчёты.

Где именно искать:

  1. Раздел «Товары»:
    • Перейдите в Каталог → Мои товары.
    • Найдите нужную позицию в списке. ID будет в колонке Артикул Ozon (не путать с вашим SKU!).
    • Если колонки нет, нажмите «Настроить таблицу» (иконка ⚙️) и добавьте поле ID товара.
  2. Раздел «Заказы»:
    • Откройте Заказы → Все заказы.
    • Кликните по номеру заказа, затем прокрутите до блока «Товары». ID будет указан рядом с названием.

⚠️ Внимание: Если вы работаете по модели FBO (хранение на складе Ozon), ID товара может отличаться от того, что вы видите в 1С или другой учётной системе. Всегда используйте актуальный идентификатор из Личного кабинета, так как Ozon может переназначать ID при изменении карточки товара.

Способ 3: Массовый экспорт ID через Excel (для больших каталогов)

Если у вас сотни или тысячи товаров, копировать ID вручную неэффективно. В этом случае поможет экспорт каталога в Excel.

Инструкция:

  1. В Личном кабинете перейдите в Каталог → Мои товары.
  2. Нажмите кнопку Экспорт (в правом верхнем углу).
  3. Выберите формат Excel (.xlsx) и дождитесь формирования файла (может занять до 10 минут для больших каталогов).
  4. Откройте скачанный файл. ID товаров будут в колонке A (или в колонке с заголовком product_id).

В экспортированном файле также будут полезные данные:

  • 📌 sku — ваш внутренний артикул.
  • 📌 name — название товара.
  • 📌 price — текущая цена.
  • 📌 stock — остатки на складе.

⚠️ Внимание: Озон ограничивает частоту экспорта: не чаще 1 раза в 30 минут. Если вам нужно обновить данные срочно, используйте API (способ 4). Также учитывайте, что в экспорте могут отсутствовать товары, находящиеся на модерации или в статусе Неактивен.

Что делать, если в Excel-файле нет колонки с ID?

Если в экспортированном файле отсутствует колонка product_id, проверьте:

1. Формат экспорта: выберите "Полный отчёт", а не "Краткий".

2. Настройки фильтра: возможно, вы экспортировали только часть товаров (например, только активные).

3. Права доступа: у вашей учётной записи должны быть права на просмотр полных данных каталога.

Если проблема сохраняется, обратитесь в поддержку Ozon с указанием ошибки MISSING_PRODUCT_ID_COLUMN

Способ 4: Получение ID через API Ozon Seller (для автоматизации)

Для продавцов, использующих 1С, Bitrix24 или собственные CRM-системы, ручной поиск ID не подходит. В этом случае поможет API Ozon Seller — программный интерфейс для автоматизации.

Как получить ID через API:

  1. Авторизуйтесь в Личном кабинете и сгенерируйте API-ключ (раздел Настройки → API).
  2. Используйте метод /v1/product/info/list для получения списка всех ваших товаров с ID. Пример запроса: 
    curl -X POST 'https://api-seller.ozon.ru/v1/product/info/list' \

    -H 'Client-Id: {ваш_client_id}' \
    

    -H 'Api-Key: {ваш_api_key}' \
    

    -H 'Content-Type: application/json' \
    

    -d '{"limit": 1000}'

  3. В ответе сервера ищите поле "product_id" для каждого товара.

Если вам нужен ID для конкретного товара, используйте метод /v2/product/info с указанием вашего SKU: 

curl -X POST 'https://api-seller.ozon.ru/v2/product/info' \

-H 'Client-Id: {ваш_client_id}' \


-H 'Api-Key: {ваш_api_key}' \


-H 'Content-Type: application/json' \


-d '{"sku": 123456789, "product_id": []}'

⚠️ Внимание: API Ozon имеет лимиты на количество запросов:

  • 🔄 /v1/product/info/list — не более 100 запросов в минуту.
  • 🔄 /v2/product/info — не более 50 запросов в минуту.

При превышении лимита API вернёт ошибку 429 Too Many Requests. Используйте паузы между запросами или настройте кэширование данных.

💡

Для дропшиппинга и автоматической синхронизации остатков всегда используйте API, а не ручной экспорт. Это позволит избежать ошибок типа STOCK_MISMATCH (несовпадение остатков) и штрафов от Ozon.

Способ 5: ID в мобильном приложении Ozon (скрытые функции)

Официальное мобильное приложение Ozon не отображает ID товара напрямую, но есть обходные пути. Они работают как для покупателей, так и для продавцов (в приложении Ozon Seller).

Для покупателей (обычное приложение Ozon):

  1. Откройте карточку товара.
  2. Нажмите кнопку «Поделиться» (иконка ⤴️).
  3. Выберите «Скопировать ссылку».
  4. Вставьте ссылку в любой текстовый редактор (например, Блокнот) и извлеките ID из неё (как в Способе 1).

Для продавцов (приложение Ozon Seller):

  1. Перейдите в раздел «Товары».
  2. Найдите нужную позицию и нажмите на неё.
  3. Прокрутите вниз до блока «Детали товара». ID будет указан в поле Идентификатор Ozon.
  4. Если поля нет, нажмите «Ещё» (⋮) → «Поделиться» → скопируйте ссылку и извлеките ID из неё.

⚠️ Внимание: В мобильном приложении Ozon Seller ID может отображаться не для всех товаров, если:

  • 🔴 Товар находится на модерации.
  • 🔴 У вас недостаточно прав (например, вы менеджер с ограниченным доступом).
  • 🔴 Товар добавлен менее 24 часов назад (ID присваивается не мгновенно).

Типичные ошибки при работе с ID и как их избежать

Даже опытные продавцы сталкиваются с проблемами, связанными с идентификаторами товаров. Вот самые распространённые ошибки и способы их решения:

Ошибка Причина Решение
ID не найден в Личном кабинете Товар не прошёл модерацию или удалён Проверьте статус в разделе Товары → На модерации
ID в Excel не совпадает с ID в API Экспорт устарел (данные кэшируются) Обновите экспорт или используйте API для актуальных данных
Ошибка PRODUCT_NOT_FOUND при запросе по ID Товар архивирован или заблокирован Проверьте статус через метод /v1/product/info
ID дублируется для разных товаров Ошибка кэша браузера или API Очистите кэш или повторите запрос с другим Client-Id

Ещё одна частая проблема — потеря ID после обновления карточки товара. Ozon может присваивать новый идентификатор, если вы:

  • 🔄 Изменили категорию товара.
  • 🔄 Обновили характеристики (например, вес или габариты).
  • 🔄 Загрузили новое изображение с другим составом (например, добавили логотип).

Чтобы избежать потери ID:

  • 📌 Используйте частичное обновление через API (метод /v1/product/update).
  • 📌 Перед массовым обновлением экспортируйте текущие ID в резервную копию.
  • 📌 Не изменяйте категорию товара — лучше создайте новую карточку.

FAQ: Частые вопросы о ID товаров на Ozon

Можно ли использовать один и тот же ID для разных товаров?

Нет, каждый ID уникален и присваивается только одной карточке товара. Если вы создаёте дубликат (например, тот же товар в другом цвете), Ozon сгенерирует новый идентификатор. Использование одного ID для разных позиций приведёт к ошибке DUPLICATE_PRODUCT_ID и блокировке загрузки.

Что делать, если ID товара изменился после обновления?

Если Ozon переназначил ID (например, после смены категории), старый идентификатор станет неактуальным. Вам нужно:

  1. Найти новый ID через Личный кабинет или API.
  2. Обновить интеграции (1С, CRM) с новым идентификатором.
  3. Если товар был в заказах, уточнить у поддержки Ozon, нужно ли переназначать ID в истории заказов.

Чтобы избежать этого, не редактируйте критические параметры товара (категорию, бренд, основные характеристики).

Как узнать ID товара, если он удалён или заблокирован?

Если товар удалён из каталога, его ID можно найти:

  • В архивных отчётах (раздел Аналитика → Отчёты → Архив).
  • В письмах от Ozon (поищите уведомления о блокировке или удалении).
  • Через API, если товар удалён менее 30 дней назад (метод /v1/product/info/archived).

Если товар заблокирован, ID останется в Личном кабинете, но будет помечен статусом Заблокирован.

Нужно ли указывать ID при оформлении возврата?

Да, при оформлении возврата или обмена через поддержку Ozon обязательно указывать ID товара. Без него оператор не сможет идентифицировать позицию, особенно если у вас несколько похожих товаров (например, одна модель смартфона в разных цветах). ID нужен и для:

  • 🔄 Отслеживания статуса возврата.
  • 🔄 Подтверждения приёма товара на складе Ozon.
  • 🔄 Оспаривания штрафов за несоответствие возврата.

Если вы не знаете ID, поддержка может запросить ссылку на товар, но это замедлит обработку заявки.

Можно ли по ID товара узнать, кто его продавец?

Нет, ID товара сам по себе не содержит информации о продавце. Однако вы можете:

  1. Открыть карточку товара по ссылке https://www.ozon.ru/product/{ID}.
  2. Прокрутить до блока «Продавец» — там будет название магазина.
  3. Если товар продаётся по модели FBO, продавца может не быть видно (указано «Ozon»).

Для массовой проверки продавцов по ID используйте API-метод /v1/product/info с параметром "offer_id".
