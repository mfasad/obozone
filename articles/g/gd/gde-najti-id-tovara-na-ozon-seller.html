Как найти ID товара на Ozon Seller: все способы для продавцов

Работа с маркетплейсом Ozon требует от продавцов точности в управлении каталогом, и один из ключевых идентификаторов — это ID товара (или артикул Ozon). Без него невозможно редактировать карточки, синхронизировать остатки через API, настраивать рекламные кампании или обрабатывать возвраты. Однако новичкам не всегда очевидно, где именно искать этот уникальный номер, особенно если товаров сотни, а интерфейс Ozon Seller регулярно обновляется.

В этой статье мы разберём все актуальные способы получения ID товара — от ручного поиска в личном кабинете до автоматизированных методов через Excel и API. Вы узнаете, как найти артикул для отдельного товара, пачки позиций или даже для удалённых карточек. А ещё — почему иногда ID в личном кабинете и ID в API могут отличаться, и как это исправить.

1. Что такое ID товара на Ozon и зачем он нужен

ID товара (или product_id) — это уникальный числовой идентификатор, который Ozon присваивает каждой карточке в каталоге. Он не совпадает с вашим внутренним артикулом (SKU) и не меняется со временем, даже если вы редактируете название или описание товара.

Где требуется product_id:

  • 📦 Обновление остатков через API или Excel-файлы (без ID система не поймёт, какой именно товар обновлять).
  • 📈 Настройка рекламы в Ozon Рекламе — без ID нельзя выбрать товар для кампании.
  • 🔄 Обработка возвратов — в отчётах указывается именно product_id, а не название.
  • 🛠️ Массовые правки через SellerLab или другие сервисы интеграции.
⚠️ Внимание: Не путайте ID товара (product_id) с ID предложения (offer_id)! Последний присваивается конкретной модификации товара (например, размеру или цвету), а product_id — всей карточке в целом.

Если вы работаете по модели FBS, то product_id также потребуется для формирования этикеток и передачи данных логистическому оператору. В FBO он нужен для синхронизации остатков на складах Ozon.

2. Способ 1: Поиск ID в карточке товара (ручной метод)

Самый простой способ — найти артикул непосредственно в интерфейсе Ozon Seller. Подходит, если вам нужен ID для 1–2 товаров.

Инструкция:

  1. Авторизуйтесь в личном кабинете Ozon Seller.
  2. Перейдите в раздел Товары → Мои товары.
  3. Найдите нужный товар через поиск или фильтры.
  4. Кликните по названию товара, чтобы открыть его карточку.
  5. В адресной строке браузера вы увидите URL вида: 
    https://seller.ozon.ru/products/{цифры}/edit
    Цифры между /products/ и /edit — это и есть product_id.

Пример:

Если URL выглядит как https://seller.ozon.ru/products/12345678/edit, то ID товара — 12345678.

Этот метод работает и для активных, и для неактивных товаров, но не подходит для удалённых карточек (их нужно восстанавливать или искать ID в отчётах).

3. Способ 2: Экспорт ID через Excel-отчёт

Если вам нужны ID для десятков или сотен товаров, ручной метод слишком долгий. В этом случае поможет выгрузка каталога в Excel.

Как получить файл с ID:

  1. В Ozon Seller перейдите в Товары → Мои товары.
  2. Нажмите кнопку Экспорт (в правом верхнем углу).
  3. Выберите формат Excel (.xlsx) и дождитесь формирования отчёта (может занять до 10 минут).
  4. Скачайте файл и откройте его. Искомый product_id будет в колонке ID товара (или Product ID).

В отчёте также будут полезные данные:

  • 📌 Название товара — для сопоставления с вашим SKU.
  • 📦 Остатки — актуальные данные по складам.
  • 🏷️ Цена — текущая стоимость с учётом скидок.
  • ⚠️ Статус — активен ли товар.
⚠️ Внимание: В Excel-отчёте могут отсутствовать товары, которые были удалены или заблокированы Ozon. Для них используйте Способ 4 (API).

Если вам нужно обновить остатки или цены через этот файл, не забывайте сохранять его в формате .xlsx (а не .csv), иначе Ozon может не принять загрузку.

4. Способ 3: Получение ID через API Ozon

Для продавцов, которые автоматизируют работу с каталогом, самый надёжный способ — использовать API Ozon. Это позволяет получить ID не только для активных, но и для удаленных или архивных товаров.

Основные методы API для работы с ID:

Метод API Описание Пример запроса
ProductInfo Получение данных по одному товару, включая product_id. 
POST /v2/product/info

{


"product_id": 12345678,


"offer_id": ""


}
ProductsList Список всех товаров продавца с ID и основными параметрами. 
POST /v1/products/list

{


"limit": 1000


}
ProductInfoList Массовое получение данных по списку ID. 
POST /v3/product/info/list

{


"product_id": [12345678, 87654321]


}

Чтобы использовать API:

  1. Получите Client-ID и API-ключ в настройках Ozon Seller (Настройки → API).
  2. Используйте Postman или любой другой инструмент для отправки запросов.
  3. Для метода ProductsList укажите параметр "limit" (максимум 1000 товаров за запрос).

⚠️ Внимание: В API Ozon есть ограничение на количество запросов — 60 в минуту. При превышении лимита вы получите ошибку 429 Too Many Requests.

Если вы не знакомы с API, но хотите автоматизировать работу, рассмотрите сервисы-посредники вроде SellerLab или MyWare — они предоставляют удобный интерфейс для работы с данными Ozon без глубоких технических знаний.

5. Способ 4: ID в мобильном приложении Ozon Seller

Если вы часто работаете со смартфона, можно найти product_id и в мобильном приложении Ozon Seller (доступно для Android и iOS).

Пошаговая инструкция:

  1. Откройте приложение и авторизуйтесь.
  2. Перейдите в раздел Товары (значок коробки в нижнем меню).
  3. Найдите нужный товар через поиск или прокрутку.
  4. Тапните по товару, чтобы открыть его карточку.
  5. В правом верхнем углу нажмите на три точки (⋮) и выберите Поделиться.
  6. В открывшемся окне будет ссылка вида: 
    https://seller.ozon.ru/products/12345678/edit

    Цифры в ссылке — это product_id.

Преимущества мобильного способа:

  • 📱 Быстрый доступ к ID в любом месте (например, на складе при приёмке товара).
  • 🔍 Возможность сканировать штрихкоды товаров для быстрого поиска.
  • 📊 Синхронизация с десктопной версией — изменения вносятся сразу в оба интерфейса.

Недостаток: в мобильной версии нет массовой выгрузки ID, поэтому для большого каталога лучше использовать Способ 2 (Excel) или Способ 3 (API).

💡

Сохраните шаблон сообщения с ID товара в заметках смартфона — так вы сможете быстро копировать его при общении с поддержкой Ozon или логистическими службами.

6. Способ 5: Поиск ID в отчётах по продажам и возвратам

Если товар уже был продан или возвращён, его ID можно найти в соответствующих отчётах. Это особенно полезно для удаленных карточек, которые не отображаются в основном каталоге.

Где искать:

  • 📊 Отчёт по продажам (Аналитика → Продажи):

    - В колонке ID товара будет нужный идентификатор.

    - Можно фильтровать по дате или названию.

  • 🔙 Отчёт по возвратам (Возвраты → История возвратов):

    - ID указывается в деталях возврата рядом с названием товара.

  • 📦 Отчёт по остаткам (Склад → Остатки):

    - Подходит для товаров на FBO-складах.

Как экспортировать отчёт:

  1. Откройте нужный раздел (например, Аналитика → Продажи).
  2. Настройте фильтры (период, статус и т. д.).
  3. Нажмите Экспорт и выберите формат Excel.
  4. В скачанном файле найдите колонку Product ID или ID товара.
⚠️ Внимание: В отчётах по возвратам иногда указывается offer_id (ID предложения), а не product_id. Убедитесь, что копируете правильный идентификатор — он должен совпадать с тем, что вы видите в карточке товара.

Если товар был удалён, но есть история продаж, его ID останется в отчётах ещё 90 дней. После этого данные архивируются, и получить их можно только через поддержку Ozon.

7. Частые ошибки и как их избежать

При работе с ID товаров продавцы часто сталкиваются с типичными проблемами. Разберём самые распространённые:

Ошибка 1: Путаница между product_id и offer_id

  • Причина: В некоторых отчётах (например, по возвратам) указывается offer_id — идентификатор конкретного предложения (размера/цвета).
  • Решение: Всегда проверяйте, какой именно ID нужен. Для массовых операций (обновление цен, остатков) используется product_id.

Ошибка 2: ID не найден в Excel-отчёте

  • Причина: Товар может быть архивирован или удалён, поэтому не попадает в стандартную выгрузку.
  • Решение: Используйте API (метод ProductsList с параметром "visibility": "ALL") или проверьте отчёты по продажам.

Ошибка 3: Ошибка 404 при запросе по API

  • Причина: Неправильно указан product_id или отсутствуют права доступа.
  • Решение: Проверьте корректность ID и актуальность API-ключа в настройках.

Ошибка 4: Разные ID в личном кабинете и API

  • Причина: В редких случаях Ozon может присваивать разные идентификаторы для одного товара в разных системах (например, после миграции каталога).
  • Решение: Используйте тот ID, который отображается в API — он приоритетный для автоматизированных операций.
💡

Всегда проверяйте актуальность ID перед массовыми операциями. Если товар был удалён и восстановлен, его идентификатор мог измениться!

FAQ: Ответы на частые вопросы

Можно ли изменить ID товара на Ozon?

Нет, product_id присваивается автоматически при создании карточки и не подлежит редактированию. Если вам нужно "сменить" ID, придётся создать новую карточку и перенести на неё все данные (остатки, отзывы и т. д.) вручную.

Где найти ID для товара, который ещё не загружен на Ozon?

У товаров, которые ещё не прошли модерацию или находятся в черновиках, нет product_id. Он появляется только после публикации карточки. Для черновиков используйте внутренний SKU.

Как узнать ID товара конкурента?

Получение product_id чужих товаров запрещено правилами Ozon. Однако вы можете увидеть его в URL публичной карточки товара (например, https://www.ozon.ru/product/{ID}). Но использовать этот ID для своих операций нельзя.

Что делать, если ID товара не отображается в Excel-отчёте?

Проверьте настройки экспорта: иногда колонка ID товара скрыта. Также убедитесь, что товар не архивирован. Если проблема остаётся, запросите данные через API или обратитесь в поддержку Ozon.

Можно ли восстановить ID удалённого товара?

Да, если с момента удаления прошло не более 90 дней, ID будет доступен в отчётах по продажам или через API. После этого срока данные архивируются, и для их восстановления потребуется помощь службы поддержки.
