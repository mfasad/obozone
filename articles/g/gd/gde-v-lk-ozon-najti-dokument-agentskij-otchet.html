Почему продавцы не могут найти агентский отчёт — и где его действительно искать

Агентский отчёт — это ключевой финансовый документ для продавцов на Ozon, который подтверждает обороты, комиссии маркетплейса и итоговую сумму к выплате. Несмотря на его важность, многие партнёры тратят часы на поиски этого файла в лабиринте меню личного кабинета. Проблема усугубляется тем, что интерфейс Озон регулярно обновляется, а старые инструкции (даже 2023 года) уже неактуальны.

В этой статье мы разберём все актуальные на 2026 год способы доступа к агентскому отчёту, включая скрытые пути для FBS и FBO, нюансы формирования документов за прошлые периоды и типичные ошибки, из-за которых отчёт может «пропадать» из системы. Вы узнаете, как скачать файл в формате Excel или PDF, почему иногда данные в нём не совпадают с реальными выплатами — и что с этим делать.

Предупреждаем сразу: если вы работаете через Ozon Global или используете Ozon Рекламу как отдельный инструмент, алгоритм поиска отчёта будет другим. В этой инструкции речь идёт исключительно о стандартном агентском отчёте для продавцов на схеме FBS/FBO в российском сегменте маркетплейса.

Способ 1: Стандартный путь через «Финансы» — для большинства продавцов

Это основной и самый надёжный метод, который работает для 90% партнёров. Отчёт формируется автоматически после закрытия расчётного периода (обычно 5-е число месяца за предыдущий период). Вот пошаговая инструкция:

  1. Авторизуйтесь в личном кабинете продавца и перейдите в раздел Финансы (значок кошелька в левом меню).

  2. В подменю выберите пункт Отчёты → Агентские отчёты.

  3. Откроется таблица со всеми сгенерированными документами. По умолчанию они сортируются по дате от новых к старым.

  4. Найдите нужный период (например, «Агентский отчёт за январь 2026») и нажмите на кнопку Скачать в правой части строки.

Файл будет доступен в формате .xlsx (для Excel) или .pdf. Обратите внимание: если кнопка Скачать неактивна (серого цвета), значит отчёт ещё не сформирован. В этом случае проверьте статус в колонке Статус — там может быть указано «В обработке» или «Ошибка формирования».

Важный нюанс: в агентском отчёте отображаются только завершённые заказы. Если покупатель оформил возврат или отменил заказ после оплаты, эти суммы будут учтены в следующих периодах. Это частая причина расхождений между отчётом и реальными выплатами.

Способ 2: Альтернативный маршрут через «Аналитика» — для FBS-продавцов

Если стандартный путь не сработал (например, из-за технических сбоев), попробуйте найти отчёт через раздел Аналитика. Этот метод особенно полезен для продавцов на схеме FBS, так как здесь отображаются дополнительные данные по логистике.

Инструкция:

  • 📊 Перейдите в Аналитика → Финансовая аналитика.
  • 📅 В фильтрах выберите нужный период (месяц/квартал).
  • 📄 В блоке «Отчёты» найдите вкладку Агентские отчёты (может называться «Финансовые отчёты» в новых версиях интерфейса).
  • 🔍 Используйте поиск по дате или номеру отчёта, если список слишком длинный.

Здесь же можно скачать расширенную версию отчёта с детализацией по каждому заказу — это полезно для бухгалтерии или аудита. Однако учтите, что такой файл может весить 100+ МБ, если у вас высокий оборот.

Предупреждение: в разделе Аналитика иногда отображаются предварительные данные, которые могут отличаться от финального агентского отчёта. Всегда сверяйте цифры с документом из раздела Финансы.

Способ 3: Через API или экспорт данных — для продвинутых пользователей

Если вам нужно автоматизировать получение отчётов (например, для интеграции с 1С или другими системами учёта), можно использовать Ozon Seller API. Этот метод требует технических навыков, но позволяет получать данные в структурированном виде без ручного скачивания.

Основные шаги:

  1. Получите доступ к API Озон для продавцов (нужен токен авторизации).

  2. Используйте метод GET /v1/finance/transaction/totals для получения финансовых данных.

  3. Для детализации по заказам подходит метод POST /v2/finance/transaction/list.

Пример запроса для получения агентского отчёта за март 2026:

curl -X POST 'https://api-seller.ozon.ru/v2/finance/transaction/list' \

-H 'Client-Id: YOUR_CLIENT_ID' \


-H 'Api-Key: YOUR_API_KEY' \


-H 'Content-Type: application/json' \


-d '{


"date": {


"from": "2026-03-01T00:00:00Z",


"to": "2026-03-31T23:59:59Z"


},


"operation_type": ["agent_commission"],


"limit": 1000


}'

Обратите внимание: через API вы получите сырые данные, которые нужно будет самостоятельно преобразовать в отчётный формат. Для этого можно использовать Python-скрипты или инструменты вроде Power Query в Excel.

Какие данные можно выгрузить через API?

Через Ozon Seller API доступны следующие финансовые данные:

- Транзакции по заказам (суммы, комиссии, возвраты)

- Логистические расходы (доставка, хранение на складах FBS)

- Выплаты и удержания (включая штрафы и бонусы)

- Данные по рекламным кампаниям (если подключены)

Однако готового агентского отчёта в PDF/Excel через API получить нельзя — только сырые данные для самостоятельной обработки.

Типичные ошибки и как их избежать

Даже опытные продавцы сталкиваются с проблемами при работе с агентскими отчётами. Вот самые распространённые ошибки и способы их решения:

  • ❌ Отчёт не формируется → Проверьте, закрыт ли расчётный период (например, за январь отчёт появится не раньше 5 февраля). Если дата прошла, а отчёта нет, обратитесь в поддержку с указанием ID вашего аккаунта.
  • ❌ Данные в отчёте не совпадают с выплатами → Сверьте даты: выплаты могут задерживаться на 1–3 дня из-за банковских обработок. Также проверьте раздел Удержания в личном кабинете — там могут быть штрафы или компенсации.
  • ❌ Нельзя скачать отчёт за прошлый год → Озон хранит отчёты только за последние 12 месяцев. Для более старых периодов запросите архив в поддержку.
  • ❌ Файл повреждён при скачивании → Попробуйте открыть его через Google Sheets или пересохраните в другом формате. Если не помогает, запросите повторную генерацию отчёта.

Если вам нужно сравнить данные из агентского отчёта с выпиской из банка, экспортируйте оба файла в CSV и используйте функцию ВПР (VLOOKUP) в Excel для автоматической сверки сумм.

Особое внимание уделите колонке Тип операции в отчёте. Там могут быть не только продажи, но и:

  • 💰 Возвраты (отрицательные суммы)
  • 📦 Логистические услуги (доставка, хранение)
  • 📢 Рекламные расходы (если используете Ozon Рекламу)
  • ⚠️ Штрафы (за нарушение правил маркетплейса)

Что делать, если отчёт потерялся или не совпадает с реальностью

Ситуации, когда агентский отчёт «исчезает» или содержит ошибочные данные, встречаются чаще, чем хотелось бы. Вот алгоритм действий для таких случаев:

⚠️ Внимание: Если разница между отчётом и фактическими выплатами превышает 10 000 ₽, немедленно обратитесь в поддержку с детализацией расхождений. В письме укажите:
  • Номер отчёта и период
  • Сумму по отчёту и сумму на счёте
  • Скриншоты выписки из банка

Без этих данных вашу заявку могут проигнорировать.

Шаги для разрешения конфликта:

  1. Проверьте раздел Удержания в личном кабинете (путь: Финансы → Удержания). Там могут быть неучтённые штрафы или компенсации покупателям.

  2. Сверьте данные с отчётом по Ozon Рекламе (если используете). Рекламные расходы иногда дублируются в основном отчёте.

  3. Экспортируйте детализацию по заказам за тот же период (раздел Заказы → История заказов) и сравните суммы вручную.

  4. Если ошибка подтвердилась, создайте тикет в поддержку с темой: «Расхождения в агентском отчёте №[номер] за [период]».

Срок рассмотрения таких обращений — до 5 рабочих дней. Если ответ не пришёл, напишите повторно или позвоните на горячую линию (номер для продавцов: 8 800 333-70-00, выбор опции «Финансовые вопросы»).

💡

Озон не несёт ответственности за ошибки в агентских отчётах, если вы не сообщили о них в течение 30 дней с даты формирования документа. После этого срока исправить данные будет практически невозможно.

FAQ: Частые вопросы про агентские отчёты на Озон

Можно ли получить агентский отчёт за период до регистрации на Озон?

Нет, отчёты формируются только с момента подключения вашего аккаунта к маркетплейсу. Если вам нужны данные за прошлые периоды (например, для налоговой), запросите выписку по счёту в банке, где были привязаны выплаты от Озон.

Почему в отчёте нет данных по некоторым заказам?

В агентский отчёт попадают только завершённые и оплаченные заказы. Если покупатель отменил заказ, вернул товар или оплата ещё не прошла (например, при рассрочке), эти суммы будут учтены в следующих периодах. Также проверьте фильтры по датам — возможно, заказы попали в другой отчётный месяц.

Как скачать отчёт за текущий месяц, если он ещё не сформирован?

Текущие данные можно посмотреть в разделе Аналитика → Финансовая аналитика, но это будут предварительные цифры. Финальный агентский отчёт появится только после закрытия периода (обычно 5-го числа следующего месяца). Для срочного анализа экспортируйте данные по заказам вручную.

Можно ли интегрировать агентские отчёты с 1С или другими системами?

Да, но для этого потребуется:

  1. Настроить Ozon Seller API (нужен программист).
  2. Создать обработчик, который будет преобразовывать JSON-данные в формат, совместимый с вашей системой учёта.
  3. Автоматизировать выгрузку (например, через cron-задачи или Zapier).

Готовых решений для 1С нет, но можно использовать отраслевые решения для маркетплейсов.

Что делать, если в отчёте указаны штрафы, которых не должно быть?

Штрафы в агентском отчёте появляются за нарушение правил Озон (например, за просрочку отгрузки, некачественное описание товара или высокий процент возвратов). Чтобы оспорить штраф:

  1. Найдите его в разделе Финансы → Удержания и кликните на номер операции.
  2. Проверьте причину (она указана в колонке «Комментарий»).
  3. Если штраф начислен ошибочно, обратитесь в поддержку с доказательствами (скриншотами заказов, перепиской с покупателем и т.д.).

Срок оспаривания — 14 дней с даты начисления.
