Где бухгалтеру посмотреть отгруженные на склад Озон товары: все способы

Контроль отгрузок на склад Ozon — критически важная задача для бухгалтера, особенно если компания работает по схеме FBO (отгрузка со своего склада) или FBS (хранение на площадке маркетплейса). Без точных данных о движении товаров невозможно корректно закрыть период, сверить остатки с фактическими поставками или подтвердить расходы для налогового учёта. Проблема в том, что информация об отгрузках разбросна: часть данных есть в личном кабинете Ozon Seller, часть — в системах логистики, а некоторые сведения приходится запрашивать у менеджеров склада.

В этой статье разберём все доступные способы проверки отгруженных товаров — от стандартных отчётов в кабинете продавца до скрытых функций API и интеграций с 1С. Особое внимание уделим нюансам, которые часто упускают: например, как отличить фактическую отгрузку от запланированной, или почему данные в отчёте "Отгрузки" могут не совпадать с данными кладовщиков. Также приведём готовые шаблоны запросов для бухгалтерских программ и предупредим о типичных ошибках, из-за которых сходятся остатки по учету и реальные запасы на складе Ozon.

1. Личный кабинет Ozon Seller: где искать данные об отгрузках

Основной источник информации для бухгалтера — раздел Логистика → Отгрузки в личном кабинете Ozon Seller. Здесь отображаются все созданные вами или логистической службой Ozon отгрузки, но важно понимать: не все из них являются фактически выполненными. Например, отгрузка может иметь статус "Создана" или "В обработке", но товар ещё не поступил на склад.

Чтобы увидеть только завершённые отгрузки, используйте фильтры:

  • 📌 Статус: выберите "Завершена" или "Принята на складе".
  • 📅 Дата: укажите период, за который нужен отчёт (например, текущий месяц для закрытия периода).
  • 📦 Тип отгрузки: FBO (со своего склада) или FBS (на склад Ozon).

В карточке каждой отгрузки доступны:

  • 📄 Номер отгрузки (нужен для сверки с накладными).
  • 📋 Список товаров с количеством и артикулами.
  • 🚛 Дата и время приёмки на складе Ozon.
  • 💰 Стоимость логистики (если отгрузка по FBS).
💡

Если в списке отгрузок нет нужной позиции, проверьте фильтр по складу назначения — иногда отгрузки разделяются по региональным хабам Ozon (Москва, Екатеринбург, Новосибирск и др.).

Обратите внимание: в личном кабинете отображаются только отгрузки, созданные через интерфейс Ozon Seller. Если ваша компания использует API или интеграцию с 1С, часть данных может отсутствовать. В этом случае переходите к следующим способам.

2. Отчёты Ozon: какой выбрать для сверки с бухгалтерией

Ozon предоставляет несколько типов отчётов, но для бухгалтера наиболее полезны два:

  1. "Отчёт по отгрузкам" — содержит детализированную информацию о каждом товаре в отгрузке, включая артикулы, количество и статусы.
  2. "Отчёт по остаткам" — показывает текущие запасы на складах Ozon, что помогает сверить фактические остатки с бухгалтерскими данными.

Чтобы скачать отчёт:

  1. Перейдите в Аналитика → Отчёты.
  2. Выберите нужный тип отчёта.
  3. Укажите период (например, с 1 по 31 число месяца).
  4. Нажмите "Сформировать" и дождитесь письма на email с файлом .xlsx или .csv.

В отчёте "По отгрузкам" обратите внимание на колонки:

Колонка Что означает Зачем нужна бухгалтеру

sku Артикул товара Сопоставление с номенклатурой в 1С
quantity Количество отгруженных единиц Контроль объёмов поставок
warehouse_id Код склада Ozon Разделение по региональным хабам
status Статус отгрузки Фильтрация только завершённых поставок

3. API Ozon: как автоматизировать получение данных

Если ваша компания отгружает большие объёмы товаров, ручная выгрузка отчётов становится неэффективной. В этом случае стоит настроить интеграцию через API Ozon. Для бухгалтера наиболее полезны два метода:

  • 🔄 POST /v2/analytics/data — запрос данных по отгрузкам в формате JSON.
  • 📊 POST /v1/analytics/stock_on_warehouses — остатки товаров на складах.

Пример запроса для получения списка отгрузок за месяц:

{

"date_from": "2026-05-01T00:00:00Z",


"date_to": "2026-05-31T23:59:59Z",


"metrics": ["order_id", "sku", "quantity", "warehouse_id"],


"dimensions": [],


"filters": [


{"key": "status", "value": "delivered"}


]


}

Для работы с API потребуется:

  1. Получить Client-ID и API-Key в настройках личного кабинета (Настройки → API).
  2. Использовать инструменты вроде Postman или написать скрипт на Python.
  3. Настроить автоматическую выгрузку данных в 1С или Google Sheets.
Как проверить корректность API-ключа?

Если при запросе вы получаете ошибку "401 Unauthorized", убедитесь, что:

- Ключ скопирован без пробелов.

- У указанного Client-ID есть права на доступ к аналитике.

- Запрос отправляется с заголовком Authorization: Bearer {API-Key}.

Преимущество API — возможность получать данные в реальном времени и автоматически сверять их с бухгалтерским учётом. Однако для этого потребуется помощь программиста или настройка готовых коннекторов (например, Ozon API Connector для 1С).

4. Интеграция с 1С: как синхронизировать отгрузки

Для компаний, использующих 1С:Управление торговлей или 1С:ERP, оптимальный способ контроля отгрузок — настройка обмена данными с Ozon. Это позволяет:

  • 🔄 Автоматически создавать документы "Реализация товаров" при отгрузке на Ozon.
  • 📈 Сверять остатки в 1С с фактическими запасами на складах маркетплейса.
  • 💰 Формировать проводки по расходам на логистику (FBS).

Для настройки интеграции:

  1. Установите обработку "Обмен с Ozon" (доступна в 1С:Каталоге решений).
  2. Введите Client-ID и API-Key в настройках обмена.
  3. Настройте расписание синхронизации (рекомендуется ежедневное обновление).

После настройки в 1С будут автоматически создаваться:

  • 📄 Заказы покупателей (для FBO).
  • 🚚 Документы отгрузки (для FBS).
  • 📦 Акты приёмки (с датой и временем поступления на склад Ozon).

Сверьте количество товаров в документе "Отгрузка" в 1С с данными отчёта "По отгрузкам" в Ozon.

Проверьте, что артикулы (sku) в 1С совпадают с артикулами на Ozon.

Убедитесь, что дата отгрузки в 1С соответствует дате в личном кабинете Ozon.

Если после настройки интеграции данные не совпадают, проверьте:

  • 🔄 Часовой пояс: Ozon работает по UTC+0, а 1С может использовать местное время.
  • 📌 Формат артикулов: в 1С и Ozon должны использоваться одинаковые sku.
  • 🔄 Периодичность обмена: при редкой синхронизации данные могут устаревать.

5. Альтернативные способы: когда стандартные методы не работают

Иногда бухгалтеру требуются данные, которых нет в стандартных отчётах или API. В таких случаях поможет:

  • 📧 Обращение в поддержку Ozon: через чат в личном кабинете или на почту seller-support@ozon.ru. В запросе укажите:
    • Номера отгрузок (order_id).
    • Даты, за которые нужны данные.
    • Причину запроса (например, "сверка с бухгалтерским учётом").
  • 📊 Выгрузка из WMS-системы: если ваш склад использует WMS (например, 1С:WMS или MegaWMS), запросите отчёт по отгрузкам на Ozon у логистов.
  • 📱 Мобильное приложение Ozon Seller: в нём доступны уведомления о статусах отгрузок (полезно для оперативного контроля).

Если вы работаете по схеме FBS и нужны данные о списаниях товаров (например, из-за истечения срока годности или брака), запросите у Ozon "Акт списания". Этот документ требуется для бухгалтерского учёта убытков.

💡

При запросе данных у поддержки Ozon всегда указывайте конкретные даты и номера отгрузок — это ускорит обработку запроса в 2–3 раза.

Также полезно вести внутренний реестр отгрузок в Excel или Google Sheets, куда логисты или менеджеры вносят:

  • 📅 Дату и время отгрузки.
  • 📦 Номера накладных.
  • 🚛 Данные о транспортной компании (для FBO).
  • ✅ Статус приёмки на складе Ozon.

6. Типичные ошибки бухгалтеров при работе с отгрузками Ozon

Даже опытные бухгалтеры сталкиваются с проблемами при учёте отгрузок на Ozon. Вот наиболее распространённые ошибки и как их избежать:

  • 🔴 Сверка по дате создания отгрузки, а не по дате приёмки:

    В бухгалтерском учёте расходы и доходы фиксируются по факту перехода права собственности. Для FBS это момент приёмки товара на складе Ozon, а не дата создания отгрузки в личном кабинете.

  • 🔴 Игнорирование возвратов и списаний:

    Если товар был возвращён с склада Ozon или списан (например, из-за брака), это должно быть отражено в учёте. Без этих данных остатки в 1С и на складе разойдутся.

  • 🔴 Неучёт логистических расходов по FBS:

    При отгрузке по схеме FBS Ozon списывает плату за хранение и обработку товаров. Эти расходы должны учитываться в статье "Логистика" или "Услуги маркетплейсов".

  • 🔴 Разные артикулы в 1С и Ozon:

    Если в 1С и на Ozon используются разные sku, автоматическая сверка становится невозможной. Решение — настроить сопоставление артикулов в интеграции.

1. Остатки товаров в отчёте "Stock on warehouses" с данными 1С.

2. Суммы логистических расходов в отчёте "Финансы → Транзакции" с проводками в бухгалтерии.-->

Ещё одна типичная проблема — задержки в обновлении статусов отгрузок. Например, товар может быть физически принят на складе Ozon, но в личном кабинете статус ещё не обновился. В таких случаях:

  • Проверьте отчёт "По движениям остатков" — там данные обновляются чаще.
  • Свяжитесь с поддержкой Ozon и уточните статус по номеру отгрузки.

Для FBO вместо акта приёмки-передачи оформляется товарная накладная (ТОРГ-12) на покупателя. В ней указываются:

  • 📌 Данные покупателя (ФИО, адрес доставки).
  • 📦 Перечень товаров с артикулами и количеством.
  • 💰 Стоимость товаров и доставки (если оплачена покупателем).

При оформлении документов для Ozon обязательно указывайте номер отгрузки (order_id) — это упростит сверку с данными маркетплейса.

FAQ: Частые вопросы бухгалтеров об отгрузках на Ozon

Как отличить отгрузку по FBS от FBO в отчётах?

В отчёте "По отгрузкам" обратите внимание на колонку warehouse_id:

  • Если warehouse_id начинается с "WB" (например, WB-MSK-1) — это FBS (отгрузка на склад Ozon).
  • Если warehouse_id отсутствует или указан адрес покупателя — это FBO (отгрузка напрямую клиенту).

Также в личном кабинете Ozon Seller отгрузки FBS помечены значком склада (🏭), а FBO — значком грузовика (🚛).

Что делать, если в отчёте Ozon нет данных об отгрузке?

Причины отсутствия данных могут быть следующими:

  1. Отгрузка создана, но ещё не обработана логистикой Ozon (статус "В обработке").
  2. Отгрузка создана через API или стороннюю систему, и данные не синхронизированы с личным кабинетом.
  3. Ошибка при формировании отчёта (например, неверно указан период).

Решения:

  • Проверьте статус отгрузки в разделе Логистика → Отгрузки.
  • Сверьте данные с WMS-системой вашего склада.
  • Обратитесь в поддержку Ozon с номером отгрузки (order_id).
Как учесть логистические расходы по FBS?

Расходы на логистику по схеме FBS отражаются в отчёте "Финансы → Транзакции" (тип операции "Логистика"). Для бухгалтерского учёта:

  1. Скачайте отчёт за нужный период.
  2. Отфильтруйте транзакции по типу "Хранение", "Приёмка", "Комиссия за обработку".
  3. Сформируйте в 1С документ "Поступление услуг" от поставщика ООО "Озон" (ИНН 7704280371).
  4. Укажите статью затрат (например, "Услуги маркетплейсов") и сумму из отчёта.

Обратите внимание: Ozon списывает плату за хранение ежемесячно, а комиссии за обработку — по факту отгрузки.

Можно ли получить данные об отгрузках за прошлые периоды?

Да, но с ограничениями:

  • В личном кабинете Ozon Seller доступны отчёты за последние 12 месяцев.
  • Через API можно запрашивать данные за любой период, но Ozon хранит историю не более 2 лет.
  • Для более старых данных придётся обращаться в поддержку с обоснованием (например, для налоговой проверки).

Если вам нужны данные за период старше 2 лет, проверьте архивы email (на письма с отчётами) или внутренние бухгалтерские регистры.

Как сверить остатки товаров на складе Ozon с данными 1С?

Для сверки выполните следующие шаги:

  1. Скачайте отчёт "Stock on warehouses" в личном кабинете Ozon Seller.
  2. Экспортируйте данные об остатках из 1С (отчёт "Остатки товаров").
  3. Сопоставьте остатки по артикулам (sku).
  4. Если есть расхождения, проверьте:
    • Отгрузки, которые ещё не приняты на складе Ozon (статус "В пути").
    • Возвраты и списания товаров (должны быть отражены в 1С).
    • Ошибки в сопоставлении артикулов между 1С и Ozon.

Для удобства используйте сводную таблицу в Excel с формулами VLOOKUP или XLOOKUP для автоматической сверки.
