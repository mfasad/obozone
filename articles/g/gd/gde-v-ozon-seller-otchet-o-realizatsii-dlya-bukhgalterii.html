Отчёт о реализации в Ozon Селлер: где найти и как выгрузить для бухгалтерии

Работа с отчётностью на Ozon — одна из самых сложных задач для продавцов, особенно когда речь идёт о передаче данных в бухгалтерию. Отчёт о реализации (или отчёт по продажам) нужен для корректного учёта выручки, расчёта налогов и сверки с 1С, но найти его в личном кабинете Ozon Seller не так просто. В зависимости от схемы работы (FBS или FBO), формата выгрузки (Excel, CSV, XML) и периода отчётности — пути доступа к документам различаются.

В этой статье разберём, где именно в Ozon Seller лежат отчёты о реализации, как их правильно выгрузить для бухгалтерии, какие нюансы учитывать при работе с FBS и FBO, а также как автоматизировать процесс, чтобы не тратить часы на ручную обработку. Особое внимание уделим актуальным изменениям 2026 года — например, новым разделам в кабинете продавца и требованиям налоговой к электронным документам.

Что такое отчёт о реализации и зачем он нужен бухгалтерии

Отчёт о реализации — это детализированный документ, который содержит информацию о всех продажах на Ozon за выбранный период. В отличие от стандартных выписок по поступлениям, он включает:

  • 📦 Номера заказов и даты продаж;
  • 💰 Суммы выручки (включая скидки, промокоды, возмещения);
  • 📊 Статусы заказов (выполнен, отменён, возвращён);
  • 📄 Данные о комиссиях Ozon и логистических расходах;
  • 🏷️ Артикулы и наименования товаров.

Без этого отчёта бухгалтерия не сможет:

  1. Правильно отразить выручку в 1С или другой учётной системе;
  2. Рассчитать НДС (если вы плательщик) или налог по УСН;
  3. Сверить данные с банковскими выписками и актами выполненных работ от Ozon;
  4. Подготовить декларации для налоговой (например, по форме 6-НДФЛ или декларации по НДС).
⚠️ Внимание: С 2026 года Ozon обязательно требует подтверждение доходов для продавцов на FBS через электронные документы. Если вы не предоставите отчёт о реализации при запросе, маркетплейс может заблокировать выплаты.

Где в Ozon Seller находится отчёт о реализации: пошаговая инструкция

Путь к отчёту зависит от того, по какой схеме вы работаете: FBO (самостоятельная доставка) или FBS (доставка через Ozon). Ниже — актуальные маршруты на 2026 год.

Для продавцов на FBS

Если вы работаете по модели FBS, отчёт о реализации скрыт в разделе Финансы → Отчёты → Отчёт по реализации FBS. Алгоритм действий:

  1. Авторизуйтесь в Ozon Seller;
  2. Перейдите в меню Финансы (значок кошелька в левом сайдбаре);
  3. Выберите вкладку Отчёты;
  4. Найдите блок Отчёт по реализации FBS и нажмите Сформировать;
  5. Укажите период (максимум — 3 месяца за один запрос) и формат (Excel или CSV);
  6. Подтвердите запрос кнопкой Скачать.

Отчёт сформируется не мгновенно — обычно требуется от 5 до 30 минут. Статус готовности можно отследить в разделе Мои отчёты.

Для продавцов на FBO

Продавцам на FBO нужно искать отчёт в другом месте: Финансы → Отчёты → Отчёт по реализации FBO. Здесь есть важный нюанс:

  • 📌 Данные обновляются с задержкой до 24 часов;
  • 📌 В отчёте не отображаются заказы, которые ещё не переданы курьеру;
  • 📌 Для корректной выгрузки в 1С лучше выбирать формат XML.

Если отчёт не формируется или содержит ошибки, проверьте:

  • 🔹 Правильно ли указан период (нельзя запрашивать данные за будущие даты);
  • 🔹 Нет ли ограничений по вашему тарифу (например, на FBO Старт отчёты доступны только за последние 30 дней);
  • 🔹 Не блокированы ли финансовые операции из-за несоблюдения требований Ozon.

Какие данные содержатся в отчёте о реализации: разбор структуры

Отчёт о реализации — это таблица с десятками колонок, и не все из них нужны бухгалтерии. Ниже разберём ключевые поля, на которые стоит обратить внимание, и те, что можно игнорировать.

Название поля Описание Нужно ли бухгалтерии?
order_id Уникальный номер заказа ✅ Да (для сверки)
posting_number Номер отправления (для FBS) ❌ Нет (если не ведёте учёт по логистике)
product_name Наименование товара ✅ Да (для анализа ассортимента)
price Цена товара без скидок ✅ Да (для расчёта выручки)
commission Комиссия Ozon ✅ Да (для учёта расходов)

Особое внимание уделите колонкам с финансовыми данными:

  • 💵 total_discount_value — сумма скидок по промокодам;
  • 🔄 return_amount — сумма возвратов;
  • 📉 cancel_amount — сумма отменённых заказов.

Важно: В отчёте по FBS комиссия Ozon уже вычтена из суммы заказа, а в FBO — нет. Это критично для бухгалтерии, так как влияет на расчёт налоговой базы.

Как выгрузить отчёт для 1С: инструкция по преобразованию данных

Бухгалтеры часто требуют отчёт в формате, совместимом с 1С:Бухгалтерия или 1С:Управление торговлей. К сожалению, стандартная выгрузка из Ozon Seller не всегда подходит — приходится дорабатывать файлы.

Способ 1: Ручная обработка в Excel

Если у вас небольшой объём заказов (до 1000 строк), можно обойтись без автоматизации:

  1. Скачайте отчёт в формате Excel;
  2. Удалите лишние колонки (например, warehouse_name, shipping_method);
  3. Добавьте столбцы для НДС (если нужно) и статуса оплаты;
  4. Сохраните файл в формате CSV с разделителем ;;
  5. Импортируйте в 1С через Администрирование → Обмен данными → Загрузка из Excel.

Способ 2: Автоматическая выгрузка через API

Для крупных продавцов (от 5000 заказов в месяц) ручная обработка неэффективна. В этом случае используйте:

  • 🤖 Ozon API (документация — api-seller.ozon.ru);
  • 🔗 Готовые интеграции (например, МойСклад, Атол Онлайн);
  • 📊 Сервисы-коннекторы (AltoBI, RetailCRM).

Пример запроса к API для получения отчёта:

GET https://api-seller.ozon.ru/v3/finance/transaction/totals

Headers:


- Client-Id: [ваш_client_id]


- Api-Key: [ваш_api_key]


Body:


{


"date": {


"from": "2026-01-01T00:00:00Z",


"to": "2026-01-31T23:59:59Z"


}


}

⚠️ Внимание: При использовании API учитывайте лимиты запросов — Ozon блокирует аккаунты за превышение (более 1000 запросов в час). Для массовой выгрузки используйте пакетную обработку.
💡

Если бухгалтерия требует подписанные электронные документы, запросите в поддержке Ozon акт выполненных работ (АВР) в формате XML с ЭЦП. Его можно прикрепить к отчёту о реализации для налоговой.

Типичные ошибки при работе с отчётами и как их избежать

Даже опытные продавцы сталкиваются с проблемами при выгрузке отчётов. Вот самые распространённые ошибки и способы их решения:

  • 🔴 Отсутствуют данные за прошлый месяц → Проверьте, не истёк ли срок хранения отчётов (в Ozon он составляет 12 месяцев);
  • 🔴 Суммы в отчёте не сходятся с выплатами → Учтите задержки по FBS (деньги поступают на 3-5 дней позже продажи);
  • 🔴 Некорректный НДС → Настройте в Ozon Seller правильный статус налогоплательщика (Настройки → Налоги);
  • 🔴 Отчёт не открывается в Excel → Попробуйте импортировать через LibreOffice Calc или конвертировать в CSV.

Если данные в отчёте явно неверные (например, пропущены заказы), обратитесь в поддержку Ozon с указанием:

  • 📅 Периода, за который сформирован отчёт;
  • 📊 Примеров расхождений (номера заказов, суммы);
  • 📧 Скриншота ошибки.

Ответ от поддержки обычно приходит в течение 1-3 рабочих дней, но в пиковые периоды (например, после Чёрной пятницы) срок может увеличиться до недели.

Что делать, если Ozon не формирует отчёт?

Если кнопка "Сформировать" неактивна или отчёт не генерируется более суток, проверьте:

1. Нет ли задолженности перед Ozon (например, по комиссиям или штрафам).

2. Не превышен ли лимит на выгрузку (для новых аккаунтов — не более 5 отчётов в день).

3. Не блокирован ли доступ к финансовым данным (проверьте в разделе "Безопасность").

Если проблема не решена, напишите в чат поддержки с темой "Ошибка формирования отчёта по реализации" и приложите скриншот ошибки.

Как автоматизировать получение отчётов: обзор сервисов и скриптов

Ручная выгрузка отчётов отнимает время, особенно если вы работаете на нескольких маркетплейсах. Рассмотрим инструменты для автоматизации:

Сервис Функции Стоимость
AltoBI Автоматическая выгрузка отчётов в 1С, аналитика продаж От 1500 ₽/мес
RetailCRM Интеграция с Ozon, формирование УПД, актов От 2900 ₽/мес
МойСклад Синхронизация заказов, учёт товаров, отчёты для бухгалтерии От 990 ₽/мес
Самописный скрипт (Python) Выгрузка через API, конвертация в нужный формат Бесплатно (нужны навыки программирования)

Пример простого скрипта на Python для выгрузки отчёта через API:

import requests

import pandas as pd



url = "https://api-seller.ozon.ru/v3/finance/transaction/totals"


headers = {


"Client-Id": "YOUR_CLIENT_ID",


"Api-Key": "YOUR_API_KEY"


}


params = {


"date": {


"from": "2026-01-01T00:00:00Z",


"to": "2026-01-31T23:59:59Z"


}


}



response = requests.get(url, headers=headers, json=params)


data = response.json()



Конвертация в Excel


df = pd.DataFrame(data["result"])


df.to_excel("ozon_realization_report.xlsx", index=False)

Для работы скрипта потребуется библиотека pandas (устанавливается командой pip install pandas). Готовый файл можно сразу передавать бухгалтерии.

💡

Автоматизация выгрузки отчётов экономит до 10 часов в месяц и снижает риск ошибок при ручном вводе данных.

Частые вопросы по отчётам о реализации на Ozon

Можно ли получить отчёт о реализации за весь год?

Нет, Ozon Seller позволяет выгружать данные только за период до 3 месяцев. Для годового отчёта придётся скачивать файлы помесячно и объединять их вручную или через скрипт.

Чем отличается отчёт по реализации от выписки по поступлениям?

Выписка по поступлениям показывает только суммы, переведённые на ваш счёт, а отчёт по реализации содержит детализацию по каждому заказу, включая возвраты, скидки и комиссии. Для бухгалтерии нужен именно отчёт о реализации.

Как в отчёте учитываются возвраты и отмены?

Возвраты и отмены отображаются со знаком минус в колонках return_amount и cancel_amount. Их нужно учитывать при расчёте чистой выручки. Если возвратов много, проверьте колонку return_reason — она поможет выявить проблемные товары.

Можно ли в отчёте увидеть данные по промокодам и акциям?

Да, информация о скидках содержится в колонках total_discount_value (сумма скидки) и promo_code (использованный промокод). Эти данные нужны для анализа эффективности маркетинговых кампаний.

Что делать, если в отчёте не хватает заказов?

Сначала проверьте, не фильтруете ли вы данные по статусу (например, отменённые заказы могут быть скрыты). Если заказы действительно отсутствуют, обратитесь в поддержку Ozon с указанием номеров заказов и периода. Чаще всего проблема связана с задержкой обновления данных в FBS.
