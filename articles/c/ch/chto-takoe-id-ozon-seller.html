ID Ozon Seller: где найти, зачем нужен и как использовать в 2026 году

Если вы продавец на маркетплейсе Ozon или только планируете начать торговлю, рано или поздно столкнётесь с термином ID Ozon Seller. Этот уникальный идентификатор присваивается каждому продавцу при регистрации и используется во всех внутренних процессах платформы — от интеграции с API до настройки рекламных кампаний. Без него невозможно подключить некоторые сервисы, подтвердить права на аккаунт или автоматизировать работу с заказами.

Также вы узнаете, чем client_id отличается от seller_id, как восстановить доступ к идентификатору при блокировке аккаунта и какие инструменты требуют обязательного указания этого параметра.

Материал будет полезен как новичкам, которые только регистрируются на Ozon, так и опытным продавцам, столкнувшимся с необходимостью интеграции сторонних сервисов или настройки автоматизации. Все инструкции актуальны на июнь 2026 года и учитывают последние обновления платформы.

Что такое ID Ozon Seller и зачем он нужен

ID продавца (Seller ID) — это уникальный числовой или буквенно-цифровой код, который Ozon присваивает каждому зарегистрированному продавцу. Он служит внутренним идентификатором вашего аккаунта в системе и используется для:

  • 🔗 Интеграции с API — без seller_id невозможно отправлять запросы к серверу Ozon для автоматизации загрузки товаров, управления заказами или получения аналитики.
  • 📊 Настройки рекламы — идентификатор требуется для создания кампаний в Ozon Advertising, особенно при работе через партнёрские сервисы.
  • 🔄 Подключения сторонних сервисов — многие CRM, системы аналитики (Google Analytics, Яндекс Метрика) и программы для дропшиппинга запрашивают seller_id для синхронизации.
  • 📄 Официальных обращений — при обращении в поддержку Ozon или оформлении юридических документов (например, договора на FBS) может потребоваться указать ваш идентификатор.

Важно понимать, что seller_id — это не то же самое, что client_id (используется для авторизации в API) или номер магазина (отображается в карточке продавца для покупателей). Это отдельный технический параметр, который не меняется со временем и не зависит от названия вашего бренда или юридического лица.

На практике ID продавца может выглядеть как:

  • Чисто числовой код: 12345678
  • Буквенно-цифровая комбинация: SELLER-1A2B3C4D
  • Код с префиксом: ozon_seller_987654

Где найти ID продавца в личном кабинете Ozon (пошаговая инструкция)

Найти свой ID Ozon Seller можно несколькими способами — через веб-версию личного кабинета, мобильное приложение или API. Ниже приведён актуальный алгоритм для 2026 года с учётом последних обновлений интерфейса.

Способ 1: Через веб-версию личного кабинета (новый дизайн)

Если вы используете обновлённую версию Ozon Seller (введённую в 2023–2026 годах), следуйте этим шагам:

  1. Авторизуйтесь на сайте seller.ozon.ru под своим логином.
  2. В правом верхнем углу кликните на иконку профиля (аватар или первую букву имени).
  3. В выпадающем меню выберите пункт «Настройки аккаунта».
  4. На открывшейся странице перейдите во вкладку «Общая информация».
  5. Прокрутите страницу вниз до блока «Идентификаторы» — здесь будет строка ID продавца (Seller ID).

Открыть seller.ozon.ru|Кликнуть на иконку профиля|Выбрать "Настройки аккаунта"|Перейти в "Общая информация"|Найти блок "Идентификаторы"-->

Если у вас старый интерфейс (до 2023 года), путь будет другим:

  1. Зайдите в раздел «Профиль» (в левом боковом меню).
  2. Выберите подпункт «Информация о продавце».
  3. В блоке «Основные данные» найдите строку ID продавца.

Обратите внимание: в некоторых случаях Ozon может скрывать идентификатор под кнопкой «Показать» — это сделано для безопасности. Нажмите на неё, чтобы увидеть полный код.

Способ 2: Через мобильное приложение Ozon Seller

Если вы работаете с мобильного устройства:

  1. Откройте приложение Ozon Seller (доступно для iOS и Android).
  2. Тапните на иконку профиля в правом нижнем углу.
  3. Выберите «Настройки» → «Профиль продавца».
  4. Прокрутите до блока «Идентификаторы» — здесь будет ваш Seller ID.
💡

Если в мобильном приложении нет блока с идентификаторами, обновите его до последней версии в App Store или Google Play. Старые версии (до 5.0) могут не отображать эту информацию.

Способ 3: Через API (для технических специалистов)

Если вам нужен seller_id для интеграции, его можно получить программно через API-запрос:

GET https://api-seller.ozon.ru/v1/analystic/data

Headers:


- Client-Id: [ваш_client_id]


- Api-Key: [ваш_api_key]

В ответе сервера будет поле "seller_id". Этот метод подходит, если у вас уже настроен доступ к API.

Чем Seller ID отличается от Client ID и других идентификаторов

На Ozon используется несколько типов идентификаторов, и их часто путают. Разберём ключевые различия:

Тип идентификатора Назначение Где используется Пример формата
Seller ID Уникальный код продавца в системе Ozon API, рекламные кампании, интеграции с CRM 12345678 или SELLER-AB12CD34
Client ID Идентификатор для авторизации в API Запросы к API, OAuth-авторизация 789456123
API Key Секретный ключ для доступа к API Заголовки API-запросов a1b2c3d4e5f6... (32 символа)
Номер магазина Публичный идентификатор для покупателей Карточка продавца, отзывы Магазин #987654

Критическое отличие: Seller ID нельзя использовать вместо Client ID в API-запросах — это приведёт к ошибке авторизации 403 Forbidden. Эти идентификаторы связаны, но служат разным целям.

Например, при настройке интеграции с 1С или МойСклад вам потребуется:

  • 🔑 seller_id — чтобы система поняла, какой аккаунт синхронизировать.
  • 🔐 client_id + api_key — чтобы подтвердить права на доступ к данным.
💡

Если вам нужно передать данные партнёру (например, агентству для настройки рекламы), никогда не отправляйте api_key — только seller_id и client_id. Ключ API должен храниться в секрете!

Когда требуется указывать Seller ID: реальные кейсы использования

Далеко не все продавцы сталкиваются с необходимостью использовать ID Ozon Seller на начальном этапе. Однако есть ситуации, где без него не обойтись:

1. Подключение к API Ozon

Если вы автоматизируете работу с заказами, ценами или остатками через API, seller_id потребуется:

  • 📥 Для получения списка заказов (/v2/posting/fbs/list).
  • 📦 Для обновления статусов отправлений (/v1/posting/fbs/ship).
  • 💰 Для управления ценами (/v1/product/import/prices).

Пример запроса с указанием seller_id:

POST https://api-seller.ozon.ru/v2/posting/fbs/list

Headers:


Client-Id: 12345678


Api-Key: your_api_key_here


Body:


{


"seller_id": 87654321,


"since": "2026-06-01T00:00:00Z"


}

2. Настройка рекламных кампаний

При создании кампаний в Ozon Advertising через партнёрские сервисы (например, K50, AdMit) или собственные скрипты, seller_id нужен для:

  • 🎯 Привязки кампании к вашему аккаунту.
  • 📈 Получения статистики по рекламе.
  • 💸 Управления бюджетом.

Без корректного идентификатора система не сможет определить, чьи товары продвигать, и вернёт ошибку вида "Seller not found".

3. Интеграция с CRM и системами аналитики

Сервисы вроде RetailCRM, Bitrix24 или Google Analytics 4 запрашивают seller_id для:

  • 📊 Сбора данных о продажах.
  • 🔄 Синхронизации заказов.
  • 📌 Сегментации клиентов.

Например, в Google Analytics 4 при настройке потока данных для Ozon потребуется ввести:

Параметр: ozon_seller_id

Значение: 12345678

4. Обращение в поддержку Ozon

В некоторых случаях при обращении в службу поддержки (особенно по техническим вопросам) менеджеры могут запросить ваш seller_id для:

  • 🔍 Быстрого поиска вашего аккаунта в системе.
  • 🛠️ Диагностики проблем с API или интеграциями.
  • 📄 Оформления официальных запросов (например, на разблокировку товара).

Частые ошибки при работе с Seller ID и как их избежать

Даже опытные продавцы иногда сталкиваются с проблемами, связанными с ID Ozon Seller. Разберём типичные ошибки и способы их решения.

Ошибка 1: "Invalid seller_id" в API

Если при отправке запроса к API вы получаете ответ:

{

"error": {


"code": "VALIDATION_ERROR",


"message": "Invalid seller_id"


}


}

Причины могут быть следующими:

  • ❌ Вы указали client_id вместо seller_id.
  • ❌ Используете идентификатор от другого аккаунта (например, тестового).
  • ❌ В коде лишние пробелы или символы (например, "12345678 " вместо "12345678").

Решение:

  1. Проверьте, что вы скопировали seller_id без лишних символов.
  2. Убедитесь, что используете идентификатор того аккаунта, к которому подключаетесь.
  3. Если проблема сохраняется, запросите новый api_key — иногда ошибка связана с истёкшими ключами.

Ошибка 2: Не вижу Seller ID в личном кабинете

Если блок с идентификаторами отсутствует в настройках:

  • 🔄 Обновите страницу (F5) или очистите кэш браузера.
  • 📱 Проверьте, что вы используете полную версию сайта (не мобильную).
  • 🔒 Убедитесь, что у вашего пользователя есть права «Администратор» в аккаунте.
  • 📧 Обратитесь в поддержку Ozon с просьбой предоставить идентификатор (укажите email, привязанный к аккаунту).
💡

Если вы работаете через партнёрский аккаунт (например, агентство), ваш seller_id может отличаться от основного. Уточните его у менеджера, который вас подключал.

Ошибка 3: Изменился Seller ID после ребрендинга или смены юридического лица

При смене названия магазина, реквизитов или переходе на другой тип аккаунта (например, с ИП на ООО) Ozon может присвоить новый идентификатор. Это приведёт к:

  • ⚠️ Ошибкам в интеграциях.
  • ⚠️ Потере статистики в аналитических системах.
  • ⚠️ Необходимости перенастраивать рекламные кампании.

Решение:

  1. Запросите новый seller_id в поддержке Ozon.
  2. Обновите идентификатор во всех интеграциях и сервисах.
  3. Если используете API, получите новые client_id и api_key.
💡

При смене юридического лица Ozon может заблокировать старый seller_id на 30 дней. Заранее подготовьте резервные копии данных и предупредите партнёров о смене идентификатора.

Ошибка 4: Утечка Seller ID и риски безопасности

ID продавца сам по себе не даёт доступ к вашему аккаунту, но в сочетании с другими данными (например, api_key) может быть использован злоумышленниками для:

  • 📉 Кражи данных о заказах.
  • 💸 Мошенничества с рекламным бюджетом.
  • 📦 Несанкционированного изменения цен или остатков.

Рекомендации по безопасности:

  • 🔒 Никому не передавайте api_key — только seller_id и client_id.
  • 🔄 Регулярно обновляйте API-ключи (раз в 3–6 месяцев).
  • 🛡️ Используйте двухфакторную авторизацию в личном кабинете.
Что делать, если ваш Seller ID попал в руки мошенников?

1. Немедленно смените api_key в настройках API.

2. Проверьте историю изменений в личном кабинете (раздел "Журнал действий").

3. Обратитесь в поддержку Ozon с просьбой временно заблокировать старый идентификатор.

4. Если заметили подозрительные заказы или изменения цен — приостановите продажи до разрешения ситуации.

Как восстановить доступ к Seller ID при блокировке аккаунта

Если ваш аккаунт на Ozon заблокирован (например, за нарушение правил или по подозрению в мошенничестве), доступ к seller_id также будет ограничен. В этом случае:

  1. Уточните причину блокировки — письмо с объяснением должно прийти на email, привязанный к аккаунту. Типичные причины:
    • 📦 Нарушение правил размещения товаров (например, продажа запрещённых категорий).
    • 💬 Негативные отзывы или высокий уровень возвратов.
    • 📄 Проблемы с документами (несоответствие реквизитов).
  2. Подготовьте документы для разблокировки:
    • Сканы уставных документов (для ООО) или паспорта (для ИП).
    • Объяснительную записку с планом исправления нарушений.
    • Доказательства легальности товара (сертификаты, договоры с поставщиками).
  • Обратитесь в поддержку через форму обратной связи в личном кабинете или по email seller-support@ozon.ru. В письме укажите:
    • Номер заблокированного аккаунта (если известен).
    • Причину блокировки (из письма от Ozon).
    • Контактный телефон для связи.

    После разблокировки ваш ID Ozon Seller останется прежним, если не было смены юридического лица. В противном случае вам присвоят новый идентификатор.

    💡

    Если блокировка связана с технической ошибкой (например, сбоем в системе), укажите в обращении фразу: "Прошу проверить аккаунт на предмет ложного срабатывания антифрод-системы". Это ускорит рассмотрение запроса.

    Альтернативные способы получить Seller ID (если стандартные не работают)

    Если вы не можете найти seller_id через личный кабинет, попробуйте эти методы:

    Способ 1: Через письмо от Ozon

    При регистрации на платформе Ozon отправляет письмо с приветствием на email, указанный при создании аккаунта. В нём может содержаться ваш идентификатор. Ищите письма с темой:

    • «Добро пожаловать на Ozon!»
    • «Ваш аккаунт продавца активирован»
    • «Подтверждение регистрации на Ozon Seller»

    Используйте поиск по ключевым словам: seller_id, ID продавца, идентификатор.

    Способ 2: Через URL адреса в личном кабинете

    В некоторых версиях интерфейса seller_id отображается в адресной строке браузера. Например:

    https://seller.ozon.ru/app/analytics/12345678/dashboard

    где 12345678 — ваш идентификатор.

    Проверьте URL на страницах:

    • Аналитика (/analytics/...).
    • Заказы (/orders/...).
    • Товары (/products/...).

    Способ 3: Через партнёрские сервисы

    Если вы подключали сторонние сервисы (например, МойСклад, 1С, K50), ваш seller_id мог сохраниться в их настройках. Проверьте:

    • Раздел «Интеграции» или «Подключения» в сервисе.
    • Логи (журналы) синхронизации — иногда идентификатор отображается в ошибках.
    • Настройки рекламных кампаний (если используете AdMit, SellerLab и т.п.).

    Способ 4: Через техническую поддержку

    Если все методы не сработали, отправьте запрос в поддержку Ozon с просьбой предоставить seller_id. Используйте шаблон:

    FAQ: Частые вопросы о Seller ID на Ozon

    Может ли у одного продавца быть несколько Seller ID?

    Да, если у вас несколько аккаунтов на Ozon (например, для разных юридических лиц или брендов). Каждый аккаунт имеет уникальный идентификатор. Однако внутри одного аккаунта seller_id только один.

    Меняется ли Seller ID при смене названия магазина?

    Нет, идентификатор остаётся прежним при изменении названия, логотипа или описания магазина. Он привязан к аккаунту, а не к визуальным данным.

    Можно ли использовать один Seller ID для нескольких маркетплейсов (Ozon, Wildberries, Яндекс Маркет)?

    Нет, seller_id уникален для каждой платформы. На Wildberries у вас будет свой идентификатор (supplierId), на Яндекс Маркете — campaignId.

    Что делать, если при регистрации не пришло письмо с Seller ID?

    Проверьте папку «Спам» в почте. Если письма нет, обратитесь в поддержку Ozon с темой «Не получен Seller ID после регистрации» и укажите email, на который регистрировались.

    Может ли Seller ID содержать буквы или только цифры?

    Формат зависит от типа аккаунта. Обычно это:

    • Чисто цифровой код для большинства продавцов (например, 12345678).
    • Буквенно-цифровая комбинация для некоторых партнёрских программ (например, SELLER-AB12CD34).

    Если ваш вопрос не освещён в статье, проверьте официальную документацию Ozon или обратитесь в службу поддержки продавцов по телефону 8 800 333-70-00 (доб. 1 для продавцов).
