Где найти SKU ID в Ozon FBO: 5 проверенных способов для продавцов

Продавцы на Ozon, работающие по модели FBO (Fulfillment by Ozon), часто сталкиваются с необходимостью найти SKU ID — уникальный артикул товара в системе маркетплейса. Без этого идентификатора невозможно отследить остатки, обновить цены, синхронизировать данные через API или исправить ошибки в карточке. Проблема в том, что SKU ID в Ozon не всегда отображается на видном месте, а его расположение зависит от способа загрузки товара, типа аккаунта и даже версии личного кабинета.

Многие продавцы путают SKU ID с другими идентификаторами: product_id (ID карточки товара), offer_id (предложение в рамках карточки) или артикулом поставщика. Это приводит к ошибкам при работе с API, некорректным отчётам и даже блокировкам товаров. В этой статье мы разберём все актуальные способы поиска SKU ID в 2026 году — от ручного просмотра в личном кабинете до автоматического парсинга через Excel и API.

Особенность Ozon FBO в том, что продавец сам управляет логистикой, но при этом должен соблюдать строгие требования маркетплейса к маркировке и учёту товаров. SKU ID здесь играет ключевую роль: он связывает ваш физический товар с виртуальной карточкой на площадке. Без него невозможно корректно сформировать штрихкоды для этикеток, загрузить остатки или отменить заказ. Если вы только начинаете работать с FBO или переходите с FBS, эта инструкция поможет избежать типичных ошибок.

1. Что такое SKU ID в Ozon и зачем он нужен продавцу FBO

SKU ID (Stock Keeping Unit) — это уникальный цифровой или буквенно-цифровой код, который Ozon присваивает каждому товару в вашем ассортименте. В отличие от product_id (идентификатор карточки), который одинаков для всех продавцов одного и того же товара, SKU ID уникален для каждого предложения конкретного продавца.

Для продавцов FBO этот идентификатор критичен по нескольким причинам:

  • 📦 Маркировка товаров: SKU ID используется для генерации штрихкодов на этикетках, которые вы клеите на коробки перед отправкой на склад Ozon.
  • 📊 Учёт остатков: В отчётах по остаткам и движениям товаров указывается именно SKU ID, а не название или артикул поставщика.
  • 🔄 Синхронизация через API: При работе с Ozon API для обновления цен, остатков или статусов заказов требуется передавать SKU ID.
  • ⚠️ Разрешение споров: При возвратах или претензиях со стороны покупателей Ozon запрашивает SKU ID для идентификации товара.

Важно понимать, что SKU ID не совпадает с:

  • 🔢 Артикулом поставщика (ваш внутренний код товара).
  • 🆔 product_id — идентификатором карточки товара на Ozon (например, 123456789).
  • 📌 offer_id — идентификатором предложения (если у товара несколько вариантов, например, размеров или цветов).
⚠️ Внимание: Если вы загружаете товары на Ozon через Excel или API, но не указываете SKU ID явным образом, система автоматически генерирует его. Однако такой подход чреват дублями и ошибками при синхронизации. Всегда присваивайте SKU ID вручную по единой системе (например, BRAND-MODEL-COLOR-SIZE).

2. Где посмотреть SKU ID в личном кабинете Ozon (пошаговая инструкция)

Самый простой способ найти SKU ID — через интерфейс личного кабинета Ozon Seller. Однако расположение этого идентификатора зависит от того, как вы загружали товар: вручную, через Excel или API.

Способ 1. Просмотр в карточке товара

  1. Перейдите в раздел Товары → Мои товары.
  2. Найдите нужный товар в списке и кликните по его названию.
  3. В открывшейся карточке перейдите на вкладку Остатки и цены.
  4. В таблице с остатками найдите колонку SKU (или Артикул продавца, если вы его задавали при загрузке).

Способ 2. Экспорт списка товаров в Excel

  1. В разделе Товары → Мои товары нажмите кнопку Экспорт (в правом верхнем углу).
  2. Выберите формат Excel и дождитесь формирования файла.
  3. В полученной таблице найдите колонку sku — там будут указаны все SKU ID ваших товаров.

Способ 3. Через отчёт по остаткам

  1. Перейдите в Аналитика → Отчёты → Остатки товаров.
  2. Скачайте отчёт в формате Excel или CSV.
  3. В колонке sku или seller_sku будут указаны нужные идентификаторы.

Убедитесь, что товар опубликован (не в черновиках)

Проверьте, что у товара есть остатки (иначе он может не отображаться в отчётах)

Используйте фильтр по названию или артикулу для быстрого поиска

Сравните SKU ID в личном кабинете и Excel — они должны совпадать-->

Если вы не видите колонку sku в отчётах, возможно, она скрыта. В этом случае:

  1. Откройте скачанный Excel-файл.
  2. Выделите первую строку (заголовки колонок).
  3. Нажмите правой кнопкой и выберите Отменить скрытие (если колонка была скрыта).
  4. Или воспользуйтесь поиском по документу (Ctrl + F) и введите sku.
⚠️ Внимание: В некоторых версиях личного кабинета Ozon колонка со SKU ID может называться Артикул продавца или Код товара. Если вы не уверены, сверьтесь с примером: SKU ID обычно представляет собой комбинацию букв и цифр (например, ABC123-XL-BLACK), тогда как product_id — это всегда цифровой код (например, 123456789).

3. Как найти SKU ID в отчётах Ozon (Excel, CSV, API)

Если вам нужно получить SKU ID для большого количества товаров, ручной просмотр в личном кабинете будет неэффективен. В этом случае поможет экспорт отчётов или работа с Ozon API.

Отчёт «Список товаров»

Этот отчёт содержит полный перечень ваших товаров с всеми идентификаторами, включая SKU ID. Чтобы его получить:

  1. Перейдите в Аналитика → Отчёты → Список товаров.
  2. Настройте фильтры (например, по статусу Опубликован).
  3. Нажмите Сформировать отчёт и скачайте его в формате Excel.

В полученном файле ищите колонки:

  • 📌 sku — ваш SKU ID.
  • 📌 product_id — идентификатор карточки товара на Ozon.
  • 📌 offer_id — идентификатор предложения (если у товара есть вариации).

Отчёт «Движение товаров»

Этот отчёт полезен, если вам нужно связать SKU ID с конкретными заказами или возвратами. Он содержит информацию о всех движениях товаров на складах Ozon, включая:

  • 📦 Приходы (поступление товаров от вас).
  • 🛒 Отгрузки (проданные товары).
  • 🔄 Возвраты (от покупателей или с сортировочных центров).

Чтобы скачать отчёт:

  1. Перейдите в Аналитика → Отчёты → Движение товаров.
  2. Укажите период (например, последний месяц).
  3. Экспортируйте данные в Excel.

В этом отчёте SKU ID будет в колонке sku, а рядом — данные о количестве и типе операции.

API-запросы для получения SKU ID

Если вы используете Ozon API для автоматизации, SKU ID можно получить через следующие эндпоинты:

  • 📡 /v2/products/info — информация о товаре (включает SKU ID).
  • 📡 /v1/analytics/stock_on_warehouses — остатки на складах (с указанием SKU).
  • 📡 /v1/product/list — список всех ваших товаров.

Пример запроса для получения SKU ID через API: 

GET https://api-seller.ozon.ru/v2/products/info

Headers:


- Client-Id: YOUR_CLIENT_ID


- Api-Key: YOUR_API_KEY


Body:


{


"product_id": [123456789], // ID карточки товара


"offer_id": ["your_offer_id"] // Если есть вариации


}

В ответе вы получите JSON с полем "sku", где будет указан нужный идентификатор.

💡

Если вы часто работаете с API, сохраните шаблон запроса в Postman или аналогичном инструменте. Это сэкономит время при повторных проверках SKU ID.

4. Как SKU ID связан с штрихкодами и маркировкой в FBO

В модели FBO продавец самостоятельно маркирует товары перед отправкой на склад Ozon. Здесь SKU ID играет ключевую роль: на его основе генерируются штрихкоды, которые сканируются на всех этапах логистики — от приёмки до отгрузки покупателю.

Как формируется штрихкод по SKU ID

Штрихкод для Ozon FBO состоит из:

  1. Префикса (например, 20 для FBO или 22 для FBS).
  2. SKU ID (ваш уникальный артикул).
  3. Контрольной суммы (рассчитывается автоматически).

Пример:

Если ваш SKU ID — MYBRAND-TSHIRT-L, то штрихкод может выглядеть так:

20MYBRAND-TSHIRT-L[контрольная сумма]

Где взять штрихкоды для печати

Вы можете сгенерировать штрихкоды двумя способами:

  1. Через личный кабинет Ozon:
    • Перейдите в Товары → Мои товары.
    • Выберите нужные товары и нажмите Сгенерировать штрихкоды.
    • Скачайте PDF-файл с готовыми этикетками.
  • С помощью сторонних сервисов:
    • Используйте генераторы штрихkodов (например, Barcode.tec-it или Online Barcode Generator).
    • Введите SKU ID в формате 20[YOUR_SKU].
    • Выберите тип штрихкода Code 128 (рекомендуемый Ozon).

    5. Частые ошибки с SKU ID и как их избежать

    Работа с SKU ID в Ozon FBO часто сопровождается ошибками, которые ведут к проблемам с учётом товаров, блокировкам или штрафам. Вот самые распространённые из них и способы их решения:

    Ошибка 1: Дублирование SKU ID

    Если вы присвоили один и тот же SKU ID разным товарам, Ozon не сможет корректно учитывать остатки. Это приводит к:

    • ❌ Ошибкам при приёмке товаров на складе.
    • ❌ Неправильному списанию остатков при продажах.
    • ❌ Блокировке товаров из-за несоответствия данных.

    Решение:

    • Проверьте все SKU ID в отчёте Список товаров на дубли.
    • Используйте уникальные префиксы для разных категорий (например, TSHIRT-, PHONE-).
    • Если дубли уже есть, измените SKU ID через API илиExcel и обновите штрихкоды.

    Ошибка 2: SKU ID не совпадает со штрихкодом

    Если на этикетке указан один SKU ID, а в системе Ozon — другой, товар не пройдёт сканирование на складе. Это приводит к:

    • ❌ Задержкам при приёмке.
    • ❌ Потере товаров на складе.
    • ❌ Штрафам за несоблюдение требований маркировки.

    Решение:

    • Сверьте SKU ID в личном кабинете и на этикетках.
    • Перепечатайте штрихкоды, если обнаружено несоответствие.
    • Используйте генераторы штрихkodов с проверкой контрольной суммы.

    Ошибка 3: Отсутствует SKU ID в отчётах

    Иногда в отчётах Движение товаров или Остатки колонка sku пустая. Это может произойти из-за:

    • 🔄 Ошибки при экспорте данных.
    • 📥 Некорректной загрузки товаров через API.
    • 🛠 Технических работ на стороне Ozon.

    Решение:

    • Проверьте, что товар опубликован и имеет остатки.
    • Экспортируйте отчёт повторно через 10–15 минут.
    • Обратитесь в поддержку Ozon, если проблема сохраняется.
    Что делать, если SKU ID потерян для старого товара?

    Если вы не можете найти SKU ID для товара, загруженного давно, попробуйте:

    1. Проверьте архивные отчёты (например, за прошлый год).

    2. Используйте API-запрос `/v2/products/info` с указанием `product_id`.

    3. Свяжитесь с поддержкой Ozon и предоставьте `product_id` — они могут восстановить SKU ID.

    4. Если товар больше не продаётся, удалите его и создайте заново с новым SKU ID.

    Ошибка 4: SKU ID содержит запрещённые символы

    Ozon накладывает ограничения на формат SKU ID. Запрещено использовать:

    • 🚫 Пробелы (заменяйте на _ или -).
    • 🚫 Спецсимволы (@, #, $, % и др.).
    • 🚫 Кириллицу (только латиница и цифры).
    • 🚫 Слишком длинные коды (максимум 40 символов).

    Решение:

    Если ваш SKU ID не соответствует требованиям, измените его через:

    1. Личный кабинет (Товары → Редактировать → Артикул продавца).
    2. Excel-файл с последующей выгрузкой.
    3. API-запрос /v1/product/import.
    💡

    Всегда проверяйте SKU ID на соответствие требованиям Ozon ДО загрузки товаров. Это избавит от проблем с маркировкой и учётом на складах.

    6. Как изменить или обновить SKU ID в Ozon

    Если вам нужно изменить SKU ID (например, из-за ошибки или ребрендинга), сделать это можно несколькими способами. Однако важно помнить: изменение SKU ID может повлиять на учёт остатков и активные заказы. Рекомендуем выполнять эту операцию в период минимальных продаж.

    Способ 1: Редактирование в личном кабинете

    1. Перейдите в Товары → Мои товары.
    2. Найдите нужный товар и нажмите Редактировать.
    3. В поле Артикул продавца (или SKU) введите новый идентификатор.
    4. Сохраните изменения.

    Ограничения:

    • ❌ Нельзя изменить SKU ID, если по товару есть активные заказы.
    • ❌ Новый SKU ID должен быть уникальным.

    Способ 2: Массовое изменение через Excel

    1. Экспортируйте список товаров в Excel (раздел Товары → Экспорт).
    2. В колонке sku замените старые идентификаторы на новые.
    3. Сохраните файл и импортируйте его обратно через Товары → Импорт.

    Важно:

    • 📌 Не изменяйте другие колонки (например, product_id), иначе товар будет считаться новым.
    • 📌 После импорта проверьте, что SKU ID обновились в отчёте Список товаров.

    Способ 3: Обновление через API

    Для изменения SKU ID через API используйте эндпоинт: 

    POST https://api-seller.ozon.ru/v1/product/import

    Headers:
    

    - Client-Id: YOUR_CLIENT_ID
    

    - Api-Key: YOUR_API_KEY
    

    Body:
    

    {
    

    "items": [
    

    {
    

    "offer_id": "your_offer_id",
    

    "product_id": 123456789,
    

    "sku": "NEW_SKU_ID" // Новый идентификатор
    

    }
    

    ]
    

    }

    Что делать после изменения SKU ID

    После обновления идентификатора необходимо:

    1. Перепечатать штрихкоды с новым SKU ID.
    2. Обновить остатки в системе (если они сбились).
    3. Проверьте, что новый SKU ID отображается в отчётах Движение товаров и Остатки.
    ⚠️ Внимание: Если вы изменили SKU ID для товара, который уже находится на складе Ozon, свяжитесь с поддержкой, чтобы они обновили данные в своей системе. В противном случае товар может быть потерян или списан некорректно.

    7. SKU ID в FBO vs FBS: ключевые различия

    Продавцы, работающие и по FBO, и по FBS, часто путают требования к SKU ID в этих моделях. Хотя принцип уникальной идентификации остаётся тем же, есть важные различия:

    Параметр FBO (самостоятельная логистика) FBS (логистика через Ozon)
    Кто присваивает SKU ID Продавец Продавец или Ozon (если не указан)
    Требования к формату Строгие (нет пробелов, спецсимволов) Менее строгие, но рекомендуется соблюдать те же правила
    Использование в штрихкодах Обязательно (префикс 20) Обязательно (префикс 22)
    Где искать SKU ID Личный кабинет, отчёты по остаткам Личный кабинет, отчёты по заказам
    Что будет при ошибке Товар не пройдёт приёмку на складе Товар может быть потерян или списан некорректно

    Что общего:

    • 🔹 SKU ID должен быть уникальным в рамках вашего аккаунта.
    • 🔹 Используется для учёта остатков и синхронизации с API.
    • 🔹 Может содержать буквы и цифры (без кириллицы).

    Чем отличается:

    • 📦 В FBO вы сами маркируете товары, поэтому SKU ID критичен для штрихkodов.
    • 🚚 В FBS Ozon может сгенерировать SKU ID автоматически, если вы его не указали.
    • 📊 В отчётах FBO SKU ID чаще встречается в разделах по остаткам, а в FBS — в заказах и возвратах.

    Если вы переходите с FBS на FBO, обязательно проверьте все SKU ID на соответствие требованиям. Часто продавцы сталкиваются с тем, что автоматически сгенерированные в FBS идентификаторы не подходят для FBO (например, содержат пробелы или кириллицу).
