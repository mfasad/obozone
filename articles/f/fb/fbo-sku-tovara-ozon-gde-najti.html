Где найти SKU товара в FBO на Озон: подробное руководство 2026

Работа с моделью FBO (Fulfillment by Ozon) на маркетплейсе требует точного учета каждого товара — и здесь на первый план выходит SKU (Stock Keeping Unit). Этот уникальный идентификатор не только упрощает логистику, но и служит ключом к аналитике продаж, возвратам и даже спорным ситуациям с покупателями. Однако многие продавцы сталкиваются с проблемой: где вообще найти SKU в системе Озон, если товар уже загружен по схеме FBO?

Путаницу усугубляет то, что на платформе одновременно используются артикул продавца, штрихкод, ID товара в каталоге Озон и собственно SKU. При этом последний может отличаться в зависимости от того, работаете вы через личный кабинет, API или мобильное приложение. В этой статье разберем все актуальные способы поиска SKU для FBO, включая скрытые нюансы, о которых не пишут в официальной документации.

Особое внимание уделим различиям между SKU в FBS и FBO — многие продавцы ошибочно ищут код в неверном разделе личного кабинета. Также раскроем, почему SKU может «исчезать» из интерфейса после обновлений Озон и как восстановить доступ к нему через альтернативные методы.

Что такое SKU в FBO и чем он отличается от артикула

SKU (от англ. Stock Keeping Unit) — это внутренний идентификатор товара в системе Озон, который присваивается автоматически при загрузке карточки. В отличие от артикула продавца (который вы указываете сами), SKU генерируется маркетплейсом и используется для:

  • 📦 Отслеживания остатков на складах FBO
  • 🔄 Обработки возвратов и обменов
  • 📊 Формирования отчетов по продажам
  • 🚛 Маркировки грузов при приемке на склад

Важное отличие: один и тот же товар может иметь разные SKU в зависимости от схемы поставки. Например, если вы продаете iPhone 15 Pro 256GB и через FBS, и через FBO, у него будут два отдельных SKU — по одному для каждой модели логистики.

Артикул продавца (vendor code) — это ваш внутренний код товара, который вы указываете при загрузке. Он может совпадать с SKU только случайно. Озон использует артикул для сопоставления ваших данных с карточкой товара, но все операции на складах FBO ведутся исключительно по SKU.

⚠️ Внимание: Если вы измените артикул товара в личном кабинете, его SKU в FBO останется прежним. Это может привести к расхождениям в отчетах, если не синхронизировать данные через API.

Где искать SKU товара в личном кабинете Озон (пошагово)

Самый очевидный способ — найти SKU непосредственно в интерфейсе личного кабинета. Однако путь к нему не всегда интуитивно понятен. Вот актуальная инструкция для 2026 года:

  1. Перейдите в раздел Товары → Мои товары.
  2. В строке поиска введите название или артикул нужного товара.
  3. Нажмите на карточку товара, чтобы открыть её детали.
  4. Прокрутите вниз до блока Логистика.
  5. В таблице с информацией о складах найдите колонку SKU — здесь будет искомый идентификатор для FBO.

Если колонка SKU не отображается:

  • 🔧 Нажмите на иконку шестеренки (⚙️) в правом верхнем углу таблицы.
  • 📋 В списке доступных колонок найдите SKU и перетащите её в видимую область.
  • 🔄 Обновите страницу, если изменения не применились сразу.

Альтернативный путь (если SKU не видно в карточке товара):

  1. Перейдите в Склад → Остатки.
  2. Выберите фильтр Тип логистики: FBO.
  3. В результатах поиска SKU будет отображаться в отдельной колонке.

Как найти SKU через отчеты Озон (экспорт в Excel)

Если интерфейс личного кабинета не показывает SKU или вам нужно получить данные по большому количеству товаров, используйте отчеты. Этот метод особенно полезен для продавцов с ассортиментом от 100+ позиций.

Инструкция по экспорту:

  1. Перейдите в Аналитика → Отчеты → Отчет по остаткам.
  2. Установите фильтры:
    • 📅 Период: текущая дата
    • 📦 Тип логистики: FBO
    • 🏷️ Статус: Активные
  • Нажмите Сформировать отчет и дождитесь готовности (обычно 1–5 минут).
  • Скачайте файл в формате .xlsx.

    • В экспортированной таблице SKU будет в колонке с названием sku или ozon_sku. Обратите внимание, что в одном отчете могут присутствовать SKU как для FBO, так и для FBS — их можно отличить по колонке logistic_scheme.

    Название колонки в отчете Описание Пример значения
    sku Уникальный идентификатор товара в FBO 21034567891234
    offer_id ID предложения (совпадает с артикулом продавца) PROD-12345678
    logistic_scheme Схема логистики (fbo или fbs) fbo
    warehouse_id ID склада Озон, где хранится товар MSK-1
    ⚠️ Внимание: В отчетах по продажам (Отчет по реализации) SKU может отсутствовать. Для получения этого идентификатора используйте только Отчет по остаткам или Отчет по движениям товаров.

    Поиск SKU через API Озон (для технических специалистов)

    Если вы интегрированы с Озон через API, получить SKU можно с помощью следующих методов:

    1. Метод /v2/analytics/stock_on_warehouses — возвращает остатки товаров на складах FBO вместе с SKU. 

    {

    "date_from": "2026-01-01T00:00:00Z",
    

    "date_to": "2026-01-31T23:59:59Z",
    

    "warehouse_type": "fbo"
    

    }

    2. Метод /v1/product/info — позволяет получить детали товара по его product_id или offer_id, включая SKU для FBO. 

    {

    "offer_id": "YOUR_OFFER_ID",
    

    "product_id": 12345678
    

    }

    В ответе API SKU будет в поле "sku" внутри объекта "stocks": 

    {

    "stocks": {
    

    "fbo": {
    

    "sku": 21034567891234,
    

    "quantity": 15
    

    }
    

    }
    

    }

    💡

    Если в ответе API поле sku отсутствует, проверьте, что товар действительно загружен в FBO. Иногда Озон не присваивает SKU, если карточка не прошла модерацию или отсутствует остаток на складе.

    3. Метод /v1/stock/update — при обновлении остатков в запросе можно указать SKU, чтобы система вернула актуальные данные по нему.

    Что делать, если SKU не отображается или "исчез" из личного кабинета

    Ситуация, когда SKU внезапно пропадает из интерфейса, встречается после обновлений Озон или при сбоях синхронизации. Вот возможные причины и решения:

    • 🔄 Обновление интерфейса: Озон периодически меняет расположение элементов. Попробуйте:
      • Очистить кэш браузера (Ctrl+Shift+Del).
      • Использовать режим инкогнито.
      • Попробовать другой браузер (рекомендуется Google Chrome последней версии).
    • 📦 Товар переведен в FBS: Если вы изменили схему логистики с FBO на FBS, SKU для FBO станет неактивным. Проверьте текущую схему в карточке товара.
    • 🚫 Товар заблокирован или удален: SKU может скрываться, если карточка находится в статусе Заблокировано или На модерации. Верните товар в статус Активен.
    • 🔧 Ошибка синхронизации: Если SKU отсутствует в отчетах и ЛК, но товар физически есть на складе FBO, обратитесь в поддержку Озон с запросом на пересинхронизацию данных. Укажите:
      • ID товара (product_id)
      • Артикул продавца (offer_id)
      • Примерный период, когда SKU был виден в последний раз
    Как ускорить ответ поддержки по проблеме с SKU?

    В теме письма укажите: "[Срочно] Пропавший SKU в FBO — {ваш offer_id}".

    Приложите скриншот из отчета по остаткам, где видно, что товар есть на складе, но SKU отсутствует.

    Ссылка на карточку товара в ЛК (копируется из адресной строки браузера).

    Эти данные помогут специалистам быстрее идентифицировать проблему.

    Критическая информация: Если SKU пропал для товара, который уже был отгружен на склад FBO, не создавайте новую карточку с тем же артикулом! Это приведет к дублированию позиций и проблемам с учетом. Вместо этого дождитесь восстановления SKU или запросите его вручную через API.

    Как SKU используется в работе с FBO: практические примеры

    Знание SKU упрощает множество операций при работе по модели FBO. Рассмотрим реальные кейсы, где этот идентификатор незаменим:

    1. Формирование заказа на отгрузку

    При отправке товара на склад Озон вы должны указать SKU в:

    • 📄 Акте приема-передачи (колонка SKU)
    • 🏷️ Маркировке коробов (наряду со штрихкодом)
    • 📦 Транспортной накладной (если используете партнерских перевозчиков)

    Без правильного SKU склад Озон может не принять товар или зачислить его на неверную позицию.

    2. Обработка возвратов

    При возврате товара от покупателя Озон указывает SKU в:

    • 📧 Уведомлении о возврате (письмо на email)
    • 📊 Отчете по возвратам (Аналитика → Возвраты)

    По этому коду вы можете быстро найти товар в своей системе учета и принять решение о его дальнейшей судьбе (повторная продажа, утилизация и т.д.).

    3. Аналитика продаж

    В отчетах по продажам (Отчет по реализации) SKU позволяет:

    • 📈 Сегментировать данные по схеме логистики (FBO vs FBS).
    • 🔍 Отслеживать динамику продаж конкретного товара на складах Озон.
    • 💰 Сверять выручку с комиссиями маркетплейса (в некоторых отчетах SKU связывается с тарифами FBO).

    4. Работа с дефицитом и перемещениями

    Если товар заканчивается на одном складе FBO, но есть на другом, вы можете инициировать перемещение, указав SKU в запросе через:

    • 🖥️ Личный кабинет (Склад → Перемещения)
    • 🤖 API (/v1/warehouse/transfer)
    💡

    SKU в FBO — это не просто технический идентификатор, а ключ к управлению товарами на складах Озон. Без него невозможно корректно отслеживать остатки, обрабатывать возвраты или анализировать продажи по модели FBO.

    Частые ошибки при работе с SKU в FBO и как их избежать

    Даже опытные продавцы допускают ошибки, связанные с SKU. Вот самые распространенные из них и способы их предотвратить:

    1. Путаница между SKU и артикулом продавца

    ❌ Ошибка: Указывать в отчетах или запросах поддержки артикул вместо SKU.

    ✅ Решение:

    • Всегда уточняйте, какой именно идентификатор запрашивает Озон (в большинстве случаев для FBO нужен SKU).
    • В переписке с поддержкой дублируйте оба кода: SKU: 21034567891234, Артикул: PROD-12345678.

    2. Игнорирование SKU при маркировке товаров

    ❌ Ошибка: Отправлять товар на склад FBO с маркировкой только по артикулу или штрихкоду.

    ✅ Решение:

    • На каждой коробке/паллете должен быть указан SKU (можно вместе со штрихкодом).
    • Используйте генераторы этикеток, которые автоматически подтягивают SKU из отчетов Озон.

    3. Несвоевременное обновление SKU в своей системе учета

    ❌ Ошибка: Не синхронизировать SKU между Озон и своей CRM/1С, что приводит к расхождениям в остатках.

    ✅ Решение:

    • Настройте автоматическую выгрузку SKU из отчетов Озон в вашу систему (через API или парсинг Excel).
    • Периодически сверяйте SKU в отчете Отчет по остаткам с вашей базой.

    4. Использование SKU от FBS для операций в FBO

    ❌ Ошибка: Указывать в запросах на перемещение или возвраты SKU, который относится к схеме FBS.

    ✅ Решение:

    • Всегда проверяйте колонку logistic_scheme в отчетах — она должна содержать значение fbo.
    • Если сомневаетесь, запросите SKU через API с фильтром по типу логистики.

    FAQ: Ответы на частые вопросы о SKU в FBO

    Можно ли изменить SKU товара в FBO?

    Нет, SKU присваивается Озон автоматически и не подлежит редактированию. Если вам нужно "обновить" SKU (например, при изменении характеристик товара), создайте новую карточку с другим артикулом — системе будет присвоен новый SKU.

    Исключение: если SKU был присвоен ошибочно (например, дублируется с другим товаром), обратитесь в поддержку Озон с доказательствами ошибки.

    Чем отличается SKU в FBO и FBS?

    SKU в FBO и FBS — это два разных идентификатора, даже если речь идет об одном и том же товаре. Они присваиваются отдельно для каждой схемы логистики, так как товары хранятся на разных складах и управляются разными процессами.

    Пример: у вас есть наушники Sony WH-1000XM5, которые вы продаете и через FBO, и через FBS. В этом случае будет два SKU:

    • Один для остатков на складах Озон (FBO).
    • Другой для остатков на вашем собственном складе (FBS).

    Как найти SKU для товара, который еще не отгружен на FBO?

    SKU присваивается товару только после того, как он проходит модерацию и появляется в остатках на складе FBO. Если товар еще не отгружен, SKU для него не существует.

    Чтобы "зарезервировать" SKU:

    1. Создайте карточку товара в личном кабинете.
    2. Укажите схему логистики FBO.
    3. Отгрузите хотя бы 1 единицу товара на склад Озон.

    После приемки товара на склад ему будет присвоен SKU.

    Можно ли использовать один SKU для разных цветов/размеров товара?

    Нет, каждый уникальный вариант товара (например, футболка красная, размер M и футболка красная, размер L) получает свой собственный SKU в FBO. Это правило действует даже если артикул продавца у них совпадает.

    Если вы продаете товар с вариациями, в отчетах Озон вы увидите отдельные строки для каждого SKU с указанием характеристик (цвет, размер и т.д.).

    Где указывать SKU при возврате товара покупателем?

    При обработке возврата SKU нужен в следующих случаях:

    • 📧 В ответе на уведомление от Озон о возврате (если вы соглашаетесь на возврат или обмен).
    • 📦 При отправке замены покупателю (указывается в форме создания обмена в ЛК).
    • 🔄 В запросах на перемещение возвращенного товара между складами FBO.

    SKU можно найти в письме от Озон о возврате или в отчете Аналитика → Возвраты.
