Агентский отчет Озон: где его найти и как правильно использовать

Агентский отчет на Ozon — это ключевой документ для продавцов, работающих по моделям FBS или FBO. Он содержит детализированную информацию о продажах, комиссиях, возвратах и финансовых операциях, без которых невозможно вести точный учет и оптимизировать бизнес. Однако многие новички сталкиваются с проблемой: где именно искать этот отчет в личном кабинете, как его экспортировать и какие данные там содержатся.

В этой статье мы разберем все актуальные способы получения агентского отчета — от стандартного скачивания через веб-версию до автоматизации через API. Вы узнаете, какие разделы отчета наиболее важны для анализа, как избежать ошибок при работе с данными и что делать, если отчет не формируется. А для опытных продавцов мы подготовили лайфхаки по интеграции отчетов с 1С, Excel и сервисами аналитики.

Что такое агентский отчет Озон и зачем он нужен

Агентский отчет — это финансовый документ, который Ozon формирует для продавцов, выступающих в роли агентов по реализации товаров. В отличие от обычной выписки по продажам, он включает:

  • 📊 Подробную разбивку комиссий маркетплейса (включая логистику, хранение, рекламу)
  • 💰 Суммы выплат за каждый заказ с учетом возвратов и удержаний
  • 📦 Данные по логистике: стоимость доставки, штрафы за невыкуп, компенсации за поврежденные товары
  • 📅 Периодизацию по дням, неделям или месяцам (в зависимости от настроек)

Без этого отчета невозможно:

  • 📈 Провести аудит финансов и сверить выручку с реальными поступлениями на счет
  • 📉 Выявить убыточные товары (где комиссии съедают всю маржу)
  • 📑 Подготовить отчетность для налоговой (особенно для ИП и ООО на ОСНО)
  • 🤝 Урегулировать споры с Озон по некорректным удержаниям или штрафам
⚠️ Внимание: Агентский отчет отличается от "Выписки по продажам" в личном кабинете. Последняя показывает только оборот, а агентский отчет — реальную финансовую картину с учетом всех удержаний и компенсаций. Игнорирование этого документа может привести к ошибкам в бухгалтерии на десятки тысяч рублей.

Например, если вы продали товар за 5 000 ₽, но в агентском отчете указана выплата 3 800 ₽, разница в 1 200 ₽ — это комиссии, логистика и возможные штрафы. Без детализации вы не поймете, куда ушли деньги.

Где найти агентский отчет в личном кабинете Озон (пошаговая инструкция)

Основной способ получения отчета — через веб-версию личного кабинета. Вот пошаговый алгоритм:

  1. Авторизуйтесь на сайте seller.ozon.ru под своим аккаунтом продавца.
  2. В верхнем меню выберите раздел Финансы → Отчеты.
  3. В блоке Агентские отчеты нажмите кнопку Сформировать отчет.
  4. Укажите период (максимум 31 день) и формат (Excel или CSV).
  5. Нажмите Скачать — файл будет сгенерирован в течение 1–5 минут (зависит от объема данных).

Если кнопка Сформировать отчет неактивна:

  • 🔹 Убедитесь, что у вас есть подтвержденные продажи за выбранный период.
  • 🔹 Проверьте, что ваш аккаунт не заблокирован (даже частичные ограничения могут блокировать доступ к отчетам).
  • 🔹 Попробуйте сменить браузер (иногда проблемы возникают в Safari или устаревших версиях Chrome).

Скачанный файл будет иметь название вида agent_report_YYYY-MM-DD.xlsx. В нем содержатся вкладки:

  • Сводка — общие итоги по периоду
  • Заказы — детализация по каждому заказу
  • Возвраты — информация о возвращенных товарах
  • Комиссии — разбивка удержаний маркетплейса

Как получить агентский отчет через мобильное приложение Ozon Seller

Мобильное приложение Ozon Seller (доступно для iOS и Android) позволяет скачивать отчеты прямо со смартфона. Инструкция:

  1. Откройте приложение и авторизуйтесь.
  2. Перейдите в раздел Финансы (значок кошелька в нижнем меню).
  3. Тапните на Отчеты → Агентские отчеты.
  4. Выберите период (максимум 7 дней в мобильной версии!) и нажмите Сформировать.
  5. После генерации нажмите Открыть файл — он откроется в стандартном просмотрщике таблиц.

Ограничения мобильной версии:

  • 📅 Можно выбрать только 7 дней (против 31 дня в веб-версии).
  • 📱 Нет возможности экспорта в CSV — только Excel.
  • 🔄 Отчеты формируются дольше (до 10 минут в пиковые часы).

Совет: если вам нужен отчет за большой период, скачайте его через компьютер, а в мобильном приложении используйте функцию Поиск по заказам (вкладка Заказы) для оперативной проверки отдельных сделок.

💡

В мобильном приложении можно включить уведомления о новых отчетах. Для этого перейдите в Настройки → Уведомления → Финансовые отчеты и активируйте опцию "Готовность агентского отчета".

Автоматизация: получение отчетов через API Ozon

Для продавцов с большим объемом продаж (100+ заказов в день) ручное скачивание отчетов неэффективно. В этом случае поможет API Ozon, который позволяет:

  • 🤖 Автоматически загружать отчеты по расписанию (например, каждый день в 9:00).
  • 📊 Интегрировать данные с 1С, Google Sheets или Power BI.
  • 🔍 Фильтровать отчеты по статусам заказов, категориям товаров или регионам.

Чтобы настроить автоматическую выгрузку:

  1. Получите Client-ID и API-Key в личном кабинете (Настройки → API).
  2. Используйте эндпоинт: 
    POST https://api-seller.ozon.ru/v3/finance/agent/report

    с параметрами:

    {

    "date": {
    

    "from": "2026-05-01",
    

    "to": "2026-05-31"
    

    },
    

    "format": "xlsx"
    

    }

  3. Обработайте ответ сервера — ссылка на скачивание будет в поле result.file.

Пример кода на Python для автоматической выгрузки:

import requests


url = "https://api-seller.ozon.ru/v3/finance/agent/report"


headers = {


"Client-Id": "ваш_client_id",


"Api-Key": "ваш_api_key",


"Content-Type": "application/json"


}


payload = {


"date": {


"from": "2026-05-01",


"to": "2026-05-31"


},


"format": "xlsx"


}



response = requests.post(url, headers=headers, json=payload)


download_url = response.json()["result"]["file"]


Далее скачиваете файл по download_url
⚠️ Внимание: При работе с API учитывайте лимиты запросов — 1000 запросов в час для стандартного тарифа. Превышение лимита блокирует доступ на 1 час. Также Ozon может ввести дополнительные ограничения, если ваш аккаунт новый (менее 3 месяцев).
Как проверить лимиты API?

Откройте раздел Настройки → API → Лимиты в личном кабинете. Там отображается текущее использование запросов и время сброса лимита (обычно в 00:00 по МСК).

Разбор структуры агентского отчета: на что обратить внимание

Скачанный агентский отчет содержит десятки колонок, но для анализа достаточно 10–15 ключевых. Рассмотрим самые важные:

Название колонки Описание Пример значения Зачем нужен
order_id Уникальный номер заказа 1234567890 Для поиска заказа в ЛК или поддержке
posting_number Номер отправления (для FBS) RB-12345678-9 Связь с логистическими данными
product_name Название товара Смартфон Xiaomi Redmi Note 12 Анализ продаж по ассортименту
commission Комиссия Ozon (%) 15.00% Расчет реальной маржи
payout Сумма к выплате продавцу 3800.00 ₽ Сверка с банковскими поступлениями

Особое внимание уделите колонкам:

  • 🔍 return_reason — причина возврата (если есть). Частые значения: не подошло, брак, недостача.
  • 💸 logistics_cost — стоимость доставки, которую удержал Ozon (для FBS).
  • ⚠️ penalty — штрафы за нарушения (например, за позднюю отправку).

Типичная ошибка новичков — игнорировать колонку cancel_reason (причина отмены заказа). Если там часто встречается client_refused (отказ клиента), это сигнал о проблемах с описанием товара или ценой.

💡

Колонка payout показывает сумму, которую вы реально получите на счет, а не цену товара. Разница между price и payout — это ваши издержки (комиссии, логистика, штрафы).

Частые проблемы с агентскими отчетами и как их решить

Даже опытные продавцы сталкиваются с трудностями при работе с отчетами. Вот самые распространенные проблемы и их решения:

Проблема Возможная причина Решение
Отчет не формируется Нет продаж за период или аккаунт заблокирован Проверьте наличие заказов в разделе Продажи. Если блокировка — обратитесь в поддержку.
Данные в отчете не совпадают с выпиской Задержка обработки возвратов или корректировок Подождите 1–2 дня — иногда данные обновляются с задержкой.
Не хватает колонок (например, нет penalty) Устаревшая версия отчета Обновите шаблон отчета в настройках или скачайте через API.
Отчет скачивается пустым Сбой на стороне Ozon или превышен лимит запросов Попробуйте позже или разбейте период на более мелкие интервалы.

Если в отчете появились отрицательные суммы в колонке payout, это может означать:

  • 🔹 Штрафы за нарушение правил (например, за просроченную отправку).
  • 🔹 Компенсации покупателям (если товар потерялся или пришел бракованным).
  • 🔹 Ошибку в расчетах (решается через обращение в поддержку с указанием order_id).

Для проверки корректности данных сверьте:

  1. Сумму в колонке payout с поступлениями на ваш расчетный счет (допустимо расхождение до 1–2 ₽ из-за округлений).
  2. Количество заказов в отчете с данными в разделе Продажи → Заказы.
  3. Сумму комиссий с тарифами, указанными в договоре с Ozon.
💡

Если вы обнаружили расхождение более чем на 5% от суммы выплаты, сразу пишите в поддержку через раздел Помощь → Финансовые вопросы. Приложите скриншоты отчета и выписки из банка — это ускорит разбирательство.

Как интегрировать агентские отчеты с 1С, Excel и сервисами аналитики

Ручная обработка отчетов занимает много времени, поэтому продавцы автоматизируют этот процесс. Вот популярные способы интеграции:

1. Загрузка в 1С

Для интеграции с 1С:Управление торговлей или 1С:Бухгалтерия:

  1. Скачайте отчет в формате Excel.
  2. В 1С перейдите в Файл → Открыть и выберите скачанный файл.
  3. Используйте обработку Импорт из Excel (доступна в стандартных конфигурациях).
  4. Настройте правила сопоставления колонок (например, order_id → НомерЗаказа).

Готовые обработки для 1С можно найти на Infostart (поиск по запросу "Ozon агентский отчет").

2. Анализ в Excel/Google Sheets

Для визуализации данных:

  • 📊 Создайте сводную таблицу по колонкам product_name и payout, чтобы увидеть самые прибыльные товары.
  • 📉 Постройте график по колонке commission, чтобы отследить динамику комиссий.
  • 🔍 Используйте условное форматирование для выделения заказов с отрицательной маржей (где price - commission - logistics_cost < 0).

Пример формулы для расчета реальной маржи в Excel:

=([@price] -[@commission] -[@logistics_cost] -[@penalty]) * (1 -[@vat_rate])

3. Подключение к Power BI или Tableau

Для глубокой аналитики:

  1. Импортируйте отчет в Power BI Desktop через Home → Get Data → Excel.
  2. Создайте модель данных, связав таблицы Заказы и Возвраты по order_id.
  3. Постройте дашборд с ключевыми метриками:
    • 📈 Конверсия возвратов = (Количество возвратов / Количество заказов) × 100%
    • 💰 Средняя маржа = (Сумма выплат / Количество заказов) - Себестоимость
    • ⚡ Доля штрафов = (Сумма штрафов / Общий оборот) × 100%

Готовые шаблоны дашбордов для Ozon можно найти на community.powerbi.com.

💡

Автоматизация отчетности экономит до 10–15 часов в месяц. Например, интеграция с 1С позволяет избегать ручного ввода данных и снижает риск ошибок в бухгалтерии на 90%.

FAQ: Ответы на частые вопросы про агентские отчеты Ozon

Можно ли получить агентский отчет за период больше 31 дня?

Нет, в личном кабинете и через API максимальный период — 31 день. Чтобы получить данные за квартал или год, скачивайте отчеты помесячно и объединяйте их в Excel с помощью функции Power Query (вкладка Данные → Получить данные → Из файла → Объединить).

Почему в отчете нет данных по некоторым заказам?

Это может происходить по нескольким причинам:

  • Заказы еще не закрыты (статус не delivered или cancelled).
  • Заказы были оформлены через Ozon Global (для них формируется отдельный отчет).
  • Произошел сбой при формировании отчета (повторите запрос через 1–2 часа).

Проверьте статус заказов в разделе Продажи → Заказы.

Как часто обновляются данные в агентском отчете?

Данные обновляются в реальном времени, но:

  • Информация по возвратам может появляться с задержкой до 3 рабочих дней.
  • Корректировки по штрафам или компенсациям отображаются в течение 24 часов после обработки.

Если вам нужны актуальные данные по конкретному заказу, используйте API-запрос к эндпоинту /v2/finance/transaction/list.

Можно ли получить агентский отчет за прошлый год?

Да, но с ограничениями:

  • Через личный кабинет доступны отчеты за последние 12 месяцев.
  • За более ранние периоды нужно обращаться в поддержку с указанием причины (например, для налоговой проверки).
  • В некоторых случаях Ozon может предоставить архивные данные в формате CSV, но без детализации по заказам.

Совет: сохраняйте все отчеты на своем сервере или в облачном хранилище (например, Google Drive), чтобы не зависеть от ограничений Ozon.

Что делать, если в отчете ошибка (неверная сумма, отсутствует заказ)?

Алгоритм действий:

  1. Проверьте, не исправилась ли ошибка при повторном формировании отчета.
  2. Сверьте данные с разделом Продажи → Заказы (найдите проблемный order_id).
  3. Если ошибка подтверждается, создайте обращение в поддержку:
    • Тема: "Ошибка в агентском отчете".
    • Приложите: скриншот отчета, номер заказа (order_id), ожидаемую и фактическую сумму.
  • Если ответ поддержки не устраивает, эскалируйте вопрос через кнопку "Не помогло" в чате.

    • Среднее время разрешения таких обращений — 3–5 рабочих дней.
