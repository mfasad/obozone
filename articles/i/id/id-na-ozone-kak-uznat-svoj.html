Как узнать свой ID на Ozon: 5 способов для продавцов и покупателей

Ozon — крупнейший маркетплейс в России, где каждый пользователь (и покупатель, и продавец) получает уникальный идентификатор. Этот ID нужен для интеграции с API, оформления возвратов, связи с поддержкой и даже для участия в акциях. Но где его найти? В этой статье разберём все способы — от очевидных до скрытых, — чтобы вы могли быстро получить свой номер без лишних вопросов.

Важно понимать, что на Ozon существует несколько типов идентификаторов: ID продавца (для бизнеса), ID покупателя (для личных аккаунтов), а также ID заказа или ID товара. Мы сфокусируемся на первых двух, так как они чаще всего вызывают путаницу. Если вы ищете способ найти client_id для API или просто хотите уточнить свой номер в поддержке — читайте дальше.

1. Что такое ID на Ozon и зачем он нужен

Идентификатор пользователя на Ozon — это уникальный числовой или буквенно-цифровой код, который присваивается каждому аккаунту. Он используется для:

  • 🔄 Интеграции с API — без seller_id или client_id невозможно подключить автоматизированные системы управления заказами.
  • 📦 Оформления возвратов — поддержка может запросить ваш ID для ускорения процесса.
  • 💳 Участия в партнёрских программах — некоторые акции требуют указания идентификатора для начисления бонусов.
  • 🛠️ Технической поддержки — при обращении в службу помощи ваш ID ускоряет поиск профиля.

Для продавцов основной идентификатор — это seller_id (пример: 12345678). Для покупателей — client_id (может выглядеть как cl-987654321). Иногда также требуется warehouse_id (идентификатор склада) или product_id (номер товара).

Если вы продавец и планируете подключать 1С, МойСклад или другие сервисы автоматизации, без seller_id не обойтись. Для покупателей ID может понадобиться при оформлении группового заказа или спорных ситуациях с возвратом.

2. Как узнать ID продавца на Ozon (seller_id)

Если вы зарегистрированы как продавец, ваш seller_id можно найти несколькими способами. Самый простой — через личный кабинет Ozon Seller.

Способ 1: Через личный кабинет Ozon Seller

  1. Авторизуйтесь на сайте seller.ozon.ru.
  2. В правом верхнем углу кликните на иконку профиля (ваше имя или аватар).
  3. В выпадающем меню выберите Настройки аккаунта.
  4. В разделе Общая информация найдите строку ID продавца (или seller_id).

Пример отображения: 



ID продавца: 50123478


Название компании: ООО "Ромашка"

Способ 2: Через API (для технических специалистов)

Если вам нужен seller_id для настройки интеграции, его можно получить через запрос к API. Для этого:

  1. Авторизуйтесь в личном кабинете API.
  2. Перейдите в раздел Ключи API.
  3. В блоке Общая информация будет указан ваш seller_id.

Также seller_id можно увидеть в ответе на любой тестовый запрос к API (например, при проверке статуса заказа). Он будет в поле "seller_id": 12345678.

Способ 3: Через мобильное приложение Ozon Seller

Если вы пользуетесь мобильным приложением:

  1. Откройте приложение Ozon Seller.
  2. Тапните на иконку профиля в правом нижнем углу.
  3. Выберите Настройки → Информация о продавце.
  4. Найдите строку ID продавца.

В некоторых версиях приложения seller_id может отображаться в разделе О магазине или Профиль компании.

💡

Если вы не видите ID в мобильном приложении, обновите его до последней версии в App Store или Google Play. В старых версиях этот раздел мог отсутствовать.

3. Как узнать ID покупателя на Ozon (client_id)

Покупателям на Ozon реже требуется их идентификатор, но он может понадобиться для:

  • 🔄 Оформления группового заказа (например, для корпоративных клиентов).
  • 📞 Обращения в поддержку с вопросом по конкретному аккаунту.
  • 🎁 Участия в бонусных программах или розыгрышах.

Способ 1: Через личный кабинет покупателя

К сожалению, Ozon не отображает client_id в открытом доступе в веб-версии или мобильном приложении. Однако его можно найти косвенно:

  1. Авторизуйтесь на сайте ozon.ru.
  2. Перейдите в раздел Мои заказы.
  3. Откройте любой заказ и скопируйте его номер (например, 312457896).
  4. Обратитесь в поддержку Ozon с просьбой сообщить ваш client_id, указав номер заказа для подтверждения.

Способ 2: Через URL-адрес профиля

В некоторых случаях client_id можно увидеть в адресной строке браузера:

  1. Откройте раздел "Мои заказы".
  2. Нажмите F12 (или Ctrl+Shift+I), чтобы открыть инструменты разработчика.
  3. Перейдите на вкладку Network и обновите страницу.
  4. Найдите запрос с названием api/my/orders и откройте его.
  5. В ответе сервера будет поле "client_id": "cl-12345678".

Внимание: этот метод требует базовых знаний работы с браузерными инструментами. Если вы не уверены, используйте способ с обращением в поддержку.

Способ 3: Через письмо от Ozon

Иногда client_id указывается в письмах от Ozon:

  • 📧 Проверьте письма с темой "Ваш заказ №ХХХ на Ozon".
  • 🔍 В некоторых уведомлениях (например, о возврате или бонусах) может быть строка вида Client ID: cl-98765432.
  • 🔎 Используйте поиск по почте по запросу "client_id" или "cl-".
Почему Ozon скрывает client_id?

Маркетплейс ограничивает доступ к идентификаторам покупателей, чтобы предотвратить мошенничество и несанкционированный доступ к аккаунтам. В большинстве случаев client_id нужен только для технических операций (например, при интеграции с партнёрскими сервисами), поэтому его не отображают в открытом доступе.

4. Где ещё можно найти ID на Ozon

Помимо стандартных способов, есть несколько альтернативных методов получения идентификаторов. Они подойдут, если у вас нет доступа к личному кабинету или требуется seller_id/client_id для третьих лиц (например, для разработчика).

Через поддержку Ozon

Самый надёжный, но долгий способ:

  1. Перейдите в раздел помощи.
  2. Выберите тему "Вопросы по аккаунту".
  3. Напишите сообщение с просьбой предоставить ваш seller_id или client_id.
  4. Прикрепите скриншот подтверждения личности (например, страницу с личными данными в профиле).

Обычно ответ приходит в течение 1–3 рабочих дней.

Через партнёрские сервисы

Если вы подключены к сервисам вроде МойСклад, 1С или RetailCRM, ваш seller_id может отображаться в настройках интеграции. Например:

  • 📌 В МойСклад: Настройки → Интеграции → Ozon → Данные для подключения.
  • 📌 В RetailCRM: Настройки → Маркетплейсы → Ozon → Параметры подключения.

Через документацию по API

Если вы разработчик или работаете с техническим специалистом, seller_id можно найти:

  • 📄 В официальной документации Ozon API (раздел "Аутентификация").
  • 🔑 В ответе на запрос получения токена (поле "seller_id").

5. Частые ошибки при поиске ID

Многие пользователи путают разные типы идентификаторов или ищут их в неправильных разделах. Вот наиболее распространённые ошибки:

  • 🔢 Путают seller_id и client_id — первый нужен для бизнеса, второй для личных аккаунтов.
  • 📦 Ищут ID в мобильном приложении Ozon (для покупателей) — там его нет, только в веб-версии через инструменты разработчика.
  • 🔑 Пытаются использовать warehouse_id вместо seller_id — это разные вещи (склад и продавец).
  • 📧 Не проверяют папку "Спам" в почте — иногда письма от Ozon с ID попадают туда.
⚠️ Внимание: Никогда не передавайте свой seller_id или client_id третьим лицам без проверки их надёжности. Мошенники могут использовать эти данные для доступа к вашему аккаунту или заказам.

Ещё одна частая проблема — несколько аккаунтов. Если вы зарегистрированы и как покупатель, и как продавец, убедитесь, что вы авторизованы под правильным профилем. Например, seller_id не будет отображаться, если вы вошли в личный кабинет покупателя (ozon.ru), а не продавца (seller.ozon.ru).

Если вы не можете найти ID, проверьте:

  • 🔹 Правильность ввода логина/пароля (возможно, вы вошли не в тот аккаунт).
  • 🔹 Наличие прав доступа (если вы менеджер в компании, у вас может не быть доступа к настройкам).
  • 🔹 Актуальность версии браузера или приложения (в старых версиях раздел с ID мог отсутствовать).

6. Зачем может понадобиться ID на Ozon: реальные кейсы

Многие пользователи не понимают, зачем им знать свой идентификатор, пока не сталкиваются с одной из этих ситуаций:

Для продавцов

  • 🤖 Подключение робота для автоматической обработки заказов — без seller_id невозможно настроить интеграцию с 1С или Bitrix24.
  • 📊 Аналитика продаж через сторонние сервисы — например, DataLens или Power BI требуют указания seller_id для выгрузки данных.
  • 💰 Участие в программе Ozon Финансы — для получения кредита или кешбэка может потребоваться подтверждение через ID.

Для покупателей

  • 🎁 Получение бонусов по партнёрской программе — некоторые акции требуют указания client_id для начисления баллов.
  • 🔄 Оформление группового заказа — если вы покупаете товары для компании, может понадобиться привязка заказов к одному аккаунту.
  • 🛡️ Восстановление доступа к аккаунту — поддержка может запросить client_id для подтверждения личности.

Также seller_id часто требуется при:

  • 📦 Переезде на схему FBS/FBO — для настройки логистики.
  • 📈 Участии в акциях типа "Чёрная пятница" — для регистрации промо-оферты.
  • 🔧 Настройке вебхуков — для получения уведомлений о новых заказах.
⚠️ Внимание: Если вы продавец и планируете подключать Ozon API, кроме seller_id вам также потребуется client-secret и API-key. Эти данные выдаются в личном кабинете API и должны храниться в секрете!

7. Что делать, если ID не отображается или потерян

Если вы не можете найти свой идентификатор, следуйте этому алгоритму:

Для продавцов

  1. Проверьте права доступа — если вы менеджер, у вас может не быть доступа к настройкам аккаунта. Обратитесь к владельцу бизнеса.
  2. Обновите браузер или приложение — в старых версиях раздел с ID мог отсутствовать.
  3. Попробуйте другой способ — если не видите seller_id в личном кабинете, проверьте API или партнёрские сервисы.
  4. Обратитесь в поддержку — если ничего не помогает, напишите в чат поддержки продавцов.

Для покупателей

  1. Используйте инструменты разработчика (метод с F12), если у вас есть навыки работы с браузером.
  2. Проверьте почту — иногда client_id указывается в письмах о заказах.
  3. Свяжитесь с поддержкой — укажите номер телефона или заказа для подтверждения личности.

Если вы потеряли доступ к аккаунту, где был указан ID:

  • 🔄 Восстановите пароль через страницу восстановления.
  • 📞 Если восстановление не работает, позвоните в поддержку по телефону 8 800 333-70-50 (для покупателей) или 8 800 700-94-50 (для продавцов).

Если вам срочно нужен seller_id для интеграции, но вы не можете его найти, временным решением может быть:

  • 📌 Использование test_seller_id из песочницы Ozon API (только для тестирования!).
  • 📌 Обращение к партнёру, который уже подключён к API — он может помочь с настройкой.

FAQ: Частые вопросы об ID на Ozon

Можно ли узнать чужой seller_id или client_id?

Нет, Ozon не предоставляет доступ к идентификаторам других пользователей. Это конфиденциальная информация. Единственный легальный способ — если владелец аккаунта сам предоставит вам свой ID (например, для совместной работы).

Меняется ли ID при смене названия магазина или реквизитов?

Нет, seller_id и client_id присваиваются один раз при регистрации и не изменяются. Даже если вы поменяете название компании, ИНН или адрес, идентификатор останется прежним.

Что такое warehouse_id и чем он отличается от seller_id?

warehouse_id — это идентификатор склада, на котором хранятся ваши товары (например, MSC1 для Московского ЦОДа). Он нужен для работы с логистикой (FBS/FBO), но не заменяет seller_id, который относится ко всему аккаунту продавца.

Можно ли использовать один seller_id для нескольких магазинов?

Нет, каждый магазин на Ozon имеет свой уникальный seller_id. Если у вас несколько юридических лиц или брендов, для каждого будет свой идентификатор. Чтобы увидеть все свои seller_id, авторизуйтесь под каждой учётной записью отдельно.

Где хранить seller_id и client_id, чтобы не потерять?

Рекомендуем сохранять идентификаторы в защищённом месте:

  • 🔐 Менеджере паролей (1Password, KeePass).
  • 📄 Зашифрованном документе (например, в Google Docs с ограниченным доступом).
  • 📱 Примечаниях телефона с блокировкой (если это личный аккаунт).

Никогда не храните ID в открытых текстовых файлах или пересылайте их по незащищённым каналам (например, в обычных письмах).
