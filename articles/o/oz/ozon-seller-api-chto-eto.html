Ozon Seller API: что это, как подключиться и использовать для автоматизации продаж

Ozon Seller API — это мощный инструмент, который позволяет продавцам на маркетплейсе автоматизировать рутинные процессы, интегрировать собственные системы с платформой и масштабировать бизнес без лишних затрат времени. Если вы продаёте на Ozon и тратите часы на обработку заказов, обновление остатков или аналитику — API поможет сократить эти затраты до минут. Но как оно работает, сколько стоит и с чего начать?

В этой статье мы разберём всё об Ozon Seller API: от базовых понятий до практических примеров использования. Вы узнаете, какие задачи решает API, как получить доступ, какие есть ограничения и как избежать типичных ошибок при интеграции. А если вы не разработчик — мы подскажем, как найти исполнителя для настройки API под ваши нужды.

Что такое Ozon Seller API и зачем оно нужно продавцам

Ozon Seller API (Application Programming Interface) — это набор протоколов и инструментов, который позволяет внешним системам взаимодействовать с платформой Ozon. Простыми словами, это «мост» между вашим софтом (сайтом, 1С, CRM, скриптом) и маркетплейсом. С его помощью вы можете:

  • 📦 Автоматически получать новые заказы и обновлять их статусы.
  • 📊 Синхронизировать остатки товаров в реальном времени.
  • 📈 Загружать и обновлять карточки товаров массово.
  • 💰 Управлять ценами, скидками и акциями.
  • 📄 Получать отчёты по продажам, возвратам и финансам.

Без API многие процессы приходится делать вручную через личный кабинет Ozon Seller, что отнимает время и увеличивает риск ошибок. Например, если у вас 1000 товаров, обновление цен или остатков вручную может занять несколько часов. С API это занимает секунды.

Кроме того, API позволяет интегрировать Ozon с другими сервисами: 1С, Bitrix24, МойСклад, RetailCRM и даже собственными разработками. Это особенно актуально для крупных продавцов, которые работают на нескольких маркетплейсах одновременно.

Какие задачи решает Ozon Seller API: топ-5 сценариев использования

API от Ozon покрывает почти все ключевые процессы продавца. Рассмотрим самые востребованные сценарии, где автоматизация приносит максимальную пользу.

1. Автоматическая обработка заказов

С API вы можете:

  • 📥 Получать уведомления о новых заказах в реальном времени.
  • 🔄 Обновлять статусы заказов (например, «В обработке», «Отправлен», «Доставлен»).
  • 📦 Передавать данные о трек-номерах отправлений.

Это избавляет от необходимости постоянно проверять личный кабинет и снижает риск пропустить заказ.

2. Управление остатками и ценами

API позволяет:

  • 📉 Автоматически корректировать остатки товаров при продаже на других площадках.
  • 💲 Массово обновлять цены (например, при изменении курса валюты или акции).
  • 🛒 Синхронизировать данные с вашим складом или 1С.

3. Массовая загрузка и обновление товаров

Вместо того чтобы загружать товары по одному через личный кабинет, вы можете:

  • 📁 Загружать тысячи карточек за раз.
  • 🔄 Обновлять описания, фотографии и характеристики массово.
  • 🏷️ Управлять категориями и атрибутами товаров.

4. Аналитика и отчётность

Через API можно получать:

  • 📊 Данные по продажам, возвратам и отменам.
  • 💰 Финансовые отчёты (выплаты, комиссии, штрафы).
  • 📈 Статистику по трафику и конверсиям.

Это помогает строить дашборды в Google Data Studio, Power BI или других инструментах аналитики.

5. Интеграция с CRM и складскими системами

API позволяет связать Ozon с:

  • 📦 1С, МойСклад, WMS-системами.
  • 📞 CRM-системами (Bitrix24, AmoCRM, RetailCRM).
  • 🤖 Чатами и ботами для уведомлений (например, в Telegram).
💡

API экономит до 80% времени на рутинных операциях, особенно для продавцов с большим ассортиментом (1000+ товаров).

Как подключиться к Ozon Seller API: пошаговая инструкция

Чтобы начать работать с API, нужно пройти несколько этапов: от регистрации приложения до получения токена доступа. Рассмотрим процесс подробно.

Шаг 1: Регистрация в Ozon Seller

Если у вас ещё нет аккаунта продавца на Ozon, сначала зарегистрируйтесь на сайте seller.ozon.ru. Подключение API доступно только для подтверждённых продавцов.

Шаг 2: Создание приложения в кабинете разработчика

Перейдите в раздел API → Мои приложения и нажмите «Создать приложение». Заполните данные:

  • 📌 Название приложения (например, «Интеграция с 1С»).
  • 🔗 Описание (указывайте цель использования).
  • 📧 Email для связи.

Шаг 3: Получение Client ID и Client Secret

После создания приложения вы получите:

  • Client ID — уникальный идентификатор вашего приложения.
  • Client Secret — секретный ключ для аутентификации.

Внимание! Храните Client Secret в безопасности — его компрометация может привести к несанкционированному доступу к вашему аккаунту.

Шаг 4: Получение токена доступа (OAuth 2.0)

Для работы с API нужен токен доступа. Его можно получить через OAuth 2.0. Пример запроса на получение токена:

POST https://api-seller.ozon.ru/v2/auth/token

Content-Type: application/json



{


"client_id": "Ваш_Client_ID",


"client_secret": "Ваш_Client_Secret",


"grant_type": "client_credentials"


}

В ответ вы получите JSON с токеном, который нужно передавать в заголовках всех последующих запросов:

{

"access_token": "ваш_токен_доступа",


"token_type": "Bearer",


"expires_in": 3600


}

Шаг 5: Тестирование запросов

Перед использованием в боевом режиме протестируйте API в песочнице (sandbox). Это поможет избежать ошибок при работе с реальными данными.

Ограничения и лимиты Ozon Seller API

Как и любой сервис, API Ozon имеет ограничения, которые важно учитывать при разработке интеграций. Их нарушение может привести к блокировке доступа.

1. Лимиты на количество запросов

Ozon устанавливает лимиты на количество запросов в минуту/час. Например:

Тип запроса Лимит (запросов/минуту) Лимит (запросов/час)
Получение списка заказов 60 1000
Обновление статуса заказа 30 500
Получение информации о товаре 100 2000
Обновление остатков 20 300

При превышении лимитов API вернёт ошибку 429 Too Many Requests. В этом случае нужно либо оптимизировать код (например, использовать пакетные запросы), либо запросить увеличение лимитов у поддержки.

2. Ограничения по времени выполнения

Некоторые запросы имеют тайм-аут (максимальное время выполнения). Например, массовое обновление товаров не должно занимать более 30 секунд. Если запрос выполняется дольше, сервер его прервёт.

3. Ограничения по данным

При работе с API учитывайте:

  • 📌 Максимальный размер файла при загрузке товаров — 50 МБ.
  • 📌 Количество товаров в одном запросе на обновление — до 1000 штук.
  • 📌 Длина текстового поля (например, описания товара) — до 4000 символов.
💡

Используйте batch-запросы для массовых операций — это поможет сократить количество обращений к API и избежать лимитов.

Примеры кода для работы с Ozon Seller API

Рассмотрим несколько практических примеров на Python и cURL, которые помогут начать работу с API.

1. Получение списка заказов

Чтобы получить список заказов за последний день, используйте следующий запрос:

import requests


url = "https://api-seller.ozon.ru/v2/posting/fbs/list"


headers = {


"Client-Id": "Ваш_Client_ID",


"Api-Key": "Ваш_API_Key",


"Content-Type": "application/json"


}


payload = {


"dir": "ASC",


"filter": {


"since": "2026-05-01T00:00:00Z",


"to": "2026-05-02T00:00:00Z"


},


"limit": 100


}



response = requests.post(url, headers=headers, json=payload)


print(response.json())

2. Обновление остатков товара

Для обновления остатков отправьте запрос с sku и новым количеством:

url = "https://api-seller.ozon.ru/v1/product/stocks"

headers = {


"Client-Id": "Ваш_Client_ID",


"Api-Key": "Ваш_API_Key",


"Content-Type": "application/json"


}


payload = {


"stocks": [


{


"sku": 123456789,


"stock": 100


}


]


}



response = requests.post(url, headers=headers, json=payload)


print(response.json())

3. Получение информации о товаре

Чтобы получить данные о конкретном товаре, используйте его product_id:

url = "https://api-seller.ozon.ru/v1/product/info"

headers = {


"Client-Id": "Ваш_Client_ID",


"Api-Key": "Ваш_API_Key"


}


params = {


"product_id": 123456789


}



response = requests.post(url, headers=headers, params=params)


print(response.json())

Что делать, если API возвращает ошибку 403 Forbidden?

Ошибка 403 Forbidden обычно означает проблемы с аутентификацией. Проверьте:

1. Правильность Client-ID и Api-Key.

2. Срок действия токена (возможно, он истёк).

3. Права вашего приложения в кабинете Ozon Seller.

Если ошибка сохраняется, обратитесь в поддержку с логами запроса.

Типичные ошибки при работе с Ozon Seller API и как их избежать

Даже опытные разработчики иногда сталкиваются с проблемами при интеграции с API Ozon. Рассмотрим самые распространённые ошибки и способы их решения.

1. Ошибки аутентификации (401 Unauthorized)

Причины:

  • 🔑 Неправильный Client-ID или Api-Key.
  • 🕒 Истёкший токен доступа.
  • 🚫 Отсутствие прав у приложения.

Решение: проверьте данные для аутентификации и обновите токен.

2. Превышение лимитов (429 Too Many Requests)

Причины:

  • ⚡ Слишком частые запросы.
  • 📈 Неоптимизированный код (например, запросы в цикле без задержек).

Решение: используйте пакетные запросы (batch) и добавляйте задержки между вызовами.

3. Ошибки валидации данных (400 Bad Request)

Причины:

  • 📝 Неправильный формат данных (например, неверный sku или цена).
  • 📏 Превышение лимитов по символам или размерам файлов.

Решение: внимательно проверяйте структуру запроса и соответствие документации.

4. Проблемы с кодировкой

Причины:

  • 🌍 Неправильная кодировка символов (например, кириллица в UTF-8).
  • 📄 Проблемы с JSON-форматированием.

Решение: используйте utf-8 и валидируйте JSON перед отправкой.

💡

Всегда логируйте запросы и ответы API — это поможет быстро найти причину ошибки.

FAQ: Частые вопросы про Ozon Seller API

❓ Сколько стоит подключение к Ozon Seller API?

Само подключение к API бесплатное. Оплата может потребоваться только за услуги разработчика, если вы нанимаете специалиста для настройки интеграции.

❓ Можно ли использовать API для FBS и FBO?

Да, API поддерживает обе схемы: FBS (склад Ozon) и FBO (самостоятельная доставка). Однако некоторые методы могут отличаться — уточняйте в документации.

❓ Как часто обновляются данные в API?

Данные по заказам, остаткам и продажам обновляются в реальном времени (с задержкой до 5 минут). Финансовые отчёты могут обновляться реже — раз в несколько часов.

❓ Что делать, если API перестал работать?

Сначала проверьте:

  • 🔌 Статус работы API на странице статуса.
  • 🔑 Актуальность токена доступа.
  • 📡 Корректность запросов (возможно, изменилась структура API).

Если проблема не решена, обратитесь в поддержку Ozon.

❓ Можно ли через API управлять рекламными кампаниями?

Да, для этого есть отдельный Ozon Performance API. Он позволяет автоматизировать создание и управление рекламными кампаниями, получение статистики и оптимизацию ставок.
