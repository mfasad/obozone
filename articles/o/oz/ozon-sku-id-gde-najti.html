Почему SKU ID на Ozon — это ваш «паспорт товара»

Вы когда-нибудь теряли ключи от квартиры? Представьте, что SKU ID на Ozon — это точно такой же уникальный «ключ», только для вашего товара. Без него маркетплейс не сможет отличить вашу футболку с принтом «Кот учёный» от тысячи других, даже если они внешне идентичны. Но в отличие от домашних ключей, найти SKU можно несколькими способами — и мы разберём их все, от очевидных до скрытых.

Для продавцов SKU (Stock Keeping Unit) — это не просто набор цифр, а основа логистики, аналитики и даже репутации. Ошибётесь в одном символе — и ваш товар уедет на другой склад, зависнет в статусе «Неопознан» или, что хуже, будет продан по цене конкурента. А для покупателей знание SKU поможет отследить заказ, уточнить детали у поддержки или найти аналогичный товар после возврата. Так где же искать этот магический идентификатор?

Что такое SKU ID и чем он отличается от артикула продавца

Прежде чем бросаться на поиски, разберёмся с терминами. На Ozon вы столкнётесь с тремя типами идентификаторов:

  • 🔢 SKU ID (или Product ID) — уникальный номер, который присваивает Ozon каждому товару в своей базе. Например, 1234567890123. Используется для поиска в системе маркетплейса, интеграции с API, отчётах.
  • 🏷️ Артикул продавца — ваш внутренний код товара (например, FB-2026-BLUE-M). Вы указываете его при загрузке карточки, но Ozon может его игнорировать.
  • 📦 Штрихкод (EAN/UPC) — международный стандарт для розничной торговли. На Ozon не обязателен, но полезен для FBS.

Главное отличие: SKU ID генерирует Ozon автоматически при создании карточки товара, и изменить его нельзя. Артикул продавца вы придумываете сами — но он работает только внутри вашей системы.

Почему это важно? Представьте, что вы продаёте кроссовки Nike Air Max в трёх цветах. У вас один артикул для модели (NIKE-AM-2026), но Ozon присвоит каждому цвету отдельный SKU ID. Это позволяет маркетплейсу учитывать остатки по каждому варианту отдельно.

Где найти SKU ID в личном кабинете Ozon Seller

Самый надёжный способ — заглянуть в личный кабинет продавца. Вот пошаговая инструкция для разных разделов:

1. В списке товаров («Мой ассортимент»)

Перейдите в раздел Товары → Мой ассортимент. В таблице с товарами SKU ID отображается в колонке «ID» (иногда называется «Product ID»). Если колонки нет — нажмите на иконку шестерёнки ⚙️ в правом верхнем углу таблицы и добавьте её в отображение.

🔍 Совет: Используйте фильтр по названию или артикулу, чтобы быстро найти нужный товар. SKU ID копируется одним кликом по числу.

2. В карточке товара

Откройте любой товар из списка. В верхней части экрана, под названием, вы увидите блок с технической информацией. SKU ID будет указан рядом с надписью «Идентификатор товара» или «Product ID».

⚠️ Внимание: Не путайте SKU ID с Offer ID — это идентификатор конкретного предложения (например, товара в определённом размере или цвете). Они могут совпадать, если у товара нет вариаций.

3. В отчётах

Скачайте любой отчёт по товарам (например, Отчёт по остаткам или Отчёт по продажам). SKU ID будет в одной из первых колонок. Полезно, если нужно обработать данные массово (например, для интеграции с 1С).

Как узнать SKU ID через API Ozon

Если вы автоматизируете работу с маркетплейсом, SKU ID понадобится для запросов к API. Вот как его получить:

1. Используйте метод POST /v2/product/info с параметром product_id (это и есть SKU ID). Пример запроса:

{

"product_id": [1234567890123],


"offer_id": [""]


}

2. В ответе вы получите полные данные о товаре, включая SKU ID в поле product_id.

💡 Полезный лайфхак: Если вы работаете с вариациями товара (размеры, цвета), используйте offer_id — он уточняет конкретную модификацию. Например, для футболки размера M и цвета синий.

💡

Сохраните SKU ID всех ваших товаров в отдельную таблицу (Excel или Google Sheets). Это сэкономит время при работе с поддержкой или настройке рекламных кампаний.

SKU ID для покупателей: где (смотреть) в мобильном приложении и на сайте

Покупателям SKU ID нужен реже, но иногда без него не обойтись — например, чтобы:

  • 🔄 Отследить заказ, если в статусе ошибка («Товар не найден»).
  • 📞 Уточнить детали у поддержки (например, «Мне прислали не тот цвет — вот SKU ID заказанного товара»).
  • 🛒 Найти аналогичный товар после возврата.

Где искать:

На сайте Ozon (десктопная версия)

1. Откройте карточку товара.

2. Прокрутите вниз до блока «Характеристики».

3. Найдите строку «Код товара» или «Артикул» — это и есть SKU ID (иногда он скрыт под кнопкой «Показать все характеристики»).

В мобильном приложении

1. Откройте товар и тапните на три точки ⋮ в правом верхнем углу.

2. Выберите «Подробности».

3. Прокрутите до раздела «Информация о товаре» — SKU ID будет указан как Идентификатор товара.

⚠️ Внимание: В некоторых категориях (например, Электроника) SKU ID может совпадать со штрихкодом (EAN). Убедитесь, что вы копируете именно идентификатор Ozon, а не производителя.

Типичные ошибки с SKU ID и как их избежать

Даже опытные продавцы иногда путаются в идентификаторах. Вот самые распространённые ошибки и их решения:

Ошибка Причина Как исправить
Товар не находится по SKU ID Копируете Offer ID вместо Product ID Проверьте формат: SKU ID — это 13 цифр, Offer ID может содержать буквы
Остатки не обновляются В файле импорта указан неверный SKU ID Сверьте данные с отчётом Ассортимент в ЛК
Товар дублируется Создали новую карточку вместо редактирования старой Объедините карточки через поддержку, указав оба SKU ID

🔥 Критическая ошибка: Никогда не используйте SKU ID от одного товара для другого — даже если они похожи. Ozon блокирует такие манипуляции и может приостановить аккаунт за «подмену ассортимента».

Что делать, если SKU ID потерян?

Если вы удалили товар из ассортимента, его SKU ID остаётся в базе Ozon навсегда. Чтобы восстановить доступ, обратитесь в поддержку с темой «Восстановление карточки товара» и укажите старый SKU ID (если помните) или артикул продавца.

SKU ID в схеме FBS vs FBO: ключевые различия

Тип логистики влияет на то, как используется SKU ID:

  • 🚚 FBS (Fulfillment by Ozon):

    • SKU ID привязан к конкретному складу. Если вы перемещаете товар между складами Ozon, может сгенерироваться новый идентификатор (например, для Ozon Рязань и Ozon Новосибирск). Всегда уточняйте актуальный SKU ID в отчёте Остатки на складах.

  • 🏠 FBO (Fulfillment by Seller):

    • SKU ID остаётся неизменным, но вы должны самостоятельно следить за его соответствием в вашей системе учёта и на Ozon. При расхождениях (например, если вы отправили товар с неверным SKU) маркетплейс может наложить штраф за «несоответствие заказа».

📌 Важно для FBS: Если вы работаете с мультискладской схемой, один и тот же товар может иметь разные SKU ID на разных складах. Например:

  • 1234567890123 — SKU ID на складе в Москве,
  • 1234567890124 — тот же товар, но на складе в Екатеринбурге.
💡

В FBS SKU ID может меняться при перемещении товара между складами Ozon. Всегда проверяйте актуальный идентификатор перед отправкой партии.

FAQ: Частые вопросы о SKU ID на Ozon

Можно ли изменить SKU ID после создания карточки товара?

Нет, SKU ID присваивается автоматически при создании карточки и остаётся неизменным. Если вам нужно «обновить» идентификатор, создайте новую карточку (со всеми вытекающими рисками для репутации и продаж).

Чем отличается SKU ID от артикула продавца?

SKU ID — это внутренний идентификатор Ozon, а артикул продавца — ваш собственный код. Например, у вас может быть артикул TSHIRT-RED-M, а Ozon присвоит этому товару SKU ID 1234567890123. Артикул можно редактировать, SKU ID — нет.

Где взять SKU ID для товара, который ещё не загружен на Ozon?

SKU ID генерируется только после создания карточки товара. Если товар ещё не загружен, у него нет SKU ID. Вам нужно сначала добавить товар в ассортимент через ЛК или API.

Можно ли использовать один SKU ID для разных товаров?

Абсолютно нет! Это грубое нарушение правил Ozon, которое ведёт к блокировке аккаунта. Каждый товар (и даже каждая его вариация) должен иметь уникальный SKU ID.

Как найти SKU ID в чеке или накладной?

В документах для FBS (например, в акте приёмки или накладной) SKU ID указывается в колонке «Код товара» или «Product ID». В чеках для покупателей он обычно не отображается — только артикул продавца.
