Ozon SKU ID — что это такое и где его взять: полное руководство 2026

Если вы хоть раз сталкивались с продажей или покупкой товаров на Ozon, то наверняка видели загадочный набор цифр и букв вроде SKU123456789 или OZON-987654321. Это и есть SKU ID — уникальный идентификатор товара, который играет ключевую роль в работе маркетплейса. Без него невозможно добавить товар в каталог, отследить заказ или решить проблему с возвратом.

Для продавцов SKU ID на Озон — это не просто номер, а основа для управления ассортиментом, анализа продаж и интеграции с системами учета. Для покупателей — способ быстро найти нужный товар или уточнить детали заказа. Но где взять этот идентификатор? Как он формируется? И почему иногда один и тот же товар имеет несколько SKU? Разбираемся по порядку.

Что такое SKU ID на Озон и зачем он нужен

SKU (от англ. Stock Keeping Unit) — это уникальный код товара, который присваивается каждой позиции в каталоге Ozon. На других маркетплейсах (например, Wildberries или Яндекс Маркет) аналогичные идентификаторы могут называться по-другому, но суть одна: это "паспорт" товара, по которому система отличает одну модель от другой.

На Ozon SKU ID выглядит как комбинация букв и цифр, например:

  • 🔢 SKU12345678 — стандартный формат для товаров, загруженных продавцами;
  • 📦 OZON-987654321 — формат для товаров, которые Ozon закупает напрямую у брендов;
  • 🔄 FB12345678 — SKU для товаров на схеме FBS (фульфилмент от Ozon).

Основные функции SKU ID:

  • 🔍 Идентификация товара — система отличает iPhone 15 Pro 128GB от iPhone 15 Pro 256GB даже если они внешне одинаковы;
  • 📊 Учет остатков — SKU привязан к количеству товара на складе;
  • 🛒 Оформление заказов — без SKU невозможно добавить товар в корзину;
  • 🔄 Возвраты и обмены — по SKU определяется, какой именно товар вернул покупатель.

⚠️ Внимание: Один и тот же товар может иметь несколько SKU, если он продается в разных вариациях (цвет, размер, комплектация). Например, футболка Nike синего и красного цвета будут иметь разные SKU, даже если модель одна.

Где взять SKU ID на Озон: 5 способов для продавцов

Если вы продавец, то получить SKU ID можно несколькими способами в зависимости от того, как вы добавляете товары в каталог. Рассмотрим все варианты:

1. Автоматическое присвоение при загрузке товара

Когда вы добавляете новый товар через Личный кабинет продавца (ЛКП) или API, система Ozon автоматически генерирует SKU ID. Его можно найти:

  • 📋 В таблице Мой ассортимент → Товары (столбец "Артикул");
  • 📄 В карточке товара (раздел "Основная информация");
  • 📥 В файле выгрузки при экспорте каталога.

2. Ручное создание SKU для существующих товаров

Если вы продаете товар, который уже есть в каталоге Ozon (например, брендовые товары), вы можете "привязаться" к существующему SKU. Для этого:

  1. Перейдите в Мой ассортимент → Добавить товар;
  2. Введите название или арт. производителя в строку поиска;
  3. Выберите нужный товар из выпадающего списка — система подтянет существующий SKU.

Важно: Если вы продаете товар с уникальными характеристиками (например, кастомный чехол для телефона), придется создавать новый SKU.

3. SKU ID в файлах выгрузки и отчетах

SKU можно найти в следующих отчетах:

  • 📈 Отчет по остаткам (Аналитика → Остатками);
  • 📦 Отчет по заказам (Заказы → Экспорт);
  • 🔄 Отчет по возвратам (Возвраты → История).

Во всех этих файлах SKU будет в отдельном столбце (обычно называется sku, product_id или offer_id).

4. SKU ID в API Ozon

Если вы используете API Ozon для управления товарами, SKU передается в следующих методах:

  • 🔧 POST /v1/product/import — создание товара;
  • 📋 GET /v1/product/info — получение информации о товаре;
  • 📦 POST /v2/warehouse/stocks — обновление остатков.

5. SKU ID в мобильном приложении Ozon Seller

В приложении для продавцов SKU можно найти:

  • 📱 В разделе Товары → Мой ассортимент;
  • 🔍 При поиске товара по названию (SKU отображается под названием).

⚠️ Внимание: Если вы продаете товар по модели FBO (самостоятельная доставка), SKU будет отличаться от того, что используется в FBS (доставка через Ozon). Убедитесь, что вы используете правильный идентификатор для своей схемы работы.

Как формируется SKU ID на Озон: структура и примеры

SKU ID на Ozon не присваивается случайно — он имеет определенную структуру, которая помогает системе быстро идентифицировать товар. Разберем, как он формируется:

Структура SKU ID

Стандартный SKU на Ozon состоит из:

  • 🔤 Префикса — обычно SKU или OZON-;
  • 🔢 Уникального числового кода — от 6 до 10 цифр;
  • 🔤 Суффикса (опционально) — может содержать буквы для обозначения вариаций (например, -BL для синего цвета).

Примеры:

  • 📱 SKU12345678 — стандартный SKU для товара;
  • 👕 SKU12345678-BL — вариация товара (синий цвет);
  • 📦 OZON-987654321 — SKU для товара, который Ozon закупает напрямую.

Как Ozon генерирует SKU

Алгоритм генерации SKU зависит от того, как товар попадает в каталог:

  • 🔄 Для новых товаров — система присваивает следующий свободный номер в последовательности;
  • 📦 Для существующих товаров — используется SKU, который уже есть в базе;
  • 🏷️ Для вариаций — к основному SKU добавляется суффикс (например, -RED для красного цвета).

Важно: SKU не меняется со временем — он остается привязанным к товару навсегда, даже если вы удалите товар и добавите его снова.

SKU ID для покупателей: где его найти в заказе

Покупателям SKU ID может понадобиться для:

  • 🔍 Уточнения деталей заказа (например, если в корзине было несколько похожих товаров);
  • 🔄 Оформления возврата или обмена;
  • 📞 Обращения в поддержку Ozon с вопросом по товару.

Где покупатель может найти SKU:

  • 📄 В карточке товара — под названием или в разделе "Характеристики";
  • 📦 В информации о заказе — на странице заказа или в письме с подтверждением;
  • 📱 В мобильном приложении — в разделе "Мои заказы" → выберите заказ → "Детали заказа".

Частые ошибки с SKU ID и как их избежать

Работа с SKU только кажется простой — на практике продавцы и покупатели часто сталкиваются с проблемами. Разберем самые распространенные ошибки:

1. Путаница между SKU и арт. производителя

Многие продавцы думают, что артикул производителя (например, IPHONE-15-PRO-128) и SKU Ozon — это одно и то же. На самом деле:

  • 🏷️ Артикул производителя — внутренний код бренда;
  • 🔢 SKU Ozon — уникальный идентификатор в каталоге маркетплейса.

Как избежать: Всегда используйте SKU Ozon для работы с заказами, остатками и возвратами. Артикул производителя нужен только для поиска товара в каталоге.

2. Дублирование SKU для разных товаров

Если вы случайно присвоите один и тот же SKU двум разным товарам, система Ozon воспримет их как один товар. Это приведет к:

  • 📉 Путанице в остатках;
  • 🛒 Ошибкам при оформлении заказов;
  • 🔄 Проблемам с возвратами.

Как избежать: Перед загрузкой товара проверяйте, не используется ли этот SKU для другого товара. В ЛКП это можно сделать через поиск по SKU в разделе Мой ассортимент.

3. Ошибки в SKU при интеграции с 1С или другими системами

Если вы выгружаете товары из 1С, МойСклад или другой программы, может произойти:

  • 🔄 Замена символов (например, дефис - превращается в подчеркивание _);
  • 📛 Обрезка SKU (если поле в системе ограничено по длине);
  • 🔢 Потеря ведущих нулей (например, SKU00123456 становится SKU123456).

Как избежать: Настройте правильный формат экспорта SKU в вашей системе учета. Используйте текстовый формат (не числовой) для поля SKU.

Что делать, если SKU в 1С и на Ozon не совпадают?

Если в вашей системе учета используется свой формат артикулов, а на Ozon — другой, можно настроить сопоставление через дополнительное поле. Например, в 1С создать колонку "SKU_Ozon" и выгружать ее отдельно.

4. Использование SKU из других маркетплейсов

Некоторые продавцы пытаются использовать SKU с Wildberries или Яндекс Маркета на Ozon. Это приводит к ошибкам, так как:

  • 🔢 Форматы SKU на разных площадках отличаются;
  • 📛 Система Ozon не распознает чужие идентификаторы;
  • 🔄 Может произойти конфликт с уже существующими SKU в каталоге.

Как избежать: Всегда создавайте отдельные SKU для каждого маркетплейса. Если вы ведете продажи на нескольких площадках, используйте префиксы (например, OZ-SKU123456, WB-SKU123456).

💡

SKU на Ozon — это внутренний идентификатор маркетплейса. Он не должен совпадать с артикулами других площадок или производителя.

SKU ID и аналитика: как использовать данные для роста продаж

SKU — это не только технический идентификатор, но и мощный инструмент для анализа продаж. Вот как его можно использовать:

1. Отслеживание популярных товаров

С помощью SKU вы можете:

  • 📈 Видеть, какие товары продаются лучше всего (в отчетах по продажам);
  • 🔍 Анализировать динамику спроса по конкретным SKU;
  • 📊 Сравнивать продажи разных вариаций одного товара (например, какой цвет популярнее).

Пример: Если SKU SKU12345678-BL (синяя футболка) продается в 3 раза чаще, чем SKU12345678-RD (красная), стоит сделать упор на закупках синего цвета.

2. Управление остатками

SKU привязан к остаткам на складе, поэтому вы можете:

  • 📦 Видеть, какие товары заканчиваются;
  • 🔄 Автоматизировать заказы у поставщиков;
  • 🚫 Избегать оверстока (избыточных остатков).

В ЛКП остатки по SKU можно посмотреть в разделе Аналитика → Остатками.

3. Анализ возвратов

По SKU можно отследить:

  • 🔄 Какие товары возвращают чаще всего;
  • 📉 Причины возвратов (брак, несоответствие описанию и т.д.);
  • 💰 Потери от возвратов по каждому SKU.

Эти данные помогут улучшить карточки товаров или пересмотреть ассортимент.

4. Интеграция с рекламными кампаниями

В Ozon можно настроить рекламу на конкретные SKU. Это позволяет:

  • 🎯 Продвигать лучшие товары;
  • 📈 Тестировать разные вариации (например, какой цвет лучше конвертируется);
  • 💰 Оптимизировать бюджет, отключая рекламу на неликвидные SKU.

Пример: Если SKU SKU98765432 имеет высокий CTR (кликабельность), но низкую конверсию, возможно, проблема в карточке товара или цене.

FAQ: Частые вопросы о SKU ID на Ozon

Можно ли изменить SKU после создания товара?

Нет, SKU на Ozon присваивается один раз и не меняется. Если вам нужно обновить идентификатор, придется создать новый товар с новым SKU и удалить старый (или скрыть его из продажи).

Что делать, если SKU потерялся или не отображается?

Если вы не можете найти SKU:

  1. Проверьте раздел Мой ассортимент → Товары;
  2. Экспортируйте каталог в Excel — SKU будет в отдельном столбце;
  3. Обратитесь в поддержку Ozon с названием товара и арт. производителя.

Можно ли использовать один SKU для разных маркетплейсов?

Не рекомендуется. Каждый маркетплейс имеет свой формат SKU, и использование чужого идентификатора может привести к ошибкам. Лучше создавать уникальные SKU для Ozon, Wildberries, Яндекс Маркета и т.д.

Как добавить вариации товара (цвет, размер) с разными SKU?

При добавлении товара с вариациями:

  1. Создайте основной товар (например, футболка);
  2. Добавьте вариации (цвет, размер);
  3. Система автоматически сгенерирует SKU для каждой вариации (например, SKU12345678-BL, SKU12345678-RD).

Что такое FB SKU и чем он отличается от обычного?

FB SKU (например, FB12345678) — это идентификатор для товаров на схеме FBS (фульфилмент от Ozon). Он отличается от обычного SKU тем, что:

  • Используется только для товаров, которые хранятся на складах Ozon;
  • Привязан к логистическим процессам (упаковка, доставка);
  • Может содержать дополнительную информацию для склада (например, габариты).
