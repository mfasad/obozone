Где найти OZON ID: все способы для покупателей и продавцов на маркетплейсе

  • 1. OZON ID для покупателей: где смотреть в заказах

Идентификатор OZON ID — это уникальный цифровой код, который присваивается каждому товару, заказу, продавцу и даже покупателю на платформе. Без него невозможно отследить статус заказа, решить спор с поддержкой или интегрировать системы учета для бизнеса. Но где именно искать этот идентификатор? Ответ зависит от того, кто вы — покупатель, начинающий продавец или опытный селлер с автоматизированными процессами.

В этой статье мы разберем все актуальные способы поиска OZON ID в 2026 году: от очевидных (личный кабинет и мобильное приложение) до малоизвестных (API-документация и экспорт отчетов). Особое внимание уделим нюансам для разных типов пользователей — например, почему продавцы видят больше идентификаторов, чем покупатели, и как не перепутать product_id с offer_id. Также вы узнаете, что делать, если IDSuddenly "исчез" из интерфейса (спойлер: это случается после обновлений платформы).

Если вы здесь потому, что поддержка Озон запросила у вас ID для разрешения спора — welcome, вы в правильном месте. Если же вы селлер, который настраивает интеграцию с 1С или МойСклад, приготовьтесь к глубокому дайвингу в технические детали. Начнем с самого простого.

1. OZON ID для покупателей: где смотреть в заказах

Покупателям идентификатор заказа (posting_number) нужен в трех случаях: чтобы уточнить статус доставки, вернуть товар или доказать факт покупки в споре. Найти его проще всего — он отображается прямо в истории заказов, но не всегда очевидно, какую именно цифру копировать.

Вот пошаговая инструкция для десктопной и мобильной версии:

  • 🖥️ На компьютере: Перейдите в раздел Мои заказы → выберите нужный заказ → в блоке "Информация о заказе" найдите строку Номер заказа. Это и есть ваш OZON ID (формат: 123456789-1234-1234).
  • 📱 В мобильном приложении: Откройте вкладку Заказы → тапните на интересующий заказ → прокрутите до блока "Детали заказа". Идентификатор будет под названием Номер отправления.
  • 📧 В письме от Озон: В теме письма с подтверждением заказа всегда указан Posting Number (например, Ваш заказ #WB-123456789 — это НЕ ID, а номер заказа в системе Озон; нужный идентификатор ищите в теле письма).

Важный нюанс: если вы оплатили заказ картой, в выписке банка может значиться другой номер (например, OZON*1234567). Это не ID заказа, а внутренний код транзакции — его поддержка не примет для идентификации покупки.

⚠️ Внимание: Если вы видите в личном кабинете статус "Заказ сформирован", но ID еще не присвоен — подождите 10-15 минут. Идентификатор генерируется только после подтверждения оплаты.

2. OZON ID для продавцов: разница между product_id, offer_id и posting_number

Продавцам на Озон приходится работать с тремя типами идентификаторов, и путать их — критичная ошибка. Вот чем они отличаются:

Тип ID Формат Где используется Пример
product_id Цифровой, 6-8 знаков Идентификатор карточки товара (одна карточка = один ID) 12345678
offer_id Цифровой, 10+ знаков Идентификатор торгового предложения (один товар в разных цветах/размерах = разные offer_id) 9876543210
posting_number Цифры и дефисы, 16+ знаков Номер отправления (привязан к конкретному заказу покупателя) 123456789-1234-5678

Где искать каждый из них:

  • 📦 product_id: В личном кабинете продавца перейдите в Товары → Мои товары, откройте карточку товара. ID будет в адресной строке браузера после /product/ (например, .../product/12345678/edit).
  • 🔖 offer_id: В том же разделе Мои товары экспортируйте отчет "Список товаров" (формат Excel). Искомый ID будет в колонке Offer ID. Для API-интеграции используйте метод /v2/product/info.
  • 🚚 posting_number: В разделе Заказы → Все заказы откройте любой заказ. ID отображается в колонке Номер отправления.

Если вы работаете через Ozon Seller API, все три типа ID можно получить программно. Например, для получения posting_number используйте метод:

GET /v2/posting/fbs/list
⚠️ Внимание: При интеграции с 1С или МойСклад убедитесь, что в настройках синхронизации указан правильный тип ID. Например, для обновления остатков нужен offer_id, а для отмены заказа — posting_number.

3. Как найти OZON ID в мобильном приложении (скрытые способы)

Мобильное приложение Озон часто обновляется, и расположение идентификаторов может меняться. Актуальные на 2026 год лайфхаки:

Для покупателей:

  1. Откройте заказ → тапните на три точки в верхнем правом углу → выберите Подробности. В блоке Информация о заказе будет строка Номер отправления.
  2. Если заказ еще не отправлен, ID может отсутствовать. В этом случае скопируйте Номер заказа (формат WB-123456789) и уточните у поддержки, когда будет сгенерирован posting_number.

Для продавцов:

  • 📲 В приложении Ozon Seller перейдите в раздел Заказы → выберите заказ → прокрутите до блока Детали. Здесь отображаются оба ID: Номер отправления (posting_number) и Артикул товара (offer_id).
  • 🔍 Чтобы найти product_id, откройте раздел Товары → выберите товар → в адресной строке браузера (если открыли через "Открыть в браузере") будет ID.

В последних версиях приложения (сентябрь 2026) идентификаторы в разделе "Товары" скрыты за кнопкой "Еще" (три точки). Если не видите ID — обновите приложение или используйте десктопную версию.

💡

Если в мобильном приложении не отображается ID, попробуйте открыть ту же страницу в браузере через режим "Для компьютера" — иногда это решает проблему.

4. Поиск OZON ID через API и экспорт отчетов

Для продавцов с большим ассортиментом ручной поиск идентификаторов неэффективен. Автоматизировать процесс помогают:

  • 📊 Экспорт отчетов: В личном кабинете перейдите в Аналитика → Отчеты → Список товаров. В экспортированном Excel-файле будут колонки Product ID, Offer ID и Article (артикул продавца).
  • 🤖 API-запросы: Для получения posting_number используйте метод /v2/posting/fbs/list с параметром since (дата начала периода). Пример запроса: 
    {

    "dir": "ASC",
    

    "filter": {
    

    "since": "2026-01-01T00:00:00Z"
    

    },
    

    "limit": 1000
    

    }

  • 🔄 Webhooks: Настройте уведомления о новых заказах через /v1/webhook/subscription. В теле уведомления будет posting_number.

Если вы используете Ozon Global, учтите, что формат posting_number может включать префикс региона (например, KZ-123456789 для Казахстана). Это важно для корректной обработки заказов в мультирегиональных интеграциях.

⚠️ Внимание: При работе с API ограничение на количество запросов — 1000 в минуту. Если превысите лимит, получите ошибку 429 Too Many Requests. Используйте пагинацию (limit и offset) для больших выборок.

5. Где искать OZON ID в карточке товара (для селлеров и конкурентного анализа)

Продавцам часто нужно узнать product_id чужого товара — например, для анализа конкурентов или подачи жалобы на нарушение правил. Способы:

На десктопе:

  1. Откройте карточку товара на Ozon.ru.
  2. Прокрутите страницу до блока с характеристиками.
  3. Нажмите Ctrl+U (исходный код страницы) → Ctrl+F → введите "productId". Искомый ID будет в формате "productId":12345678.

В мобильном браузере:

  • 📱 Откройте карточку товара → тапните на три точки в браузере → выберите Версия для компьютера.
  • 🔍 В адресной строке после /product/ будет product_id (например, .../product/12345678).

Для массового сбора ID конкурентов используйте парсеры (например, Ozon Parser или DataCol). Учтите, что автоматизированный сбор данных может нарушать правила Озон — проверьте раздел Правила для продавцов → Запрещенные действия.

💡

Сбор чужих product_id вручную (до 50 штук в день) не нарушает правила Озон. Автоматизированный парсинг требует согласования с поддержкой.

6. Частые ошибки при работе с OZON ID и как их избежать

Даже опытные продавцы иногда путают идентификаторы или используют их не по назначению. Распространенные ошибки:

  • 🔄 Подмена offer_id на product_id: При обновлении цен через API некоторые селлеры отправляют product_id вместо offer_id, из-за чего изменения не применяются. Всегда проверяйте формат ID перед отправкой запроса.
  • 📦 Игнорирование posting_number при возвратах: Для оформления возврата нужен именно posting_number, а не номер заказа. Если указать неверный ID, заявка будет отклонена.
  • 🔍 Поиск по артикулу продавца: Внутри Озон артикул (article) — это внутренний код селлера, а не ID платформы. Для API-запросов он не подходит.

Как избежать ошибок:

  1. Всегда сверяйте формат ID с документацией API.
  2. Для критичных операций (отмена заказа, возврат) копируйте ID напрямую из личного кабинета, а не из экспортированных отчетов (там могут быть устаревшие данные).
  3. Используйте валидатор ID от Озон: GET /v1/product/info/validate?id={your_id}.
⚠️ Внимание: Если вы передаете неверный posting_number при интеграции с курьерской службой (например, СДЭК или Boxberry), заказ может быть потерян. Всегда проверяйте ID через метод /v2/posting/fbs/get перед передачей.

7. Что делать, если OZON ID не отображается или "исчез"

Иногда идентификаторы пропадают из интерфейса после обновлений Озон. Вот что проверять:

  • 🔄 Обновите страницу: Нажмите Ctrl+F5 (жесткое обновление кэша).
  • 📱 Смените устройство: Проверьте отображение ID на другом гаджете или в другом браузере.
  • 📧 Проверьте письма: Озон дублирует posting_number в письмах о статусе заказа (папка "Спам" тоже!).
  • 🛠️ Обратитесь в поддержку: Если ID не видно более суток, напишите в чат с указанием:
    • Номера заказа (формат WB-123456789)
    • Скриншота страницы без ID
    • Браузера и версии ОС

Для продавцов: если пропали product_id или offer_id в личном кабинете, проверьте:

  1. Не истек ли срок действия товара (в разделе Товары → Архив).
  2. Не заблокирован ли товар модерацией (раздел Товары → На модерации).
  3. Не изменилась ли категория товара (иногда при переносе в другую категорию ID остается, но отображается в другом разделе).

Если проблема массовая (исчезли ID у всех товаров), проверьте статус сервисов Озон на странице https://status.ozon.ru/. Возможно, идут технические работы.

FAQ: Ответы на частые вопросы про OZON ID

Можно ли по OZON ID узнать, кто покупатель?

Нет, posting_number не содержит данных о покупателе. Чтобы увидеть имя и контакты, откройте заказ в личном кабинете продавца — там будут данные только для подтвержденных заказов (в соответствии с политикой конфиденциальности Озон).

Как связаны OZON ID и штрихкод товара?

product_id и штрихкод (barcode) — разные вещи. Штрихкод присваивается производителем и может совпадать для одинаковых товаров разных продавцов. А product_id уникален для каждой карточки на Озон. Например, один и тот же iPhone 15 от разных селлеров будет иметь одинаковый штрихкод, но разные product_id.

Меняется ли OZON ID после редактирования товара?

product_id и offer_id остаются неизменными при редактировании карточки (цена, описание, фото). Но если вы удалите товар и создадите новый (даже с теми же характеристиками), ему будет присвоен новый ID. Исключение: при смене категории товар может получить новый product_id.

Можно ли восстановить удаленный OZON ID?

Если товар удален, его product_id и offer_id становятся неактивными, но остаются в базе Озон. Вы можете найти их в отчетах (раздел Аналитика → Архив товаров) или через API-запрос /v1/product/info/archive. Однако восстановить товар по старому ID нельзя — придется создавать новую карточку.

Как по OZON ID проверить оригинальность товара?

Сам по себе product_id не гарантирует оригинальность. Но вы можете:

  1. Проверьте историю изменения цен по ID через Ozon Trends (резкие скачки цены могут указывать на подделку).
  2. Сравните фотографии товара с официального сайта бренда и карточки на Озон (по product_id).
  3. Используйте сервис Brand Analytics (доступен продавцам с рейтингом выше 4.5).

Для 100% проверки запросите у продавца сертификаты (через чат Озон).
