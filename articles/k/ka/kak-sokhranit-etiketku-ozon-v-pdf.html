Как сохранить этикетку Озон в PDF: все способы для продавцов и логистов

📋 Содержание
  • Почему важно сохранять этикетки в PDF, а не в JPG/PNG
  • Способ 1: Сохранение через личный кабинет Ozon Seller (базовый метод)
  • Способ 2: Автоматизация через API Ozon (для крупных продавцов)
  • Способ 3: Использование мобильного приложения Ozon Seller
  • Способ 4: Стороние сервисы для работы с этикетками
  • Способ 5: Ручное создание PDF из Excel (для продвинутых пользователей)
  • Частые ошибки и как их избежать
  • FAQ: Ответы на частые вопросы

Этикетки Ozon — обязательный атрибут для отправки товаров на склады маркетплейса, но их сохранение в удобном формате часто вызывает вопросы. Продавцы сталкиваются с проблемами: печать напрямую из личного кабинета сбивает масштаб, а ручное копирование данных в Excel отнимает часы. В этой статье разберём все актуальные способы сохранения этикеток в PDF — от стандартных инструментов Ozon Seller до автоматизации через API и сторонние сервисы.

Особенно критично правильное оформление этикеток для продавцов на схеме FBS (когда логистику берет на себя Ozon): ошибка в штрихкоде или размере может привести к штрафам за некорректную маркировку. Мы проанализировали реальные кейсы продавцов 2026 года, где неправильное сохранение этикеток приводило к задержкам на складах до 3 дней. В статье — только проверенные методы, адаптированные под последние обновления интерфейса Ozon Seller.

Почему важно сохранять этикетки в PDF, а не в JPG/PNG

Формат PDF — единственный рекомендованный Ozon для печати этикеток, и на это есть объективные причины:

  • 📏 Сохранение масштаба: PDF гарантирует, что штрихкод отсканируется с первого раза (в JPG/PNG при печати возможны искажения до 15%).
  • 🖨️ Печать без потерь: векторный формат PDF не "пикселится" при увеличении, в отличие от растровых изображений.
  • 📊 Пакетная обработка: в одном PDF можно объединить этикетки для 100+ товаров (в Excel это требует ручного форматирования).
  • 🔒 Защита от редактирования: PDF сложнее подделать, чем изображение в графическом редакторе.

По данным Ozon за 2023 год, 37% возвратов товаров от покупателей происходило из-за нечитаемых штрихкодов на этикетках. В 80% случаев проблема была связана с печають из неправильного формата (например, сжатый JPEG с разрешением ниже 300 dpi). PDF исключает эту ошибку.

⚠️ Внимание: Если вы печатаете этикетки на термопринтере, проверьте, поддерживает ли он PDF Direct Printing. Некоторые модели (например, Zebra GX420t) требуют предварительной конвертации в ZPL-формат.

Способ 1: Сохранение через личный кабинет Ozon Seller (базовый метод)

Самый простой способ, не требующий дополнительных инструментов. Подходит для продавцов с небольшим ассортиментом (до 50 SKU в день).

  1. Перейдите в раздел Товары → Этикетки в личном кабинете Ozon Seller.
  2. Отметьте галочками нужные товары или используйте фильтр по SKU/артикулу.
  3. Нажмите кнопку Скачать этикетки и выберите формат PDF.
  4. В открывшемся окне укажите:
    • 📄 Количество копий (по умолчанию — 1, но для FBS рекомендуется 2).
    • 📦 Размер этикетки: 100×70 мм (стандарт Ozon) или 100×150 мм (для габаритных товаров).
    • 🖼️ Ориентация: Книжная (для большинства принтеров).
  • Подтвердите скачивание. Файл будет называться ozon_labels_{дата}.pdf.

    • Ограничения метода:

    • ❌ Максимум 200 этикеток за один запрос (при превышении — разбейте на партии).
    • ❌ Нет возможности кастомизировать дизайн (например, добавить логотип магазина).
    • ❌ В PDF не включаются данные о сроке годности (актуально для продовольственных товаров).

    Отфильтруйте товары по статусу "Готов к отправке"

    Проверьте корректность SKU в карточках товаров

    Выберите размер этикетки 100×70 мм для стандартных товаров

    Убедитесь, что в настройках принтера отключено "Подогнать под размер страницы"

    Способ 2: Автоматизация через API Ozon (для крупных продавцов)

    Если вы отправляете 1000+ заказов в месяц, ручное сохранение этикеток станет бутылочным горлышком. Решение — интеграция с API Ozon через метод /v2/label/print. Этот способ требует базовых знаний Python или использования сервисов-посредников типа AltoAPI.

    Пример запроса для генерации PDF с этикетками:

    POST https://api-seller.ozon.ru/v2/label/print

    Headers:
    

    - Client-Id: [ваш_client_id]
    

    - Api-Key: [ваш_api_key]
    

    Body:
    

    {
    

    "label_format": "PDF",
    

    "label_size": "100x70",
    

    "shipping_method_id": [ID_способа_доставки],
    

    "orders": [
    

    {
    

    "order_id": 123456789,
    

    "packages": [
    

    {
    

    "package_id": "PKG123",
    

    "items": [
    

    {
    

    "sku": 100500,
    

    "quantity": 2
    

    }
    

    ]
    

    }
    

    ]
    

    }
    

    ]
    

    }

    Преимущества API-метода:

    • ⚡ Скорость: генерация 1000 этикеток за 2-3 минуты (против 1-2 часов вручную).
    • 🤖 Автоматизация: можно настроить триггеры (например, создавать PDF при смене статуса заказа на "Готов к отправке").
    • 📈 Кастомизация: добавление в PDF дополнительных данных (вес, габариты, комментарии для склада).

    ⚠️ Внимание: При работе с API следите за лимитами запросов: для метода /v2/label/print действует ограничение 100 запросов в минуту. Превышение ведёт к блоку аккаунта на 1 час.
    Пример кода на Python для массового скачивания этикеток

    import requests

    url = "https://api-seller.ozon.ru/v2/label/print"

    headers = {

    "Client-Id": "ваш_client_id",

    "Api-Key": "ваш_api_key",

    "Content-Type": "application/json"

    }

    payload = {

    "label_format": "PDF",

    "label_size": "100x70",

    "orders": [...] # список заказов

    }

    response = requests.post(url, headers=headers, json=payload)

    with open("labels.pdf", "wb") as f:

    f.write(response.content)

    Способ 3: Использование мобильного приложения Ozon Seller

    Мобильное приложение Ozon Seller (доступно для Android и iOS) позволяет сохранять этикетки в PDF прямо со смартфона. Это удобно для продавцов, которые управляют бизнесом "в поле" — например, на складе или в пункте выдачи.

    Инструкция:

    1. Откройте приложение и перейдите в раздел Заказы → Готовы к отправке.
    2. Выберите заказ и нажмите Этикетка.
    3. В открывшемся окне тапните на иконку ⬇️ Скачать и выберите PDF.
    4. Файл сохранится в папку Download/OzonLabels на вашем устройстве.

    Ограничения мобильного метода:

    • ❌ Нет пакетного скачивания — только по одной этикетке за раз.
    • ❌ На iOS возможны проблемы с открытием PDF в сторонних приложениях (решается установкой Adobe Acrobat Reader).
    • ❌ Этикетки сохраняются без водяных знаков, что повышает риск подделки при передаче файлов третьим лицам.

    MoySklad От 2500 ₽/мес Синхронизация с 1С, печать этикеток партиями, контроль остатков Да (через API)

    • 🔄 Синхронизацию с Ozon: официальные партнёры (AltoAPI, RetailCRM) обновляют интеграцию автоматически при изменениях в API маркетплейса.
    • 📦 Поддержку FBS/FBO: некоторые сервисы оптимизированы только под одну схему работы.

    Способ 5: Ручное создание PDF из Excel (для продвинутых пользователей)

    Если вам нужно кастомизировать дизайн этикеток (например, добавить логотип или изменить шрифт), можно сгенерировать PDF из Excel с помощью надстройки Kutools или макросов VBA.

    Пошаговая инструкция:

    1. Экспортируйте данные этикеток из Ozon Seller в CSV (раздел Товары → Экспорт).
    2. Откройте файл в Excel и создайте шаблон этикетки:
      • 📌 Ячейка A1: Штрихкод (шрифт IDAutomationHC39M для корректного сканирования).
      • 📌 Ячейка A2: Название товара (размер шрифта — 10 пт).
      • 📌 Ячейка A3: Артикул и SKU.
  • Установите надстройку Kutools for Excel и выберите Printing → Batch Print.
  • В настройках печати укажите: 
    Формат листа: A4

    Ориентация: Альбомная
    

    Поля: 0 мм
    

    Масштаб: 100%
    

    Количество копий: 1 (для теста)

  • Нажмите Экспорт в PDF и сохраните файл.

    • Предупреждения:

    • ⚠️ Штрихкод должен быть в формате Code 128 или EAN-13 — другие типы (QR, DataMatrix) не поддерживаются складами Ozon.
    • ⚠️ Проверьте разрешение печати: минимальное требование — 300 dpi (при 150 dpi штрихкод может не считываться).

    💡

    Используйте шрифт IDAutomationHC39M для штрихкодов — он гарантирует 100% читаемость сканерами Ozon. Скачать его можно бесплатно на сайте разработчика.

    Частые ошибки и как их избежать

    Даже опытные продавцы допускают ошибки при работе с этикетками. Вот топ-5 проблем и их решения:

    1. Штрихкод не сканируется

      Причины:

      • 🔍 Некорректный формат (например, сохранён как изображение с низким разрешением).
      • 🔍 Неправильный размер этикетки (стандарт — 100×70 мм, но для некоторых категорий требуется 100×150 мм).
      • 🔍 Печать на глянцевой бумаге (сканеры Ozon лучше работают с матовыми поверхностями).

    Решение: Распечатайте тестовую этикетку на лазерном принтере с разрешением 600 dpi и проверьте сканирование бесплатным приложением Barcode Scanner.

  • В PDF отсутствуют данные о товаре

    Причина: При экспорте из Ozon Seller не были выбраны все необходимые поля. Решение — вручную добавить недостающие данные через Excel или повторить экспорт с правильными настройками.

  • Этикетки сохраняются в одном файле без разделителей

    Решение: Используйте API с параметром "separate_files": true или разбейте PDF на страницы в Adobe Acrobat (Инструменты → Организовать страницы → Разделить).

    • ⚠️ Внимание: Если вы отправляете товары на склад Ozon в Москве (Пушкино) или Санкт-Петербурге (Шушары), проверьте актуальные требования к этикеткам — они могут отличаться от общероссийских (например, обязательное указание веса брутто для FBS).

    FAQ: Ответы на частые вопросы

    Можно ли сохранять этикетки в JPG и печатать их?

    Технически да, но не рекомендуется. Формат JPG сжимает изображение, что может привести к искажению штрихкода. Если альтернативы нет, используйте максимальное качество (100% JPEG) и разрешение не ниже 300 dpi. Перед отправкой на склад обязательно проверьте сканирование этикетки.

    Как сохранить этикетки для возвратов (обратная логистика)?

    Для возвратов используйте специальный тип этикеток — Return Label. Их можно сгенерировать в личном кабинете в разделе Возвраты → Этикетки. В PDF обратной этикетки обязательно должны быть:

    • 📦 Номер заказа и причина возврата.
    • 🔄 Уникальный штрихкод обратной доставки (начинается с RL).
    • 📍 Адрес склада возврата (указывается автоматически).

    Можно ли редактировать PDF с этикетками после сохранения?

    Да, но с осторожностью. Для редактирования используйте Adobe Acrobat Pro или Foxit PDF Editor. Категорически нельзя:

    • ❌ Изменять штрихкод (даже если он кажется некорректным).
    • ❌ Менять размеры этикетки более чем на ±2 мм.
    • ❌ Добавлять постскриптумы или рекламу (это нарушает правила Ozon).

    Как проверить, что PDF с этикетками соответствует требованиям Ozon?

    Используйте чек-лист проверки:

    1. Откройте PDF в Adobe Acrobat Reader и увеличьте масштаб до 400% — штрихкод должен оставаться чётким.
    2. Распечатайте тестовую этикетку на принтере, который будете использовать для основной партии.
    3. Проверьте сканирование штрихкода приложением Barcode Scanner (доступно для Android/iOS).
    4. Сверьте данные на этикетке с информацией в личном кабинете (SKU, артикул, вес).

    Что делать, если этикетки не сохраняются в PDF (ошибка 500 в личном кабинете)?

    Ошибка 500 обычно связана с перегрузкой серверов Ozon. Попробуйте:

    • 🔄 Обновите страницу через 5-10 минут.
    • 🌐 Используйте другой браузер (Chrome или Firefox показывают лучшую стабильность).
    • 📱 Попробуйте сохранить этикетки через мобильное приложение.
    • 📧 Если ошибка повторяется более часа, напишите в поддержку Ozon с указанием: 
      Тема: Ошибка 500 при сохранении этикеток

      Текст: Укажите ID заказов, время ошибки и скриншот консоли браузера (F12 → Console).
