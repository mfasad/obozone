Как обнулить остатки на Ozon Seller: все способы и нюансы

  • Почему важно правильно обнулять остатки на Ozon
  • Способы обнуления остатков в личном кабинете Ozon Seller
  • Обнуление остатков при работе по FBS и FBO: ключевые различия
  • Частые ошибки при обнулении остатков и как их избежать
  • Как обнулить остатки для архивных или удалённых товаров
  • Автоматизация обнуления остатков: сервисы и инструменты
  • FAQ: Частые вопросы про обнуление остатков на Ozon

Обнуление остатков на Ozon Seller — критически важная процедура для продавцов, работающих по моделям FBO (отгрузка со своего склада) и FBS (отгрузка через склады Ozon). Неправильное управление остатками ведёт к штрафам за недоступные товары, падению рейтинга магазина и даже блокировке аккаунта. В этой статье разберём все актуальные способы обнуления остатков в 2026 году, включая ручные и автоматизированные методы, а также типичные ошибки, которые допускают новички.

Особенность Ozon в том, что платформа жёстко контролирует соответствие заявленных остатков реальному наличию товаров. Если вы указали 10 единиц товара, но на деле их только 5, система зафиксирует недоотгрузку и наложит санкции. Обнуление остатков требуется в случаях: завершения продаж сезонного товара, смены поставщика, технических ошибок синхронизации или перехода на другую схему работы (например, с FBO на FBS). Далее — пошаговые инструкции для каждого сценария.

Почему важно правильно обнулять остатки на Ozon

Маркетплейс Ozon использует сложную систему штрафов за несоответствие остатков. Основные риски при неправильном обнулении:

  • 📉 Штраф 500–5 000 ₽ за каждый случай недоотгрузки (если товар числится в наличии, но отсутствует на складе).
  • ⚠️ Блокировка карточки товара на 3–7 дней при системных расхождениях.
  • 📊 Снижение рейтинга магазина в поисковой выдаче (вплоть до исключения из топовых позиций).
  • 🚫 Приостановка работы аккаунта при регулярных нарушениях (более 3 случаев за месяц).

Кроме того, Ozon автоматически резервирует товары под заказы в момент их оформления. Если вы не обнулили остатки вовремя, система может «заблокировать» несуществующие единицы, что приведёт к отменам заказов и негативным отзывам. Например, если у вас осталось 2 товара, но вы забыли обнулить остатки после продажи последней единицы, следующий покупатель увидит товар как доступный, а вы получите штраф за отмену.

⚠️ Внимание: Обнуление остатков не равно удалению товара из каталога. После обнуления карточка остаётся активной, но скрывается из поиска покупателей. Чтобы полностью удалить товар, используйте функцию «Архивировать» в личном кабинете.

Способы обнуления остатков в личном кабинете Ozon Seller

Существует три основных метода обнуления остатков, каждый из которых подходит для разных ситуаций:

  1. Ручное обнуление — через интерфейс личного кабинета (подходит для небольшого ассортимента).
  2. Массовое обнуление — с помощью файла Excel (оптимально для 100+ товаров).
  3. API-интеграция — автоматическое обновление остатков через программный интерфейс (для крупных продавцов).

Рассмотрим каждый способ подробно.

1. Ручное обнуление остатков (для 1–50 товаров)

Это самый простой метод, который не требует технических навыков. Подходит для небольших магазинов или разовых корректировок. Инструкция:

  1. Авторизуйтесь в личном кабинете Ozon Seller.
  2. Перейдите в раздел Товары → Мои товары.
  3. Найдите нужный товар через поиск или фильтры (можно отфильтровать по артикулу или названию).
  4. Нажмите на строку товара → выберите Редактировать остатки.
  5. В поле Количество введите 0 и сохраните изменения.

После сохранения остатки обнулятся в течение 5–15 минут. Чтобы ускорить процесс, можно воспользоваться кнопкой Обновить остатки сейчас в верхнем меню.

2. Массовое обнуление через Excel (для 50–10 000 товаров)

Если у вас большой ассортимент, ручное обнуление займёт слишком много времени. В этом случае используйте выгрузку/загрузку файла Excel. Алгоритм:

  1. В личном кабинете перейдите в Товары → Импорт/Экспорт → Экспорт остатков.
  2. Скачайте шаблон файла (формат .xlsx).
  3. Откройте файл и в столбце Quantity (или Количество) проставьте 0 для всех товаров, которые нужно обнулить.
  4. Сохраните файл и загрузите его обратно через Импорт остатков.

Важные нюансы:

  • 🔹 Файл должен быть в формате .xlsx (не .csv или .xls).
  • 🔹 Не изменяйте структуру шаблона (названия столбцов, порядок данных).
  • 🔹 Максимальный размер файла — 10 МБ (при превышении разбейте на части).
  • 🔹 Обработка файла может занять до 2 часов (в пиковые нагрузки).
⚠️ Внимание: Если в файле указаны товары, которых нет в вашем каталоге, система проигнорирует эти строки. Перед загрузкой убедитесь, что все SKU (артикулы) корректны.

3. Автоматическое обнуление через API (для 10 000+ товаров)

Крупные продавцы используют API Ozon Seller для автоматической синхронизации остатков. Это позволяет обнулять тысячи товаров за секунды, но требует технической настройки. Основные шаги:

  1. Получите API-key в личном кабинете (Настройки → API).
  2. Используйте метод POST /v2/products/stocks для обновления остатков.
  3. В теле запроса передайте массив товаров с параметром "stock": 0.

Пример запроса (на языке Python):

import requests


url = "https://api-seller.ozon.ru/v2/products/stocks"


headers = {


"Client-Id": "Ваш_Client_ID",


"Api-Key": "Ваш_API_Key",


"Content-Type": "application/json"


}


data = {


"stocks": [


{"sku": 12345678, "stock": 0},


{"sku": 87654321, "stock": 0}


]


}



response = requests.post(url, headers=headers, json=data)


print(response.json())

Преимущества API:

  • ⚡ Мгновенное обновление остатков (без ожидания синхронизации).
  • 🤖 Автоматизация процесса (можно интегрировать с 1С, MoySklad или другими системами учёта).
  • 📊 Возможность массовых операций (до 100 000 товаров в одном запросе).

Недостатки:

  • 🛠️ Требуются навыки программирования или помощь разработчика.
  • 🔒 Ограничение по количеству запросов (1000 в минуту для стандартного тарифа).

Обнуление остатков при работе по FBS и FBO: ключевые различия

Процедура обнуления остатков отличается в зависимости от схемы работы: FBO (отгрузка со своего склада) или FBS (отгрузка через склады Ozon). Разберём нюансы для каждой модели.

Параметр FBO (отгрузка со своего склада) FBS (отгрузка через Ozon)
Скорость обнуления Мгновенно (после сохранения) До 24 часов (требуется синхронизация со складом Ozon)
Штрафы за несоответствие 500–5 000 ₽ за недоотгрузку 1 000–10 000 ₽ + блокировка товара
Дополнительные действия Нет Требуется подтверждение от логистики Ozon
Возможность отката Да (можно вернуть остатки вручную) Нет (требуется согласование с Ozon)

При работе по FBS обнуление остатков — более сложный процесс, так как товары физически хранятся на складах Ozon. Если вы обнулили остатки в личном кабинете, но на складе ещё есть товар, система может:

  • 📦 Вернуть товар на ваш адрес (за ваш счёт).
  • 💰 Начислить штраф за некорректное управление остатками.
  • 🚫 Заблокировать возможность работы по FBS на 30 дней.

Чтобы избежать проблем, перед обнулением остатков по FBS:

  1. Сверьте реальные остатки на складе Ozon через отчёт Складские остатки.
  2. Если товар ещё есть на складе, сначала закажите возврат через раздел Логистика → Возвраты.
  3. Только после подтверждения возврата обнуляйте остатки в личном кабинете.
💡

Перед обнулением остатков по FBS проверьте, не участвует ли товар в программе "Гарантированная доставка". Если участвует, сначала исключите его из программы, иначе Ozon продолжит резервировать товар под заказы.

Частые ошибки при обнулении остатков и как их избежать

Даже опытные продавцы допускают ошибки при работе с остатками. Вот самые распространённые из них и способы их предотвращения:

  • 🔄 Обнуление без проверки активных заказов → Перед обнулением откройте раздел Заказы → Активные и убедитесь, что на товар нет резервов.
  • 📥 Использование устаревшего шаблона Excel → Всегда скачивайте свежий шаблон из личного кабинета (структура файла может меняться).
  • 🤖 Автоматическое обнуление без тестирования → Перед массовым обнулением через API протестируйте запрос на 1–2 товарах.
  • 📦 Обнуление FBS-товаров без возврата со склада → Это приводит к штрафам за "потерю" товара на складе Ozon.
  • ⏳ Игнорирование времени синхронизации → После обнуления подождите 1–2 часа перед повторными действиями (система может не успеть обновить данные).

Критическая ошибка: обнуление остатков для товаров, участвующих в акциях или распродажах. В этом случае Ozon блокирует изменение остатков до конца акции. Чтобы избежать проблем, сначала исключите товар из акции в разделе Акции → Мои акции.

Ещё одна типичная проблема — расхождение остатков между вашей системой учёта и Ozon. Например, если вы ведёте учёт в 1С, а в Ozon остатки не синхронизируются, используйте логи обновлений (раздел Товары → История изменений), чтобы отследить, когда и кто внёс последние правки.

Что делать, если остатки не обнулились?

Если после всех действий остатки не обновились:

1. Проверьте, нет ли ошибок в файле Excel (например, неверный формат ячеек).

2. Убедитесь, что у вас достаточно прав доступа (роль "Администратор" или "Менеджер товаров").

3. Обратитесь в поддержку Ozon через чат в личном кабинете, приложив скриншот ошибки.

4. Если проблема в API, проверьте код ответа сервера (ошибки 4xx или 5xx указывают на проблемы с запросом).

Как обнулить остатки для архивных или удалённых товаров

Если товар уже архивирован или удален из каталога, но остатки по нему всё ещё числятся в системе, потребуются дополнительные действия. Ситуация возникает, когда:

  • 🗃️ Товар архивирован, но остатки не были обнулены до этого.
  • 🗑️ Товар удалён, но в отчётах Ozon всё ещё виден остаток.
  • 🔄 Произошла ошибка синхронизации при удалении.

Инструкция для архивных товаров:

  1. Верните товар из архива: Товары → Архив → Восстановить.
  2. Обнулите остатки любым из описанных выше способов.
  3. Снова архивируйте товар.

Если товар удалён окончательно, восстановить его нельзя. В этом случае:

  1. Создайте новый товар с тем же SKU (если артикул не занят).
  2. Обнулите остатки для нового товара.
  3. Удалите товар повторно (если он не нужен).

Для удалённых товаров с остатками также можно использовать отчёт по расхождениям (Аналитика → Остатки → Расхождения). Если система показывает остатки по несуществующему товару, обратитесь в поддержку с запросом на принудительное обнуление (указывайте SKU и дату удаления).

Для продавцов с большим ассортиментом ручное обнуление остатков неэффективно. На рынке есть специализированные сервисы, которые помогают автоматизировать процесс:

Сервис Функционал Стоимость Интеграция с Ozon

  • 🔗 Надёжность интеграции (частота обрывов связи с API Ozon).

FAQ: Частые вопросы про обнуление остатков на Ozon

Можно ли обнулить остатки для товара, который участвует в акции?

Нет. Если товар включён в акцию (например, "Выгодная цена" или "Распродажа"), Ozon блокирует изменение остатков до её завершения. Чтобы обнулить остатки, сначала исключите товар из акции в разделе Акции → Мои акции.

Сколько времени занимает обнуление остатков по FBS?

При работе по FBS обнуление остатков может занять до 24 часов, так как требуется синхронизация с логистическим центром Ozon. Если товар физически есть на складе, его сначала нужно вернуть (через раздел Логистика → Возвраты), и только потом обнулять остатки.

Что будет, если обнулить остатки, но на складе ещё есть товар?

Если вы работаете по FBO, ничего критичного не произойдёт (просто товар скрывается из продажи). Но если вы работаете по FBS, Ozon начислит штраф за расхождение остатков (от 1 000 ₽) и может инициировать возврат товара на ваш адрес за ваш счёт.

Как обнулить остатки для всех товаров сразу?

Для массового обнуления используйте Excel-файл:

  1. Скачайте шаблон остатков в личном кабинете.
  2. В столбце Quantity проставьте 0 для всех строк.
  3. Загрузите файл обратно через Импорт остатков.

Для 100 000+ товаров лучше использовать API.

Можно ли отменить обнуление остатков?

Да, но только если товар ещё не был продан или зарезервирован под заказ. Для этого:

  1. Верните нужное количество в поле Остатки в личном кабинете.
  2. Если товар архивирован, сначала восстановите его.

Для FBS отмена обнуления возможна только после согласования с логистическим центром Ozon.
