Как добавить или изменить количество товара на Ozon Seller: все способы 2026

  • 1. Способы изменения остатков на Ozon: какой выбрать?

Работа с остатками товара на Ozon — одна из ключевых задач для продавца, напрямую влияющая на видимость карточек и объем продаж. Неправильно указанное количество может привести к штрафам за отмены заказов или упущенной выручке из-за искусственного дефицита. В 2026 году платформа предлагает несколько способов корректировки остатков: от ручного редактирования в личном кабинете до автоматизации через API и YML-фиды.

Эта статья покрывает все актуальные методы — от базовых для новичков до продвинутых для крупных продавцов с тысячами SKU. Мы разберем нюансы работы в схеме FBS (когда товар хранится у вас) и FBO (когда логистикой занимается Ozon), а также типичные ошибки, из-за которых остатки могут «слетать» или блокироваться системой. Если вы только начинаете продавать на маркетплейсе или хотите оптимизировать процесс — здесь найдете проверенные решения.

1. Способы изменения остатков на Ozon: какой выбрать?

Выбор метода зависит от объема ассортимента, технических возможностей и схемы работы (FBS/FBO). Рассмотрим все варианты с их плюсами и минусами:

  • 🖥️ Ручной ввод в личном кабинете — подходит для небольшого количества товаров (до 50 SKU). Просто, но долго при масштабировании.
  • 📤 Массовая загрузка через Excel/XLSX — оптимально для 50–5 000 позиций. Требует аккуратности при формировании файла.
  • 🤖 API Ozon Seller — автоматизация для крупных продавцов (от 5 000 SKU). Нужны навыки программирования или интеграция с 1С/MoySklad.
  • 🔄 YML-фид — синхронизация остатков через XML-файл. Подходит для мультиканальных продавцов (например, если вы также торгуете на Wildberries или Яндекс Маркете).

Важно: в схеме FBO остатки обновляются автоматически после приемки товара на складе Ozon. В FBS продавец несет полную ответственность за актуальность данных. При расхождении фактических и заявленных остатков более чем на 10% маркетплейс может приостановить продажи по проблемным артикулам.

2. Пошаговая инструкция: как изменить количество товара вручную

Самый простой способ — редактирование напрямую в личном кабинете. Подходит для корректировки 1–2 позиций или экстренных изменений. Следуйте алгоритму:

  1. Авторизуйтесь в личном кабинете Ozon Seller.
  2. Перейдите в раздел Товары → Мои товары.
  3. Найдите нужный товар через поиск по артикулу, названию или SKU.
  4. Нажмите на строку с товаром — откроется карточка редактирования.
  5. В блоке Остатки введите новое количество и сохраните изменения кнопкой Применить.

Ограничения ручного метода:

  • ⏳ Максимум 50 редактирований в час (при превышении лимит сбрасывается через 60 минут).
  • 🔒 Нельзя массово изменить остатки для нескольких товаров одновременно.
  • 📉 При большом ассортименте процесс занимает часы.

Критическая ошибка: если вы продаете товар в схеме FBS, но по ошибке указали схему FBO, Ozon заблокирует возможность изменения остатков до устранения несоответствия. В этом случае обратитесь в поддержку через раздел Помощь → Написать в поддержку с темой «Ошибка схемы логистики».

3. Массовое обновление остатков через Excel: инструкция и шаблон

Для продавцов с ассортиментом от 50 позиций ручное редактирование неэффективно. Массовая загрузка через XLSX-файл экономит время, но требует точности. Скачайте актуальный шаблон в личном кабинете:

  1. Перейдите в Товары → Импорт/Экспорт → Шаблоны.
  2. Выберите шаблон Обновление информации о товаре (версия для остатков).
  3. Заполните обязательные колонки:
    • SKU — уникальный идентификатор товара (обязательно!).
    • Количество — новое значение остатка (целое число).
    • Склад — укажите FBS или FBO (если не указано, будет применено к текущей схеме товара).
  • Сохраните файл в формате .xlsx (не .csv!).
  • Загрузите через Товары → Импорт/Экспорт → Загрузить файл.

    • Типичные ошибки при загрузке:

    Ошибка Причина Как исправить
    Invalid SKU Опечатка в артикуле или товар архивирован Проверьте SKU в карточке товара или восстановите архивный товар
    Quantity must be integer Указано дробное число (например, 5.5) Округлите до целого значения
    Warehouse mismatch Схема FBS/FBO в файле не совпадает с текущей Укажите корректную схему или измените её в карточке товара
    File too large Файл превышает 50 МБ или содержит >50 000 строк Разбейте на несколько файлов
    💡

    Перед массовой загрузкой экспортируйте текущие остатки через Товары → Экспорт → Остатки. Это поможет избежать ошибок и отследить изменения.

    4. Автоматизация через API: для кого подходит и как настроить

    Если ваш ассортимент превышает 5 000 SKU или вы синхронизируете остатки с 1С, MoySklad, Bitrix24, оптимальное решение — API Ozon Seller. Этот метод требует технических навыков или помощи разработчика, но полностью исключает ручной труд.

    Преимущества API:

    • ⚡ Мгновенное обновление остатков (без задержек на обработку файлов).
    • 🔄 Синхронизация с вашей учетной системой в реальном времени.
    • 📊 Возможность настройки автоматических правил (например, резервирование 10% от остатка).

    Как подключить API:

    1. Получите Client-ID и API-Key в разделе Настройки → Интеграции → API.
    2. Используйте эндпоинт для обновления остатков: 
      POST https://api-seller.ozon.ru/v2/products/stocks

      Headers:
      

      - Client-Id: [ваш_client_id]
      

      - Api-Key: [ваш_api_key]
      

      Body (JSON):
      

      {
      

      "stocks": [
      

      {
      

      "offer_id": "SKU12345",
      

      "stock": 100,
      

      "warehouse_id": 1  // 1 для FBS, 2 для FBO
      

      }
      

      ]
      

      }

    3. Обработайте ответ сервера. Успешный статус: 200 OK.

    Для тестирования используйте песочницу Ozon API. Обратите внимание: лимиты на запросы — 100 вызовов в минуту. При превышении API вернет ошибку 429 Too Many Requests.

    Пример кода на Python для обновления остатков

    import requests

    url = "https://api-seller.ozon.ru/v2/products/stocks"

    headers = {

    "Client-Id": "your_client_id",

    "Api-Key": "your_api_key",

    "Content-Type": "application/json"

    }

    data = {

    "stocks": [

    {"offer_id": "SKU001", "stock": 50, "warehouse_id": 1},

    {"offer_id": "SKU002", "stock": 30, "warehouse_id": 2}

    ]

    }

    response = requests.post(url, headers=headers, json=data)

    print(response.json())

    5. Особенности работы с остатками в схеме FBO

    В модели FBO (Fulfillment by Ozon) логистикой занимается маркетплейс, но продавец все равно должен контролировать остатки. Главное отличие от FBS: фактическое количество товара на складе Ozon может не совпадать с тем, что вы указали в системе. Вот ключевые нюансы:

    • 📦 Приемка товара: после доставки на склад Ozon в течение 1–3 дней проводит приемку. Только после неё остатки становятся доступны для продажи.
    • 🔄 Автоматическая корректировка: если при приемке обнаружат расхождения (например, вы отправили 100 шт., а приняли 95), система автоматически обновит остатки.
    • ⚠️ Блокировка при несоответствии: если разница между вашими данными и фактическим количеством превышает 10%, Ozon может заблокировать возможность редактирования остатков до выяснения причин.

    Как избежать проблем в FBO:

    1. Перед отправкой товара на склад Ozon сверьте количество с данными в Заказы → Отгрузки.
    2. Используйте Этикетки Ozon для маркировки коробов — это ускорит приемку.
    3. Отслеживайте статус приемки в разделе Склад → Приемки.
    ⚠️ Внимание: Если вы отправили товар на склад Ozon, но не создали отгрузку в личном кабинете, система не сможет сопоставить поступившие единицы с вашими SKU. Это приведет к «потере» товара на складе. Всегда формируйте отгрузку до отправки физических коробов!

    6. Типичные ошибки и как их избежать

    Даже опытные продавцы сталкиваются с проблемами при работе с остатками. Вот самые распространенные ошибки и способы их предотвращения:

    Ошибка Последствия Решение
    Указание остатков без учета резерва под заказы Отмены заказов из-за нехватки товара, штрафы Всегда резервируйте 5–10% от остатка на случай пиковых продаж
    Несвоевременное обновление после продажи офлайн Расхождение фактических и заявленных остатков Настройте автоматическую синхронизацию с вашей кассой/1С
    Использование Excel с формулами или объединенными ячейками Ошибка загрузки файла Экспортируйте данные в «чистый» XLSX без формул
    Игнорирование уведомлений о блокировке остатков Остановка продаж по проблемным SKU Проверяйте раздел Проблемы ежедневно

    Самая опасная ошибка — умышленное занижение остатков для создания искусственного дефицита. Ozon отслеживает такие манипуляции через анализ динамики продаж и может:

    • Понизить позицию товара в выдаче.
    • Лишить участия в акциях (например, «Выгодная цена»).
    • Приостановить продажи по бренду.
    💡

    Регулярно сверяйте остатки в личном кабинете с фактическими данными на складе. Расхождения более 10% — повод для блокировки аккаунта.

    7. Как отслеживать и анализировать остатки

    Контроль остатков — не разовая задача, а постоянный процесс. Ozon предоставляет несколько инструментов для анализа:

    • 📊 Отчет «Остатки» (Аналитика → Остатки): показывает динамику по каждому SKU за выбранный период.
    • 🚨 Уведомления о низких остатках: настройте в Настройки → Уведомления (порог срабатывания — от 1 до 100 ед.).
    • 🔍 Фильтр по статусу: в разделе Товары можно отобразить только товары с остатком ≤5 или =0.

    Продвинутая аналитика:

    Используйте Ozon BI (доступно продавцам с оборотом от 1 млн ₽/мес) для построения дашбордов. Например, можно отслеживать:

    • Скорость оборачиваемости остатков по категориям.
    • Долю «мертвого» товара (лежит на складе >90 дней).
    • Влияние остатков на позицию в поиске (товары с остатком >50 ед. ранжируются выше).

    Для автоматизации мониторинга подключите Google Sheets к Ozon API через Apps Script. Пример скрипта для выгрузки остатков:

    Пример скрипта для Google Sheets

    function getOzonStocks() {

    const url = "https://api-seller.ozon.ru/v1/product/info/stocks";

    const headers = {

    "Client-Id": "YOUR_CLIENT_ID",

    "Api-Key": "YOUR_API_KEY"

    };

    const response = UrlFetchApp.fetch(url, {headers: headers});

    const data = JSON.parse(response.getContentText());

    return data.result.items;

    }

    ⚠️ Внимание: Если вы продаете товары с истекающим сроком годности (например, косметику или продукты), Ozon требует ежемесячного подтверждения остатков через раздел Склад → Товары с истекающим сроком. При пропуске подтверждения товар будет снят с продажи.

    Часто задаваемые вопросы

    Можно ли указать дробное количество товара (например, 0.5 кг)?

    Нет, Ozon принимает только целые числа в поле «Количество». Для штучных товаров с весом укажите количество упаковок (например, 1 упаковка = 0.5 кг, тогда в остатках будет 2 ед. = 1 кг). Для сыпучих товаров (мука, крупы) используйте фасовку: 1 ед. = 1 кг.

    Что делать, если остатки обнулились сами собой?

    Причины автоматического обнуления:

    1. Истек срок годности товара (для категорий «Продукты», «Косметика»).
    2. Товар попал под санкции (например, блокировка бренда).
    3. Технический сбой на стороне Ozon (проверьте статус систем на странице статуса).

    Решение: проверьте раздел Проблемы в личном кабинете или обратитесь в поддержку с указанием SKU.

    Как резервировать товар под заказы, чтобы не продавать «в минус»?

    Ozon автоматически резервирует товар под оформленные заказы, но для дополнительной защиты:

    • В схеме FBS: укажите в остатках на 10–15% меньше фактического количества.
    • В схеме FBO: используйте параметр reserve в API (например, "stock": 100, "reserve": 10).
    • Для ручной корректировки: экспортируйте остатки, уменьшите значения в Excel и загрузите обратно.

    Сколько времени занимает обновление остатков после загрузки файла?

    Срок обработки зависит от загруженности системы:

    • Файлы до 1 000 строк: 5–30 минут.
    • Файлы 1 000–10 000 строк: 1–3 часа.
    • Файлы >10 000 строк: до 6 часов (рекомендуется разбивать на части).

    Статус обработки можно отследить в разделе Товары → Импорт/Экспорт → История загрузок.

    Можно ли вернуть прежние остатки, если ошибся при обновлении?

    Да, если ошибка обнаружена быстро:

    1. Для ручного изменения: просто введите корректное значение в карточке товара.
    2. Для массовой загрузки: создайте новый файл с правильными данными и загрузите его (последние данные перезапишут предыдущие).
    3. Если прошло >24 часа и заказы уже оформлены: свяжитесь с поддержкой для ручной корректировки.

    Важно: если из-за ошибки были отменены заказы, Ozon может наложить штраф за нарушение SLA (соглашения об уровне обслуживания).
