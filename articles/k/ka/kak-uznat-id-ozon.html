Где и как узнать ID товара на Ozon: все методы для продавцов и покупателей

Каждый товар на маркетплейсе Ozon имеет уникальный идентификатор — ID, который используется для поиска, аналитики, интеграции с API и решения спорных ситуаций. Без этого кода невозможно отследить продукцию в личном кабинете, оформить возврат или синхронизировать данные с 1С. Проблема в том, что платформа не всегда показывает ID на видном месте: его приходится искать в неочевидных разделах интерфейса или извлекать из ссылок вручную.

В этой статье мы разберём 7 проверенных способов, как найти ID товара на Ozon — от базовых (через карточку товара) до продвинутых (парсинг URL и работа с API). Отдельно остановимся на нюансах для продавцов и покупателей, а также на том, как отличить артикул продавца (SKU) от внутреннего ID маркетплейса. Если вы регулярно работаете с большим каталогом, сохраните эту инструкцию в закладки — она сэкономит часы на поиск кода вручную.

1. Что такое ID товара на Ozon и зачем он нужен

ID товара (или product ID) — это уникальный числовой код, который Ozon присваивает каждой карточке при создании. Он отличается от:

  • 🔖 Артикула продавца (SKU) — внутреннего кода, который вы указываете при загрузке товара (например, SKU-12345-AB).
  • 📦 Штрихкода (EAN/UPC) — международного идентификатора продукта, который печатается на упаковке.
  • 🔗 Ссылки на товар — URL-адреса страницы, где ID может быть частью пути.

ID нужен для:

  • 📊 Аналитики продаж — отслеживания динамики по конкретному товару в Ozon Seller или сторонних сервисах (DataLens, Retail Rocket).
  • 🔄 Интеграции с API — обновления цен, остатков или описаний через программный интерфейс.
  • ⚖️ Решения споров — обращения в поддержку с указанием точного кода товара.
  • 📦 Логистики — идентификации товара при возвратах или перемещении по складам Ozon.
⚠️ Внимание: ID товара на Ozon может меняться, если карточка была удалена и создана заново. Не используйте старые коды для API-запросов — это приведёт к ошибке 404 Not Found.

Для покупателей ID полезен, чтобы:

  • 🛒 Быстро найти товар по прямой ссылке (если сохранить ID).
  • 🔄 Отследить статус возврата или обмена.
  • 📞 Указать код при обращении в поддержку по проблеме с заказом.

2. Как узнать ID товара через карточку на сайте Ozon

Самый простой способ — извлечь ID прямо из адресной строки браузера. Этот метод работает и для продавцов, и для покупателей.

Пошаговая инструкция:

  1. Откройте карточку товара на сайте Ozon.ru.
  2. Посмотрите на URL в адресной строке. Он будет выглядеть так:
    https://www.ozon.ru/product/{ID}-{nazvanie-tovara}/
  3. Скопируйте числовой код перед первым дефисом — это и есть ID.

Примеры:

  • Ссылка: https://www.ozon.ru/product/smartfon-apple-iphone-15-pro-max-256gb-naturalnyy-titan-146819024/

    → ID: 146819024

  • Ссылка: https://www.ozon.ru/product/umnye-chasy-apple-watch-series-9-41-mm-alyuminievyy-korpus-sportivnyy-remeshok-170387536/

    → ID: 170387536

Если в ссылке нет числа перед дефисом (например, https://www.ozon.ru/product/naimenovanie-tovara-1234567/), значит:

  • 🔍 Товар новый, и ID ещё не сгенерирован (подождите 10–15 минут).
  • 🚫 Карточка удалена или заблокирована (проверьте статус в личном кабинете).
⚠️ Внимание: В мобильном приложении Ozon ID в URL не отображается. Используйте браузерную версию или методы из следующих разделов.

3. Поиск ID в личном кабинете Ozon Seller

Продавцы могут найти ID товара прямо в панели управления Ozon Seller. Этот метод надёжнее, чем парсинг URL, так как показывает актуальные данные даже для скрытых или заблокированных карточек.

Способ 1. Через раздел «Товары»:

  1. Авторизуйтесь в Ozon Seller.
  2. Перейдите в Товары → Список товаров.
  3. Найдите нужный товар в таблице и кликните по его названию.
  4. В открывшейся карточке ID будет указан в блоке Основная информация (поле «Идентификатор товара»).

Способ 2. Через экспорт данных:

  1. В разделе Товары нажмите Экспорт → Экспортировать все товары.
  2. В полученном файле .xlsx ID будет в колонке «product_id».

Способ 3. Через API (для продвинутых пользователей):

Если вы используете Ozon API, отправьте запрос:

GET https://api-seller.ozon.ru/v2/product/info

Headers:


- Client-Id: {ваш_client_id}


- Api-Key: {ваш_api_key}


Body:


{


"product_id": [12345678], // здесь укажите предполагаемый ID


"offer_id": ["ваш_SKU"]


}

Метод Где искать ID Подходит для
Раздел «Товары» Карточка товара → «Идентификатор товара» Всех продавцов
Экспорт в Excel Колонка product_id Продавцов с большим каталогом
API-запрос Ответ сервера, поле product_id Разработчиков и автоматизаторов
💡

Если в личном кабинете отображается ошибка при открытии карточки товара, попробуйте очистить кеш браузера или использовать режим инкогнито. Часто проблема решается перезагрузкой страницы (Ctrl + F5).

4. Как найти ID в мобильном приложении Ozon

В официальном приложении Ozon (для iOS/Android) ID товара не отображается в интерфейсе, но его можно извлечь двумя способами:

Способ 1. Через «Поделиться»:

  1. Откройте карточку товара в приложении.
  2. Нажмите кнопку Поделиться (значок стрелки вверх).
  3. Выберите Копировать ссылку.
  4. Вставьте ссылку в любой текстовый редактор и найдите число перед первым дефисом (аналогично методу с браузером).

Способ 2. Через браузерную версию:

  1. В мобильном браузере (Chrome, Safari) откройте сайт Ozon.ru.
  2. Найдите товар через поиск.
  3. Скопируйте ID из URL (как описано в разделе 2).
⚠️ Внимание: В приложении Ozon Seller (для продавцов) ID отображается в карточке товара в блоке Основная информация, но только в последних версиях (обновите приложение, если поля нет).

Если ни один из способов не сработал, попробуйте:

  • 📱 Установить браузерное расширение для парсинга URL (например, Tampermonkey с пользовательским скриптом).
  • 🖥️ Использовать ПК-версию сайта через режим «Для компьютера» в мобильном браузере.

5. Поиск ID через API Ozon (для разработчиков)

Если вы автоматизируете работу с Ozon через API, ID товара можно получить программно. Этот метод подходит для массовой обработки каталога.

Базовые запросы для получения ID:

1. Получение списка всех товаров с ID:

GET https://api-seller.ozon.ru/v2/product/info/list

Headers:


- Client-Id: {ваш_client_id}


- Api-Key: {ваш_api_key}


Body:


{


"limit": 1000, // максимальное количество товаров в ответе


"offset": 0    // смещение для пагинации


}

В ответе ищите поле "product_id" для каждого товара.

2. Поиск ID по SKU (артикулу продавца):

POST https://api-seller.ozon.ru/v1/product/info

Headers:


- Client-Id: {ваш_client_id}


- Api-Key: {ваш_api_key}


Body:


{


"offer_id": ["ваш_SKU_1", "ваш_SKU_2"],


"product_id": []


}

В ответе будет массив с данными, где product_id — искомый идентификатор.

3. Поиск по названию товара:

POST https://api-seller.ozon.ru/v1/product/search

Headers:


- Client-Id: {ваш_client_id}


- Api-Key: {ваш_api_key}


Body:


{


"filter": {


"visibility": "ALL" // или "VISIBLE", "INVISIBLE"


},


"search": "название вашего товара",


"page": 1,


"page_size": 10


}

Ошибки при работе с API:

Код ошибки Причина Решение
401 Unauthorized Неверный Api-Key или Client-Id Проверьте ключи в личном кабинете Ozon Seller
404 Not Found Товар удалён или ID неверный Обновите данные или проверьте ID через личный кабинет
429 Too Many Requests Превышен лимит запросов (60 в минуту) Добавьте задержку между запросами или используйте пакетную обработку
Пример ответа API с product_id

{

"result": {

"items": [

{

"offer_id": "SKU-12345",

"product_id": 170387536, // <-- это и есть ID

"name": "Apple Watch Series 9",

"visibility": "VISIBLE"

}

]

}

}

Для удобства используйте инструменты вроде Postman или Insomnia для тестирования API-запросов. Также можно написать простой скрипт на Python с библиотекой requests:

import requests


url = "https://api-seller.ozon.ru/v2/product/info/list"


headers = {


"Client-Id": "ваш_client_id",


"Api-Key": "ваш_api_key"


}


params = {"limit": 100}



response = requests.get(url, headers=headers, params=params)


data = response.json()


for item in data["result"]["items"]:


print(f"SKU: {item['offer_id']}, ID: {item['product_id']}")

6. Альтернативные способы: парсинг и сторонние сервисы

Если стандартные методы не сработали, можно воспользоваться дополнительными инструментами.

1. Парсинг HTML-кода страницы:

ID товара часто хранится в мета-тегах или данных data- атрибутах. Как его найти:

  1. Откройте карточку товара в браузере.
  2. Нажмите Ctrl + U (Просмотр кода страницы) или F12 (Инструменты разработчика).
  3. Введите в поиск по коду (Ctrl + F) запрос: product_id или data-product-id.
  4. Найденное число и будет ID.

2. Сторонние сервисы для продавцов:

  • 📊 SellerLab — показывает ID в отчётах по товарам.
  • 📈 OzonStat — аналитика с указанием product_id.
  • 🔄 MyWare — синхронизация с 1С, где ID отображается в карточке товара.

3. Расширения для браузера:

  • 🦊 Ozon Assistant (Chrome) — добавляет кнопку копирования ID в карточке товара.
  • 🔍 Ozon Tools (Firefox) — парсит ID и другие данные.
⚠️ Внимание: Используйте сторонние сервисы осторожно. Некоторые из них могут запрашивать доступ к вашему аккаунту Ozon Seller. Перед подключением проверьте отзывы и политику конфиденциальности.
💡

Самый надёжный способ — комбинация личного кабинета и API. Парсинг и сторонние сервисы удобны, но могут давать сбой при изменении структуры сайта.

7. Частые ошибки и как их избежать

При поиске ID товара пользователи часто сталкиваются с типичными проблемами. Разберём самые распространённые:

1. Путают ID и SKU:

  • 🔢 ID — уникальный код Ozon (например, 170387536).
  • 🏷️ SKU — ваш внутренний артикул (например, IPHONE-15-PRO-128GB).

Как отличить: ID всегда числовой, SKU может содержать буквы и символы.

2. Копируют ID с ошибками:

  • 🚫 Ошибочно: 170387536/ (со слешем).
  • ✅ Правильно: 170387536 (чистое число).

3. Не обновляют кеш:

Если вы недавно изменили товар, ID в личном кабинете может отображаться с задержкой. Очистите кеш браузера или используйте режим инкогнито.

4. Ищут ID в архивных товарах:

Для удалённых или архивных карточек ID можно найти только через API или экспорт данных. В интерфейсе Ozon Seller они не отображаются.

5. Используют устаревшие ссылки:

Если товар был перенесён в другой раздел или изменён, его URL мог поменяться. Всегда проверяйте актуальность ссылки.

Ошибка Причина Решение
ID не найден в URL Товар новый или ссылка битая Подождите 10–15 минут или проверьте через личный кабинет
API возвращает 404 Товар удалён или ID неверный Проверьте статус товара в Ozon Seller
В Excel-файле нет колонки product_id Неправильный шаблон экспорта Выберите формат «Полные данные» при экспорте
💡

Если вы часто работаете с ID товара, создайте в Google Таблицах или Excel отдельный лист с соответствиями SKU → product_id. Это сэкономит время при массовых операциях.

FAQ: Ответы на частые вопросы

Можно ли узнать ID товара, если я покупатель, а не продавец?

Да. Для этого откройте карточку товара в браузере и скопируйте число из URL (как описано в разделе 2). Также ID можно найти в истории заказов: перейдите в Мои заказы, выберите товар и посмотрите ссылку на его карточку.

Что делать, если ID товара изменился?

Это происходит, если карточка была удалена и создана заново. В таком случае:

  1. Обновите ID в своих системах (1С, CRM, скрипты).
  2. Если товар был в заказах, уточните у поддержки Ozon, не повлияет ли изменение на логистику.
  3. Для API используйте поиск по SKU, а не по product_id.
Как найти ID для товара, которого нет в продаже?

Если товар архивирован или заблокирован:

  1. Попробуйте найти его через экспорт данных в Ozon Seller (включите фильтр «Все товары»).
  2. Используйте API-запрос с параметром "visibility": "ALL".
  3. Обратитесь в поддержку Ozon с просьбой предоставить ID (укажите SKU или название товара).
Можно ли по ID товара узнать, кто его продаёт?

Нет, ID сам по себе не содержит информации о продавце. Однако вы можете:

  1. Открыть карточку товара по ID (добавив его в URL: https://www.ozon.ru/product/{ID}-.../).
  2. Посмотреть название магазина в блоке «Продавец».
  3. Если вы продавец, используйте API-запрос /v1/product/info с указанием product_id — в ответе будет поле seller_id.
Как массово получить ID для всех своих товаров?

Самые эффективные способы:

  1. Экспорт через Ozon Seller: Товары → Экспорт → Экспортировать все товары (ID будет в колонке product_id).
  2. API-запрос: используйте эндпоинт /v2/product/info/list с пагинацией (указывайте limit и offset).
  3. Скрипт на Python: напишите парсер, который пройдётся по всем страницам вашего каталога и соберёт ID (пример кода есть в разделе 5).

Для больших каталогов (10 000+ товаров) рекомендуется использовать API с разбивкой на пакеты по 1000 запросов.
