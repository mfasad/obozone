Откуда идёт заказ с Ozon: как продавцу узнать город, ПВЗ или курьерскую службу

Вы продаёте на Ozon и хотите понять, откуда concretno идёт заказ покупателя? Это критично для логистики, если вы работаете по модели FBO (самостоятельная отправка), или для анализа спроса по регионам. Но даже на FBS (склад Озона) знание геолокации покупателя помогает оптимизировать запасы и рекламные кампании.

Проблема в том, что Ozon не всегда показывает исходный пункт заказа в личном кабинете. Иногда виден только финальный пункт выдачи (ПВЗ), а сам покупатель мог оформить заказ из другого города — например, отправив подарок родственникам. В этой статье разберём 5 проверенных способов, как определить реальный источник заказа: от анализа данных в Личном кабинете до использования сторонних сервисов.

Важно: часть методов работает только для продавцов с доступом к Ozon Seller API или расширенной аналитике. Но мы также приведём решения для новичков без технических навыков.

1. Проверка данных в личном кабинете Ozon Seller

Самый очевидный способ — изучить информацию о заказе в интерфейсе Ozon Seller. Вот что нужно сделать:

  • 📌 Перейдите в раздел Заказы → Все заказы.
  • 🔍 Найдите нужный заказ по номеру или фильтру (например, по статусу"Ожидает отправки").
  • 📋 Кликните на номер заказа, чтобы открыть детальную карточку.

В карточке заказа обратите внимание на следующие поля:

  • 📍 Адрес доставки — здесь может быть указан ПВЗ (пункт выдачи заказов) или адрес курьерской доставки. Но это не всегда совпадает с местом оформления заказа!
  • 🏠 Город покупателя — иногда отображается в блоке"Информация о покупателе". Если поле пустое, попробуйте другие методы.
  • 📦 Тип доставки —"Курьер","ПВЗ","Почта России". Это подскажет, как именно заказ дойдёт до клиента.
💡

Если в карточке заказа нет города покупателя, попробуйте экспортировать данные в Excel через кнопку"Экспорт" в правом верхнем углу. В некоторых версиях отчётов эта информация присутствует.

Ограничение метода: Ozon может скрывать реальный город покупателя, если тот использует функцию"Подарок" или оформляет заказ в другой регион. В таком случае переходите к следующим способам.

2. Анализ номера заказа: что скрыто в коде

Каждый заказ на Ozon имеет уникальный номер, который содержит зашифрованную информацию. Например, номер вида 2134567890-123456 можно расшифровать:

  • 🔢 Первые 10 цифр — идентификатор заказа.
  • 📌 Последние 6 цифр (после тире) — код ПВЗ или маршрута курьера.

Чтобы определить регион по коду ПВЗ:

  1. Скопируйте последние 6 цифр из номера заказа.
  2. Перейдите на сайт points.ozon.ru (официальная карта ПВЗ Озона).
  3. Введите код в поле поиска. Система покажет адрес пункта выдачи и город.

Если заказ курьерский, код после тире может указывать на логистический хаб, откуда отправляется посылка. Например, коды 101 часто относятся к Москве, 202 — к Санкт-Петербургу, а 380*** — к Краснодару.

3. Использование Ozon Seller API для точной геолокации

Для продавцов с техническими навыками или доступом к разработчикам есть более точный метод — запрос данных через Ozon Seller API. Этот способ позволяет получить:

  • 🌍 Город оформления заказа (не всегда совпадает с адресом доставки).
  • 📦 Информацию о складе, с которого будет отправлен заказ (для FBS).
  • 🚚 Данные о курьерской службе (если доставка не через ПВЗ).

Пример запроса для получения информации о заказе:

GET https://api-seller.ozon.ru/v2/posting/fbo/list

Headers:


- Client-Id: [Ваш Client-ID]


- Api-Key: [Ваш API-ключ]



Query params:


- limit=10


- offset=0


- status=awaiting_packaging

В ответе ищите поля:

  • "delivery_method" — способ доставки.
  • "warehouse_id" — идентификатор склада (для FBS).
  • "region" или "city" — регион покупателя.

Если вы не знакомы с API, можно использовать готовые сервисы-коннекторы, например, AltoBI или RetailCRM, которые автоматически парсят и визуализируют данные о заказах, включая геолокацию.

Как получить доступ к Ozon Seller API?

Для этого нужно зарегистрировать приложение в личном кабинете Ozon Seller (раздел"API и Интеграции"), указать Client-ID и сгенерировать Api-Key. Подробная инструкция есть в официальной документации.

4. Геолокация по IP-адресу покупателя (продвинутый метод)

Этот способ подходит для продавцов, которые хотят анализировать реальное местоположение покупателя в момент оформления заказа. Для этого нужно:

  1. Интегрировать на сайт (если продаёте через него) или в чат-бот сервис геолокации по IP, например, IPAPI или MaxMind.
  2. Настроить сбор IP-адресов покупателей при переходе с Ozon (например, через UTM-метки или пиксель).
  3. Сопоставить IP с заказами в CRM.

Пример ответа от IPAPI для IP 95.108.231.7:

{

"status":"success",


"country":"Russia",


"countryCode":"RU",


"region":"MOW",


"regionName":"Moscow",


"city":"Moscow",


"zip":"101000",


"lat": 55.7522,


"lon": 37.6156


}

Важно: сбор IP-адресов должен соответствовать политике конфиденциальности Ozon и законодательству РФ (ФЗ-152). Не используйте этот метод без согласия покупателей!

💡

Геолокация по IP даёт наиболее точные данные, но требует технической настройки и соблюдения законов о персональных данных.

5. Анализ отчётов Ozon: скрытые данные о регионах

В личном кабинете Ozon Seller есть раздел Аналитика → Отчёты, где можно найти агрегированные данные о заказах по регионам. Вот как это работает:

  1. Перейдите в Аналитика → Отчёты → Отчёт по продажам.
  2. Выберите период (например, последнюю неделю).
  3. В настройках отчёта добавьте группировку по полю "Регион покупателя".
  4. Экспортируйте данные в Excel или Google Sheets.

В полученной таблице будут столбцы:

Поле Описание Пример значения
order_id Номер заказа 2134567890-123456
region Регион покупателя Москва и Московская область
delivery_type Тип доставки courier или pickup_point
warehouse Склад отправки (для FBS) МСК-1 (Балашиха)

Если в отчёте нет колонки с регионом, попробуйте:

  • 🔄 Обновить данные (иногда информация подгружается с задержкой).
  • 📊 Использовать отчёт "Отчёт по логистике" — там может быть больше деталей.

6. Альтернативные способы: чат с покупателем и внешние сервисы

Если все вышеперечисленные методы не сработали, остаются"ручные" варианты:

  • 💬 Написать покупателю через чат в заказе (вежливо уточните, из какого он города — иногда клиенты отвечают).
  • 📦 Посмотреть этикетку (для FBO): если заказ уже собран, на наклейке может быть указан город назначения.
  • 🌐 Использовать сервисы отслеживания вроде Track24 или Почта России (если доставка через них). Введите трек-номер — система покажет маршрут.

Пример сообщения покупателю:

⚠️ Внимание: Не спрашивайте у покупателя адрес или персональные данные — это нарушает правила Ozon и может привести к блокировке аккаунта. Достаточно вежливо уточнить город для уточнения деталей доставки.

Здравствуйте! Подскажите, пожалуйста, из какого вы города? Это поможет нам точнее подобрать способ доставки. Спасибо!

Если покупатель не отвечает, попробуйте проанализировать его профиль (если заказ оформлен через аккаунт Ozon). Иногда в отзывах или вопросах к товарам пользователи указывают свой город.

Частые ошибки и как их избежать

При определении источника заказа продавцы часто сталкиваются с следующими проблемами:

  • 🔍 Путают город покупателя и город доставки — например, покупатель из Екатеринбурга отправляет подарок в Казань. В этом случае в заказе будет указан Казань, но для анализа спроса важен Екатеринбург.
  • 📦 Игнорируют тип доставки — заказы с ПВЗ и курьерские имеют разную логистику. Для FBS это критично, так как товар может уйти на другой склад.
  • 🚫 Нарушают конфиденциальность — сбор IP или личных данных без согласия может привести к жалобам и блокировке.

Чтобы избежать ошибок:

⚠️ Внимание: Если вы работаете по модели FBS, не пытайтесь перенаправлять заказы на другой склад самостоятельно — это нарушает договор с Ozon. Используйте геолокацию только для анализа и планирования запасов.

  • 📌 Всегда сверяйте данные из нескольких источников (ЛК, API, отчёты).
  • 🔄 Регулярно обновляйте отчёты — информация о регионах может появляться с задержкой.
  • 📊 Используйте CRM-системы для автоматизации сбора данных (например, МойСклад или 1С:УНФ).

FAQ: Ответы на частые вопросы

Можно ли узнать точный адрес покупателя?

Нет, Ozon скрывает точные адреса покупателей для защиты персональных данных. Вы видите только город (иногда) и адрес ПВЗ/курьера. Для FBO адрес доставки указывается на этикетке после оплаты заказа.

Почему в карточке заказа нет информации о городе покупателя?

Это может происходить по нескольким причинам:

  • Покупатель оформил заказ как"Подарок" и указал другой адрес доставки.
  • Заказ сделан через мобильное приложение с включённым VPN (геолокация сбивается).
  • Ozon скрывает данные для некоторых типов доставки (например,"Почта России").

Попробуйте использовать API или отчёты для уточнения.

Как понять, с какого склада FBS уйдёт заказ?

Для FBS склад определяется автоматически по алгоритмам Ozon, но вы можете увидеть его в:

  • Отчёте по логистике (колонка warehouse).
  • API-запросе к методу /v2/posting/fbs/list (поле warehouse_id).
  • Этикетке заказа (если товар уже собран).

На выбор склада влияют: регион покупателя, наличие товара на ближайших хабах и тарифы доставки.

Можно ли фильтровать заказы по региону в личном кабинете?

Да, в разделе Заказы есть фильтр по региону, но он работает не всегда корректно. Лучше экспортировать данные в Excel и фильтровать там. Для этого:

  1. Нажмите"Экспорт" в правом верхнем углу.
  2. Выберите формат XLSX.
  3. Откройте файл и отсортируйте по колонке"Регион".

Что делать, если заказ идёт из другого региона, чем мой склад?

Если вы работаете по FBO и видите, что заказ идёт в другой регион:

  • Проверьте тарифы доставки — возможно, выгоднее отправить через СДЭК или Boxberry.
  • Уточните у покупателя, готов ли он ждать дольше (если отправляете из другого города).
  • Для FBS этот вопрос не актуален — Ozon сам перемещает товар между складами.

Если такие заказы частые, рассмотрите возможность открытия склада в другом регионе или подключения FBS.
