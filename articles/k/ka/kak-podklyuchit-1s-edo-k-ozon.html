Интеграция 1С ЭДО с Озон: полное руководство для продавцов

  • FAQ: Частые вопросы по подключению 1С ЭДО к Озон

Подключение электронного документооборота (ЭДО) между 1С и маркетплейсом Озон — обязательный шаг для продавцов, работающих по моделям FBS и FBO. Без корректной настройки обмена счета-фактуры, УПД и акты не будут автоматически передаваться в налоговую, что грозит штрафами до 10 000 ₽ за каждый неподанный документ. В этой статье разберём все этапы интеграции: от регистрации в личном кабинете Озон до тестирования обмена, а также рассмотрим типичные ошибки и способы их устранения.

Важно понимать, что с 2026 года Озон ужесточил требования к ЭДО: теперь все продавцы с оборотом свыше 5 млн ₽ в квартал обязаны использовать электронные документы. Даже если ваш бизнес меньше, подключение ЭДО упрощает бухгалтерию и ускоряет возврат НДС. Мы собрали актуальные инструкции с учётом последних обновлений API Озон и новых версий 1С:Предприятие 8.3.

Почему продавцам на Озон обязательно подключать 1С ЭДО

С 1 июля 2026 года Озон перешёл на обязательный электронный документооборот для всех партнёров, работающих по схеме FBO (отгрузка со склада Озон). Для FBS (самостоятельная логистика) ЭДО также рекомендовано, но не строго обязательно — однако без него вы не сможете:

  • 📄 Автоматически формировать счета-фактуры и УПД при отгрузке товаров;
  • 💰 Быстро получать возврат НДС (без ЭДО срок проверки увеличивается до 3 месяцев);
  • ⚡ Синхронизировать данные о возвратах и списаниях с 1С;
  • 🔄 Обмениваться актами сверки с Озон без ручного ввода.

По данным ФНС, за 2023 год более 30% продавцов на маркетплейсах получили штрафы за несвоевременную сдачу электронных документов. Средняя сумма штрафа — 7 000 ₽ за документ. Подключение ЭДО через 1С позволяет избежать этих рисков и сэкономить до 40 часов в месяц на ручной обработке бумаг.

Требования к системе перед подключением

Перед настройкой интеграции проверьте, соответствует ли ваша система минимальным требованиям:

Компонент Минимальная версия Рекомендуемая версия
1С:Предприятие 8.3.10.2561 8.3.20+ (с поддержкой ЭДО 3.0)
Конфигурация 1С УТ 11.4 / ERP 2.4 УТ 11.5.6+ / ERP 2.5.5+
Операционная система Windows 7 / Linux Windows 10/11 или Ubuntu 20.04+
API Озон v1 (устаревает с 01.01.2026) v2 (обязательно для новых подключений)

Если ваша версия 1С старше 8.3.15, обновите платформу через Панель управления → Программы и компоненты или обратитесь к партнёру 1С. Для работы с API Озон v2 потребуется установить расширение OzonEDO (скачать можно в личном кабинете ЭДО).

⚠️ Внимание: С 1 января 2026 года Озон отключит поддержку API v1. Если вы используете старую версию интеграции, обновите её заранее, иначе обмен документами прекратится.

Пошаговая инструкция: подключение 1С ЭДО к Озон

Процесс интеграции состоит из 5 ключевых этапов. Следуйте инструкции строго по порядку, чтобы избежать ошибок синхронизации.

Шаг 1. Регистрация в личном кабинете ЭДО Озон

Перейдите на портал edo.ozon.ru и авторизуйтесь под учётной записью продавца. Если у вас ещё нет доступа:

  1. Нажмите Зарегистрироваться и выберите тип Юридическое лицо/ИП;
  2. Заполните данные ИНН, ОГРН и email (должен совпадать с почтой в личном кабинете Озон Селлер);
  3. Подтвердите регистрацию по SMS или через Госуслуги;
  4. Дождитесь письма с логином и паролем (приходит в течение 1 часа).

После регистрации в разделе Настройки → Профиль проверьте, что указаны:

  • 📌 Правильный ИНН и КПП;
  • 📌 Активный сертификат ЭЦП (если нет — закажите в Контуре или Такском);
  • 📌 Email для уведомлений (должен отличаться от основной почты Озон).

Шаг 2. Настройка обмена в 1С

Откройте вашу базу 1С и перейдите в Администрирование → Обмен с маркетплейсами → Настройки ЭДО. Если раздела нет, установите обработку OzonEDO.dt (скачать в партнёрском кабинете).

В форме настройки укажите:



Адрес сервера ЭДО: https://api-seller.ozon.ru/v2/edo


Клиентский идентификатор: [ваш Client-ID из ЛК Озон]


Секретный ключ: [ваш API-key из ЛК Озон]


Формат документов: УПД (универсальный передаточный документ)

Для проверки соединения нажмите Тест подключения. Если появится ошибка 403 Forbidden, убедитесь, что:

  • 🔑 API-key скопирован без пробелов;
  • 🔑 В личном кабинете Озон разрешён доступ к API (раздел Безопасность → API-ключи);
  • 🔑 Сервер 1С имеет белый IP-адрес (если используете облако, проверьте настройки фаервола).

Шаг 3. Сопоставление номенклатуры

Чтобы документы формировались корректно, необходимо сопоставить товары из 1С с артикулами Озон. Для этого:

  1. Экспортируйте список товаров из Озон в формате CSV (Товары → Экспорт);
  2. Импортируйте файл в 1С через Администрирование → Обмен данными → Импорт номенклатуры;
  3. Проверьте совпадение артикулов в колонках VendorCode (Озон) и Код товара (1С).

Если артикулы не совпадают, создайте правило сопоставления в Настройки обмена → Сопоставление номенклатуры. Для товаров с вариантами (цвет/размер) используйте формат:



[Артикул Озон]_[Характеристика]


Пример: SHIRT123_Red_L

Шаг 4. Настройка автоматического обмена

Чтобы документы отправлялись в Озон без ручного запуска, настройте регламентное задание:

  1. Перейдите в Администрирование → Регламентные задания;
  2. Создайте новое задание с типом ОбменЭДОСОзон;
  3. Установите расписание: ежедневно в 09:00 и 18:00;
  4. В параметрах укажите Автоотправка = Да и Логировать ошибки = Да.

Для проверки работы задания forced-отправьте тестовый документ:



Процедура ТестОтправкиЭДО()


ОбменЭДОСОзон.ОтправитьДокументы(Истина);


КонецПроцедуры

Шаг 5. Тестирование и запуск

Перед полноценной работой выполните тестовый обмен:

  1. Создайте в 1С тестовый заказ на 1 товар;
  2. Проведите документ Реализация товаров и услуг;
  3. Вручную отправьте УПД через Действия → Отправить в ЭДО;
  4. Проверьте статус документа в личном кабинете Озон ЭДО (раздел Исходящие).

Если документ успешно принят (статус Подписан), настройка завершена. В противном случае изучите раздел с ошибками.

💡

После успешного теста включите автоматический обмен в регламентных заданиях и проверьте логи раз в неделю на наличие ошибок.

Типичные ошибки и их решения

Даже при корректной настройке могут возникать сбои. Мы собрали самые распространённые ошибки и способы их устранения:

Код ошибки Описание Решение
EDO-1001 Неверный формат документа Проверьте шаблон УПД в Настройки ЭДО → Шаблоны. Используйте версию 5.01.
EDO-2003 Не найден контрагент Сопоставьте ИНН Озон (7743017600) с карточкой контрагента в 1С.
EDO-3005 Ошибка подписи Обновите сертификат ЭЦП в Настройки → Сертификаты.
API-403 Отказано в доступе Проверьте актуальность API-key в личном кабинете Озон.

Если ошибка не указана в таблице, изучите лог обмена в Администрирование → Журнал обмена. Для сложных случаев обратитесь в поддержку Озон с указанием:

  • 📌 Даты и времени ошибки;
  • 📌 ID документа в 1С;
  • 📌 Скриншота ошибки из журнала.
⚠️ Внимание: Если после обновления 1С до версии 8.3.20+ перестали отправляться документы, проверьте настройки совместимости в Параметры → Совместимость → Использовать ЭДО 3.0. В новых версиях по умолчанию включается протокол ЭДО 3.0, который требует дополнительной настройки сертификатов.

Оптимизация работы с ЭДО: советы экспертов

Чтобы минимизировать ручную работу и избежать ошибок, следуйте этим рекомендациям:

  • 🔄 Автоматизируйте сверку: Настройте еженедельную выгрузку актов сверки из Озон в 1С через Обмен → Импорт актов. Это сократит время на проверку расхождений с 2 часов до 15 минут.
  • 📊 Используйте отчёты: В 1С создайте отчёт Анализ ЭДО с фильтрами по датам и статусам документов. Пример запроса:

  • 🔒 Контролируйте доступ: Ограничьте права на отправку документов только для бухгалтеров и логистов. В 1С это настраивается в Администрирование → Права доступа → Роли.

Критическая ошибка большинства продавцов: игнорирование статуса "Ожидает подписи" в исходящих документах. Если документ не подписан Озон в течение 3 дней, он автоматически аннулируется, и его придётся создавать заново. Чтобы избежать этого, настройте уведомления в Настройки ЭДО → Уведомления на email и SMS.

Создайте в 1С отдельную папку для документов Озон с префиксом "OZ_" (например, "OZ_УПД-001 от 01.01.2026"). Это упростит поиск и аудит.

Альтернативные способы интеграции

Если стандартная настройка через 1С вызывает сложности, рассмотрите альтернативные варианты:

  • 🔗 Облачные сервисы: Контур.Диадок, СБИС или Такском предлагают готовые коннекторы для Озон. Стоимость — от 1 500 ₽/мес. Преимущество: не требуется обновлять 1С.
  • 🤖 API-интеграция: Если у вас собственная IT-команда, можно написать скрипт на Python для обмена через API Озон v2. Пример кода для отправки УПД: 
    

    import requests
    


    url = "https://api-seller.ozon.ru/v2/edo/document"
    

    headers = {
    

    "Client-Id": "ваш_client_id",
    

    "Api-Key": "ваш_api_key",
    

    "Content-Type": "application/json"
    

    }
    

    data = {
    

    "document_type": "universal_transfer_document",
    

    "document_data": {...}
    

    }
    

    response = requests.post(url, headers=headers, json=data)
  • 📦 1С:EDO: Модуль от 1С для работы с ЭДО (стоимость — 12 000 ₽). Поддерживает все маркетплейсы, включая Озон, Wildberries и Яндекс Маркет.

Для продавцов с оборотом свыше 50 млн ₽/мес рекомендуем комбинированный подход: основной обмен через 1С, а резервный канал — через Диадок. Это страхует от сбоев при обновлениях платформы.

FAQ: Частые вопросы по подключению 1С ЭДО к Озон

Можно ли подключить ЭДО без 1С?

Да, вы можете использовать сторонние сервисы (Контур.Диадок, СБИС) или ручной ввод документов в личном кабинете Озон ЭДО. Однако без 1С вам придётся вручную заполнять все реквизиты (ИНН, суммы, номенклатуру), что занимает в 5–10 раз больше времени.

Сколько стоит подключение ЭДО на Озон?

Сама регистрация в Озон ЭДО бесплатна. Однако могут потребоваться затраты на:

  • Обновление 1С (от 5 000 ₽);
  • Сертификат ЭЦП (от 1 500 ₽/год);
  • Облачный сервис (от 1 500 ₽/мес, если не используете 1С).

Что делать, если документ завис в статусе "Обработка"?

Это типичная ошибка при перегрузке API Озон. Подождите 2–3 часа и проверьте статус заново. Если статус не изменился:

  1. Проверьте лог обмена в 1С;
  2. Отправьте документ повторно через Действия → Повторить отправку;
  3. Если ошибка повторяется, обратитесь в поддержку Озон с ID документа.

Нужно ли подключать ЭДО для FBS?

Для схемы FBS (самостоятельная логистика) ЭДО не обязательно, но крайне рекомендовано. Без него вы не сможете:

  • Автоматически формировать счета-фактуры для покупателей;
  • Получать акты сверки с Озон;
  • Участвовать в программе Озон Премиум (требуется 100% ЭДО).

Как отменить ошибочно отправленный документ?

Если документ ещё не подписан Озон (статус Ожидает подписи), вы можете отозвать его через Действия → Отозвать документ в личном кабинете Озон ЭДО. Если документ уже подписан, придётся создавать корректировочный УПД.
