Как посмотреть ID склада Озон: все рабочие методы для продавцов FBS и FBO

Работа с логистикой на Ozon требует точности: один неверный идентификатор склада может привести к задержкам отгрузки, штрафам или даже блокировке товара. ID склада (или warehouse_id) — это уникальный цифровой код, который присваивается каждому пункту приёмки и хранения товаров на маркетплейсе. Без него невозможно корректно сформировать этикетки, загрузить остатки через API или настроить автоматическую отправку заказов в системе FBS.

Проблема в том, что Ozon не всегда показывает этот идентификатор на видном месте. В зависимости от схемы работы (FBO, FBS, rFBS) и типа склада (собственный, партнёрский, перевалочный) способы получения warehouse_id отличаются. В этой статье мы разберём все актуальные методы — от ручного поиска в личном кабинете до автоматизированного парсинга через API, а также расскажем, как избежать типичных ошибок при работе с идентификаторами.

Что такое ID склада Озон и зачем он нужен

ID склада (или warehouse_id) — это уникальный числовой код, который присваивается каждому логистическому объекту Ozon. Он используется для:

  • 📦 Точной привязки товара к месту хранения (особенно важно для FBS, где товар может лежать на нескольких складах одновременно).
  • 📋 Формирования корректных этикеток и накладных при отгрузке.
  • 🔄 Автоматической синхронизации остатков через API или Excel-шаблоны.
  • 🚚 Настройки маршрутов доставки в системах WMS (Warehouse Management System).

Без этого идентификатора маркетплейс не сможет однозначно определить, на какой именно склад отправлять ваш товар — а это чревато задержками до 3–5 дней и штрафами за несвоевременную отгрузку. Например, если вы укажете в накладной warehouse_id склада в Подольске, а фактически отправите товар в Чехов, система не сможет автоматически принять посылку, и её придётся перемаркировать вручную.

Важно понимать, что ID склада ≠ адрес склада. Один физический адрес может соответствовать нескольким warehouse_id (например, если на территории работает несколько операторов), и наоборот — один warehouse_id может объединять несколько зданий (в случае крупных логистических хабов).

Где взять ID склада: официальные источники

Самый надёжный способ получить актуальный warehouse_id — использовать официальные инструменты Ozon. Рассмотрим все доступные варианты.

Способ 1: Личный кабинет Seller (раздел «Логистика»)

Это самый простой метод для продавцов, работающих по схеме FBS или rFBS. Инструкция:

  1. Авторизуйтесь в личном кабинете Seller.
  2. Перейдите в раздел Логистика → Склады.
  3. В таблице найдите нужный склад и кликните по его названию.
  4. В открывшейся карточке скопируйте значение поля ID склада (или warehouse_id).

Обратите внимание: в этом разделе отображаются только те склады, с которыми у вас уже есть активное взаимодействие (например, куда вы ранее отправляли товар). Если вам нужен warehouse_id нового склада, его придётся искать другими способами.

Сверьте название склада с адресом в договоре|Убедитесь, что статус склада "Активен"|Проверьте, что ID начинается с цифры 1–9 (не с 0)|Скопируйте ID без пробелов и дополнительных символов-->

Способ 2: API Ozon (для автоматизации)

Если вы используете API для интеграции с Ozon, получить список складов с их warehouse_id можно через запрос:

GET https://api-seller.ozon.ru/v2/warehouse/list

В ответе придет JSON-массив со всеми доступными складами, их статусами и идентификаторами. Пример ответа:

{

"result": [


{


"warehouse_id": 123456,


"name": "Озон Чехов (Основной)",


"is_rfbs": false,


"status": "active"


},


{


"warehouse_id": 789012,


"name": "Озон Подольск (Перевалочный)",


"is_rfbs": true,


"status": "active"


}


]


}

Этот метод подходит для продавцов с большим ассортиментом, которые нуждаются в автоматической синхронизации данных. Однако для работы с API требуется токен доступа и базовые навыки программирования (или готовое решение от 1C, МойСклад и т.п.).

💡

Если в ответе API отсутствует нужный склад, проверьте права вашего токена. Для работы с логистикой требуется scope logistics.

Способ 3: Excel-шаблоны для загрузки товаров

При загрузке товаров через Excel-файлы (например, для массового обновления остатков) Ozon часто требует указать warehouse_id в соответствующей колонке. Чтобы узнать актуальные идентификаторы:

  1. Скачайте шаблон для загрузки остатков в разделе Товары → Импорт/экспорт.
  2. Откройте файл и найдите лист с названием Склады (или Warehouses).
  3. В этом листе будут перечислены все доступные склады с их warehouse_id.

Важно: не используйте устаревшие шаблоны. Ozon регулярно обновляет структуру файлов, и в старых версиях могут отсутствовать актуальные склады. Всегда скачивайте свежий шаблон перед работой.

Неофициальные способы получения ID склада

Иногда официальные методы не работают: например, склад ещё не добавлен в ваш аккаунт, или вы работаете через партнёра, который не предоставляет доступ к API. В таких случаях можно воспользоваться альтернативными источниками.

Способ 4: Через мобильное приложение Ozon Seller

В мобильном приложении Ozon Seller (доступно для iOS и Android) можно найти warehouse_id следующим образом:

  1. Откройте приложение и авторизуйтесь.
  2. Перейдите в раздел Логистика → Мои склады.
  3. Выберите нужный склад и тапните по его названию.
  4. В открывшейся карточке прокрутите вниз до блока Техническая информация — там будет указан ID склада.

Преимущество этого способа — возможность быстро найти идентификатор даже в дороге. Однако в мобильной версии не всегда отображаются все склады (например, перевалочные или временно закрытые).

Что делать, если склада нет в списке?

Если в мобильном приложении или личном кабинете не отображается нужный склад, это может означать:

1. Склад временно закрыт на технические работы (проверьте у менеджера Ozon).

2. У вас нет прав на работу с этим складом (нужно оформить доступ через поддержку).

3. Склад ещё не добавлен в ваш аккаунт (требуется заключить дополнительное соглашение).

Способ 5: Парсинг публичных данных (для опытных пользователей)

В некоторых случаях warehouse_id можно найти в открытых источниках, например:

  • 📄 В PDF-инструкциях от Ozon для новых партнёров (иногда там приводятся примеры идентификаторов).
  • 💬 В чатах и форумах продавцов (например, в Telegram-каналах по FBS).
  • 🔍 Через поиск по коду ошибок (если Ozon вернул ошибку с упоминанием warehouse_id, его можно извлечь из лога).

Однако этот метод не рекомендуется для постоянного использования, так как:

  • Идентификаторы могут устареть (например, при реорганизации складов).
  • Некорректный warehouse_id приведёт к сбоям в логистике.
  • Ozon может заблокировать аккаунт за использование неофициальных данных.
💡

Всегда сверяйте полученный warehouse_id с официальными источниками. Даже если идентификатор выглядит корректно, он может относиться к другому региону или типу склада.

Как проверить корректность ID склада

Ошибка в warehouse_id может стоить дорого: от задержки доставки до штрафов за несвоевременную отгрузку. Чтобы избежать проблем, всегда проверяйте идентификатор перед использованием.

Проверка через тестовый запрос в API

Если вы работаете через API, отправьте тестовый запрос на получение информации о складе:

GET https://api-seller.ozon.ru/v1/warehouse/{warehouse_id}

Замените {warehouse_id} на ваш идентификатор. Если склад существует и доступен для вас, в ответе придут его название, адрес и статус. Если же вернётся ошибка 404 Not Found или 403 Forbidden, значит:

  • 🔴 warehouse_id неверный или устарел.
  • 🔴 У вашего аккаунта нет прав на работу с этим складом.

Проверка через поддержку Ozon

Самый надёжный, но и самый долгий способ — обратиться в поддержку с вопросом:

⚠️ Внимание: В запросе обязательно укажите:
  • Название склада (как в договоре).
  • Тип склада (FBS, rFBS, перевалочный).
  • Регион (например, "Московская область").

Без этих данных менеджер не сможет однозначно идентифицировать склад.

Ответ от поддержки обычно приходит в течение 1–2 рабочих дней. Если сроки поджимают, уточните у менеджера возможность ускоренной обработки запроса.

Типичные ошибки при работе с ID склада

Даже опытные продавцы иногда допускают ошибки при работе с идентификаторами складов. Вот самые распространённые из них и способы их избежать:

Ошибка Последствия Как избежать
Указание warehouse_id от другого склада Товар будет отправлен не по адресу, задержка до 5 дней Всегда сверяйте название склада с warehouse_id в личном кабинете
Использование устаревшего идентификатора Ошибка при загрузке остатков, блокировка товара Периодически обновляйте список складов через API или шаблоны
Копирование warehouse_id с пробелами или символами Сбой при формировании этикеток или накладных Используйте функции TRIM() в Excel для очистки данных
Путаница между FBS и rFBS складами Товар будет обработан по неправильной схеме (например, как возврат) Проверяйте флаг is_rfbs в ответе API

Одна из самых коварных ошибок — использование ID перевалочного склада вместо основного. Перевалочные склады (хабы) предназначены для временного хранения и сортировки товаров, и если вы укажете их в накладной, ваш товар может застрять на этапе распределения. Всегда уточняйте тип склада перед отгрузкой!

Создайте в Google Sheets или Excel отдельный лист со всеми актуальными warehouse_id и обновляйте его раз в месяц. Это сэкономит время при формировании отгрузок.

Частые вопросы о работе с ID склада Ozon

Мы собрали ответы на самые популярные вопросы, которые возникают у продавцов при работе с идентификаторами складов.

Можно ли использовать один и тот же warehouse_id для разных типов товаров?

Да, warehouse_id привязан к складу, а не к товару. Вы можете отправлять на один склад товары разных категорий, брендов или габаритов — если они соответствуют правилам хранения этого склада (например, не требуют специальных условий, как лекарства или продукты).

Исключение: если у вас заключён отдельный договор на хранение определённой группы товаров (например, Ozon Fresh), то для неё может быть выделен отдельный склад с уникальным warehouse_id.

Что делать, если Ozon изменил warehouse_id моего склада?

Такие случаи бывают при реорганизации логистической сети (например, при объединении или разделение складов). В этом случае:

  1. Проверьте актуальный warehouse_id через API или личный кабинет.
  2. Обновите идентификатор во всех интеграциях (1С, МойСклад, Excel-шаблоны).
  3. Свяжитесь с менеджером Ozon, чтобы уточнить, нужно ли перенаправлять уже отправленные товары.

Обычно Ozon заранее уведомляет продавцов о таких изменениях по email, но письма могут попадать в спам.

Как узнать warehouse_id для нового склада, куда я ещё не отправлял товар?

Если склад ещё не добавлен в ваш аккаунт, вам нужно:

  1. Заключить дополнительное соглашение на работу с этим складом (через менеджера Ozon).
  2. После подписания договора склад появится в личном кабинете и API в течение 1–2 дней.

Без договора вы не сможете получить warehouse_id, даже если знаете адрес склада.

Можно ли отправлять товар на склад с warehouse_id, который отмечен как "неактивный"?

Нет, отправка на неактивный склад приведёт к тому, что товар не будет принят, а вам придётся организовывать обратную логистику за свой счёт. Статус "неактивный" может означать:

  • Склад временно закрыт на ремонт.
  • Склад больше не принимает товары от новых продавцов.
  • У вашего аккаунта приостановлены права на работу с этим складом.

Уточните причину у менеджера Ozon перед планированием отгрузки.

Где хранить warehouse_id, чтобы не потерять?

Рекомендуем вести таблицу со всеми актуальными идентификаторами в одном из сервисов:

  • 📖 Google Sheets (с разделением доступа для команды).
  • 📱 Notion или ClickUp (для интеграции с другими бизнес-процессами).
  • 💾 Локальный Excel-файл с резервными копиями.

Не храните warehouse_id только в личных заметках или переписках — при смене менеджера или утере доступа к почте данные могут быть потеряны.

Заключение: как не потерять товар из-за ошибки в ID склада

Работа с warehouse_id на Ozon требует внимания к деталям, но при правильном подходе это не вызывает сложностей. Главные правила:

  • 🔍 Всегда проверяйте актуальность идентификатора перед отгрузкой.
  • 📑 Ведите собственную базу warehouse_id и обновляйте её раз в 1–2 месяца.
  • 🤝 Уточняйте у менеджера Ozon любые сомнения по поводу склада.
  • 🚫 Никогда не используйте идентификаторы из непроверенных источников.

Если вы автоматизируете работу через API, настройте уведомления об изменениях в статусах складов. Это поможет избежать ситуаций, когда товар отправляется на закрытый или перегруженный склад.

Помните: ошибка в одном символе warehouse_id может обернуться задержкой доставки на неделю и штрафами. Но если следовать инструкциям из этой статьи, вы сведете риски к минимуму и сможете сосредоточиться на развитии бизнеса, а не на решении логистических проблем.
