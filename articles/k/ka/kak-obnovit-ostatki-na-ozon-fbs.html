Как быстро обновить остатки на Ozon FBS: 5 проверенных способов

Почему важно оперативно обновлять остатки на Ozon FBS

Работа по модели FBS (Fulfillment by Seller) на Ozon требует от продавца полного контроля над логистикой — от хранения до доставки. Одним из ключевых аспектов здесь является актуализация остатков. Если вы не обновляете данные вовремя, рискуете столкнуться с ошибками при оформлении заказов, штрафами за невыполненные обязательства или даже блокировкой товара. Например, покупатель может оплатить товар, которого на самом деле нет в наличии — это приводит к отменам заказов и падению рейтинга продавца.

С другой стороны, Ozon приоритезирует товары с актуальными остатками в поисковой выдаче. Это значит, что чем чаще вы синхронизируете данные, тем выше шансы, что ваши позиции будут показываться выше конкурентов. Особенно критично это для категорий с высокой конкуренцией — электроники, одежды или товаров для дома. В этой статье разберём все актуальные способы обновления остатков, включая ручные и автоматизированные методы, а также типичные ошибки, которые допускают продавцы.

1. Обновление остатков через личный кабинет Ozon Seller

Самый простой способ — использовать веб-интерфейс Ozon Seller. Он подходит для небольших каталогов (до 1000 SKU) или разовых правок. Чтобы обновить остатки:

  1. Авторизуйтесь в личном кабинете и перейдите в раздел Товары → Остатки.
  2. Выберите нужные позиции с помощью фильтров (например, по артикулу или названию).
  3. Нажмите Редактировать и введите новое количество в колонке Доступно для продажи.
  4. Сохраните изменения кнопкой Применить.

Обратите внимание: изменения вступают в силу не мгновенно. В пиковые часы (с 10:00 до 14:00 по МСК) синхронизация может занимать до 30 минут. Если вам нужно срочно заблокировать товар (например, при нехватке на складе), используйте функцию Приостановить продажу в том же разделе.

⚠️ Внимание: При ручном вводе легко допустить опечатку в артикуле или количестве. Всегда сверяйте данные с вашей учётной системой (1С, МойСклад) перед сохранением. Озон не восстанавливает ошибочно обновлённые остатки — придётся вносить правки повторно.

2. Массовое обновление через Excel-файл

Если у вас сотни или тысячи позиций, редактировать их по одной неэффективно. Для таких случаев Ozon предоставляет возможность массового импорта остатков через Excel. Алгоритм действий:

  1. Скачайте шаблон файла в разделе Товары → Импорт/Экспорт → Скачать шаблон для остатков.
  2. Заполните колонки:
    • 📌 offer_id — уникальный идентификатор товара на Ozon (обязательно!).
    • 📦 stock — новое количество доступных единиц.
    • 🏷️ warehouse_id — ID склада (если используете несколько складов FBS).
  • Сохраните файл в формате .xlsx или .csv (разделитель — запятая).
  • Загрузите его обратно в раздел Импорт/Экспорт → Загрузить остатки.

    • Система проверит файл на ошибки и покажет отчёт. Типичные проблемы при загрузке:

    Ошибка Причина Решение
    Invalid offer_id Неверный или отсутствующий идентификатор товара Проверьте offer_id в личном кабинете или экспортируйте актуальный список товаров
    Stock value is negative Отрицательное значение в колонке stock Укажите ноль или положительное число
    Warehouse not found Указан несуществующий warehouse_id Проверьте ID склада в разделе Склады FBS
    💡

    Перед массовой загрузкой сделайте резервную копию текущих остатков. Для этого экспортируйте данные в Excel через раздел Товары → Экспорт → Остатки.

    Если файл прошёл валидацию, остатки обновятся в течение 1–2 часов. Для ускорения процесса можно разделить большой каталог на несколько файлов по 500–1000 строк.

    3. Автоматизация через API Ozon Seller

    Для крупных продавцов с динамично меняющимися остатками оптимальным решением станет интеграция через API. Это позволяет синхронизировать данные в реальном времени, избегая ручного труда. Основные методы API для работы с остатками:

    • 🔄 POST /v2/products/stocks — обновление остатков для одного или нескольких товаров.
    • 📊 GET /v1/analytics/stocks — получение текущих остатков (для сверки).
    • 🚫 POST /v1/product/info/stocks/block — временная блокировка товара.

    Пример запроса для обновления остатков (на языке Python с использованием библиотеки requests):

    import requests


    url = "https://api-seller.ozon.ru/v2/products/stocks"
    

    headers = {
    

    "Client-Id": "ваш_client_id",
    

    "Api-Key": "ваш_api_key",
    

    "Content-Type": "application/json"
    

    }
    

    payload = {
    

    "stocks": [
    

    {
    

    "offer_id": "123456789",
    

    "stock": 10,
    

    "warehouse_id": 1
    

    }
    

    ]
    

    }
    


    response = requests.post(url, headers=headers, json=payload)
    

    print(response.json())

    Ключевые моменты при работе с API:

    • 🔐 Используйте HTTPS и никогда не передавайте Api-Key в открытом виде.
    • ⏱️ Ограничение на количество запросов — 1000 в минуту (при превышении вернётся ошибка 429 Too Many Requests).
    • 📈 Для крупных каталогов реализуйте пакетную обработку (например, обновляйте остатки партиями по 100 SKU).
    Как получить Client-Id и Api-Key?

    Чтобы получить доступ к API, перейдите в личный кабинет Ozon Seller → Настройки → Интеграции → API-ключи. Сгенерируйте новый ключ и сохраните его в безопасном месте (например, в менеджере паролей). Один аккаунт может иметь несколько ключей с разными правами доступа.

    ⚠️ Внимание: При интеграции через API учитывайте задержку синхронизации между вашей системой учёта и Ozon. Например, если вы обновляете остатки в 1С и сразу отправляете данные на маркетплейс, возможны расхождения из-за неучтённых заказов. Решение — настраивать крон-задачи с интервалом в 5–10 минут.

    4. Синхронизация с 1С, МойСклад и другими системами учёта

    Многие продавцы используют 1С, МойСклад, Bitrix24 или RetailCRM для управления запасами. Чтобы избежать двойной работы, можно настроить автоматическую синхронизацию между этими системами и Ozon. Рассмотрим популярные способы:

    1С:Предприятие

    Для интеграции потребуется:

    1. Установить расширение Ozon Seller из Маркетплейса 1С.
    2. Настроить обмен данными через HTTP-сервис или REST API.
    3. Сопоставить поля артикул в 1С с offer_id на Ozon.

    МойСклад

    В МойСклад есть встроенный модуль для работы с маркетплейсами:

    1. Перейдите в Настройки → Интеграции → Ozon.
    2. Авторизуйтесь через Ozon Seller и выберите склады для синхронизации.
    3. Настройте расписание обновлений (рекомендуется каждые 2–4 часа).

    Преимущества автоматизированной синхронизации:

    • ⏳ Экономия времени — нет нужды вручную экспортировать/импортировать данные.
    • 📉 Снижение риска ошибок (например, несовпадения артикулов).
    • 🔄 Возможность настроить обратную синхронизацию — когда заказы с Ozon автоматически резервируют товар в вашей системе.
    💡

    При настройке интеграции обязательно тестируйте синхронизацию на небольшой группе товаров (5–10 SKU), прежде чем подключать весь каталог.

    5. Обновление остатков через мобильное приложение Ozon Seller

    Если вам нужно срочно скорректировать остатки в дороге или на складе, можно воспользоваться мобильным приложением Ozon Seller (доступно для iOS и Android). Функционал здесь ограничен, но основные действия выполнить можно:

    1. Откройте приложение и авторизуйтесь.
    2. Перейдите в раздел Товары → Остатки.
    3. Найдите нужный товар через поиск или фильтры.
    4. Нажмите на позицию и выберите Редактировать остаток.
    5. Введите новое значение и сохраните.

    Ограничения мобильной версии:

    • 🚫 Нет возможности массового редактирования (только по одному товару).
    • 📱 Отсутствует поддержка загрузки Excel-файлов.
    • ⏳ Синхронизация может занимать до 1 часа (в зависимости от нагрузки на сервер).

    Приложение удобно для экстренных правок, но не подходит для регулярного управления большим каталогом. Например, если вы заметили расхождение остатков во время инвентаризации на складе, можно оперативно заблокировать товар прямо с телефона.

    6. Типичные ошибки и как их избежать

    Даже опытные продавцы иногда допускают ошибки при обновлении остатков. Вот наиболее распространённые проблемы и способы их решения:

    Ошибка Последствия Как избежать
    Неучтённые заказы в обработке Продажа товара, который уже зарезервирован под другой заказ Всегда вычитайте из остатков количество единиц в статусе "В обработке" или "В сборке"
    Разные остатки на нескольких складах FBS Покупатель видит товар в наличии, но его нет на ближайшем складе Используйте warehouse_id для привязки остатков к конкретному складу
    Задержка синхронизации Товар продаётся в минус (остаток обновился слишком поздно) Настройте автоматическое обновление через API с интервалом 15–30 минут
    Опечатки в offer_id Остатки обновляются для чужих товаров Проверяйте данные через экспорт текущих остатков перед массовым импортом

    Ещё одна распространённая проблема — несоответствие остатков в разных системах. Например, в 1С указано 10 единиц товара, а на Ozon — 8. Это может происходить из-за:

    • 🔄 Неучтённых возвратов или отмен заказов.
    • 📦 Ошибок при приёмке товара на склад.
    • ⏳ Задержки синхронизации между системами.

    Решение: еженедельно проводите сверку остатков между Ozon и вашей учётной системой. Для этого экспортируйте данные из личного кабинета и сравнивайте их с отчётами из 1С/МойСклад.

    7. Как проверить, что остатки обновлены корректно

    После внесения изменений важно убедиться, что они применились без ошибок. Вот как это сделать:

    1. Проверка в личном кабинете:
      • Перейдите в Товары → Остатки и найдите обновлённые позиции.
      • Убедитесь, что в колонке Доступно для продажи отображается правильное значение.
    2. Экспорт данных:
      • Скачайте актуальный отчёт по остаткам через Товары → Экспорт → Остатки.
      • Сравните его с вашим исходным файлом (если обновляли через Excel).
    3. Проверка через API:
      • Выполните запрос GET /v1/analytics/stocks с указанием offer_id.
      • Сверьте полученные данные с ожидаемыми значениями.
  • Тестовый заказ:
    • Создайте тестовый заказ на товар с ограниченным остатком (например, 1 шт.).
    • Убедитесь, что после оформления остаток уменьшился до 0.

      • Если остатки не обновляются:

      1. Проверьте Журнал операций в личном кабинете на наличие ошибок.
      2. Убедитесь, что у вас достаточно прав доступа (например, роль "Администратор" или "Менеджер товаров").
      3. Обратитесь в поддержку Ozon, прикрепив скриншоты ошибок и лог файла (если обновляли через Excel).
      💡

      Для ускорения проверки используйте фильтр по дате последнего обновления. В личном кабинете в разделе Остатки можно отсортировать товары по колонке Последнее изменение.

      FAQ: Частые вопросы об обновлении остатков на Ozon FBS

      Можно ли обновить остатки ночью, чтобы избежать пиковой нагрузки?

      Да, но учтите, что технические работы на стороне Ozon чаще всего проводятся с 02:00 до 05:00 по МСК. В это время синхронизация может занять больше времени или временно недоступна. Оптимальное время для массовых обновлений — с 20:00 до 23:00.

      Что делать, если остатки обновляются с задержкой?

      Задержки до 2–3 часов считаются нормальными в пиковые периоды (например, во время распродаж). Если синхронизация не происходит более 6 часов:

      1. Проверьте статус сервисов Ozon на странице https://status.ozon.ru/.
      2. Обратитесь в поддержку с указанием offer_id проблемных товаров.
    Как обновить остатки для товаров с вариациями (размерами, цветами)?

    Для товаров с вариациями (например, футболки разных размеров) используйте:

    • В Excel: укажите offer_id каждой вариации отдельно.
    • В API: передавайте массив объектов с уникальными offer_id для каждой комбинации.

      • Пример структуры для API:

      {

      "stocks": [
      

      {
      

      "offer_id": "123456789_1", // вариация "Чёрный, S"
      

      "stock": 5
      

      },
      

      {
      

      "offer_id": "123456789_2", // вариация "Чёрный, M"
      

      "stock": 10
      

      }
      

      ]
      

      }

    Можно ли откатить ошибочное обновление остатков?

    Нет, Ozon не предоставляет функцию отката. Если вы ошиблись:

    1. Срочно загрузите новый файл с корректными данными.
    2. Если товар был продан в минус — свяжитесь с покупателем и предложите альтернативу или компенсацию.
    Нужно ли обновлять остатки для товаров, которые временно недоступны?

    Да. Если товар отсутствует на складе, установите остаток 0. Это лучше, чем оставлять старые данные, так как:

    • Покупатели не увидят товар в поиске (если остаток = 0).
    • Вы избежите штрафов за невыполненные заказы.

    Чтобы вернуть товар в продажу, просто обновите остаток до актуального значения.
