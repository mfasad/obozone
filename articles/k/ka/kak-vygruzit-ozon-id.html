Как выгрузить Ozon ID: 5 проверенных способов для продавцов

Работа с маркетплейсом Ozon требует регулярного обмена данными о товарах, и одним из ключевых идентификаторов является Ozon ID — уникальный номер продукта в системе. Без него невозможно обновлять карточки, синхронизировать остатки или анализировать продажи через внешние сервисы. Однако не все продавцы знают, как правильно выгрузить эти идентификаторы, особенно если речь идет о тысячах позиций.

В этой статье мы разберем все актуальные способы получения Ozon ID в 2026 году — от ручного копирования до автоматизированных решений через API. Вы узнаете, какие инструменты подходят для небольших каталогов, а какие оптимальны для крупных продавцов с сотнями тысяч SKU. Особое внимание уделим типичным ошибкам, которые приводят к потере данных или блокировке аккаунта.

Что такое Ozon ID и зачем он нужен

Ozon ID (или product_id в API) — это уникальный числовой идентификатор товара в базе маркетплейса. Он присваивается автоматически при создании карточки и никогда не изменяется, даже если вы редактируете название или артикул. Этот ID используется для:

  • 🔄 Синхронизации остатков через FBS или FBO
  • 📊 Экспорта статистики продаж в аналитические системы
  • 🔧 Массового редактирования карточек через Excel или API
  • 🤝 Интеграции с 1С, МойСклад или другими ERP-системами

Без Ozon ID невозможно точно идентифицировать товар в системе — например, если у вас несколько вариантов одного продукта (разные цвета или размеры), их ID будут отличаться. Важно понимать, что это не то же самое, что sku (ваш внутренний артикул) или offer_id (идентификатор торгового предложения).

⚠️ Внимание: Никогда не путайте Ozon ID с item_id из заказов — это разные сущности. Первый относится к товару, второй — к конкретной покупке в корзине.

Способ 1: Ручная выгрузка через личный кабинет Ozon Seller

Самый простой, но и самый трудоемкий метод — скачать данные прямо из интерфейса Ozon Seller. Он подходит для небольших каталогов (до 500 товаров) или разовых задач. Вот как это сделать:

  1. Авторизуйтесь в личном кабинете и перейдите в раздел Товары → Мои товары.
  2. Нажмите кнопку Экспорт в правом верхнем углу.
  3. Выберите формат Excel (.xlsx) и отметьте галочкой поле ID товара на Ozon.
  4. Дождитесь формирования файла (может занять до 10 минут) и скачайте его.

В полученном файле Ozon ID будет в колонке A (или в отдельном столбце с названием product_id). Обратите внимание, что в экспорте не всегда присутствуют все поля — если вам нужны дополнительные данные (например, категории или атрибуты), их придется выгружать отдельно.

Способ 2: Выгрузка через Excel-шаблон Ozon

Если вам нужно не только получить ID, но и обновить данные по товарам, удобнее использовать официальный шаблон Ozon. Этот метод подходит для массового редактирования и поддерживает до 10 000 строк за одну загрузку.

Инструкция:

  1. Скачайте актуальный шаблон в разделе Товары → Импорт/экспорт → Скачать шаблон.
  2. Заполните колонку Ваш SKU (артикулы ваших товаров).
  3. Сохраните файл в формате .xlsx (не .csv!) и загрузите его через Импорт → Обновить товары.
  4. После обработки скачайте отчет об ошибках — в нем будет колонка с Ozon ID для успешных позиций.

Преимущество этого метода в том, что вы одновременно можете обновлять цены, остатки или описания. Однако есть нюансы:

  • 🚫 Если в шаблоне ошибка хотя бы в одной ячейке, весь импорт может провалиться.
  • ⏳ Обработка большого файла занимает от 30 минут до нескольких часов.
  • 📌 В отчете Ozon ID будет только для тех товаров, которые успешно прошли валидацию.

Способ 3: Автоматизированная выгрузка через Ozon API

Для продавцов с большими каталогами (от 10 000 товаров) или тех, кто использует собственные IT-решения, оптимальным вариантом будет работа через API Ozon Seller. Этот метод требует технических навыков или помощи разработчика, но дает максимальную гибкость.

Основные этапы:

  1. Получите Client-ID и API-Key в разделе Настройки → API.
  2. Используйте эндпоинт /v2/product/info/list для запроса данных по товарам.
  3. В теле запроса укажите массив product_id или offer_id (если нужны варианты).
  4. Обработайте ответ, где для каждого товара будет поле "id": 12345678 (это и есть Ozon ID).

Пример запроса на Python:

import requests


url = "https://api-seller.ozon.ru/v2/product/info/list"


headers = {


"Client-Id": "ваш_client_id",


"Api-Key": "ваш_api_key",


"Content-Type": "application/json"


}


payload = {


"product_id": [12345678, 87654321],  # или "offer_id"


"sku": [100001, 100002]  # опционально


}



response = requests.post(url, json=payload, headers=headers)


print(response.json())

Ограничения API:

  • 🔒 Лимит: 100 запросов в минуту (при превышении — блокировка на 5 минут).
  • 📦 В одном запросе можно передать до 1000 product_id.
  • 🛡️ Для работы с API обязательна двухфакторная аутентификация в личном кабинете.
Как обойти лимит API при выгрузке больших каталогов?

Используйте пагинацию: разбивайте массив ID на пачки по 1000 штук и делайте запросы с задержкой в 1-2 секунды.

Для критически важных задач запрашивайте увеличение лимитов через поддержку Ozon (нужны веские основания).

Кэшируйте полученные данные локально, чтобы не запрашивать одни и те же ID повторно.

Если вы используете 1С, МойСклад или банковские эквайринговые системы (например, Альфа-Банк), Ozon ID можно получить через их встроенные интеграции. Рассмотрим популярные варианты:

Сервис Как получить Ozon ID Ограничения
1С:Управление торговлей Через обработку "Обмен с Ozon" в разделе "Интернет-магазины". ID подтягиваются автоматически при синхронизации. Требуется доработка типовой конфигурации для массовой выгрузки.
МойСклад В настройках интеграции с Ozon включите опцию "Синхронизировать ID товаров". Данные обновляются раз в сутки. Бесплатный тариф ограничивает количество синхронизируемых товаров (до 1000).
Альфа-Банк (эквайринг) В личном кабинете мерчанта в разделе "Маркетплейсы" доступен экспорт транзакций с привязкой к Ozon ID. ID показываются только для товаров, по которым были продажи.

Преимущество этого метода в том, что вы получаете данные в привычном интерфейсе и можете сразу их использовать для аналитики или бухгалтерии. Однако есть риски:

⚠️ Внимание: При синхронизации через 1С или МойСклад Ozon ID могут дублироваться, если в вашей базе есть товары-связки (комплекты). Всегда сверяйте итоговое количество уникальных ID с реальным ассортиментом.

Способ 5: Альтернативные методы (парсинг, поддержка Ozon)

Если стандартные способы не подходят, можно воспользоваться альтернативными вариантами — но они требуют осторожности:

  • 🤖 Парсинг страниц товаров: С помощью скриптов (например, на Python + BeautifulSoup) можно извлекать Ozon ID из URL карточек. Однако это нарушает правила Ozon и может привести к блокировке.
  • 📧 Запрос в поддержку: Через тикет в Помощь → Обратиться в поддержку можно запросить выгрузку ID для всех ваших товаров. Обычно отвечают в течение 1-3 дней.
  • 📊 Отчеты по продажам: В разделе Аналитика → Отчеты некоторые отчеты (например, "Детализация продаж") содержат Ozon ID. Минус — только для проданных товаров.

Самый легальный из альтернативных способов — запрос в поддержку. Пример формулировки тикета:

Тема: Запрос выгрузки Ozon ID для всего каталога

Текст:


Добрый день!


Просим предоставить файл со всеми Ozon ID наших товаров (всего ~X позиций) в формате Excel.


Необходимо для интеграции с внутренней системой учета.


Спасибо!

💡

Если вам срочно нужен Ozon ID для одного товара, откройте его карточку на Ozon, скопируйте цифры из URL после "/product/" — это и есть нужный идентификатор. Например, в ссылке https://www.ozon.ru/product/смартфон-xiaomi-redmi-note-12-128gb-154327689/ ID равен 154327689.

Типичные ошибки и как их избежать

При выгрузке Ozon ID продавцы часто сталкиваются с проблемами, которые ведут к потере данных или ошибкам в интеграциях. Вот самые распространенные:

  1. Путаница между product_id и offer_id:

    product_id — это ID карточки товара, а offer_id — ID конкретного варианта (например, размер или цвет). Если вы обновите остатки по product_id для товара с вариантами, система не поймет, к какому именно offer_id относится изменение.

  2. Использование устаревших ID:

    Если товар был удален и создан заново, его Ozon ID изменится. Всегда проверяйте актуальность данных перед массовыми операциями.

  3. Превышение лимитов API:

    При автоматической выгрузке через API легко нарваться на ограничение в 100 запросов/минуту. Используйте паузы между пачками запросов.

Чтобы минимизировать риски:

  • 🔍 Перед массовыми операциями тестируйте процесс на 5-10 товарах.
  • 📋 Ведите журнал изменений, где фиксируйте дату последней выгрузки ID.
  • 🔄 Регулярно (раз в месяц) обновляйте локальную базу ID, так как они могут меняться при модификации карточек.
💡

Всегда сверяйте количество выгруженных Ozon ID с реальным числом товаров в вашем каталоге. Расхождения более чем на 5% — повод проверить настройки фильтров или обратиться в поддержку.

FAQ: Частые вопросы о выгрузке Ozon ID

Можно ли получить Ozon ID для товаров, которые еще не опубликованы?

Нет. Ozon ID присваивается только после модерации и публикации товара. Для черновиков этот идентификатор не генерируется. Если вам нужно заранее подготовить интеграцию, используйте свои внутренние SKU, а после публикации замените их на Ozon ID.

Что делать, если в выгрузке не хватает Ozon ID для некоторых товаров?

Причины могут быть разные:

  • Товар удален или заблокирован модерацией.
  • В вашем аккаунте недостаточно прав (например, вы менеджер с ограниченным доступом).
  • Ошибка в фильтрах при экспорте (например, выгружаются только активные товары).

Проверьте статус товаров в личном кабинете и повторите выгрузку без фильтров.

Можно ли использовать Ozon ID из чужого аккаунта для своих товаров?

Нет, это невозможно. Ozon ID уникален для каждого продавца и товара. Даже если вы продаете аналогичный товар, его ID будет другим. Попытки использовать чужие идентификаторы приведут к ошибкам при загрузке данных.

Как часто обновляются Ozon ID?

Ozon ID статичен и не изменяется в течение всего жизненного цикла товара, кроме случаев:

  • Товар был удален и создан заново (в этом случае Assign новый ID).
  • Произошла смена категории товара с полной перемодерацией.

В остальных случаях (редактирование описания, цены, фото) ID остается прежним.

Есть ли плагины для WordPress/WooCommerce, которые автоматически подтягивают Ozon ID?

Да, например:

  • Ozon WooCommerce Integration (официальный плагин от Ozon).
  • WP All Import с аддоном для Ozon.

Эти решения позволяют синхронизировать ID при импорте товаров, но требуют настройки API-доступа.
