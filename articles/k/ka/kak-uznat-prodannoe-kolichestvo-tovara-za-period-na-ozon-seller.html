Как продавцу на Ozon узнать точное количество проданных товаров за период

Продажи на Ozon — это не только прибыль, но и данные, которые помогают принимать стратегические решения. Знание точного количества проданных товаров за день, неделю или месяц позволяет оптимизировать запасы, анализировать спрос и корректировать маркетинговые кампании. Однако многие продавцы сталкиваются с проблемой: где именно в личном кабинете Ozon Seller искать эти цифры, особенно если речь идёт о ретроспективе за несколько месяцев?

В этой статье мы разберём все доступные способы получения статистики продаж — от стандартных отчётов до скрытых функций и API. Вы узнаете, как выгрузить данные в Excel, какие фильтры использовать для точного анализа, и почему иногда цифры в разных разделах личного кабинета могут не совпадать. А ещё — как автоматизировать сбор статистики, чтобы не тратить время на ручной экспорт.

1. Стандартный отчёт «Продажи» в личном кабинете Ozon Seller

Самый очевидный и быстрый способ — использовать встроенный раздел «Аналитика» → «Продажи». Здесь собираются данные о всех заказах, включая количество проданных единиц товара. Чтобы получить точные цифры за нужный период:

  1. Перейдите в Личный кабинет → Аналитика → Продажи.
  2. Установите диапазон дат в правом верхнем углу (например, 01.01.2026 — 31.01.2026).
  3. В таблице выберите колонку Количество товаров — здесь отображается суммарное число проданных единиц.

⚠️ Внимание: По умолчанию отчёт показывает данные по FBO и FBS вместе. Если вы работаете только по одной схеме, используйте фильтр Тип логистики в левом сайдбаре.

В этом отчёте также доступны дополнительные метрики:

  • 📊 Выручка — сумма продаж без учёта комиссий.
  • 📦 Количество заказов — сколько уникальных покупок было совершено.
  • 🔄 Возвраты — сколько товаров вернули (вычитается из общего количества продаж).
💡

Если вам нужны данные по конкретному товару, используйте фильтр Артикул или Название товара в этом же отчёте.

2. Детализация по товарам: отчёт «Товары и цены»

Если вам важно узнать, сколько именно единиц конкретного товара было продано (например, для анализа хитов и аутсайдеров), перейдите в раздел Аналитика → Товары и цены. Здесь данные группируются по SKU (артикулам), что удобно для:

  • 📈 Сравнения продаж разных модификаций одного товара.
  • 🛒 Анализа остатков на складе.
  • 💰 Корректировки цен на основе спроса.

Чтобы выгрузить данные:

  1. Выберите период и примените фильтры (например, по категории или бренду).
  2. Нажмите Экспорт в Excel — файл будет содержать колонку Количество проданных единиц.

⚠️ Внимание: В этом отчёте не учитываются отменённые заказы. Если вам нужна «чистая» статистика, используйте отчёт Продажи с фильтром Статус заказа = Выполнен.

3. Экспорт данных через «Архив заказов»

Если вам нужна максимально детализированная информация (например, для бухгалтерии или анализа пиковых дней продаж), используйте Архив заказов. Здесь можно выгрузить полный список заказов с указанием количества товаров в каждом.

Инструкция:

  1. Перейдите в Заказы → Архив заказов.
  2. Установите период и фильтры (например, Статус = Выполнен).
  3. Нажмите Экспорт → Excel.

В полученном файле будет колонка Количество товаров, которую можно суммировать для расчёта общего объёма продаж. Этот метод подходит, если вам нужны:

  • 📅 Данные по дням (например, для анализа сезонности).
  • 📌 Информация о покупателях (ID заказа, дата, время).
  • 🔍 Детализация по регионам доставки.

⚠️ Внимание: При большом количестве заказов (более 10 000 строк) Ozon может разбить экспорт на несколько файлов. Скачивайте их все и объединяйте в одном документе.

4. Использование Ozon API для автоматизации

Если вы ведёте продажи в большом масштабе или хотите интегрировать данные с другими системами (например, 1С или Google Sheets), наилучший вариант — Ozon API. Через него можно получать статистику продаж в реальном времени без ручного экспорта.

Для этого:

  1. Получите API-ключ в настройках личного кабинета (Настройки → Интеграции → API).
  2. Используйте метод GET /v2/analytics/sales для получения данных о продажах.
  3. Для детализации по товарам используйте GET /v2/analytics/data с параметром metrics=items_sold.

Пример запроса для получения количества проданных товаров за месяц:

curl -X GET 'https://api-seller.ozon.ru/v2/analytics/sales?date_from=2026-01-01&date_to=2026-01-31' \

-H 'Client-Id: {ваш_client_id}' \


-H 'Api-Key: {ваш_api_key}'

⚠️ Внимание: API имеет ограничения по количеству запросов (обычно 1000 в час). Для крупных продавцов может потребоваться увеличение лимита через поддержку.

Как получить расширенные права на API?

Для увеличения лимита запросов или доступа к дополнительным методам (например, отчётам по возвратам) напишите в поддержку Ozon с темой "Запрос на расширение прав API". Приложите обоснование (например, количество товаров в каталоге или объём продаж).

5. Скрытые возможности: отчёт «Движение товаров»

Мало кто знает, но в Ozon Seller есть скрытый отчёт, который показывает не только продажи, но и все движения товара (поступления, списания, возвраты). Это полезно, если вам нужно:

  • 🔄 Отследить баланс остатков.
  • 📉 Понять причины расхождений между фактическими продажами и данными в отчётах.
  • 🛠️ Выявить ошибки склада (например, потери или неправильные списания).

Как найти этот отчёт:

  1. Перейдите в Склад → Движение товаров.
  2. Выберите период и фильтр Тип операции = Продажа.
  3. Экспортируйте данные в Excel — колонка Количество покажет проданные единицы.

Важно: Этот отчёт учитывает только фактические списания со склада, поэтому данные могут отличаться от отчёта «Продажи» (например, если заказ был отменён после списания).

6. Альтернативные способы: Google Sheets + Ozon Keeper

Если вам нужно автоматизировать сбор данных без программирования, можно использовать:

1. Ozon Keeper — бесплатный инструмент от Ozon для мониторинга продаж. Он позволяет:

  • 📊 Видеть продажи в реальном времени.
  • 🔔 Настраивать уведомления о низких остатках.
  • 📥 Экспортировать данные в Google Sheets.

2. Интеграция с Google Sheets через Apps Script:

  1. Создайте новый Google Sheet.
  2. Перейдите в Extensions → Apps Script.
  3. Вставьте скрипт для запроса к Ozon API (примеры есть в документации Ozon).
  4. Настройте автоматическое обновление данных (например, раз в день).
💡

Автоматизация через Google Sheets подходит для продавцов, которые ведут учёт в облаке и хотят видеть актуальные данные без ручного экспорта.

7. Почему данные в отчётах могут не совпадать?

Многие продавцы сталкиваются с ситуацией, когда цифры в отчёте Продажи и Архив заказов различаются. Причины могут быть следующими:

1. Учёт отменённых заказов:

  • Отчёт Продажи по умолчанию показывает все заказы, включая отменённые.
  • В Архиве заказов можно фильтровать по статусу Выполнен.

2. Возвраты и частичные возвраты:

  • Если покупатель вернул часть заказа, в отчёте Товары и цены это может не отразиться.
  • Для точного учёта используйте отчёт Возвраты (Аналитика → Возвраты).

3. Задержки обновления данных:

  • Некоторые отчёты (например, Движение товаров) обновляются с задержкой до 24 часов.
  • API даёт данные в реальном времени, но требует настройки.

⚠️ Внимание: Если расхождения критичны (более 5%), запросите сверку у поддержки Ozon через тикет с темой «Расхождение в данных по продажам». Приложите скриншоты отчётов и укажите период.

FAQ: Частые вопросы о продажах на Ozon

Можно ли увидеть продажи по конкретному ПВЗ?

Да, в отчёте Архив заказов есть колонка Пункт выдачи. Экспортируйте данные в Excel и фильтруйте по нужному ПВЗ.

Как узнать продажи за прошлый год?

В личном кабинете доступны данные за последние 13 месяцев. Для более ранних периодов запросите архив у поддержки Ozon.

Почему в отчёте «Товары и цены» нет данных по некоторым SKU?

Это может быть связано с:

  • 🔴 Товар удалён из каталога.
  • 🔴 SKU был изменён (исторические данные привязаны к старому артикулу).
  • 🔴 Продажи были по FBS, а фильтр установлен на FBO.
Можно ли получить данные по продажам в разрезе времени (по часам)?summary>

Да, но только через Ozon API с параметром granularity=hour. В личном кабинете детализация доступна только по дням.

Как посчитать продажи по акции (например, во время «Чёрной пятницы»)?

Используйте фильтр Тип заказа = Акционный в отчёте Продажи или экспортируйте данные из Архива заказов с указанием периода акции.
