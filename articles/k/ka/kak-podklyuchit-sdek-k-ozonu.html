Подключение SDK Ozon: полная инструкция для разработчиков и продавцов

Интеграция с Ozon SDK открывает продавцам доступ к автоматизации бизнес-процессов: от управления заказами до синхронизации остатков в реальном времени. Однако процесс подключения часто вызывает вопросы — особенно у тех, кто впервые работает с API маркетплейса. В этой статье разберём все этапы подключения SDK к Ozon, включая регистрацию приложения, настройку токенов, тестирование запросов и решение типичных ошибок.

Важно понимать, что SDK (Software Development Kit) — это не просто набор инструментов, а полноценный мост между вашей системой учета (1С, Bitrix24, самописным CRM) и платформой Ozon. Без правильной настройки интеграции вы рискуете столкнуться с расхождениями в остатках товаров, задержками обработки заказов или даже блокировкой аккаунта за нарушение SLA. Мы сосредоточимся на актуальных требованиях 2026 года, включая изменения в авторизации и новые эндпоинты API для схем FBS и FBO.

Что такое SDK Ozon и зачем оно нужно продавцам

SDK (Software Development Kit) от Ozon — это программный комплекс, который позволяет автоматизировать взаимодействие с платформой через API. В отличие от ручного ввода данных в личном кабинете, SDK даёт возможность:

  • 📦 Синхронизировать остатки товаров в реальном времени (критично для схемы FBS, где штрафы за несоответствие достигают 500₽ за позицию).
  • 📝 Автоматически принимать и обрабатывать заказы, включая генерацию этикеток и передачу трек-номеров.
  • 🔄 Обновлять статусы заказов (от "В обработке" до "Доставлен") без участия оператора.
  • 📊 Получать аналитику по продажам, возвратам и штрафам в структурированном виде.

На практике SDK устраняет необходимость вручную копировать данные между системами. Например, при поступлении заказа на Ozon информация моментально попадает в вашу 1С или MoySklad, где формируется накладная и печатается этикетка. Это сокращает время обработки с 30+ минут до нескольких секунд и снижает риск ошибок.

Важно: SDK не заменяет личный кабинет Ozon Seller, а дополняет его. Некоторые операции (например, оспаривание штрафов или настройка акций) по-прежнему требуют ручного вмешательства. Однако 90% рутинных задач можно делегировать системе.

Подготовка к подключению: регистрация приложения в Ozon

Прежде чем приступать к технической интеграции, необходимо зарегистрировать своё приложение в экосистеме Ozon. Этот шаг обязателен — без него вы не получите client_id и client_secret, которые нужны для авторизации запросов.

Инструкция по регистрации:

  1. Перейдите в раздел "Мои приложения" в личном кабинете Ozon Seller.
  2. Нажмите "Создать приложение" и заполните форму:
    • 📌 Название приложения — укажите понятное имя (например, "Интеграция с 1С для [Название вашей компании]").
    • 🔗 Redirect URI — URL, на который Ozon будет перенаправлять после авторизации (например, https://ваш-домен.ru/ozon/callback).
    • 📝 Описание — кратко укажите цель интеграции (например, "Автоматическая синхронизация заказов и остатков").
  • После сохранения вы получите client_id (идентификатор приложения) и client_secret (секретный ключ). Сохраните их в надёжном месте — при утере придётся создавать новое приложение.
    • 💡

    Не используйте один и тот же client_id для тестового и боевого окружений. Создайте отдельные приложения для песочницы (sandbox) и продакшена.

    На этапе регистрации многие продавцы допускают ошибку, указывая неверный Redirect URI. Этот адрес должен быть доступен из интернета и обработан вашим сервером. Если вы тестируете интеграцию локально, используйте инструменты вроде ngrok для создания временного туннеля.

    Получение токенов доступа: OAuth 2.0 в действии

    Авторизация в API Ozon построена на протоколе OAuth 2.0. Это означает, что для каждого запроса к SDK вам потребуется действующий токен доступа (access_token). Процесс его получения состоит из нескольких шагов:

    Шаг 1. Формирование ссылки для авторизации

    Сгенерируйте URL по шаблону:

    https://id.ozon.ru/oauth/authorize?

    response_type=code&
    

    client_id=ВАШ_CLIENT_ID&
    

    redirect_uri=ВАШ_REDIRECT_URI&
    

    scope=api&
    

    state=произвольная_строка

    Откройте эту ссылку в браузере, где вы авторизованы в личном кабинете Ozon Seller. После подтверждения прав система перенаправит вас на указанный redirect_uri с кодом авторизации (code) в параметрах URL.

    Шаг 2. Обмен кода на токен

    Отправьте POST-запрос на эндпоинт https://id.ozon.ru/oauth/token с следующими параметрами:

    grant_type=authorization_code&

    code=ПОЛУЧЕННЫЙ_КОД&
    

    client_id=ВАШ_CLIENT_ID&
    

    client_secret=ВАШ_CLIENT_SECRET&
    

    redirect_uri=ВАШ_REDIRECT_URI

    В ответ вы получите JSON с access_token и refresh_token. Первый используется для авторизации запросов к API, второй — для обновления токена по истечении срока действия (обычно 1 час).

    Что делать, если токен истёк?

    Если access_token просрочен, отправьте POST-запрос на https://id.ozon.ru/oauth/token с параметрами: 

    grant_type=refresh_token&

    refresh_token=ВАШ_REFRESH_TOKEN&
    

    client_id=ВАШ_CLIENT_ID&
    

    client_secret=ВАШ_CLIENT_SECRET

    Это вернёт новый access_token без повторной авторизации пользователя.

    ⚠️ Внимание: Никогда не храните client_secret и токены в открытом виде в коде или репозиториях. Используйте защищённые переменные окружения или сервисы вроде AWS Secrets Manager.

    Настройка SDK: выбор библиотеки и базовая конфигурация

    Ozon официально поддерживает SDK для нескольких языков программирования: Python, PHP, Java и Go. Выбор зависит от стека технологий вашей компании. Ниже рассмотрим базовую настройку на примере Python (наиболее популярный вариант среди продавцов).

    Установка библиотеки:

    pip install ozon-seller-api

    Пример базовой конфигурации:

    from ozon_seller import Client


    client = Client(
    

    client_id="ВАШ_CLIENT_ID",
    

    api_key="ВАШ_ACCESS_TOKEN",  # или используйте refresh_token для автоматического обновления
    

    sandbox=True  # False для боевого режима
    

    )

    Параметр sandbox=True включает режим песочницы, где вы можете тестировать запросы без риска повлиять на реальные данные. Всегда начинайте интеграцию с тестового окружения, даже если у вас уже есть опыт работы с API.

    Изучены лимиты API (например, 60 запросов в минуту для остатков)|Создан отдельный пользователь в личном кабинете для интеграции|Настроено логирование ошибок|Проверена актуальность версии SDK (не ниже 2.1.0)-->

    Для других языков процесс аналогичен:

    • 🐍 Python: ozon-seller-api (официальная библиотека).
    • 🐘 PHP: ozon-seller-php-sdk (неофициальная, но поддерживаемая сообществом).
    • ☕ Java: com.ozon:seller-api-client (Maven-пакет).

    Основные методы API: что можно автоматизировать

    SDK Ozon предоставляет доступ к десяткам эндпоинтов, но для старта достаточно освоить 5 ключевых групп методов:

    Категория Эндпоинт Назначение Пример использования
    Управление товарами /v1/product/info Получение информации о товаре Проверка актуальных характеристик перед обновлением
    Остатки /v1/product/stocks Обновление остатков на складах Синхронизация с 1С каждые 15 минут
    Заказы /v1/posting/fbs/list Получение списка заказов (FBS) Автоматическое принятие новых заказов
    Статусы /v1/posting/fbs/ship Отметка заказа как "Отправлен" Передача трек-номера курьеру
    Аналитика /v1/analytics/sales Отчёты по продажам Еженедельный экспорт данных в Excel

    Рассмотрим подробнее обновление остатков — одну из самых востребованных операций. Для этого используется эндпоинт /v1/product/stocks с телом запроса в формате:

    {

    "stocks": [
    

    {
    

    "offer_id": "123456",  // ваш артикул
    

    "stock": 10,           // новое количество
    

    "warehouse_id": 1      // ID склада (1 для FBS, другие для FBO)
    

    }
    

    ]
    

    }

    ⚠️ Внимание: При работе со схемой FBS обновляйте остатки не реже чем раз в 2 часа. В противном случае Ozon может приостановить продажи товара до ручной синхронизации.

    Тестирование и отладка: как избежать ошибок

    Даже при правильной настройке SDK ошибки неизбежны. Чаще всего продавцы сталкиваются со следующими проблемами:

    • 🚫 Ошибка 401 Unauthorized — истёк access_token или неверные client_id/client_secret.
    • 🔄 Ошибка 429 Too Many Requests — превышен лимит запросов (60 в минуту для большинства эндпоинтов).
    • 📦 Ошибка 400 Bad Request — неверный формат данных (например, отрицательный остаток).
    • 🔗 Ошибка 500 Internal Server Error — проблемы на стороне Ozon (повторите запрос позже).

    Для отладки используйте:

    1. Логирование запросов/ответов (библиотека logging в Python).
    2. Инструменты вроде Postman для ручного тестирования эндпоинтов.
    3. Песочницу (sandbox=True) — здесь ошибки не влияют на реальные данные.
    💡

    Настройте уведомления в Telegram или Slack о критических ошибках SDK. Например, при падении синхронизации остатков более 3 раз подряд.

    Переход в боевой режим: чек-лист перед запуском

    Перед отключением режима песочницы (sandbox=False) обязательно выполните следующие шаги:

    Проверены лимиты API и настроены задержки между запросами|Синхронизация остатков протестирована на 100+ товарах|Настроено резервное копирование базы данных|Подготовлен план действий при сбое (контакты поддержки Ozon)|Обучен персонал работе с новой системой-->

    Критические моменты:

    • 🔒 Безопасность: Убедитесь, что access_token и refresh_token хранятся в защищённом хранилище (не в коде!).
    • ⚡ Производительность: При большом количестве товаров (10 000+) используйте пакетную обработку (не более 100 позиций в одном запросе).
    • 📈 Мониторинг: Настройте дашборды в Grafana или Datadog для отслеживания успешности запросов.

    ⚠️ Внимание: В первые 24 часа после перехода в боевой режим Ozon может временно ограничивать количество запросов. Начните с минимальной нагрузки (например, обновляйте остатки раз в 4 часа) и постепенно увеличивайте частоту.

    💡

    Наиболее частая причина блокировки аккаунта при интеграции SDK — превышение лимитов API. Всегда implementируйте задержки между пакетными запросами (например, 1 запрос в секунду).

    FAQ: ответы на частые вопросы

    Можно ли использовать одно приложение для нескольких аккаунтов Ozon Seller?

    Нет, каждое приложение привязано к одному аккаунту продавца. Если у вас несколько магазинов на Ozon, зарегистрируйте отдельное приложение для каждого.

    Как часто нужно обновлять остатки через SDK?

    Для схемы FBS — не реже чем раз в 2 часа. Для FBO достаточно 1 раза в сутки, но рекомендуется синхронизироваться чаще (каждые 6 часов), чтобы избежать продажи отсутствующих товаров.

    Что делать, если SDK перестал работать после обновления Ozon?

    Проверьте официальную документацию на предмет изменений в API. Часто достаточно обновить версию SDK (например, pip install --upgrade ozon-seller-api). Если проблема сохраняется, обратитесь в поддержку с логами ошибок.

    Можно ли через SDK управлять ценами на товары?

    Да, для этого используется эндпоинт /v1/product/price. Однако учтите, что частое изменение цен (чаще 1 раза в час) может привести к ограничениям со стороны Ozon.

    Как интегрировать SDK с 1С?

    Самый надёжный способ — использовать промежуточный сервер (например, на Python или PHP), который будет:

    1. Получать данные из 1С через REST API или COM-соединение.
    2. Форматировать их под требования Ozon SDK.
    3. Отправлять запросы к API Ozon и обрабатывать ответы.

    Готовые решения предлагают Атотрейд, МойСклад и другие сервисы.
