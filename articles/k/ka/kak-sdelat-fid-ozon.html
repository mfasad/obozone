Как правильно сделать фид для Ozon: руководство с примерами и лайфхаками

Загрузка товаров на Ozon через фид (feed) — это основа успешного старта для любого продавца. Без корректно оформленного файла с данными ваши карточки не пройдут модерацию, а товары не появятся в каталоге. Но как избежать ошибок при создании фида, какие форматы поддерживаются в 2026 году, и как ускорить обработку данных? В этой статье разберём всё по шагам — от выбора структуры до загрузки и исправления типичных проблем.

Многие продавцы теряют дни на переделку фидов из-за мелочей: неправильный разделитель в CSV, лишние пробелы в артикулах или отсутствие обязательных атрибутов. Мы собрали актуальные требования Ozon Seller, примеры заполнения для разных категорий товаров и проверенные способы автоматической генерации фидов — от Excel до специализированных сервисов вроде MoySklad или 1С. А ещё расскажем, как проверить файл перед загрузкой, чтобы не получить отказ от модераторов.

Что такое фид для Ozon и зачем он нужен

Фид (от англ. product feed) — это структурированный файл с данными о товарах, который продавец загружает в личный кабинет Ozon Seller. Он содержит всю необходимую информацию для создания карточек: от названий и цен до характеристик, фотографий и остатков на складе. Без фида добавить товары на маркетплейс можно только вручную — по одному, что нереально для ассортимента от 50 позиций.

Почему фид критичен для продавца:

  • 📦 Массовая загрузка: добавить сотни товаров за несколько минут вместо недель ручного ввода.
  • 🔄 Автоматическое обновление: синхронизировать цены, остатки и описания без повторной загрузки.
  • 🎯 Точность данных: исключить ошибки, которые неизбежны при ручном копировании.
  • ⚡ Скорость модерации: корректный фид проходит проверку быстрее, чем карточки, созданные через интерфейс.

На Ozon фид используется для:

  • 📌 Первичной загрузки нового ассортимента.
  • 🔄 Обновления существующих товаров (цены, остатки, описания).
  • 📊 Создания промо-акций и скидок через массовое редактирование.
  • 🚀 Подключения к программам Ozon Express или Premium.
⚠️ Внимание: Если вы продаёте товары в категориях Электроника, Одежда или Красота, фид должен включать расширенные атрибуты (например, размерную сетку или сертификаты). Без них карточки не пройдут модерацию.

Форматы фидов, которые поддерживает Ozon в 2026 году

Ozon принимает фиды в трёх основных форматах. Выбор зависит от вашего ассортимента, технических возможностей и предпочтений:

Формат Расширение Плюсы Минусы Когда использовать
CSV .csv Простота создания в Excel, поддерживается всеми сервисами Чувствителен к разделителям, ошибки при кодировке Для небольших ассортиментов (до 1000 товаров)
XLSX .xlsx Удобен для ручного редактирования, поддерживает формулы Большой размер файла, медленнее обрабатывается Если вы работаете в Excel и не планируете автоматическую выгрузку
XML .xml Подходит для сложных структур, автоматическая генерация из 1С Требует знаний разметки или специальных программ Для крупных продавцов с интеграцией через API

С 2023 года Ozon также поддерживает JSON-фиды, но этот формат пока рекомендован только для продавцов, использующих Ozon API для автоматической синхронизации. Для большинства задач достаточно CSV или XLSX.

Важно: независимо от формата, файл должен соответствовать техническим требованиям маркетплейса. Например, максимальный размер фида — 50 МБ, а количество товаров в одном файле — до 50 000 позиций.

Обязательные поля в фиде Ozon: что нельзя пропустить

Каждый фид должен содержать обязательные атрибуты, без которых товар не пройдёт модерацию. Их список зависит от категории, но есть базовые поля, актуальные для всех товаров:

  • 🔖 Offer ID — уникальный идентификатор товара в вашей системе (не более 50 символов).
  • 🏷️ Name — название товара (до 255 символов, без спецсимволов).
  • 💰 Price — цена в рублях (целое число без копеек).
  • 📦 Quantity — остаток на складе (если 0, товар скрывается из каталога).
  • 📸 Image — ссылка на главное фото (разрешение не менее 800×800 px).
  • 🔍 Category ID — идентификатор категории Ozon (например, 17035434 для смартфонов).
  • 📝 Description — описание товара (от 50 символов, с ключевыми словами).

Для некоторых категорий требуются дополнительные обязательные поля:

  • 👕 Одежда/Обувь: Size, Color, Material.
  • 💊 Аптека: Certificate, Expiration Date.
  • 📱 Электроника: Brand, Model, Warranty.
  • 🍎 Продукты: Shelf Life, Weight.

Пример заполнения обязательных полей для товара "Беспроводные наушники Xiaomi Redmi Buds 4":

Offer ID,Name,Price,Quantity,Image,Category ID,Description,Brand,Model

⚠️ Внимание: Если в фиде отсутствует Category ID или он указан неверно, товар автоматически отклоняется. Уточните актуальные идентификаторы категорий в справочнике Ozon.

Как создать фид для Ozon: пошаговая инструкция

Создать фид можно несколькими способами — вручную, через Excel, с помощью сервисов или автоматически из 1С. Разберём каждый вариант с примерами.

Способ 1: Ручное заполнение в Excel (для небольшого ассортимента)

Если у вас до 100 товаров, проще всего создать фид в Excel:

  1. Скачайте шаблон фида с сайта Ozon.
  2. Заполните обязательные поля (см. раздел выше).
  3. Добавьте дополнительные атрибуты для вашей категории (например, Size для одежды).
  4. Сохраните файл в формате CSV (разделитель — запятая).
  5. Проверьте кодировку: должна быть UTF-8.

Способ 2: Автоматическая генерация из 1С или MoySklad

Для крупных продавцов ручное заполнение не подходит. Используйте интеграцию с:

  • 📊 1С:Предприятие — через модуль Обмен с Ozon (требуется настройка).
  • 📦 MoySklad — в разделе "Интеграции" выберите Ozon и настройте выгрузку.
  • 🔗 API Ozon — для полностью автоматической синхронизации (нужен программист).

Пример настройки выгрузки в MoySklad:

  1. Перейдите в Настройки → Интеграции → Ozon.
  2. Авторизуйтесь через Ozon Seller.
  3. Выберите склад, с которого будет идти выгрузка остатков.
  4. Настройте расписание автоматического обновления (например, раз в час).

Способ 3: Сервисы для генерации фидов

Если нет 1С или времени на Excel, используйте специализированные сервисы:

  • 🛒 Pricelabs — для динамического ценообразования и выгрузки на несколько маркетплейсов.
  • 📈 DataFeedWatch — оптимизация фидов под требования Ozon.
  • 🔄 Lengow — автоматическая синхронизация с Ozon, Wildberries и др.

Стоимость таких сервисов — от 1 000 ₽/мес, но они экономят время и снижают риск ошибок.

💡

Если вы загружаете фид впервые, используйте режим "Тестовая загрузка" в личном кабинете Ozon. Это позволит проверить файл на ошибки без создания реальных карточек.

Типичные ошибки в фидах и как их исправить

Даже опытные продавцы сталкиваются с отказами при загрузке фидов. Вот самые распространённые ошибки и способы их устранения:

Ошибка Причина Как исправить
Invalid Offer ID Дублирующийся или слишком длинный идентификатор Используйте уникальные ID до 50 символов (например, brand_model_color)
Image not found Ссылка на фото недоступна или ведёт на 404 Проверьте URL через URLVoid, загрузите фото на надёжный хостинг
Price is not valid Цена содержит копейки или отрицательное значение Укажите цену в рублях целым числом (например, 1999, а не 1999.99)
Category not found Неверный Category ID или категория заблокирована Уточните актуальный ID в справочнике Ozon
Description too short Описание меньше 50 символов Дополните текст ключевыми словами и характеристиками товара

Чтобы избежать ошибок, всегда проверяйте фид перед загрузкой:

  1. Откройте файл в текстовом редакторе (например, Notepad++) и убедитесь, что разделители (, или ;) стоят правильно.
  2. Проверьте кодировку: должна быть UTF-8 без BOM.
  3. Используйте валидатор Ozon для автоматической проверки.
⚠️ Внимание: Если в фиде есть товары с Quantity = 0, они не будут отображаться в каталоге, но останутся в вашей базе. Это нормально — так Ozon экономит место на витрине.
Что делать, если фид не загружается вообще?

Если при загрузке фида вы видите ошибку "Файл повреждён" или "Неверный формат", проверьте:

1. Расширение файла — оно должно точно соответствовать формату (например, .csv, а не .csv.txt).

2. Разделители — в Excel при сохранении в CSV выберите "Разделитель — запятая".

3. Кодировку — конвертируйте файл в UTF-8 через Notepad++ или онлайн-конвертеры.

Если проблема остаётся, попробуйте загрузить файл через другой браузер (например, Chrome вместо Safari).

Как загрузить фид в личный кабинет Ozon

Когда фид готов, остаётся загрузить его в Ozon Seller. Вот пошаговая инструкция:

  1. Авторизуйтесь в личном кабинете.
  2. Перейдите в раздел Товары → Импорт товаров.
  3. Нажмите Загрузить фид и выберите файл с компьютера.
  4. Укажите тип загрузки:
    • 🆕 Новые товары — если загружаете ассортимент впервые.
    • 🔄 Обновление существующих — для изменения цен, остатков или описаний.
  • Нажмите Загрузить и дождитесь обработки (может занять до 24 часов).

    • После загрузки проверьте статус обработки в разделе История загрузок. Если есть ошибки, Ozon пришлёт отчёт с указанием проблемных строк. Исправьте их и загрузите фид повторно.

    Совет: если вы обновляете цены или остатки чаще чем раз в день, настройте автоматическую выгрузку через API или сервисы вроде MoySklad. Это сэкономит время и снизит риск ошибок.

    💡

    Всегда сохраняйте копию загруженного фида на случай, если потребуется откат изменений. Ozon не хранит историю ваших файлов дольше 30 дней.

      • Преимущества (чем ваш товар лучше аналогов).
      • Гарантию и условия возврата.

    Частые вопросы о фидах Ozon

    Сколько времени занимает обработка фида?

    Стандартный срок — до 24 часов. В пиковые периоды (например, перед "Чёрной пятницей") обработка может занять до 48 часов. Если фид не обработался дольше, проверьте его на ошибки или обратитесь в поддержку.

    Можно ли обновлять цены и остатки без полной выгрузки фида?

    Да, для этого используйте частичное обновление. В фиде укажите только поля Offer ID, Price и Quantity, а при загрузке выберите тип "Обновление существующих товаров".

    Что делать, если товар не прошёл модерацию?

    Проверьте письмо от Ozon с причиной отказа. Чаще всего проблемы связаны с:

    • Несоответствием фото требованиям (например, водяные знаки).
    • Отсутствием обязательных атрибутов (например, сертификата для электроники).
    • Неуникальным описанием (копипаст с других сайтов).

    Исправьте ошибки и загрузите фид повторно.

    Как добавить варианты товара (например, разные цвета)?summary>

    Для товаров с вариациями (цвет, размер) используйте поле Group ID. Все варианты одного товара должны иметь одинаковый Group ID, но разные Offer ID. Пример: 

    Offer ID,Group ID,Name,Color

    tshirt_black,tshirt_123,Футболка хлопковая чёрная,Чёрный
    

    tshirt_white,tshirt_123,Футболка хлопковая белая,Белый

    Можно ли загружать фид через API?

    Да, Ozon API позволяет автоматизировать загрузку и обновление фидов. Для этого нужен доступ к API (запросите его в личном кабинете) и техническая реализация (можно через 1С или специализированные сервисы). Подробная документация — здесь.
