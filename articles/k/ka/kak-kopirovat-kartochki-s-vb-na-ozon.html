Как скопировать карточки товаров с Wildberries на Ozon: 5 проверенных способов

Введение: зачем переносить карточки с WB на Озон?

Перенос товарных карточек с Wildberries на Ozon — задача, с которой сталкивается каждый второй мультимаркетплейсовый продавец. По данным Data Insight за 2026 год, 68% продавцов, работающих одновременно на обеих площадках, тратят до 40% времени на дублирование контента. Причины очевидны: расширение аудитории, диверсификация рисков и увеличение продаж за счёт кросс-платформенного присутствия.

Однако просто скопировать описание, фото и характеристики «как есть» не получится. Платформы имеют разные требования к оформлению карточек, ограничения по контенту и даже алгоритмы модерации. Например, Ozon строже относится к ключевым словам в заголовках, а Wildberries разрешает больше символов в описании. В этой статье разберём 5 способов переноса карточек — от ручного ввода до автоматизации через API, — а также расскажем, как избежатьных ошибок и блокировок.

Важно: ни один из методов не гарантирует 100% успех. Даже при использовании специализированных сервисов до 15% карточек может быть отклонено модерацией Ozon из-за несоответствия правилам площадки. Поэтому перед массовым переносом рекомендуем протестировать 2-3 товара.

Способ 1: Ручной перенос через личный кабинет Ozon

Самый трудоёмкий, но и самый надёжный метод — ввод данных вручную через интерфейс Ozon Seller. Подходит для небольшого ассортимента (до 50 товаров) или уникальных позиций, где важна точность.

Алгоритм действий:

  • 📌 Откройте карточку товара на Wildberries и скопируйте:
    • Название (укорачивайте до 120 символов — лимит Ozon)
    • Описание (уберите упоминания WB, замените на Ozon)
    • Характеристики (приведите к формату Ozon: например, размеры указывайте в см, а не в дюймах)
    • Фото (скачайте оригиналы в максимальном разрешении — Ozon требует не менее 800×800 px)
  • 🔄 В личном кабинете Ozon Seller перейдите в Товары → Добавить товар
  • 📝 Заполните поля, адаптируя контент под требования Ozon (см. таблицу ниже)
  • 🔍 Проверьте карточку на ошибки с помощью встроенного валидатора перед отправкой на модерацию

Преимущества метода: полный контроль над контентом, минимальный риск отклонения модерацией. Недостатки: временные затраты (в среднем 15-20 минут на одну карточку) и человеческий фактор.

Способ 2: Экспорт/импорт через Excel (для FBS/FBO)

Если у вас более 50 товаров, ручной ввод станет неэффективным. В этом случае поможет экспорт данных из Wildberries в Excel с последующим импортом в Ozon. Метод подходит для продавцов на моделях FBS и FBO.

Пошаговая инструкция:

  1. В личном кабинете Wildberries перейдите в Товары → Экспорт и скачайте файл в формате .xlsx.
  2. Откройте файл в Excel или Google Sheets и удалите колонки, не нужные для Ozon:
    • 🚫 Колонки с внутренними ID WB (например, nm_id)
    • 🚫 Данные о остатках на складах WB
    • 🚫 Цены (их придётся пересчитать с учётом комиссий Ozon)
  • Добавьте обязательные для Ozon колонки:
    • 📦 Вес брутто (г) — если не указан, товар не пройдёт модерацию
    • 📏 Габариты (см) — длина × ширина × высота
    • 🔖 Ключевые слова — до 5 слов через запятую
    • Сохраните файл в формате .csv (кодировка UTF-8!) и импортируйте через Товары → Импорт в Ozon Seller.

      • Обратите внимание: Ozon требует строгого соответствия формату CSV. Например, если в ячейке с ценой будет указано 1 000 руб. вместо 1000, импорт завершится ошибкой. Используйте функцию Найти и заменить в Excel, чтобы убрать лишние символы.

      💡

      Перед импортом проверьте файл через сервис CSVLint — он покажет синтаксические ошибки, которые могут прервать загрузку.

      Способ 3: Парсинг карточек с помощью сервисов

      Для автоматизации переноса можно использовать специализированные парсеры, такие как ParserOK, DataCol или Zennoposter. Эти инструменты позволяют скачивать данные с Wildberries в структурированном виде и адаптировать их под Ozon.

      Как работает парсинг:

      1. Настройте задачу в парсере:
        • 🔗 Укажите URL карточек Wildberries (можно загрузить список)
        • 📌 Выберите поля для экспорта (название, описание, фото, характеристики)
        • 🤖 Задайте правила трансформации (например, замена «WB» на «Ozon» в текстах)
    • Запустите парсинг. Сервис соберёт данные и сгенерирует файл для импорта в Ozon.
    • Проверьте результат: некоторые парсеры могут неправильно распознавать таблицы характеристик или пропускать фото.

    Стоимость парсинга варьируется от 0.5 руб./карточка (при оптовых заказах) до 5 руб./карточка (разовые задачи). Главный риск — блокировка аккаунта Wildberries за чрезмерную нагрузку. Чтобы этого избежать:

    • 🛡️ Используйте прокси-сервера (рекомендуем Luminati или Oxylabs)
    • ⏱️ Настраивайте задержки между запросами (≥3 секунды)
    • 🔄 Парсьте не более 200 карточек в час с одного IP

Для легального переноса лучше запросить данные у поставщика или использовать официальные инструменты синхронизации.

    Способ 4: Синхронизация через API (для опытных продавцов)

    Наиболее продвинутый метод — интеграция через API обеих площадок. Подходит для крупных продавцов с собственными IT-ресурсами или тех, кто использует 1C, МойСклад и другие системы учёта.

    Как настроить синхронизацию:

    1. Получите доступ к API:
      • 🔑 Зарегистрируйте приложение в личном кабинете WB и получите API-key
      • 🔑 Аналогично получите ключи в Ozon Seller
  • Напишите скрипт (или используйте готовые решения вроде Alba API, RetailCRM), который будет:
    • 📥 Выгружать данные из Wildberries по API
    • 🔄 Трансформировать их под формат Ozon
    • 📤 Загружать на Ozon через их API
    • Настройте автоматическое обновление (например, раз в сутки) для синхронизации остатков и цен.

    Пример запроса к API Wildberries для получения данных о товаре:

    GET https://suppliers-api.wildberries.ru/content/v1/cards/list

    Headers:

    Преимущества API-интеграции:

    • ⚡ Автоматизация: нет нужды вручную обновлять карточки
    • 🔄 Синхронизация: цены и остатки обновляются в реальном времени
    • 📊 Аналитика: можно собирать данные о продажах с обеих площадок в одной системе
    ⚠️ Внимание: При работе с API Ozon учитывайте лимиты запросов:
    • 🔢 100 запросов в минуту для метода /v1/product/list
    • 🔢 50 запросов в минуту для метода /v1/product/update

    Превышение лимитов приводит к блокировке API-ключа на 24 часа.

    Способ 5: Использование сервисов-коннекторов (Alba, МойСклад, RetailCRM)

    Если вы не готовы разрабатывать собственные скрипты для API, можно воспользоваться готовыми сервисами-коннекторами. Они предоставляют интерфейс для настройки синхронизации между площадками без программирования.

    Популярные сервисы:

    • 🔗 Alba — поддерживает WB, Ozon, Яндекс Маркет и др. Стоимость от 1 500 руб./мес.
    • 📦 МойСклад — удобен для тех, кто ведёт учёт в этой системе. Интеграция с Ozon бесплатна, но за парсинг WB нужно платить отдельно.
    • 🛒 RetailCRM — подходит для крупных продавцов с большим ассортиментом. Тарифы от 3 000 руб./мес.

    Как настроить синхронизацию в Alba:

    1. Подключите аккаунты Wildberries и Ozon в личном кабинете Alba.
    2. Создайте правило синхронизации:
      • 🔄 Укажите направление: WB → Ozon
      • 📌 Выберите поля для переноса (название, описание, фото и т.д.)
      • 🔧 Настройте трансформацию данных (например, замена текста, изменение форматов)
  • Запустите тестовую синхронизацию для 1-2 товаров.
  • Проверьте результат в личном кабинете Ozon и при необходимости скорректируйте правила.

    • Сервисы-коннекторы экономят время, но имеют ограничения:

    • 💰 Дополнительные расходы на подписку
    • ⚙️ Не все поля могут синхронизироваться корректно (например, сложные таблицы характеристик)
    • ⏳ Задержки обновления (до 1-2 часов для некоторых сервисов)
    💡

    Сервисы-коннекторы оптимальны для продавцов с ассортиментом от 100 товаров. Для меньшего количества дешевле обойдётся ручной перенос или парсинг.

    Типичные ошибки и как их избежать

    Даже при использовании автоматизированных инструментов до 30% карточек может быть отклонено модерацией Ozon. Рассмотрим TOP-5 ошибок и способы их предотвращения:

    1. Неуникальное описание

      Ozon требует уникальность текстов ≥90%. Если вы просто скопируете описание с Wildberries, карточка будет отклонена. Решение:

      • 📝 Перепишите текст своими словами или используйте сервисы вроде Text.ru для проверки уникальности.
      • 🤖 Для массовой обработки подойдёт Synonymizer или SpinRewriter (но проверяйте результат вручную!).
    2. Несоответствие категорий

      Категории товаров на WB и Ozon не всегда совпадают. Например, «Электроника» на WB может разбиваться на «Аудиотехника», «Видеотехника» и «Гаджеты» в Ozon. Решение:

      • 🔍 Используйте дерево категорий Ozon для правильного сопоставления.
      • 📊 В сервисах-коннекторах настройте маппинг категорий (например, «Обувь» на WB → «Обувь → Кроссовки» на Ozon).
  • Неправильные габариты или вес

    Ozon строго контролирует эти параметры, особенно для модели FBO. Если вес указан неверно, товар может быть оштрафован за несоответствие при приёмке на складе. Решение:

    • ⚖️ Взвесьте товар самостоятельно или запросите данные у поставщика.
    • 📏 Для габаритов используйте формулу: (длина + 2 см) × (ширина + 2 см) × (высота + 2 см) (учитывая упаковку).

    Другие распространённые ошибки:

    Фото с водяными знаками WB Отклонение карточки Удалите водяные знаки в Photoshop или загрузите оригиналы от поставщика

    FAQ: Ответы на частые вопросы

    Можно ли перенести отзывы с Wildberries на Ozon?

    Нет, Ozon запрещает перенос отзывов с других площадок. Это считается манипуляцией рейтинга и может привести к блокировке аккаунта. Однако вы можете:

    • 📌 Просить покупателей Ozon оставлять отзывы после покупки (например, через email-рассылку).
    • 💬 Использовать положительные отзывы с WB в описании товара (с пометкой «Отзыв с другой площадки»).
    Как перенести карточки, если у меня FBS на WB и FBO на Ozon?

    В этом случае нужно:

    1. Экспортировать данные из WB (как в Способе 2).
    2. Добавить в файл колонки, обязательные для FBO:
      • Вес брутто (г)
      • Габариты упаковки (см)
      • Срок годности (если применимо)
  • Указать в личном кабинете Ozon, что товар будет отгружаться со своего склада (FBO).

    • Обратите внимание: для FBO Ozon требует предоставлять сертификаты на товары (если они подлежат обязательной сертификации). Их также нужно будет загрузить в личный кабинет.

    Что делать, если Ozon отклонил карточку после переноса?

    Причины отклонения указаны в личном кабинете (Товары → На модерации → Отклонённые). Чаще всего это:

    • 📝 Неуникальное описание — перепишите текст.
    • 🖼️ Неподходящие фото — замените на изображения без водяных знаков, с белым фоном.
    • 🏷️ Неверная категория — выберите другую из списка Ozon.

    После исправлений отправьте карточку на модерацию повторно. Если проблема повторяется, обратитесь в поддержку Ozon с указанием ID товара и скриншотом ошибки.

    Можно ли автоматически обновлять цены на Ozon при изменении цен на WB?

    Да, это возможно с помощью:

    • 🔄 API-интеграции (см. Способ 4)
    • 🤖 Сервисов-коннекторов (Alba, RetailCRM)
    • 📊 1C/МойСклад — если ведёте учёт в этих системах

    При настройке учитывайте:

    • 💰 Комиссия Ozon выше, чем на WB (до 15% против 5-10%).
    • ⚖️ Цена на Ozon должна быть не ниже минимальной рекомендованной (иначе товар не будет участвовать в акциях).
    Как перенести карточки с WB на Ozon бесплатно?

    Бесплатные способы:

    • 🖥️ Ручной ввод (подходит для ≤50 товаров).
    • 📑 Excel + импорт — если умеете работать с формулами и макросами.
    • 🤖 Парсинг через бесплатные расширения (например, Web Scraper для Chrome), но это рискованно из-за возможной блокировки.

    Платные сервисы начинаются от 500 руб./мес. (например, тариф Alba Lite). Для большого ассортимента это оправданные траты — экономия времени перекрывает стоимость подписки.
