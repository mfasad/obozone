Как продавцу на Озон проверить остатки товаров: все способы + решения ошибок

Продавцы на Ozon часто сталкиваются с ситуацией, когда реальные остатки товаров на складе не совпадают с данными в личном кабинете. Это приводит к проблемам: отменённым заказам, штрафам за недопоставку или, наоборот, к избыточному хранению. В 2026 году платформа обновила интерфейс и логику работы с остатками, поэтому старые инструкции могут не сработать.

В этой статье — 5 проверенных способов проверить остатки на Ozon Seller, включая скрытые функции личного кабинета, API-методы и решения для FBS/FBO. А также разберём, почему данные могут отличаться от фактических и как это исправить без обращения в поддержку.

1. Где посмотреть остатки в личном кабинете Ozon Seller

Основной способ — через раздел Товары → Остатки. Здесь отображаются:

  • 📦 Доступно для продажи — количество товаров, которое видит покупатель.
  • 🚛 В пути — заказы, которые ещё не доехали до ПВЗ или склада Ozon.
  • ⚠️ Резерв — товары, забронированные под заказы, но ещё не оплаченные.
  • 📉 Невозвращаемый брак — списанные единицы (требует подтверждения).

Чтобы увидеть детализированную информацию по конкретному SKU:

  1. Перейдите в Товары → Каталог.
  2. Найдите нужный товар через поиск или фильтры.
  3. Кликните по SKU — откроется карточка с вкладкой Остатки.
💡

Если остатки не обновляются более 24 часов, попробуйте вручную запустить синхронизацию через кнопку Обновить данные в правом верхнем углу раздела.

Важно: в личном кабинете показываются только подтверждённые остатки. Если вы отправили товар на склад Ozon по FBS, но он ещё не принят — эти единицы не будут учтены. Для FBS-схемы актуальные данные появляются только после приёмки (обычно в течение 1–3 дней).

2. FBS vs FBO: где искать остатки в каждой схеме

Логика отображения остатков зависит от модели работы с Ozon:

Схема Где хранится товар Как обновляются остатки Где смотреть в ЛК
FBS На складах Ozon Автоматически после приёмки Товары → Остатки (вкладка "На складе Ozon")
FBO На вашем складе Вручную или через API Товары → Остатки (вкладка "На вашем складе")
RBS На складе поставщика Через интеграцию с 1С/ERP Требует отдельной настройки в Настройки → Интеграции

Для FBO ключевой нюанс: остатки в личном кабинете — это ваша декларация, а не фактическое количество. Если вы указали 100 единиц, но реально их 50, Ozon будет принимать заказы на 100, пока не получит жалобу от покупателя о недопоставке. Чтобы избежать штрафов, синхронизируйте данные хотя бы раз в сутки.

Для FBS критично следить за статусом В пути. Если товар больше 5 дней висит в этом статусе, проверьте:

  • 📄 Корректность заполненной товарной накладной (особенно поля Количество и SKU).
  • 🚚 Данные транспортной компании — иногда Ozon не получает трек-номер.
  • 📧 Уведомления от Ozon — возможно, требуется подтвердить приёмку.

3. Как обновить остатки вручную (для FBO и FBS)

Если автоматическая синхронизация дала сбой, остатки можно обновить самостоятельно:

Зайти в Товары → Остатки

Выбрать нужные SKU галочками

Нажать Изменить остатки (кнопка вверху)

Ввести актуальные цифры и сохранить

Подтвердить изменения в всплывающем окне-->

Для FBS ручное обновление доступно только в двух случаях:

  1. Если товар ещё не отправлен на склад Ozon (статус Ожидает отправки).
  2. Если Ozon вернул товар на ваш склад (статус Возврат).

Во всех других случаях остатки FBS обновляет только система после физической приёмки. Если вы видите неактуальные данные, напишите в поддержку с темой "Несовпадение остатков FBS" и приложите:

  • 📋 Скриншот раздела Остатки с датой.
  • 📄 Товарную накладную (если товар в пути).
  • 📧 Письмо от транспортной компании с трек-номером.
Что делать, если кнопка "Изменить остатки" неактивна?

Это означает, что товар находится в статусе На модерации или Заблокирован. Проверьте раздел Товары → Модерация и устраните причину блока (чаще всего это некорректные фотографии или описание). После разблокировки кнопка станет активной.

4. Проверка остатков через API (для продвинутых пользователей)

Если вы работаете с большим каталогом (1000+ SKU), ручное обновление неэффективно. В этом случае используйте API Ozon Seller. Основные методы:

  • 🔄 GET /v2/analytics/stock_on_warehouses — остатки по складам FBS.
  • 📊 GET /v1/product/info/stocks — остатки по SKU (включая резерв).
  • ⚡ POST /v1/product/info/stocks — обновление остатков (для FBO).

Пример запроса для получения остатков FBS (на языке Python):

import requests


url = "https://api-seller.ozon.ru/v2/analytics/stock_on_warehouses"


headers = {


"Client-Id": "Ваш_Client_ID",


"Api-Key": "Ваш_API_ключ",


"Content-Type": "application/json"


}


params = {


"warehouse_id": 101000060001,  # ID склада (например, Москва)


"limit": 1000


}



response = requests.get(url, headers=headers, params=params)


print(response.json())

Для FBO обновление остатков через API выглядит так:

url = "https://api-seller.ozon.ru/v1/product/info/stocks"

data = {


"stocks": [


{


"sku": 123456789,


"stock": 50  # новое количество


}


]


}


response = requests.post(url, headers=headers, json=data)

💡

API позволяет обновлять остатки пачками до 1000 SKU за один запрос. Это в 10 раз быстрее, чем ручное редактирование в личном кабинете.

⚠️ Внимание: При работе с API учитывайте лимиты:

🔴 Внимание: Ozon блокирует аккаунты за превышение лимитов API (более 60 запросов в минуту). Используйте time.sleep(1) между пачками обновлений, чтобы избежать бана.

5. Почему остатки на Ozon не совпадают с реальными

Разница между данными в личном кабинете и фактическими остатками возникает по 7 основным причинам:

  1. Задержка приёмки FBS — товар доехал до склада, но Ozon ещё не обработал его (до 72 часов).
  2. Резервирование под заказы — в личном кабинете показано "10 шт.", но 3 из них забронированы под неоплаченные корзины.
  3. Возвраты от покупателей — товар вернулся на склад, но ещё не прошёл проверку качества.
  4. Брак и списания — Ozon списал единицы по акту, но не уведомил продавца.
  5. Ошибки интеграции — если вы используете 1С или МойСклад, возможны сбои синхронизации.
  6. Ручное редактирование — кто-то из вашей команды изменил остатки напрямую в ЛК, не обновляя складскую систему.
  7. Технические работы — Ozon иногда приостанавливает обновление данных (обычно ночью).

Чтобы найти причину, проверьте:

  1. Раздел Отчёты → Логи изменений остатков (доступен по запросу в поддержку).
  2. Вкладку Возвраты в карточке товара — там могут быть неучтённые единицы.
  3. Электронную почту, связанную с аккаунтом — Ozon отправляет уведомления о списаниях.
💡

Если разница в остатках превышает 10% от общего количества, Ozon может заблокировать товар по подозрению в "недопоставке". Чтобы этого избежать, еженедельно сверяйте данные через выгрузку отчёта stocks.csv в разделе Отчёты.

6. Как исправить несовпадение остатков: пошаговый план

Если вы обнаружили расхождения, действуйте по алгоритму:

  1. Шаг 1. Экспортируйте данные

    Скачайте отчёт Остатки товаров в формате CSV через Отчёты → Товары → Остатки.

  2. Шаг 2. Сверьте с складской системой

    сравните файлы из Ozon и вашей ERP (1С, МойСклад, Bitrix).

  3. Шаг 3. Выявите расхождения

    Обратите внимание на SKU с разницей более 5 единиц — их нужно проверять в первую очередь.

  4. Шаг 4. Обновите данные

    Для FBO: загрузите корректировку через Товары → Остатки → Загрузить файл.

    Для FBS: свяжитесь с поддержкой и приложите акты приёмки/списания.

  5. Шаг 5. Настройте автоматическую синхронизацию

    Подключите интеграцию с вашей складской системой (инструкции в разделе Настройки → Интеграции).

Если расхождения критичные (более 20% от общего количества), Ozon может приостановить продажи товара до выяснения причин. В этом случае:

  • Создайте тикет в поддержку с темой "Расхождение остатков — срочно".
  • Приложите:
    • 📊 Сверочный отчёт из вашей ERP.
    • 📄 Акт инвентаризации (если проводили).
    • 📧 Переписку с транспортной компанией (для FBS).

7. Частые ошибки и как их избежать

Опытные продавцы на Ozon сталкиваются с типичными проблемами при работе с остатками:

Ошибка Причина Решение
Остатки "зависли" на одном значении Сбой синхронизации с API Перезагрузите токен API в Настройки → Безопасность
Товар есть на складе, но в ЛК "0 шт." Ozon не принял поставку Проверьте статус в Поставки → История
После обновления остатки сбросились Конфликт при пакетном импорте Обновляйте остатки по одному SKU за раз
Отрицательные остатки Возврат от покупателя не учтён Проверьте раздел Возвраты → Ожидают обработки

⚠️ Внимание: Если вы используете мультискладскую схему (несколько складов FBS), остатки могут дублироваться:

🔴 Внимание: При работе с несколькими складами Ozon суммирует остатки по всем точкам хранения. Если вы продаёте товар с двух складов (например, Москва и Санкт-Петербург), в личном кабинете будет показана общая сумма, а не распределение по регионам. Чтобы увидеть детализацию, используйте отчёт /v2/analytics/stock_on_warehouses через API.

Ещё одна распространённая ошибка — неучтённые резервы. Например, покупатель добавил товар в корзину, но ещё не оплатил. В этом случае:

  • В личном кабинете остаток уменьшится на количество в резерве.
  • Если покупатель не оплатит заказ в течение 24 часов, резерв автоматически снимется.
  • Чтобы увидеть резервы, экспортируйте отчёт Заказы → Резервы.

FAQ: Ответы на частые вопросы

Могу ли я увидеть остатки по конкретному ПВЗ?

Нет, Ozon не предоставляет детализацию остатков по пунктам выдачи. Вы можете увидеть только общие остатки по региону (например, "Москва и область"). Для точного контроля используйте схему FBO или настройте мультискладскую логистику.

Как часто обновляются остатки в личном кабинете?

Для FBO — мгновенно после ручного обновления или через API. Для FBS — в течение 1–3 часов после приёмки товара на складе. В пиковые периоды (например, во время распродаж) обновление может задерживаться до 6 часов.

Что делать, если остатки обнулились без причины?

Скорее всего, сработал автоматический алгоритм Ozon из-за:

  • 📉 Длительного отсутствия продаж (товар не продавался более 30 дней).
  • ⚠️ Жалоб покупателей на недопоставку.
  • 🔄 Технического сбоя (проверьте статус сервисов на status.ozon.ru).

Чтобы восстановить остатки, загрузите актуальный файл через Товары → Остатки → Загрузить файл.

Можно ли скрыть товар, если остаток 0?

Да, это делается автоматически, если:

  1. Вы включили настройку Скрывать товары с нулевым остатком в Настройки → Витрина.
  2. Товар не участвует в акциях (акционные товары отображаются даже при нулевом остатке).

Чтобы скрыть товар вручную, переведите его в статус Не продаётся в карточке товара.

Как проверить остатки по штрихкоду?

Ozon не поддерживает поиск остатков по штрихкоду напрямую. Вам нужно:

  1. Найти SKU товара через раздел Товары → Каталог (используйте фильтр по штрихкоду).
  2. Перейти в карточку товара и открыть вкладку Остатки.

Если штрихкод не привязан к SKU, сделайте это через Товары → Импорт/Экспорт → Обновить штрихкоды.
