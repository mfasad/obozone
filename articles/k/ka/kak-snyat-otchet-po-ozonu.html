Как правильно снять отчёт с Озон: все способы и нюансы

Работа на маркетплейсе Ozon требует постоянного контроля за продажами, остатками и финансами. Без регулярного анализа отчётности невозможно принимать взвешенные решения: от корректировки цен до оптимизации логистики. Однако многие продавцы сталкиваются с трудностями при попытке выгрузить данные — система предлагает десятки типов отчётов, скрывает нужные настройки в неочевидных разделах, а иногда и вовсе выдаёт ошибки при экспорте.

Эта статья поможет разобраться, как снять отчёт с Озон максимально быстро и без технических сбоев. Мы рассмотрим все актуальные способы выгрузки (вручную и через API), разберёмся с форматами файлов, научимся настраивать фильтры для точных данных и решим типичные проблемы — от «пустых» отчётов до ошибок авторизации. Особое внимание уделим отличиям между FBS и FBO, так как алгоритмы формирования отчётности для этих схем работы принципиально разные.

Важно: инструкция актуальна для личного кабинета продавца версии 2026 года (интерфейс обновлялся в марте). Если вы работаете через старую версию или мобильное приложение, некоторые пункты меню могут отличаться — об этом мы предупредим отдельно.

Какие отчёты можно снять с Озон: полный список

Маркетплейс предоставляет более 40 видов отчётности, но для повседневной работы продавцу достаточно 5–7 основных. Все отчёты делятся на три категории:

  • 📊 Финансовые: выручка, комиссии, возврат средств, выплаты. Например, Отчёт по выплатам или Движение денежных средств.
  • 📦 Логистические: остатки на складах, движение товаров, возвраты. Ключевой — Отчёт по остаткам FBS.
  • 📈 Аналитические: продажи по товарам, трафик, конверсии. Популярный — Отчёт по продажам (детализированный).

🔹 Важный нюанс: отчёты для схем FBS и FBO формируются по-разному. Например, в FBO вы не увидите данных по остаткам на складах Озон — вместо этого будут поля для самостоятльного указания запасов. А в FBS отсутствует отчёт по заказам на доставку (так как логистику контролирует маркетплейс).

Пошаговая инструкция: как снять отчёт вручную

Рассмотрим универсальный алгоритм для выгрузки любого отчёта через личный кабинет. Инструкция подходит для всех типов отчётности, кроме API (об этом — в следующем разделе).

  1. Авторизуйтесь в личном кабинете продавца и перейдите в раздел Отчёты (в верхнем меню).

  2. В левом боковом меню выберите категорию отчёта (например, Финансы или Логистика).

  3. Найдите нужный отчёт в списке и кликните на его название. Откроется страница с настройками.

  4. Задайте параметры:

    • 📅 Период: выберите даты (максимальный диапазон — 3 месяца).
    • 🔍 Фильтры: укажите SKU, склады, статусы заказов и т.д.
    • 📄 Формат: XLSX (рекомендуется) или CSV.

  • Нажмите Сформировать отчёт. Система покажет статус обработки.

  • После завершения формирования кликните Скачать — файл сохранится на ваш компьютер.

    • Указать корректный период (не более 3 месяцев)

    ⚠️ Внимание: если отчёт не формируется дольше 10 минут, проверьте:

    1. Не превышен ли лимит запросов (в бесплатном тарифе — до 5 отчётов в час).

    2. Нет ли технических работ на стороне Озон (информация отображается в верхней панели ЛК).

    3. Корректно ли указан период — система не формирует отчёты за будущие даты.

    💡 Совет для ускорения: если вам нужны данные за большой период (например, год), разбейте запрос на несколько частей по 2–3 месяца. Это сократит время формирования и уменьшит риск сбоя.

    Как настроить фильтры для точных данных

    Без правильных фильтров отчёт будет содержать лишнюю информацию или, наоборот, пропустит важные данные. Рассмотрим ключевые параметры настройки на примере Отчёта по продажам (детализированного):

    • 🔖 SKU/Артикул: укажите конкретные товары (через запятую) или оставьте поле пустым для всех позиций.
    • 🏠 Склады: выберите Все или конкретные склады (например, Озон Рязань). Для FBO этот фильтр отсутствует.
    • 📦 Статусы заказов: отметьте только Доставлен и Возврат, чтобы исключить отменённые заказы.
    • 💰 Тип оплаты: разделите данные по Предоплата и Постоплата (актуально для FBS).

    📌 Пример настройки для анализа продаж конкретного товара:

    1. Период: 01.06.2026 — 30.06.2026.
    2. SKU: TShirt-Black-M, TShirt-Black-L.
    3. Склады: Озон Москва, Озон Санкт-Петербург.
    4. Статусы: только Доставлен.
    Как сохранить шаблон фильтров?

    В Озон нет функции сохранения шаблонов фильтров, но можно обойти это ограничение:

    1. После настройки фильтров скопируйте URL страницы — он содержит все параметры.

    2. Сохраните ссылку в закладках браузера с понятным названием (например, "Продажи чёрных футболок").

    3. При следующем визите откройте закладку — фильтры подгрузятся автоматически.

    ⚠️ Внимание: фильтр Тип логистики (FBS/FBO) появляется только в отчётах, где это актуально (например, в Остатки по складам его нет, а в Продажи по товарам — есть). Не путайте его с фильтром Склады!

    Скачивание отчётов через API: для продвинутых пользователей

    Если вам нужны данные в реальном времени или автоматизация выгрузки, используйте API Озон. Этот метод требует технических навыков, но позволяет:

    • 🤖 Получать отчёты по расписанию (например, каждый день в 9:00).
    • 📊 Интегрировать данные с 1С, Excel или Google Sheets.
    • ⚡ Скачивать большие объёмы данных без ограничений личного кабинета.

    🔧 Как подключиться к API:

    1. Получите Client-ID и API-Key в разделе Настройки → Интеграции → API.
    2. Используйте документацию Озон: https://docs.ozon.ru/api/seller/.
    3. Для выгрузки отчётов используйте метод POST /v2/analytics/data.

    📝 Пример запроса для получения данных по продажам:

    curl -X POST 'https://api-seller.ozon.ru/v2/analytics/data'

    -H 'Client-Id: YOUR_CLIENT_ID'
    

    -H 'Api-Key: YOUR_API_KEY'
    

    -H 'Content-Type: application/json'
    

    -d '{
    

    "date_from": "2026-06-01",
    

    "date_to": "2026-06-30",
    

    "metrics": ["revenue", "orders"],
    

    "dimensions": ["sku", "day"],
    

    "filters": {
    

    "offer_id": ["123456", "789012"]
    

    },
    

    "limit": 10000
    

    }'

    ⚠️ Внимание: при работе с API учитывайте лимиты:

    1. До 100 запросов в минуту для одного Client-ID.

    2. Максимальный период запроса — 90 дней.

    3. Данные обновляются с задержкой до 24 часов (для некоторых метрик).

    💡

    Если вы не программист, но хотите автоматизировать выгрузку, используйте сервисы-посредники вроде Zapier или Make (ex-Integromat). Они позволяют настраивать интеграции с Озон API через визуальный интерфейс.

    Типичные ошибки и как их исправить

    Даже опытные продавцы сталкиваются с проблемами при скачивании отчётов. Рассмотрим самые частые ошибки и их решения:

    Ошибка Причина Решение
    Отчёт не формируется (крутится бесконечно) Слишком большой период или перегрузка серверов Разбейте запрос на меньшие интервалы (например, по неделям)
    Файл повреждён (не открывается в Excel) Сбой при скачивании или несовместимость форматов Попробуйте формат CSV или повторите скачивание
    Доступ запрещён (ошибка 403) Недостаточно прав или истёк сеанс Перезайдите в ЛК или проверьте роль пользователя
    Нет данных (пустой отчёт) Некорректные фильтры или отсутствие продаж Проверьте фильтры (особенно SKU и период)

    🔍 Разбор сложного случая: если отчёт формируется, но содержит неполные данные (например, отсутствуют продажи за последние 2 дня), причины могут быть следующими:

    • 🕒 Задержка обновления: некоторые метрики (например, возврат средств) обновляются с lagом до 48 часов.
    • 📌 Ошибка сегментации: если вы указали конкретный склад, но товар продавался с другого, данные не попадут в отчёт.
    • 🔄 Сбой синхронизации: редко, но случаются технические ошибки на стороне Озон. В этом случае поможет обращение в поддержку с указанием ID отчёта.

    1. Актуальность вашего тарифного плана (некоторые отчёты доступны только на PRO).

    2. Наличие долгов перед Озон — при блокировке аккаунта часть функций ограничивается.

    3. Корректность часового пояса в настройках ЛК (должен совпадать с московским временем).-->

    Автоматизация: сервисы для работы с отчётами Озон

    Ручная выгрузка отнимает время, особенно если вам нужны данные из нескольких отчётов. К счастью, есть инструменты для автоматизации:

    • 📱 Мобильные приложения:
      • Ozon Seller (официальное) — базовая аналитика в телефоне.
      • Мой Склад — интеграция с отчётами по остаткам.
    • 💻 Десктопные решения:
      • 1С:Управление торговлей — синхронизация с отчётами Озон через плагины.
      • Excel + Power Query — автоматическая выгрузка и обработка данных.
    • ☁️ Облачные сервисы:
      • Google Apps Script — бесплатная автоматизация для Google Sheets.
      • Zoho Analytics — визуализация данных из Озон.

    🔹 Пример настройки в Google Sheets:

    1. Откройте Google Таблицы и создайте новый файл.
    2. Перейдите в Расширения → Apps Script.
    3. Вставьте код для выгрузки данных через API (примеры есть в открытых репозиториях).
    4. Настройте триггер для автоматического обновления (например, каждый день в 10:00).

    ⚠️ Внимание: при использовании сторонних сервисов:

    1. Не передавайте API-Key ненадёжным платформам — это может привести к утечке данных.

    2. Проверяйте лимиты сервисов: некоторые бесплатные тарифы позволяют выгружать не более 1000 строк в день.

    3. Тестируйте интеграции на песчаной версии (sandbox) API, если она доступна.

    Советы по анализу полученных данных

    Скачать отчёт — половина дела. Главное — правильно его проанализировать. Вот несколько практических советов:

    • 📉 Сравнивайте периоды: анализируйте данные за аналогичные дни недели (например, понедельник 3 июня и понедельник 10 июня), чтобы исключить влияние «выходного эффекта».
    • 🔍 Ищите аномалии: резкое падение продаж по одному SKU может указывать на проблемы с карточкой товара или остатками.
    • 💰 Считайте чистую прибыль: вычтите из выручки комиссии Озон, стоимость доставки и возвраты — это даст реальную картину доходности.
    • 📦 Контролируйте оборот складов: если товар лежит дольше 30 дней, рассмотрите акции или смену ценовой стратегии.

    📊 Пример анализа отчёта по продажам:

    1. Отсортируйте товары по убыванию количества продаж.
    2. Выделите топ-20% позиций — они дают 80% выручки (принцип Парето).
    3. Проверьте средний чек по каждому SKU: если он ниже рынка, возможно, стоит поднять цену.
    4. Сравните конверсию (соотношение просмотров к продажам) — если она ниже 2%, оптимизируйте карточку товара.

    💡 Продвинутый приём: объединяйте данные из разных отчётов. Например, сопоставляйте Отчёт по продажам и Отчёт по трафику, чтобы понять, какие источники переходов (поиск, рекомендации, реклама) приносят больше конверсий. Для этого:

    1. Экспортируйте оба отчёта в Excel.
    2. Используйте функцию ВПР (VLOOKUP) для связывания данных по SKU.
    3. Постройте сводную таблицу с разбивкой по источникам трафика.

    FAQ: ответы на частые вопросы

    Можно ли скачать отчёт за весь год?

    Нет, максимальный период — 3 месяца. Для годового анализа разбейте запрос на кварталы и объедините данные вручную или через Excel.

    Почему в отчёте по остаткам не совпадают цифры с реальными запасами?

    Это типичная проблема для FBS. Причины:

    • Задержка обновления данных (до 24 часов).
    • Товары в статусе «на контроле качества» или «в пути».
    • Ошибки при приёмке на складе Озон.

    Для точности сверяйте данные с отчётом Движение товаров по складам.

    Как экспортировать отчёт в Google Sheets автоматически?

    Используйте Google Apps Script с API Озон. Пример кода: 

    function getOzonReport() {

    const url = 'https://api-seller.ozon.ru/v2/analytics/data';
    

    const options = {
    

    method: 'post',
    

    headers: {
    

    'Client-Id': 'YOUR_CLIENT_ID',
    

    'Api-Key': 'YOUR_API_KEY',
    

    'Content-Type': 'application/json'
    

    },
    

    payload: JSON.stringify({
    

    date_from: '2026-06-01',
    

    date_to: '2026-06-30',
    

    metrics: ['revenue'],
    

    dimensions: ['sku', 'day']
    

    })
    

    };
    

    const response = UrlFetchApp.fetch(url, options);
    

    const data = JSON.parse(response.getContentText());
    

    // Далее обработайте данные и запишите в Sheet
    

    }

    Не забудьте заменить YOUR_CLIENT_ID и YOUR_API_KEY на реальные значения.

    Что делать, если отчёт не скачивается из-за ошибки 500?

    Ошибка 500 указывает на сбой сервера Озон. Действия:

    1. Подождите 10–15 минут и повторите попытку.
    2. Попробуйте другой браузер (например, Chrome вместо Safari).
    3. Очистите кэш или используйте режим инкогнито.
    4. Если проблема сохраняется, напишите в поддержку Озон с указанием:
      • Типа отчёта.
      • Времени попытки скачивания.
      • Скриншота ошибки.

    Как узнать, какой отчёт нужен для налоговой отчётности?

    Для налоговой используйте:

    • Отчёт по выплатам — подтверждение доходов.
    • Отчёт по комиссиям — подтверждение расходов.
    • Акт выполненных работ (формируется в разделе Документы) — для подтверждения услуг Озон.

    Все отчёты должны быть за период, совпадающий с налоговым (обычно календарный месяц). Экспортируйте их в формате PDF с электронной подписью Озон.
