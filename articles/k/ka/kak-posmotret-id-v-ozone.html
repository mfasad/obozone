Почему ID в Озоне так важны — и где их искать

Каждый товар, заказ или даже категория на Ozon имеет уникальный идентификатор — ID. Это не просто набор цифр: от правильного использования этих кодов зависит скорость обработки заказов, точность аналитики и даже рейтинг продавца. Например, ID карточки товара (SKU) в Озоне автоматически формируется при загрузке через Excel, но его можно редактировать только до первой продажи.

Покупателям ID помогают отследить статус возврата или жалобы, а продавцам — синхронизировать остатки с 1С, выгружать отчёты по продажам или настраивать рекламные кампании. Проблема в том, что на платформе одновременно существуют 4 типа идентификаторов (артикул, SKU, ID заказа, ID карточки товара), и их легко перепутать. Далее разберём, где именно искать каждый из них — с пошаговыми скриншотами и лайфхаками для ускорения работы.

1. Где найти ID карточки товара (Product ID) в личном кабинете Озона

Этот идентификатор присваивается автоматически при создании карточки товара и не изменяется даже после редактирования. Он нужен для прямой ссылки на товар, интеграции с CRM или настройки рекламы.

Чтобы его найти:

  • 📌 Перейдите в раздел Товары → Мои товары в личном кабинете Ozon Seller.
  • 🔍 В таблице с товарами кликните на название нужного продукта — откроется его карточка.
  • 🔗 В адресной строке браузера будет URL вида https://seller.ozon.ru/products/{ID}, где {ID} — это и есть Product ID (например, 123456789).
  • 📋 Альтернативный способ: экспортируйте список товаров в Excel через кнопку Выгрузить — ID будет в колонке product_id.

⚠️ Внимание: Не путайте Product ID с SKU! Первый присваивается системой и неизменен, а второй (артикул) вы задаёте сами при загрузке товара.

💡

Скопируйте Product ID и добавьте его в шаблон ссылки https://www.ozon.ru/product/{ID}/ — так вы получите прямую ссылку на товар для покупателей.

2. Как узнать SKU (артикул) товара в Озоне

SKU (Stock Keeping Unit) — это ваш внутренний артикул, который вы указываете при загрузке товара. Он может совпадать с заводским артикулом производителя или быть уникальным для вашего магазина. В отличие от Product ID, SKU можно редактировать (но с осторожностью — это повлияет на остатки!).

Где его посмотреть:

  • 📦 В личном кабинете: Товары → Мои товары → [Выберите товар] → Раздел "Характеристики".
  • 📊 В отчётах: скачайте выгрузку по остаткам (Склад → Остатки) — SKU будет в колонке sku.
  • 🔄 В Excel-шаблоне для загрузки товаров: колонка Артикул (обязательна к заполнению!).

3. Как найти номер заказа (Order ID) для отслеживания

Номер заказа нужен покупателям для отслеживания посылки, а продавцам — для обработки возвратов или связи с поддержкой. Он формируется автоматически при оформлении покупки и состоит из 14 цифр.

Способы найти Order ID:

  • 📬 В письме от Озона: тема письма будет "Заказ №{ID} оформлен".
  • 🛒 В личном кабинете покупателя: Мои заказы → [Выберите заказ] (ID в заголовке).
  • 📦 Для продавцов: Заказы → Все заказы → [Кликните на заказ] (ID в URL или колонке Номер заказа).

⚠️ Внимание: Если покупатель просит вас найти заказ по номеру телефона, используйте поиск в разделе Заказы → Фильтр → Номер телефона. Но никогда не делитесь Order ID с третьими лицами — это может привести к мошенничеству с возвратами!

Что делать, если Order ID не отображается?

Если номер заказа не виден в личном кабинете, проверьте:

1) Не используете ли вы тестовый режим (песочницу) — там ID формируются по другим правилам.

2) Не блокируется ли номер адблоком (отключите расширения браузера).

3) Обновите страницу или очистите кэш (Ctrl+F5).

4. Как найти ID товара в мобильном приложении Озона

В мобильном приложении интерфейс упрощён, но все ID доступны. Главное — знать, где искать:

Для покупателей:

  • 📱 Откройте карточку товара → тапните на три точки (⋮) → Поделиться → скопируйте ссылку. ID будет после /product/.
  • 🛍️ В заказе: откройте заказ → прокрутите до блока Информация о товаре → там будет Артикул (это SKU).

Для продавцов:

  • 📊 В приложении Ozon Seller: Товары → [Выберите товар] → Детали → прокрутите до ID товара.
  • 📦 В заказах: Заказы → [Выберите заказ] → Товары → тапните на товар → увидите SKU и Product ID.

5. Где взять ID для API Озона (для автоматизации)

Если вы подключаете Ozon API для интеграции с 1С, Bitrix24 или другими системами, вам понадобятся:

  • 🔑 Client-ID и API-Key (для аутентификации) — берутся в Настройки → Интеграции → API.
  • 📦 Product ID или SKU (для работы с товарами).
  • 📦 Warehouse ID (идентификатор склада) — найдёте в Склад → Настройки → Склады.

Пример запроса к API для получения информации о товаре:

GET https://api-seller.ozon.ru/v1/product/info/{Product_ID}

Headers:


- Client-Id: ваш_client_id


- Api-Key: ваш_api_key

⚠️ Внимание: Не храните API-Key в открытом виде! Используйте переменные окружения или защищённые хранилища (например, AWS Secrets Manager). При утечке ключа немедленно сгенерируйте новый в личном кабинете.

💡

Для тестирования API используйте песочницу (sandbox) Озона — там не нужны реальные Client-ID и API-Key, а данные не влияют на продажи.

6. Частые ошибки при работе с ID в Озоне

Даже опытные продавцы иногда путают идентификаторы или используют их неправильно. Вот самые распространённые ошибки:

  • 🔄 Изменение SKU после продаж: это приводит к рассинхронизации остатков и ошибкам при сборке заказов. Если нужно поменять артикул, создайте новую карточку товара.
  • 📋 Копирование Product ID вместо SKU в Excel-шаблоны: система не примет такой файл. Всегда проверяйте колонку Артикул.
  • 🔗 Использование короткой ссылки (например, ozon.ru/{short-id}) для API: она не содержит реальный Product ID. Берите ID только из URL карточки товара.
  • 📦 Игнорирование Warehouse ID при работе с FBS: без него нельзя указать, с какого склада отгружать товар.

Чтобы избежать проблем:

  1. Всегда делайте тестовую выгрузку в песочницу перед массовым обновлением товаров.
  2. Используйте Проверку файла в личном кабинете перед загрузкой Excel.
  3. Сверяйте SKU в отчётах по остаткам и заказам — они должны совпадать.

FAQ: Ответы на частые вопросы про ID в Озоне

Можно ли изменить Product ID после создания товара?

Нет, Product ID присваивается системой один раз и не редактируется. Если вам нужно поменять идентификатор (например, для переноса товара в другой аккаунт), создайте новую карточку с другим названием или артикулом.

Где взять ID для возврата товара?

Для оформления возврата нужен Order ID (номер заказа) и SKU товара. Их можно найти:

  • В письме от Озона о заказе.
  • В личном кабинете покупателя в истории заказов.
  • В чеке (если покупка была офлайн в ПВЗ).

Если вы продавец, ID возврата также будет в разделе Возвраты → Все возвраты.

Что делать, если SKU дублируется?

Дублирующиеся SKU приводят к ошибкам при загрузке товаров. Чтобы исправить:

  1. Экспортируйте список товаров в Excel (Товары → Выгрузить).
  2. Найдите дубли в колонке sku (можно использовать условное форматирование).
  3. Измените артикулы для дублей (например, добавьте суффикс -1, -2).
  4. Загрузите обновлённый файл обратно.

⚠️ Если товар уже продавался, лучше создать новую карточку с уникальным SKU, а старую архивировать.

Как найти ID категории товара?

ID категории нужен для загрузки товаров через API или Excel. Чтобы его узнать:

  1. Перейдите в Товары → Добавить товар.
  2. Выберите нужную категорию в дереве.
  3. ID категории отобразится в URL после category_id= (например, category_id=12345).

Список всех категорий с ID также можно получить через API-метод /v1/category/tree.

Почему в Excel не отображается колонка с Product ID?

Если вы экспортировали товары, но в файле нет колонки product_id, проверьте:

  • Выбран ли правильный шаблон выгрузки (должен быть Полный отчёт по товарам).
  • Не скрыта ли колонка в Excel (попробуйте развернуть все столбцы).
  • Не используете ли вы устаревшую версию шаблона (обновите его в личном кабинете).

Если проблема остаётся, попробуйте выгрузить данные через API или обратитесь в поддержку с указанием ошибки.
