Как посмотреть остатки на складах Ozon: 5 рабочих способов для продавцов и покупателей

Проверка остатков на складах Ozon — критически важная задача как для продавцов, так и для покупателей. Для бизнеса это вопрос логистики, предотвращения штрафов за нехватку товаров и оптимизации запасов. Для клиентов — возможность убедиться в наличии товара перед оформлением заказа и выбрать самый быстрый способ доставки. Однако не все знают, что информация об остатках на складах Ozon обновляется в реальном времени и доступна через несколько каналов.

В этой статье мы разберём все актуальные способы проверки остатков — от официальных инструментов в личном кабинете до альтернативных методов с использованием API и сторонних сервисов. Особое внимание уделим нюансам работы с FBS (складская модель) и FBO (самостоятельная доставка), а также объясним, почему данные об остатках могут отличаться в разных источниках. Если вы продавец, вы научитесь избегать ошибок при синхронизации запасов; если покупатель — сможете точно узнать, когда товар появится на складе рядом с вами.

1. Официальные способы проверки остатков для продавцов

Продавцы на Ozon имеют доступ к расширенной аналитике остатков через личный кабинет. Основные инструменты зависят от выбранной модели работы:

  • 📦 FBS (складская модель) — остатки отображаются в разделе «Товары» с разбивкой по складам Ozon. Здесь же можно увидеть резервы под заказы и свободные единицы.
  • 🚚 FBO (самостоятельная доставка) — продавец сам контролирует остатки в своей системе, но должен синхронизировать их с Ozon через API или вручную.
  • 🔄 Комбинированная модель — если часть товаров на складах Ozon, а часть — у продавца, данные будут разделены по вкладкам.

Чтобы посмотреть остатки в личном кабинете:

  1. Перейдите в раздел Товары → У меня на складах.
  2. Выберите нужный товар и кликните на строку с названием.
  3. В открывшейся карточке перейдите на вкладку Остатки.
  4. Система покажет распределение товара по складам Ozon (например, Москва-1, Санкт-Петербург-2) и резервы под заказы.
💡

Если вы продавец и видите расхождения между фактическими остатками и данными в личном кабинете, обязательно запустите синхронизацию запасов вручную через кнопку «Обновить» в карточке товара. Это поможет избежать штрафов за нехватку товаров при оформлении заказов.

Для FBS-модели Ozon предоставляет детализацию по каждому складу, включая:

  • 📍 Географическое расположение (город и адрес склада).
  • 📦 Физический остаток — реальное количество товара на складе.
  • 🔒 Резерв под заказы — единицы, которые уже забронированы под оформленные, но ещё не собранные заказы.
  • ✅ Доступно к продаже — свободные единицы, которые можно заказать.
Что такое "виртуальный склад" в Ozon?

В личном кабинете продавца может отображаться "виртуальный склад" — это не физическое хранилище, а обобщённая группа складов в одном регионе. Например, "Москва (виртуальный)" объединяет несколько реальных складов в Московской области. Такие остатки показываются суммарно, без привязки к конкретному адресу.

2. Как покупателю узнать остатки товара на складах Ozon

Покупатели не имеют прямого доступа к детализированным остаткам на складах, но могут использовать косвенные методы. Самый простой способ — посмотреть время доставки в карточке товара. Если товар доступен к заказу с доставкой "завтра" или "через 2 дня", это означает, что он есть на складе в вашем регионе. Если же указан срок 5-7 дней, скорее всего, товар придётся везти из другого города.

Более точная информация доступна через:

  • 📱 Мобильное приложение Ozon — в карточке товара иногда отображается надпись "Есть на складе" или "Ограниченное количество".
  • 🌐 Веб-версия сайта — при выборе города доставки система может показать уточнение "Доставим из [город]".
  • 📧 Чат с поддержкой — операторы Ozon могут уточнить наличие товара на конкретном складе по запросу (но не всегда предоставляют эту информацию).

Пример: если вы видите в карточке товара строку "Доставим из Екатеринбурга за 3 дня", это означает, что товар физически находится на складе в Екатеринбурге, и курьерская служба рассчитала время транспортировки до вашего города.

3. Проверка остатков через API Ozon (для продавцов и разработчиков)

Для автоматизации работы с остатками продавцы могут использовать API Ozon. Это особенно актуально для крупных магазинов с тысячами SKU, где ручная проверка невозможна. Основные методы API для работы с остатками:

  • 🔄 /v2/analytics/stock_on_warehouses — возвращает остатки по всем складам с разбивкой по товарам.
  • 📊 /v1/product/info/stocks — показывает актуальные остатки для конкретного SKU.
  • 📦 /v1/warehouse/list — список всех складов Ozon с идентификаторами (нужно для привязки остатков к конкретному складу).

Пример запроса для получения остатков по конкретному товару (используется client-id и api-key из личного кабинета):

GET https://api-seller.ozon.ru/v1/product/info/stocks

Headers:


Client-Id: ваш_client_id


Api-Key: ваш_api_key


Content-Type: application/json



Body:


{


"offer_id": "12345678",  // ID вашего товара


"warehouse_id": 10100005 // ID склада (опционально)


}

Важно: данные API обновляются с задержкой до 15 минут. Если вам нужны остатки в реальном времени (например, для синхронизации с 1С), используйте вебхуки Ozon, которые отправляют уведомления при изменении запасов.

4. Экспорт остатков в Excel и анализ данных

Для удобства анализа продавцы могут экспортировать остатки в Excel или Google Sheets. В личном кабинете Ozon это делается через раздел Аналитика → Отчёты → Остатки на складах. Система позволяет выбрать:

  • 📅 Период (текущий день, неделя, месяц).
  • 📦 Склады (все или конкретные).
  • 📊 Формат (XLSX или CSV).

С экспортированными данными можно:

  • 📈 Строить графики динамики остатков по дням/неделям.
  • 🔔 Настраивать уведомления о критическом уровне запасов (например, через условное форматирование в Excel).
  • 🔄 Синхронизировать с 1С или другой учётной системой.

Экспорт остатков в Excel полезен для прогнозирования спроса. Например, если товар на складе в Москве заканчивается каждые 3 дня, можно заранее перевести часть запасов из Санкт-Петербурга, чтобы избежать дефицита.

5. Почему данные об остатках могут отличаться?

Частая проблема — расхождения между остатками в личном кабинете, API и реальным положением дел на складе. Основные причины:

⚠️ Внимание: Если вы видите в личном кабинете остаток 10 единиц, а при оформлении заказа система показывает ошибку "Недостаточно товара", это может означать, что часть единиц заблокирована под неоплаченные заказы или находится в зоне приёмки склада.
  • ⏳ Задержка синхронизации — данные в личном кабинете обновляются каждые 10-30 минут, а API может отставать до 15 минут.
  • 🔄 Резервирование под заказы — товар может быть физически на складе, но забронирован под оформленный заказ (в этом случае он не отображается как "доступный").
  • 🚛 Товар в пути — если вы отправили товар на склад Ozon, но он ещё не прошёл приёмку, он не будет учитываться в остатках.
  • 🔧 Технические работы — иногда Ozon проводит инвентаризацию, и на это время остатки могут быть скрыты или неточны.

Чтобы минимизировать риски:

  1. Настройте автоматическую синхронизацию остатков между вашей учётной системой и Ozon (через API или интеграции с 1С).
  2. Используйте буферный запас — держите на складе на 10-15% больше товаров, чем показывают продажи за последние 7 дней.
  3. Проверяйте остатки в пиковые часы (с 10:00 до 14:00 по МСК), когда большинство заказов оформляется.

6. Альтернативные способы проверки остатков

Помимо официальных инструментов, существуют альтернативные методы, которые могут быть полезны в специфических ситуациях:

  • 🤖 Парсинг страниц Ozon — некоторые сервисы (например, Ozon Parser или DataCol) позволяют собирать данные об остатках по публичным страницам товаров. Однако это нарушает правила Ozon и может привести к блокировке аккаунта.
  • 📊 Сервисы аналитики — платформы вроде Sellerboard или Peak агрегируют данные об остатках и продажах, но их точность зависит от качества интеграции с Ozon.
  • 📱 Мобильные приложения для продавцов — например, Ozon Seller или Мой Склад предоставляют удобные дашборды с остатками.
⚠️ Внимание: Использование неофициальных методов парсинга или скрейпинга данных Ozon может привести к блокировке аккаунта за нарушение пользовательского соглашения. Озон активно борется с автоматизированным сбором данных, поэтому для легальной работы используйте только официальное API.

Если вам нужны данные по остаткам конкурентов, единственный легальный способ — анализ публичной информации:

  • 📉 Динамика цен — если цена на товар резко выросла, это может означать дефицит.
  • ⭐ Отзывы покупателей — иногда в комментариях пишут о долгой доставке из-за нехватки товара на складе.
  • 📦 Изменение статуса доставки — если раньше товар доставлялся за 2 дня, а теперь за 7, вероятно, его привезли из другого региона.

7. Частые ошибки при работе с остатками и как их избежать

Даже опытные продавцы допускают ошибки, которые ведут к штрафам или потерянным продажам. Рассмотрим самые распространённые:

Ошибка Последствия Как избежать
Несинхронизированные остатки между своей системой и Ozon Штраф за отмену заказа из-за нехватки товара (до 50% от стоимости товара) Настройте автоматическую синхронизацию через API или 1С
Игнорирование резервов под заказы Продажа товара, который уже забронирован под другой заказ Учитывайте поле reserved_amount в API или колонку "В резерве" в Excel-отчёте
Неучёт времени доставки товара на склад Ozon Товар числится в остатках, но физически ещё не поступил на склад Отправляйте товар на склад заранее и отслеживайте статус приёмки в личном кабинете
Работа без буферного запаса Дефицит товара в пиковые дни (например, во время распродаж) Держите на складе запас в 10-15% от среднедневных продаж

Одна из самых опасных ошибок — продажа товара, который уже забронирован под другой заказ. Это происходит, когда продавец не учитывает поле reserved_amount в API или не обновляет остатки вручную после оформления заказа. В результате Ozon может оштрафовать за отмену заказа по вине продавца.

Чтобы этого избежать:

  1. Всегда проверяйте свободные остатки (free_to_sell_amount), а не общие.
  2. Настройте автоматическое обновление остатков при изменении статуса заказа.
  3. Используйте систему уведомлений о низком уровне запасов (например, через Excel или специализированные сервисы).

Частые вопросы об остатках на складах Ozon

Можно ли увидеть остатки товара на конкретном складе Ozon, если я покупатель?

Нет, покупатели не имеют доступа к детализированным остаткам по складам. Однако можно косвенно оценить наличие товара по времени доставки: если товар доставляется быстро (1-3 дня), он скорее всего есть на складе в вашем регионе. Также иногда в карточке товара отображается надпись "Есть на складе" или "Ограниченное количество".

Как часто обновляются остатки в личном кабинете продавца?

Данные в личном кабинете обновляются каждые 10-30 минут, но в пиковые часы (например, во время распродаж) задержка может увеличиваться до 1 часа. Для самых актуальных данных используйте API Ozon — оно обновляется каждые 5-15 минут. Если вам нужны остатки в реальном времени, настройте вебхуки, которые будут присылать уведомления при изменении запасов.

Что делать, если остатки в личном кабинете не совпадают с реальными?

Если вы видите расхождения, сначала запустите ручную синхронизацию в карточке товара (кнопка "Обновить"). Если проблема остаётся:

  1. Проверьте, не находится ли часть товара в зоне приёмки склада (статус "В пути").
  2. Убедитесь, что товар не заблокирован за неоплаченные заказы (поле reserved_amount в API).
  3. Свяжитесь с службой поддержки Ozon через чат в личном кабинете — они могут провести проверку.

Если расхождения критические (например, в личном кабинете 50 единиц, а на складе 10), это может быть технической ошибкой — в этом случае поддержка Ozon обязана её исправить.

Можно ли узнать остатки конкурентов на Ozon?

Ozon не предоставляет официальных инструментов для просмотра остатков других продавцов. Однако можно использовать косвенные методы:

  • Отслеживать динамику цен — если цена резко выросла, возможно, товар заканчивается.
  • Читать отзывы покупателей — иногда там пишут о долгих сроках доставки из-за нехватки товара.
  • Использовать сервисы аналитики (например, Sellerboard), которые оценивают остатки на основе изменений рейтинга и доступности товара.

Важно: парсинг данных Ozon для получения остатков конкурентов нарушает пользовательское соглашение и может привести к блокировке аккаунта.

Как перевести товар с одного склада Ozon на другой?

Самостоятельно перемещать товары между складами Ozon нельзя — это делает служба логистики маркетплейса. Однако вы можете:

  1. Запросить перемещение через поддержку Ozon (указываете SKU, текущий и целевой склады).
  2. Отправить новую партию товара на нужный склад (через раздел "Поступления" в личном кабинете).
  3. Использовать сервис "Перемещение запасов" в личном кабинете (доступен не для всех категорий товаров).

Срок перемещения обычно составляет 3-7 дней, в зависимости от расстояния между складами. Услуга может быть платной — тарифы уточняйте у менеджера Ozon.
