Как посмотреть, сколько раз купили товар на Ozon: все способы для продавцов и покупателей

Почему данные о продажах важны для работы на Ozon

Статистика продаж на Ozon — это не просто цифры, а основа для принятия решений. Для продавцов количество заказов показывает востребованность товара, помогает корректировать ценовую политику и планировать закупки. Например, если ваш товар продаётся по 100 штук в месяц, а запасов осталось на 50 — пора срочно договариваться с поставщиком. Покупателям же данные о популярности помогают оценить надёжность продавца: товар с сотнями продаж вызывает больше доверия, чем новинка без отзывов.

Однако Ozon не показывает количество продаж напрямую в карточке товара — эту информацию нужно искать в аналитике или через специальные инструменты. В зависимости от вашей роли (продавец, покупатель, аналитик) и уровня доступа способы получения данных будут разными. В этой статье мы разберём все актуальные методы — от базовых до продвинутых, включая скрытые фишки личного кабинета и работу с API.

Важно понимать, что некоторые данные доступны только продавцам, а покупатели могут видеть лишь косвенные показатели (рейтинги, отзывы, позиции в выдаче). Если вы продавец, но ещё не подключили Ozon Seller или Ozon Business, часть функций будет для вас закрыта. Об этом мы тоже расскажем подробнее.

Способ 1: Аналитика в личном кабинете Ozon Seller

Самый надёжный и точный способ — использовать встроенную аналитику в личном кабинете продавца. Этот метод подходит только для владельцев товаров или их официальных представителей. Чтобы увидеть количество продаж:

  1. Авторизуйтесь на сайте seller.ozon.ru.
  2. Перейдите в раздел Аналитика → Продажи.
  3. Выберите нужный временной период (день, неделя, месяц, год).
  4. В таблице найдите столбец Количество заказов — здесь и будет искомая цифра.

Если вам нужна детализация по конкретному товару, используйте фильтр по SKU или названию. Обратите внимание: данные обновляются с задержкой до 24 часов, поэтому сегодняшние продажи могут отобразиться только завтра.

💡

Чтобы сравнить динамику продаж по неделям, экспортируйте данные в Excel через кнопку Выгрузить в правом верхнем углу. Это поможет выявить сезонные пики спроса.

Для продавцов на модели FBS (когда товар хранится на складе Ozon) доступна дополнительная аналитика по остаткам и оборотам. Если же вы работаете по схеме FBO (самостоятельная доставка), часть метрик может отсутствовать.

⚠️ Внимание: Если вы продавец с несколькими аккаунтами, данные по продажам в личном кабинете показываются только по текущему профилю. Чтобы увидеть общую картину, придётся переключаться между аккаунтами или использовать Ozon API.

Способ 2: Использование Ozon API для автоматизированного сбора данных

Для тех, кто управляет большим ассортиментом или ведёт аналитику по нескольким аккаунтам, ручной сбор статистики через личный кабинет неэффективен. В этом случае поможет Ozon API — интерфейс программирования, который позволяет автоматически получать данные о продажах, остатках и других метриках.

Чтобы начать работу с API, вам понадобится:

  • 🔑 Ключ API (генерируется в личном кабинете в разделе Настройки → API).
  • 💻 Базовые знания Python, JavaScript или другого языка для отправки запросов.
  • 📊 Понимание структуры JSON-ответов от Ozon.

Пример запроса для получения количества продаж по конкретному SKU:

GET https://api-seller.ozon.ru/v2/analytics/data

Headers:


Client-Id: [ваш_client_id]


Api-Key: [ваш_api_key]



Body:


{


"date_from": "2026-01-01",


"date_to": "2026-01-31",


"metrics": ["ordered_units"],


"dimensions": ["sku"],


"filters": {


"sku": [123456789]  // Замените на ваш SKU


}


}

В ответе вы получите массив с данными по каждому товару, где ordered_units — это и есть количество проданных единиц. Для визуализации можно подключить результаты к Google Data Studio или Power BI.

Как получить Client-ID и API-Key?

1. Перейдите в Личный кабинет → Настройки → API.

2. Нажмите Создать ключ и укажите название (например, "Аналитика продаж").

3. Скопируйте сгенерированные Client-ID и Api-Key — они понадобятся для авторизации запросов.

4. Никому не передавайте эти данные! Компрометация ключа может привести к утечке информации или блокировке аккаунта.

⚠️ Внимание: Ozon ограничивает количество запросов к API — обычно это 1000 запросов в минуту. При превышении лимита вы получите ошибку 429 Too Many Requests. Чтобы избежать блокировки, используйте паузы между запросами или кэшируйте данные.

Способ 3: Сторонние сервисы для анализа продаж

Если вы не продавец, а покупатель или конкурент, официальные методы вам недоступны. В этом случае помогут сторонние сервисы, которые собирают и анализируют данные с Ozon. Наиболее популярные инструменты:

Сервис Функционал Стоимость Ограничения
Ozon Insights Анализ продаж, динамика цен, рейтинг товаров От 1 500 ₽/мес Нет данных по FBS-товарам
DataLens Визуализация продаж, сравнение с конкурентами Бесплатно (с ограничениями) Задержка данных до 3 дней
SellerBoard Мониторинг продаж, прибыли, возвратов От 990 ₽/мес Требуется подключение к API
EcomTrack Отслеживание позиций в поиске, анализ спроса От 2 900 ₽/мес Нет детализации по регионам

Большинство сервисов требуют авторизации через Ozon API, поэтому для полноценной работы вам всё равно понадобится доступ к аккаунту продавца. Бесплатные версии обычно имеют ограничения по количеству запросов или глубине анализа.

Например, DataLens позволяет построить дашборд с графиками продаж по категориям, но не покажет точные цифры по конкретному товару без подключения к API. А Ozon Insights даёт данные по топовым товарам в нише, но не раскрывает информацию по малоизвестным позициям.

Способ 4: Косвенные показатели для покупателей

Если вы покупатель и хотите оценить популярность товара перед покупкой, прямой доступ к статистике продаж у вас отсутствует. Однако есть несколько косвенных индикаторов, которые помогут сделать выводы:

  • ⭐ Количество отзывов. Чем их больше, тем выше вероятность, что товар хорошо продаётся. Например, 500 отзывов за месяц говорят о высоком спросе.
  • 📦 Рейтинг продавца. В карточке товара отображается процент положительных отзывов и количество заказов за последний месяц (например, "98% положительных отзывов из 1 200 заказов").
  • 🔝 Позиция в поисковой выдаче. Товары с высокими продажами обычно занимают топовые позиции по релевантным запросам.
  • 🏷️ Наличие акционных меток. Если товар часто участвует в распродажах (например, "Топ продаж" или "Выбор Ozon"), это косвенно подтверждает его популярность.

Также обратите внимание на динамику цен. Если цена на товар стабильно растёт, это может означать высокий спрос и малые остатки. Например, если в январе товар стоил 2 000 ₽, а в феврале — уже 2 500 ₽, вероятно, его активно раскупают.

💡

Косвенные показатели не дают точных цифр, но помогают оценить тенденции. Например, товар с 10 отзывами в день продаётся примерно в 5–10 раз чаще, чем с 1 отзывом в день (учитывая, что отзывы оставляет только 10–20% покупателей).

Ещё один лайфхак: посмотрите на даты отзывов. Если они распределены равномерно (например, по 5–10 штук каждый день), спрос стабильный. Если отзывы идут "волнами" (например, 50 штук за один день, а потом неделю тишины), это может говорить о сезонности или разовых акциях.

Способ 5: Парсинг данных (для продвинутых пользователей)

Для тех, кто готов погрузиться в технические детали, есть метод сбора данных через парсинг — автоматическое извлечение информации с страниц Ozon. Этот способ требует знаний в области веб-скрейпинга и может нарушать правила маркетплейса, поэтому используйте его на свой страх и риск.

Основные этапы парсинга:

  1. Выберите инструмент: Python с библиотеками BeautifulSoup или Scrapy, либо специализированные программы вроде ParseHub.
  2. Настройте сбор данных с карточек товаров (например, количество отзывов, рейтинг, позицию в выдаче).
  3. Запустите скрипт с учетом ограничений Ozon на частоту запросов (не более 1 запроса в секунду).
  4. Обработайте полученные данные и сопоставьте их с косвенными метриками (например, "1 отзыв = 5–10 продаж").

Пример простого скрипта на Python для сбора количества отзывов:

import requests

from bs4 import BeautifulSoup



url = "https://www.ozon.ru/product/пример-товара-123456789/"


headers = {'User-Agent': 'Mozilla/5.0'}



response = requests.get(url, headers=headers)


soup = BeautifulSoup(response.text, 'html.parser')



reviews_count = soup.find('span', {'class': 'reviews-count'}).text


print(f"Количество отзывов: {reviews_count}")

💡

Чтобы избежать блокировки, используйте прокси-серверы и имитируйте поведение реального пользователя (например, случайные задержки между запросами).

⚠️ Внимание: Парсинг может нарушать пользовательское соглашение Ozon. При массовом сборе данных ваш IP может быть заблокирован. Для легального анализа лучше использовать официальный API.

Частые ошибки и как их избежать

При работе со статистикой продаж на Ozon многие сталкиваются с типичными проблемами. Вот самые распространённые из них и способы их решения:

  • 🔄 Данные не обновляются. В личном кабинете и API статистика может отставать на 1–2 дня. Если вам нужны актуальные цифры, проверяйте продажи вручную через Заказы → Новые.
  • 📉 Расхождения в цифрах. Количество заказов в аналитике может отличаться от реальных продаж из-за возвратов или отмен. Сверяйте данные с разделом Финансы → Выплаты.
  • 🔒 Нет доступа к API. Убедитесь, что ваш тариф на Ozon Seller поддерживает работу с API. Для некоторых категорий (например, Premium) доступ открывается автоматически.
  • 🌍 Региональные различия. Данные по продажам в личном кабинете показываются по всем регионам. Чтобы увидеть распределение, используйте фильтр по Складам в аналитике.

Ещё одна частая ошибка — игнорирование сезонности. Например, если вы анализируете продажи новогодних товаров в январе, цифры будут завышены, а в июле — занижены. Всегда сравнивайте данные с аналогичным периодом прошлого года.

FAQ: Ответы на популярные вопросы

Можно ли увидеть продажи конкурентов?

Прямой доступ к данным конкурентов возможен только через Ozon API (при наличии согласия продавца) или сторонние сервисы вроде DataLens. Однако точные цифры вы не получите — только оценки на основе отзывов и рейтинга. Например, если у конкурента 1 000 отзывов за месяц, можно предположить, что он продаёт около 5 000–10 000 единиц товара (учитывая, что отзывы оставляет 10–20% покупателей).

Почему в аналитике показано 100 продаж, а денег пришло на 90?

Разница возникает из-за возвратов, отмен заказов или удержаний комиссии. В аналитике учитываются все заказы, включая те, которые später были отменены. Чтобы увидеть чистые продажи, сверяйте данные с разделом Финансы → Выплаты, где указаны только подтверждённые транзакции.

Как узнать продажи по конкретному региону?

В личном кабинете Ozon Seller перейдите в Аналитика → Продажи и примените фильтр по Региону доставки. Если вам нужна детализация по городам, используйте API с параметром dimensions": ["region"]. Учтите, что данные по FBS-товарам распределяются по складам, а не по конечным точкам доставки.

Можно ли экспортировать историю продаж за несколько лет?

Да, но с ограничениями. В личном кабинете доступна история за последний год. Для более ранних данных придётся использовать API или обратиться в поддержку Ozon с обоснованным запросом (например, для аудита или судебного разбирательства). Сторонние сервисы обычно хранят данные не дольше 2–3 лет.

Почему у товара много просмотров, но мало продаж?

Это может быть связано с несколькими факторами:

  • Высокая цена по сравнению с конкурентами.
  • Слабое описание или фотографии, которые не убеждают покупателей.
  • Низкий рейтинг продавца или отрицательные отзывы.
  • Товар попадает в выдачу по нерелевантным запросам (например, пользователи ищут одно, а находят другое).

Проанализируйте карточку товара и сравните её с лидерами категории.
