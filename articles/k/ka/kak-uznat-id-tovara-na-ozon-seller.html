Где и как найти ID товара на Ozon Seller: все способы для продавцов

Работа с маркетплейсом Ozon требует точности в каждом параметре — от названия товара до его уникального идентификатора. ID товара (или артикул) на платформе — это ключ к управлению карточкой, аналитике продаж и интеграции с системами учета. Без него невозможно обновить цену через Excel, связать товар с рекламной кампанией или исправить ошибку в описании через API.

Новички часто путают ID товара с SKU (внутренним артикулом продавца) или штрихкодом, что приводит к ошибкам при загрузке каталога. В этой статье разберем, где именно искать идентификатор в личном кабинете Ozon Seller, как его извлечь из выгрузок и почему иногда система выдает несколько ID на один товар. А еще — раскроем скрытый способ получения ID для товаров, которые еще не прошли модерацию.

Что такое ID товара на Ozon и зачем он нужен

Уникальный идентификатор (Product ID) — это числовой код, который Ozon присваивает каждому товару в каталоге. Он отличается от:

  • 📦 SKU — вашего внутреннего артикула (например, SKU-12345), который вы указываете при загрузке;
  • 🏷️ Штрихкода — международного кода товара (EAN, UPC);
  • 🔗 Ссылки на карточку — URL-адреса страницы товара.

ID присваивается автоматически после модерации и используется для:

  • 📊 Аналитики: отслеживания продаж, остатков и конверсии в Ozon Statistics;
  • 🔄 Обновлений: массового изменения цен, описаний или фотографий через Excel или API;
  • 🛒 Рекламы: привязки товара к кампаниям в Ozon Advertising;
  • 📦 Логистики: идентификации товара на складах FBS.
⚠️ Внимание: Если вы удалите товар и загрузите его заново, Ozon присвоит ему новый ID. Старый идентификатор станет неактивным и не будет отображаться в выгрузках.

Способ 1: Как найти ID в карточке товара в личном кабинете

Самый простой метод — извлечь идентификатор прямо из интерфейса Ozon Seller. Для этого:

  1. Авторизуйтесь в личном кабинете и перейдите в раздел Товары → Мой каталог.
  2. Найдите нужный товар через поиск или фильтры (по названию, SKU или статусу).
  3. Кликните по названию товара, чтобы открыть его карточку.

В верхней части страницы, под названием товара, вы увидите блок с технической информацией. Искомый ID будет обозначен как Идентификатор товара или Product ID (например, 123456789). Скопируйте это число — оно понадобится для дальнейших операций.

💡

Если товар не прошел модерацию, его ID можно найти в разделе Товары → На модерации. Кликните по строке товара — идентификатор отобразится в URL-адресе браузера после символов product/ (например, .../product/987654321).

Если карточка товара открывается в новом окне (в режиме предварительного просмотра), посмотрите адресную строку браузера. ID будет часть пути после /product/:

https://seller.ozon.ru/app/product/123456789/edit

Способ 2: Получение ID через Excel-выгрузку каталога

Для массовой работы с товарами удобнее использовать выгрузку в формате Excel. Это позволит получить ID всех позиций одновременно. Инструкция:

  1. В личном кабинете перейдите в Товары → Мой каталог.
  2. Нажмите кнопку Выгрузить (в правом верхнем углу) и выберите формат Excel.
  3. В открывшемся окне отметьте галочкой поле Идентификатор товара (Product ID) и подтвердите выгрузку.

Файл будет сформирован в течение 1–5 минут (в зависимости от размера каталога). В полученной таблице ищите колонку product_id — там будут перечислены все идентификаторы. Обратите внимание: если товар не прошел модерацию, в этой колонке будет пусто.

Идентификаторы (product_id) совпадают с карточками товаров|Отсутствуют дубликаты ID|Нет пустых ячеек в колонке product_id (кроме товаров на модерации)|Формат ID — числовой, без букв и символов-->

Тип выгрузки Где искать ID Примечания

Заказы Колонка product_id в детализации Позволяет связать продажу с конкретным товаром
Рекламные кампании Колонка item_id Может совпадать с product_id или быть уникальным

Способ 3: Извлечение ID из URL-адреса публичной карточки

Если товар уже опубликован на Ozon, его ID можно найти через публичную страницу. Этот метод полезен, когда нет доступа к личному кабинету (например, если вы работаете с подрядчиком).

  1. Откройте карточку товара на сайте Ozon.ru.
  2. Посмотрите адресную строку браузера. ID будет часть пути после /product/ или /p/.

Примеры:

  • Старый формат URL: https://ozon.ru/product/kompyuter-123456789/ → ID = 123456789;
  • Новый формат URL: https://ozon.ru/p/abc123def456/ → здесь ID зашифрован, используйте способ 1 или 2.
⚠️ Внимание: В новых URL-адресах Ozon использует хэшированные идентификаторы (например, p/abc123...). Они не совпадают с product_id из личного кабинета и не подходят для работы через API или Excel.

Способ 4: Получение ID через API Ozon Seller

Для автоматизации процессов (например, синхронизации с 1С или другими системами учета) используйте API Ozon Seller. Чтобы получить Product ID, выполните следующие шаги:

  1. Сгенерируйте Client-ID и API-Key в разделе Настройки → Интеграции → API.
  2. Используйте метод POST /v2/product/info с параметром offer_id (ваш SKU) или product_id (если известен).

Пример запроса для получения информации о товаре (включая ID):

curl -X POST \

'https://api-seller.ozon.ru/v2/product/info' \


-H 'Client-Id: YOUR_CLIENT_ID' \


-H 'Api-Key: YOUR_API_KEY' \


-H 'Content-Type: application/json' \


-d '{


"offer_id": "VASh_SKU_123",


"product_id": 0,


"sku": 0


}'

В ответе будет блок "product_id" с нужным идентификатором. Этот метод подходит для массового получения ID, но требует навыков работы с API.

Что делать, если API возвращает ошибку 404?

Ошибка 404 при запросе /v2/product/info означает, что товар не найден. Проверьте:

1) Правильность указанного offer_id (SKU);

2) Статус товара — он должен быть на модерации или опубликован;

3) Права доступа вашего API-Key (в настройках интеграций).

Если товар удален, его ID больше недействителен и не будет возвращен через API.

Способ 5: Как узнать ID товара, который еще не прошел модерацию

Мало кто знает, но Ozon присваивает предварительный ID товару сразу после загрузки — еще до модерации. Его можно извлечь из:

  • 📄 Страницы редактирования: в URL-адресе после /product/ (например, .../product/987654321/edit);
  • 📥 Файла импорта: если загружали товары через Excel, в ответе сервера будет колонка product_id (даже для немодерированных позиций);
  • 🔍 Журнала операций: в разделе Товары → Журнал найдите запись о загрузке и кликните по ней.

Этот ID можно использовать для:

  • 🔄 Обновления данных до модерации (через API или повторную загрузку Excel);
  • 📊 Отслеживания статуса в журнале (например, "На проверке", "Требуется доработка").
⚠️ Внимание: Предварительный ID может измениться после модерации, если Ozon объединит ваш товар с существующей карточкой в каталоге. Всегда проверяйте актуальность идентификатора после публикации.

Частые ошибки при работе с ID товара и как их избежать

Даже опытные продавцы сталкиваются с проблемами, связанными с идентификаторами. Рассмотрим типичные ситуации и решения:

Ошибка Причина Решение
ID не найден в Excel-выгрузке Товар не прошел модерацию или удален Проверьте статус в Товары → На модерации
Дублирующиеся ID в каталоге Озон объединил ваши товары с существующими карточками Используйте offer_id (SKU) для точной идентификации
API возвращает ошибку "Product not found" Неверный product_id или недостаточно прав Проверьте ID через личный кабинет и права API-Key
ID в URL публичной страницы не совпадает с product_id Новый формат ссылок Ozon использует хэши Используйте только product_id из личного кабинета

Еще одна распространенная проблема — путаница между product_id и offer_id:

  • product_id — уникальный идентификатор карточки товара на Ozon;
  • offer_id — ваш внутренний артикул (SKU), который вы указываете при загрузке.

Для массовых операций (например, обновления цен) часто требуется связка product_id + offer_id. Если вы используете только один из них, система может не идентифицировать товар.

💡

Всегда проверяйте актуальность ID перед массовыми операциями. Удаленные или архивированные товары теряют свои идентификаторы, и попытка обновить их приведет к ошибке.

FAQ: Ответы на частые вопросы о ID товара на Ozon

Можно ли изменить ID товара после публикации?

Нет, Product ID присваивается автоматически и не подлежит редактированию. Если вам нужно "сменить" ID, удалите товар и загрузите его заново — системе будет присвоен новый идентификатор. Учтите, что при этом потеряется история продаж и отзывов.

Почему у одного товара два разных ID?

Это происходит, если Ozon объединил вашу карточку с существующей в каталоге. Например, вы загрузили товар с теми же характеристиками, что и другой продавец. В этом случае система присваивает общий product_id, но сохраняет ваш offer_id (SKU) для внутреннего учета.

Где найти ID для товара, который продается в комплекте?

Для комплектов (наборов) Ozon создает отдельную карточку с уникальным product_id. Его можно найти:

  • В личном кабинете в разделе Товары → Комплекты;
  • В Excel-выгрузке по комплектам (колонка product_id);
  • Через API, используя метод /v1/product/complex/info.
Как связать ID товара с заказом покупателя?

В выгрузке заказов (раздел Заказы → Выгрузить) ищите колонку items. В ней будет массив с данными о товарах, включая product_id и offer_id. Пример структуры:

{

"items": [


{


"product_id": 123456789,


"offer_id": "SKU-123",


"name": "Название товара",


"quantity": 1


}


]


}

Можно ли использовать штрихкод (EAN) вместо Product ID?

Нет, штрихкод и Product ID — это разные сущности. Штрихкод (EAN/UPC) нужен для идентификации товара в физическом мире (например, при сканировании на складе), а product_id — для работы с системой Ozon. Однако в некоторых API-методах можно искать товар по штрихкоду, но ответ все равно будет содержать product_id.
