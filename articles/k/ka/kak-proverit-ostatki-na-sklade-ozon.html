Как проверить остатки на складе Ozon: все способы для продавцов в 2026 году

Контроль остатков на складах Ozon — критически важная задача для любого продавца, работающего по моделям FBS (склад Ozon) или FBO (самостоятельная доставка). От точности этих данных зависит не только избежание штрафов за недопоставку, но и репутация магазина: покупатели видят актуальность товаров в карточках, а алгоритмы маркетплейса учитывают коэффициент доступности при ранжировании.

В 2026 году Ozon обновлял интерфейсы личного кабинета и API, поэтому старые инструкции могут не работать. В этой статье — 5 проверенных способов получить актуальные остатки, включая скрытые фишки для массовой выгрузки и автоматизации. Мы разберём нюансы для разных типов складов (Ozon Рязань, Ozon Новосибирск, Ozon Краснодар и др.), а также покажем, как избежать расхождений между вашей системой учёта и данными маркетплейса.

Если вы продаёте через FBO, контроль остатков ложится на ваши плечи — здесь важно синхронизировать данные с 1С или другим софтом. Для FBS задача усложняется: нужно учитывать резервы под заказы, товары в пути на склад и возможные потери при приёмке. Ошибка в остатках может привести к автоматическому снижению лимитов на загрузку новых позиций — это одна из частых причин блокировки аккаунтов начинающих продавцов.

1. Проверка остатков через личный кабинет Ozon Seller

Самый простой способ — использовать веб-версию личного кабинета. Он подходит для разовых проверок и небольших каталогов (до 1 000 SKU). В 2026 году Ozon обновил раздел «Товары», добавив фильтры по складам и статусам остатков.

Чтобы увидеть актуальные данные:

  1. Авторизуйтесь на сайте seller.ozon.ru.
  2. Перейдите в раздел Товары → Остатки.
  3. В фильтре «Склад» выберите нужный регион (например, Ozon Москва (FBS) или Ваш склад (FBO)).
  4. Обратите внимание на колонки:
    • «Доступно» — товары, готовые к продаже;
    • «В резерве» — позиции, закреплённые под заказы;
    • «Недоступно» — брак, утерянные или заблокированные товары.

⚠️ Внимание: Данные в личном кабинете обновляются с задержкой до 3 часов. Если вы только что отправили товар на склад Ozon, он может не отобразиться сразу. Для срочной проверки используйте API (раздел 3).

Для удобства можно экспортировать остатки в Excel:

  1. Нажмите кнопку Экспорт в правом верхнем углу.
  2. Выберите формат XLSX или CSV.
  3. В полученном файле будут колонки с артикулами, названиями и остатками по каждому складу.
    4. 💡

    Если остатки в личном кабинете и вашей системе учёта расходятся более чем на 5%, запустите инвентаризацию через раздел Склады → Инвентаризация. Это поможет выявить потери или ошибки приёмки.

    2. Массовая выгрузка остатков в Excel: шаблоны и нюансы

    Для продавцов с большим ассортиментом (от 1 000 SKU) ручная проверка через личный кабинет неэффективна. Ozon позволяет выгружать остатки массово — это экономит время и снижает риск ошибок.

    Способы массовой выгрузки:

    • 📄 Стандартный экспорт из раздела Товары → Остатки (ограничение: до 50 000 строк за раз).
    • 🔄 Автоэкспорт через Настройки → Интеграции → Экспорт данных (настраивается расписание).
    • 📂 Выгрузка по API (для технически подкованных продавцов).

    3. Проверка остатков через API Ozon: руководство для разработчиков

    Для крупных продавцов и тех, кто интегрирован с 1С или другими системами учёта, API Ozon — оптимальный способ получить актуальные остатки в реальном времени. В 2026 году Ozon обновил документацию, добавив новые эндпоинты для работы со складами.

    Основные методы API для проверки остатков:

    • 📦 /v2/analytics/stock_on_warehouses — остатки по всем складам;
    • 📦 /v1/product/info/stocks — остатки по конкретным артикулам;
    • 📦 /v2/warehouse/stocks — расширенная информация с учётом резервов.

    Пример запроса на Python для получения остатков: 

    import requests


    url = "https://api-seller.ozon.ru/v2/analytics/stock_on_warehouses"
    

    headers = {
    

    "Client-Id": "YOUR_CLIENT_ID",
    

    "Api-Key": "YOUR_API_KEY",
    

    "Content-Type": "application/json"
    

    }
    


    response = requests.post(url, headers=headers)
    

    stocks = response.json()
    


    for item in stocks["result"]["items"]:
    

    print(f"Артикул: {item['offer_id']}, Склад: {item['warehouse_name']}, Остаток: {item['free_to_sell_amount']}")

    ⚠️ Внимание: При работе с API учитывайте лимиты:

    • До 100 запросов в минуту для эндпоинтов аналитики;
    • До 1 000 позиций в одном запросе к /v1/product/info/stocks;
    • Обязательная аутентификация по Client-Id и Api-Key.

Для тестирования запросов удобно использовать Postman. Скачайте коллекцию официальных запросов Ozon по ссылке.

Как получить Client-Id и Api-Key для API Ozon?

1. Перейдите в Настройки → Безопасность → API-ключи в личном кабинете.

2. Нажмите Создать ключ и укажите название (например, «Выгрузка остатков»).

3. Скопируйте Client-Id и Api-Key — они понадобятся для запросов.

⚠️ Храните ключи в безопасности! При утечке сразу отзывайте их в личном кабинете.

4. Проверка остатков в мобильном приложении Ozon Seller

Мобильное приложение Ozon Seller (доступно для iOS и Android) позволяет проверять остатки в дороге или на складе. Функционал ограничен по сравнению с веб-версией, но удобен для оперативного контроля.

Как пользоваться:

  1. Скачайте приложение из App Store или Google Play.
  2. Авторизуйтесь под своим аккаунтом продавца.
  3. Перейдите в раздел Товары → Остатки.
  4. Используйте фильтры по складам или поиск по артикулу.

Преимущества мобильного приложения:

  • 🔔 Уведомления о критических остатках (настраиваются в Профиль → Настройки → Уведомления);
  • 📱 Быстрый доступ к истории изменений остатков;
  • 📸 Возможность сканировать штрихкоды для поиска товара.

⚠️ Внимание: В мобильной версии нет функции массовой выгрузки. Для анализа больших объёмов данных используйте веб-версию или API.

💡

Мобильное приложение удобно для оперативного контроля, но не заменяет полноценный анализ через личный кабинет или API. Используйте его как дополнение к основным инструментам.

5. Типичные ошибки и как их избежать

Даже опытные продавцы сталкиваются с расхождениями в остатках. Вот самые распространённые проблемы и способы их решения:

🔹 Ошибка 1: Расхождение между вашей системой учёта и Ozon

  • 📌 Причина: Артикулы в 1С и на Ozon не совпадают.
  • 🛠 Решение: Проведите сверку артикулов через Товары → Импорт/Экспорт → Скачать прайс-лист.

🔹 Ошибка 2: Товар есть на складе, но не отображается в остатках

  • 📌 Причина: Товар находится в статусе «На приёмке» или «В карантине».
  • 🛠 Решение: Проверьте статус в разделе Склады → Приёмка.

🔹 Ошибка 3: Остатки обновляются с задержкой

  • 📌 Причина: Технические работы на складе Ozon или сбои API.
  • 🛠 Решение: Используйте эндпоинт /v1/info/warehouse для проверки статуса склада.

🔹 Ошибка 4: Резерв превышает реальные остатки

  • 📌 Причина: Ошибка в настройках резервирования или «зависшие» заказы.
  • 🛠 Решение: Запустите пересчёт резервов через поддержку Ozon.

Если проблема не решается — обратитесь в поддержку Ozon через раздел Помощь → Техническая поддержка. В сообщении укажите:

  • Артикулы проблемных товаров;
  • Скриншоты остатков из личного кабинета;
  • Дата и время, когда была замечена ошибка.

6. Автоматизация контроля остатков: инструменты и сервисы

Для продавцов с ассортиментом от 5 000 SKU ручной контроль остатков нереален. На помощь приходят специализированные сервисы и собственные решения на базе API.

🛠 Готовые сервисы для автоматизации:

  • 📊 МойСклад + интеграция с Ozon (подходит для FBO);
  • 🤖 Sellerboard — аналитика остатков и прогнозирование спроса;
  • 📈 Peak — мониторинг остатков в реальном времени с уведомлениями;
  • 🔄 1С:Управление торговлей с модулем для Ozon.

🛠 Самостоятельные решения:

  • 🐍 Скрипты на Python с использованием библиотеки requests для запросов к API;
  • 📂 Google Apps Script для автообновления Google Sheets;
  • 🖥 Power Automate (от Microsoft) для интеграции с Excel.

Пример автоматизации через Google Sheets:

  1. Создайте таблицу с колонками: Артикул, Склад, Остаток.
  2. Используйте функцию =IMPORTDATA для подгрузки данных из API.
  3. Настройте триггеры для автоматического обновления (например, раз в час).

⚠️ Внимание: При автоматизации учитывайте лимиты API Ozon. Если вы превысите количество запросов, аккаунт может быть временно заблокирован.

💡

Автоматизация контроля остатков снижает риск ошибок на 80% и экономит до 10 часов в неделю для среднего продавца.

FAQ: Частые вопросы о проверке остатков на Ozon

🔍 Как часто обновляются остатки в личном кабинете Ozon?

Данные в разделе Товары → Остатки обновляются каждые 3 часа. Для складов FBS в пиковые периоды (например, во время распродаж) задержка может увеличиться до 6 часов. Если нужны актуальные данные в реальном времени, используйте API.

📦 Почему в остатках отображается ноль, хотя товар есть на складе?

Это может происходить по нескольким причинам:

  • Товар находится в статусе «На приёмке» (ещё не обработан складом);
  • Товар заблокирован из-за нарушений (например, несоответствие описания);
  • Ошибка синхронизации между вашей системой учёта и Ozon.

Проверьте статус товара в разделе Склады → Приёмка или обратитесь в поддержку.

🔄 Можно ли вернуть товар со склада Ozon, если остатки не продаются?

Да, но есть нюансы:

  • Для FBS можно инициировать возврат через раздел Склады → Возвраты. Комиссия за возврат — от 50 рублей за позицию.
  • Для FBO вы управляете остатками самостоятельно — просто снизьте количество в личном кабинете.
  • Если товар лежит на складе Ozon более 180 дней, маркетплейс может списать его или вернуть за ваш счёт.

Подробнее о правилах возврата читайте в документации Ozon.

📊 Как экспортировать остатки по нескольким складам одновременно?

В личном кабинете:

  1. Перейдите в Товары → Остатки.
  2. В фильтре «Склад» выберите Все склады или отметьте нужные галочками.
  3. Нажмите Экспорт и выберите формат XLSX.

В полученном файле будет отдельная колонка для каждого склада с остатками.

⚠️ Что делать, если остатки в Ozon и моей 1С не сходятся?

Алгоритм действий:

  1. Проведите полную инвентаризацию через раздел Склады → Инвентаризация.
  2. Сверьте артикулы в вашей системе и на Ozon (возможно, где-то добавлен лишний символ).
  3. Проверьте логи интеграции — иногда ошибки возникают при автоматической выгрузке.
  4. Если расхождение критичное (более 10%), обратитесь в поддержку Ozon с детальным описанием проблемы.

Для профилактики настройте еженедельную сверку остатков.
