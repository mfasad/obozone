Скрипт для автоматической выгрузки продаж с Ozon в Google Таблицу: инструкция 2026

Автоматизация рутинных процессов — ключевой фактор успеха для продавцов на Ozon. Ежедневная выгрузка данных о заказах, статусах доставки и финансовых транзакциях вручную отнимает часы рабочего времени, увеличивает риск ошибок и мешает оперативно анализировать продажи. Решение проблемы лежит на поверхности: Google Apps Script позволяет создать скрипт, который будет автоматически pulling данные из личного кабинета Ozon Seller и записывать их в Google Таблицу по расписанию.

Но как это работает на практике? Многие продавцы ошибочно считают, что для такой автоматизации нужны глубокие знания программирования или дорогостоящие CRM-системы. На самом деле достаточно базовых навыков работы с JavaScript и понимания структуры API Ozon. В этой статье мы разберём весь процесс — от получения API-ключей до настройки триггеров для автоматического обновления данных. Вы узнаете, как выгружать не только основные параметры заказов, но и дополнительные метрики: время обработки, причины отмен, данные по возвратам и даже аналитику по категориям товаров.

Важно: скрипт, описанный ниже, подходит для продавцов на моделях FBS и FBO. Он не требует установки дополнительного ПО — всё работает в браузере через Google Таблицы. Если вы используете Ozon Global или другие международные платформы маркетплейса, потребуются дополнительные настройки API, которые мы также кратко рассмотрим.

Прежде чем приступить к технической части, давайте разберёмся, какие именно данные можно выгружать и зачем это нужно. Например, автоматизация выгрузки статусов заказов поможет оперативно реагировать на задержки логистики, а анализ причин отмен позволит снизить их количество. А если подключить к таблице Google Data Studio, вы получите интерактивную дашборд с визуализацией ключевых метрик в реальном времени.

1. Какие данные можно выгружать из Ozon в Google Таблицу

Скрипт для выгрузки продаж может работать с несколькими типами данных, доступными через API Ozon. Основные категории, которые чаще всего запрашивают продавцы:

  • 📦 Информация о заказах: номер заказа, дата создания, статус (awaiting_packaging, shipped, delivered и др.), сумма, количество товаров, данные покупателя (с учётом ограничений конфиденциальности).
  • 💰 Финансовые транзакции: суммы выплат, комиссии Ozon, удержанные налоги, даты зачислений на счёт. Эти данные помогают сверять выручку с бухгалтерскими отчётами.
  • 🚚 Логистические метрики: время обработки заказа, даты передачи курьеру, статус доставки, причины задержек. Особенно актуально для продавцов на FBS, где логистика управляется маркетплейсом.
  • 🔄 Возвраты и отмены: причины возвратов (несоответствие описанию, брак, передумал и т.д.), суммы возмещений, статус обработки возврата. Эти данные критичны для улучшения качества товаров и описаний.
  • 📊 Аналитика по товарам: какие позиции продаются лучше, средний чек по категориям, динамика продаж по дням/неделям. Это основа для принятия решений о закупках и маркетинговых акциях.

Пример структуры данных, которые можно выгрузить в таблицу:

2. Подготовка: регистрация приложения и получение API-ключей

Прежде чем писать скрипт, необходимо получить доступ к API Ozon. Для этого нужно зарегистрировать приложение в личном кабинете продавца. Вот пошаговая инструкция:

  1. Перейдите в личный кабинет Ozon Seller и авторизуйтесь.
  2. Откройте раздел Настройки → API-ключи.
  3. Нажмите кнопку «Создать ключ» и заполните форму:
    • 📌 Укажите название приложения (например, Выгрузка заказов в Google Sheets).
    • 🔗 В поле «Redirect URI» введите https://script.google.com/macros/d (это временный URL, его можно будет изменить позже).
    • ✅ Выберите необходимые разрешения (scope). Для выгрузки заказов достаточно галочек напротив orders и finance.
  • После сохранения вы получите два ключа: Client ID и Client Secret. Сохраните их в надёжном месте — они понадобятся для авторизации скрипта.

    • ⚠️ Внимание: Никогда не делитесь Client Secret с третьими лицами и не публикуйте его в открытых репозиториях (например, на GitHub). Это может привести к компрометации вашего аккаунта на Ozon. Если ключи случайно стали известны посторонним, немедленно отзовите их в личном кабинете и сгенерируйте новые.

    Также обратите внимание на ограничения API Ozon:

    • 🔄 Лимит запросов: до 60 запросов в минуту для большинства методов.
    • ⏳ Время жизни токена: 1 час (после истечения нужно повторно авторизоваться).
    • 📅 Данные о заказах доступны за последние 90 дней (для более ранних периодов потребуется архивная выгрузка).
    💡

    Если вы планируете выгружать данные чаще чем раз в час, настройте в скрипте автоматическое обновление токена доступа. Для этого используйте функцию refreshToken с сохранением нового токена в Properties Service Google Apps Script.

    3. Создание Google Таблицы и базовой структуры

    Теперь подготовим Google Таблицу, в которую будут записываться данные. Создайте новую таблицу и назовите её, например, Ozon Sales Data 2026. Рекомендуемая структура листов:

    • 📄 «Заказы» — основные данные о заказах (ID, дата, статус, сумма).
    • 📄 «Товары» — детализация по товарам в заказах (артикул, количество, цена).
    • 📄 «Финансы» — транзакции, комиссии, выплаты.
    • 📄 «Возвраты» — информация об отменах и возвратах.

    Пример заголовков для листа «Заказы»:

    A1 B1 C1 D1 E1 F1
    order_id order_date status total_amount items_count shipping_date

    Чтобы избежать ошибок при записи данных, зафиксируйте первую строку (заголовки) и настройте форматирование:

    1. Выделите первую строку → Вид → Заморозить → 1 строка.
    2. Для столбцов с датами (order_date, shipping_date) установите формат Дата.
    3. Для столбцов с денежными суммами (total_amount) выберите формат Финансовый.

    ⚠️ Внимание: Если в таблице уже есть данные, скрипт по умолчанию будет дописывать новые строки в конец. Чтобы избежать дублирования, добавьте в скрипт проверку на уникальность order_id перед записью. Альтернативный вариант — создавать новый лист для каждого дня (например, Заказы_2026-05-20).

    4. Написание скрипта для выгрузки данных

    Переходим к самому техническому этапу — созданию скрипта. Откройте вашу Google Таблицу, затем выберите Расширения → Apps Script. Откроется редактор кода, где мы будем писать скрипт.

    Ниже приведён базовый код для авторизации и выгрузки заказов. Обратите внимание: этот код использует OAuth 2.0 для аутентификации, поэтому потребуется одноразовая авторизация через браузер. 

    

    // Конфигурация API Ozon
    

    const CLIENT_ID ='ВАШ_CLIENT_ID'; // Замените на ваш Client ID
    

    const CLIENT_SECRET ='ВАШ_CLIENT_SECRET'; // Замените на ваш Client Secret
    

    const REDIRECT_URI ='https://script.google.com/macros/d'; // Можно оставить без изменений
    


    // ID вашей Google Таблицы (возьмите из URL таблицы)
    

    const SPREADSHEET_ID ='ВАШ_SPREADSHEET_ID';
    


    // Функция для получения токена доступа
    

    function getOzonToken {
    

    const service = getOzonService;
    

    if (!service.hasAccess) {
    

    const authorizationUrl = service.getAuthorizationUrl;
    

    Logger.log('Откройте эту ссылку для авторизации:' + authorizationUrl);
    

    throw new Error('Требуется авторизация. Ссылка в логах.');
    

    }
    

    return service.getAccessToken;
    

    }
    


    // Функция для выгрузки заказов
    

    function fetchOrders {
    

    const token = getOzonToken;
    

    const url ='https://api-seller.ozon.ru/v3/orders/list';
    


    const headers = {
    

    'Client-Id': CLIENT_ID,
    

    'Api-Key': token,
    

    'Content-Type':'application/json'
    

    };
    


    const params = {
    

    'method':'POST',
    

    'headers': headers,
    

    'payload': JSON.stringify({
    

    "dir":"ASC",
    

    "filter": {
    

    "since":"2026-05-01T00:00:00Z", // Дата начала периода
    

    "to":"2026-05-20T23:59:59Z" // Дата конца периода
    

    },
    

    "limit": 1000,
    

    "offset": 0,
    

    "translit": true,
    

    "with": {
    

    "analytics_data": true,
    

    "financial_data": true
    

    }
    

    })
    

    };
    


    const response = UrlFetchApp.fetch(url, params);
    

    const data = JSON.parse(response.getContentText);
    

    return data.result.orders;
    

    }
    


    // Функция для записи данных в таблицу
    

    function writeOrdersToSheet(orders) {
    

    const sheet = SpreadsheetApp.openById(SPREADSHEET_ID).getSheetByName('Заказы');
    

    const rows = orders.map(order => [
    

    order.order_id,
    

    new Date(order.created_at),
    

    order.status,
    

    order.total_amount,
    

    order.products.length,
    

    order.shipping_date? new Date(order.shipping_date): null
    

    ]);
    


    // Дописываем данные в конец таблицы
    

    sheet.getRange(sheet.getLastRow + 1, 1, rows.length, rows[0].length).setValues(rows);
    

    }
    


    // Основная функция для запуска
    

    function updateOzonOrders {
    

    const orders = fetchOrders;
    

    writeOrdersToSheet(orders);
    

    Logger.log(`Выгружено ${orders.length} заказов.`);
    

    }
    


    // Служебная функция для OAuth
    

    function getOzonService {
    

    return OAuth2.createService('Ozon')
    

    .setAuthorizationBaseUrl('https://id.ozon.ru/oauth/authorize')
    

    .setTokenUrl('https://id.ozon.ru/oauth/token')
    

    .setClientId(CLIENT_ID)
    

    .setClientSecret(CLIENT_SECRET)
    

    .setScope('orders')
    

    .setRedirectUri(REDIRECT_URI)
    

    .setPropertyStore(PropertiesService.getUserProperties);
    

    }

    Этот скрипт выполняет следующие действия:

    1. Авторизуется в API Ozon через OAuth 2.0.
    2. Запрашивает список заказов за указанный период (в примере — с 1 по 20 мая 2026 года).
    3. Преобразует данные в удобный формат и записывает их в лист «Заказы».

    ⚠️ Внимание: При первом запуске скрипт запросит авторизацию. Следуйте инструкциям в логах (откройте ссылку для авторизации и разрешите доступ). После успешной авторизации токен сохранится, и повторный ввод данных не потребуется (до истечения срока действия токена).

    Как изменить период выгрузки?

    Чтобы выгрузить данные за другой период, измените параметры since и to в функции fetchOrders. Формат даты должен соответствовать YYYY-MM-DDTHH:MM:SSZ (например, 2026-01-01T00:00:00Z для 1 января 2026 года).

    6. Расширенные возможности: аналитика и интеграции

    Базовая выгрузка данных — только первый шаг. Чтобы превратить сырые данные в полезную аналитику, можно расширить функционал скрипта:

    • 📈 Автоматическое построение графиков: Используйте =QUERY или Google Data Studio для визуализации динамики продаж, среднего чека, конверсии.
    • 🔔 Уведомления о проблемах: Настройте отправку email или сообщений в Telegram, если статус заказа долго не меняется (например, более 24 часов в статусе awaiting_packaging).
    • 🔄 Синхронизация с 1С или CRM: Экспортируйте данные из Google Таблицы в другие системы через API или CSV-выгрузку.
    • 🛒 Анализ ассортимента: Добавьте в скрипт расчёт ABC-анализа для определения самых и наименее продаваемых товаров.

    Пример кода для отправки уведомления в Telegram при задержке обработки заказа: 

    

    function checkDelayedOrders {
    

    const sheet = SpreadsheetApp.openById(SPREADSHEET_ID).getSheetByName('Заказы');
    

    const data = sheet.getDataRange.getValues;
    

    const now = new Date;
    

    const delayedOrders =;
    


    // Проверяем заказы в статусе awaiting_packaging старше 24 часов
    

    for (let i = 1; i < data.length; i++) {
    

    if (data[i][2] ==='awaiting_packaging') {
    

    const orderDate = new Date(data[i][1]);
    

    const hoursDiff = (now - orderDate) / (1000  60  60);
    

    if (hoursDiff > 24) {
    

    delayedOrders.push({
    

    order_id: data[i][0],
    

    hours: Math.round(hoursDiff)
    

    });
    

    }
    

    }
    

    }
    


    if (delayedOrders.length > 0) {
    

    const message = `⚠️ Задержка обработки заказов:\n${delayedOrders.map(o => `#${o.order_id}: ${o.hours} часов`).join('\n')}`;
    

    sendTelegramMessage(message); // Функция для отправки в Telegram
    

    }
    

    }

    Для интеграции с Telegram потребуется создать бота через @BotFather и получить chat_id и token. Эти данные используются в функции sendTelegramMessage, которую можно найти в документации Telegram API.

    7. Типичные ошибки и их решение

    При работе со скриптом вы можете столкнуться с следующими проблемами:

    Ошибка Причина Решение
    Invalid Client ID or Secret Неверные учётные данные API. Проверьте CLIENT_ID и CLIENT_SECRET в скрипте. При необходимости сгенерируйте новые ключи в ЛК Ozon.
    Token expired Истёк срок действия токена (1 час). Добавьте в скрипт автоматическое обновление токена (см. раздел 4).
    Exceeded rate limit Превышен лимит запросов (60 в минуту). Уменьшите частоту запуска скрипта или разбейте выгрузку на части с задержкой Utilities.sleep(1000).
    Sheet not found Лист с указанным именем не найден. Проверьте название листа в таблице и в скрипте (регистр важен!).

    ⚠️ Внимание: Если скрипт перестал работать после обновления API Ozon, проверьте актуальность используемых эндпоинтов (url) и структуру ответа. Маркетплейс периодически вносит изменения в API, что может приводить к ошибкам. Следите за обновлениями в документации Ozon API.

    8. Альтернативные способы выгрузки данных

    Если работа с Google Apps Script кажется сложной, рассмотрите альтернативные варианты автоматизации:

    • 🔗 Сервисы-коннекторы: Zapier, Make (ex-Integromat), Albato предлагают готовые интеграции между Ozon и Google Sheets без программирования. Минус — платные тарифы при большом объёме данных.
    • 📥 Экспорт в CSV: В личном кабинете Ozon Seller можно вручную выгружать отчёты в CSV/Excel, а затем импортировать их в Google Таблицы через Файл → Импорт.
    • 🖥️ Локальные скрипты на Python: Если вы знакомы с Python, можно использовать библиотеку requests для работы с API Ozon и gspread для записи в Google Sheets.
    • 📊 CRM-системы: Сервисы вроде МойСклад, Битрикс24 или RetailCRM имеют готовые интеграции с Ozon и могут автоматически синхронизировать данные.

    FAQ: Частые вопросы о выгрузке данных с Ozon

    ❓ Как выгрузить данные за период больше 90 дней?

    Стандартное API Ozon предоставляет данные только за последние 90 дней. Для выгрузки более ранних заказов:

    1. Используйте ручной экспорт через личный кабинет (Аналитика → Отчёты → История заказов).
    2. Обратитесь в поддержку Ozon с запросом на архивную выгрузку (платно, от 1000 ₽ за запрос).
    3. Если у вас сохранены старые CSV-файлы, импортируйте их в Google Таблицу через Файл → Импорт.

    Для автоматизации архивной выгрузки можно настроить ежемесячное сохранение данных в отдельные листы таблицы.

    ❓ Можно ли выгрузить персональные данные покупателей (ФИО, телефон, адрес)?

    Нет, API Ozon не предоставляет персональные данные покупателей из-за политики конфиденциальности. Доступны только:

    • ID заказа;
    • Статус;
    • Сумма;
    • Дата создания;
    • Город доставки (без точного адреса).
