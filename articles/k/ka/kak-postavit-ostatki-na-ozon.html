Как поставить остатки на Ozon: все способы для продавцов в 2026 году

Выставление остатков на Ozon — критически важный процесс для любого продавца, который напрямую влияет на видимость товаров, скорость продаж и рейтинг в маркетплейсе. Неправильно указанные остатки ведут к автоотменам заказов, штрафам за недопоставку и потере доверия покупателей. В 2026 году Ozon ужесточил требования к актуализации складских данных: теперь за просроченные остатки (более 24 часов) продавцы получают пеню 1% от стоимости неотгруженного товара.

В этом руководстве мы разберём все актуальные способы обновления остатков — от ручного ввода в личном кабинете до автоматизации через API и Excel-шаблоны. Особое внимание уделим нюансам для схем FBS (склад Ozon) и FBO (самостоятельная логистика), а также типичным ошибкам, из-за которых товары «проваливаются» в поиске. Если вы только начинаете работать с маркетплейсом или хотите оптимизировать процесс, эта статья поможет избежать потерь продаж из-за технических накладок в остатках.

1. Способы обновления остатков на Ozon в 2026 году

На сегодняшний день Ozon поддерживает 5 официальных способов актуализации остатков. Выбор метода зависит от объёма ассортимента, технической оснащённости и схемы работы (FBS/FBO). Вот краткий обзор:

  • 📝 Ручной ввод в личном кабинете — подходит для небольших каталогов (до 50 SKU).
  • 📊 Загрузка через Excel — оптимально для 50–5 000 товаров.
  • 🤖 API-интеграция — для крупных продавцов с автоматизированными системами учёта.
  • 🔄 Синхронизация с 1С/МойСклад — через готовые коннекторы.
  • 📦 Автообновление при приёмке на склад FBS — только для товаров на складах Ozon.

Каждый метод имеет ограничения и особенности. Например, ручной ввод позволяет обновлять остатки с точностью до единицы, но занимает много времени, а API требует навыков программирования или услуг разработчика. В следующих разделах мы подробно разберём каждый вариант.

2. Ручной ввод остатков в личном кабинете Ozon

Это самый простой, но и самый трудоёмкий способ. Он подходит для продавцов с минимальным ассортиментом (до 50 позиций) или для экстренных правок. Чтобы обновить остатки вручную:

  1. Авторизуйтесь в личном кабинете продавца.
  2. Перейдите в раздел Товары → Список товаров.
  3. Найдите нужный товар через поиск или фильтры.
  4. Нажмите на Редактировать (иконка карандаша).
  5. В поле Остаток введите актуальное количество и сохраните изменения.

⚠️ Внимание: При ручном вводе остатки обновляются не мгновенно, а в течение 10–30 минут. Если вы продаёте по схеме FBO, учитывайте время на доставку товара до покупателя — Ozon блокирует продажу позиции, если остаток равен нулю, но заказ уже оплачен.

Проверять остатки перед пиковыми продажами (черная пятница, распродажи)

Главный минус ручного метода — высокий риск ошибок. Например, продавец может забыть обновить остаток после отгрузки или указать неверное количество. Для минимизации рисков рекомендуем:

  • 📅 Вести Google Таблицу с актуальными остатками и сверять её с данными в ЛК раз в день.
  • 🔔 Настраивать уведомления о низком остатке (в Настройках → Уведомления).
  • 🔍 Использовать фильтр Остаток ≤ 3 для оперативного контроля.

3. Загрузка остатков через Excel: пошаговая инструкция

Для продавцов с ассортиментом от 50 до 5 000 позиций оптимальным решением станет загрузка остатков через Excel-файл. Этот метод позволяет обновлять данные пакетно, экономя время. Вот как это работает:

Шаг 1. Скачайте шаблон:

  • Перейдите в Товары → Импорт/Экспорт → Шаблоны.
  • Выберите шаблон Остатки товаров и скачайте его.

Шаг 2. Заполните файл:

  • В колонке SKU укажите артикулы товаров (обязательно!).
  • В колонке Остаток введите актуальное количество.
  • Для FBS добавьте колонку Склад (указывайте код склада Ozon, например, MSK-1).

Шаг 3. Загрузите файл:

  • Вернитесь в Импорт/Экспорт → Импорт остатков.
  • Выберите заполненный файл и нажмите Загрузить.
  • Дождитесь обработки (обычно занимает 5–15 минут).
Что делать, если файл не загружается?

Наиболее частые ошибки при загрузке Excel:

- Неправильный формат SKU (проверьте регистр и символы).

- Отсутствует колонка "Склад" для FBS (обязательна с 2026 года).

- Превышен лимит строк (максимум 50 000 строк на файл).

- Несоответствие типов данных (остаток должен быть целым числом).

Если ошибка повторяется, скачайте список кодов ошибок Ozon и сверьте с вашим случаем.

⚠️ Внимание: С 1 марта 2026 года Ozon ввёл новое правило: если остаток товара не обновлялся более 24 часов, система автоматически снижает его видимость в поиске на 30%. Это означает, что даже при наличии товара на складе он может не показываться покупателям!

Тип ошибки Причина Решение

4. Автоматизация через API: для крупных продавцов

Если ваш ассортимент превышает 5 000 SKU или вы используете собственную систему учёта (например, 1С), ручное обновление остатков станет неэффективным. В этом случае API Ozon — оптимальное решение. Оно позволяет:

  • 🔄 Синхронизировать остатки в реальном времени.
  • 📡 Получать уведомления о изменениях статуса товаров.
  • 🤖 Автоматически блокировать продажу при нулевом остатке.

Для подключения API:

  1. Получите Client-ID и API-ключ в разделе Настройки → API.
  2. Используйте метод POST /v2/products/stocks для обновления остатков.
  3. Настройте автоматическую отправку данных из вашей ERP-системы.

Пример запроса для обновления остатка:

curl -X POST 'https://api-seller.ozon.ru/v2/products/stocks'

-H 'Client-Id: {ваш_client_id}'


-H 'Api-Key: {ваш_api_key}'


-H 'Content-Type: application/json'


-d '{


"stocks": [


{


"offer_id": "ваш_SKU_1",


"stock": 10,


"warehouse_id": 1000001  // ID склада (для FBS)


}


]


}'

💡

Если вы не программист, используйте готовые сервисы для работы с API Ozon, например, Alto, МойСклад или RetailCRM. Они предлагают визуальные конструкторы для настройки интеграции без кода.

⚠️ Внимание: При работе с API учитывайте лимиты запросов:

- 100 запросов в минуту для метода обновления остатков.

- 5 000 SKU в одном запросе (при превышении разбейте на партии).

Если лимиты превышены, Ozon вернёт ошибку 429 Too Many Requests.

5. Особенности обновления остатков для FBS и FBO

Схема работы (FBS или FBO) напрямую влияет на процесс управления остатками. Рассмотрим ключевые различия:

📦 FBS (склад Ozon)

  • 🔄 Остатки обновляются автоматически при приёмке товара на склад.
  • 📍 Необходимо указывать warehouse_id (код склада) при загрузке.
  • ⚡ Время обработки изменений: 1–2 часа (из-за логистических процессов).

🚚 FBO (самостоятельная логистика)

  • ⏱ Остатки обновляются мгновенно после загрузки.
  • 📦 Ответственность за актуальность данных лежит на продавце.
  • ⚠️ При несовпадении фактического и указанного остатка — штраф 500 ₽ за заказ.

Для FBS критично правильно указывать складской код. Например, если товар физически находится на складе MSK-1, но вы укажете SPB-2, система не сможет резервировать его для заказов. Список актуальных складов можно скачать в разделе Логистика → Склады.

💡

Для FBS остатки обновляются в два этапа: сначала в системе Ozon (через ЛК/API), затем — при фактической приёмке на складе. Всегда сверяйте данные с отчётом Остатки на складах FBS!

6. Типичные ошибки и как их избежать

Даже опытные продавцы сталкиваются с проблемами при обновлении остатков. Вот TOP-5 ошибок и способы их решения:

  1. Остатки не обновляются:

    - Причина: Неправильный формат файла или ошибки в SKU.

    - Решение: Проверьте шаблон на соответствие требованиям Ozon.

  2. Товар исчез из поиска:

    - Причина: Остаток не обновлялся более 24 часов.

    - Решение: Настройте автоматическое обновление через API или 1С.

  3. Автоотмены заказов:

    - Причина: Фактический остаток меньше указанного.

    - Решение: Ведите учёт в реальном времени (например, через МойСклад).

  4. Ошибка "Склад не найден":

    - Причина: Неверный warehouse_id для FBS.

    - Решение: Уточните код склада в разделе Логистика.

  5. Зависание загрузки файла:

    - Причина: Большой объём данных (>50 000 строк).

    - Решение: Разбейте файл на части по 10 000 строк.

Ещё одна распространённая проблема — расхождение остатков между вашей системой учёта и Ozon. Чтобы этого избежать:

  • 🔄 Синхронизируйте данные не реже 1 раза в час в пиковые периоды.
  • 📊 Используйте отчёт История изменений остатков для аудита.
  • 🤖 Настройте вебхуки на события изменения остатков (доступно через API).

7. Как проверить, что остатки обновлены корректно

После загрузки остатков обязательно проверьте, что изменения применились. Вот 3 способа контроля:

  1. Ручная проверка в ЛК:

    - Перейдите в Товары → Список товаров.

    - Отфильтруйте по Дате изменения остатка.

  2. Отчёт "Остатки":

    - Скачайте его в Аналитика → Отчёты → Остатки товаров.

    - Сравните с вашими данными.

  3. API-запрос:

    - Используйте метод GET /v1/product/info/stocks для проверки текущих остатков.

⚠️ Внимание: Если вы работаете по FBS, учитывайте, что остатки в отчёте могут отличаться от фактических на 1–2 часа из-за времени обработки на складе. Для точных данных используйте отчёт Остатки на складах FBS (детализированный).

Для удобства рекомендуем создать Google Таблицу с формулами сравнения остатков из вашей системы и Ozon. Пример формулы для автоматического выделения расхождений:

=IF(A2<>B2, "РАСХОЖДЕНИЕ", "OK")

где A2 — ваш остаток, B2 — остаток из отчёта Ozon.

FAQ: Частые вопросы об остатках на Ozon

🔹 Можно ли обновить остатки ночью, чтобы не блокировать продажи?

Да, но учитывайте, что пиковое время обработки запросов на серверах Ozon — с 00:00 до 03:00 по МСК. В этот период обновление может занять до 1 часа. Для критичных товаров лучше выбирать время с 04:00 до 06:00.

🔹 Что будет, если не обновлять остатки более 3 дней?

Согласно правилам Ozon, при отсутствии обновлений более 72 часов:

  • Товар исключается из поиска и рекомендаций.
  • Начисляется пеня 1% от стоимости неотгруженных товаров в день.
  • При повторных нарушениях — блокировка возможности участия в акциях.

🔹 Как обновлять остатки для товаров с вариациями (размеры, цвета)?

Для товаров с вариациями (parent-child) необходимо:

  1. Указывать остатки для каждой вариации отдельно (по SKU дочернего товара).
  2. В Excel-файле заполнять колонку Offer ID (это SKU вариации).
  3. Для FBS добавлять колонку Warehouse ID для каждой вариации.

Пример структуры файла:

Parent SKU | Offer ID (SKU вариации) | Остаток | Склад

PROD-001   | PROD-001-BLACK-S       | 5       | MSK-1


PROD-001   | PROD-001-BLACK-M       | 3       | MSK-1

🔹 Почему после обновления остатки сбросились на ноль?

Это типичная ошибка при:

  • Загрузке пустого файла (проверьте, что в колонке Остаток указаны значения).
  • Несоответствии SKU в файле и системе (даже лишний пробел сбрасывает остаток).
  • Технических работах на стороне Ozon (проверьте статус сервисов здесь).

Чтобы восстановить остатки, загрузите файл повторно с корректными данными.

🔹 Нужно ли обновлять остатки для товаров в статусе "На модерации"?

Нет, остатки для товаров на модерации обновлять нельзя. Система игнорирует такие запросы. Дождитесь одобрения карточки товара (статус Активен), после чего обновите остатки.
