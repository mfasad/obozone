Как передать штрихкод Озона: все способы для FBS и FBO

Передача штрихкодов на Ozon — обязательный этап для продавцов, работающих по моделям FBS (склад продавца) и FBO (склад Озона). Без корректного штрихкодирования товар не пройдет приемку, не попадет на витрину или будет возвращен с логистического хаба. В этой статье разберем все актуальные способы передачи штрихкодов — от ручного ввода в личном кабинете до автоматической загрузки через API, а также типичные ошибки, из-за которых товары «зависают» на проверке.

Особенность работы с Ozon в том, что маркетплейс требует уникальные штрихкоды для каждого SKU (артикула). Если вы продаете товар в нескольких вариациях (цвет, размер), каждому варианту нужен отдельный код. При этом сам штрихкод может быть как производительским (если он уже нанесен на упаковку), так и маркетплейсовским (сгенерированным Озоном). Разберемся, какой вариант выбрать и как избежать дублирования.

1. Виды штрихкодов на Озоне: какой передавать?

На Ozon используются три типа штрихкодов:

  • 📦 Производительский (EAN-13, UPC, ISBN) — уже нанесен на товар заводом-изготовителем. Подходит, если код уникален и не используется другими продавцами на площадке.
  • 🏷️ Маркетплейсовский (OZON) — генерируется автоматически в личном кабинете. Используется, если у товара нет заводского кода или он дублируется.
  • 🔢 Внутренний (SKU) — присваивается продавцом для учета. На Озоне не используется как основной, но может потребоваться для интеграции с 1С или МойСклад.

Какой выбрать? Ozon рекомендует приоритетно использовать производительские коды, если они:

  • 🔹 Соответствуют стандарту EAN-13 (13 цифр) или UPC (12 цифр).
  • 🔹 Не дублируются в карточках других продавцов (проверить можно через Каталог → Поиск по штрихкоду).
  • 🔹 Нанесены типографским способом (не рукописные или наклейки).

Если хотя бы одно условие не выполняется — генерируйте маркетплейсовский код. Например, для товаров handmade, расходных материалов без заводской маркировки или продуктов с истекающим сроком годности (где штрихкод может повторяться у разных партий).

2. Как передать штрихкод через личный кабинет Озона

Самый простой способ — ручной ввод в интерфейсе Seller Lab. Подходит для небольшого ассортимента (до 50 SKU). Инструкция:

  1. Перейдите в раздел Товары → Мои товары.
  2. Найдите нужный товар и нажмите на его название.
  3. В блоке Штрихкоды кликните Добавить штрихкод.
  4. Выберите тип:
    • 📌 Производительский — введите код вручную (например, 4607001234567).
    • 📌 OZON — система сгенерирует уникальный код автоматически.
  • Сохраните изменения кнопкой Применить.

    • После добавления штрихкод появится в списке через 5–15 минут. Чтобы ускорить обработку, используйте кнопку Обновить статус в верхнем меню.

    ☑️ Проверка перед сохранением штрихкода

    Выполнено: 0 / 4

    Важно: если товар уже находится на складе FBO, изменять штрихкод можно только через поддержку. В противном случае товар будет возвращен как «неопознанный».

    3. Массовая загрузка штрихкодов через Excel

    Для ассортимента от 50 позиций удобнее использовать шаблон Excel. Скачать его можно в разделе Товары → Импорт/Экспорт → Шаблоны для загрузки (выберите Штрихкоды).

    Как заполнить файл:

    1. Столбец A — SKU (артикул вашего товара).
    2. Столбец B — Тип штрихкода (EAN, UPC или OZON).
    3. Столбец C — сам Штрихкод (для типа OZON оставьте пустым — система сгенерирует его автоматически).

    Пример корректной строки:

    SKUТип штрихкодаШтрихкод
    TV-001-BLACKEAN4607001234567
    TV-001-WHITEOZON
    CABLE-5MUPC012345678912

    После заполнения:

    1. Сохраните файл в формате .xlsx (не .csv!).
    2. Вернитесь в Импорт/Экспорт → Загрузить штрихкоды.
    3. Прикрепите файл и дождитесь обработки (до 1 часа).
    💡

    Если при загрузке появляется ошибка "Некорректный формат", проверьте, чтобы в ячейках не было скрытых символов (например, пробелов после цифр). Используйте функцию ТРИМ в Excel для очистки данных.

    4. Автоматическая передача штрихкодов через API

    Для крупных продавцов (1000+ SKU) оптимально использовать API Озона. Это позволяет интегрировать передачу штрихкодов с 1С, МойСклад или другими системами учета.

    Основной метод для работы со штрихкодами — POST /v2/product/info/barcode. Пример запроса:

    {

    "items": [
    

    {
    

    "offer_id": "TV-001-BLACK",
    

    "barcode": "4607001234567",
    

    "barcode_type": "EAN_13"
    

    },
    

    {
    

    "offer_id": "CABLE-5M",
    

    "barcode_type": "OZON"
    

    }
    

    ]
    

    }

    Параметры:

    • 📌 offer_id — ваш SKU.
    • 📌 barcode — штрихкод (не указывайте для типа OZON).
    • 📌 barcode_type — возможные значения:
      • EAN_13, EAN_8, UPC — для производительских кодов.
      • OZON — для генерации маркетплейсовского кода.

    Ответ от API будет содержать сгенерированные коды для типа OZON:

    {

    "result": {
    

    "items": [
    

    {
    

    "offer_id": "CABLE-5M",
    

    "barcode": "2000000123456",  // сгенерированный Ozon
    

    "barcode_type": "OZON"
    

    }
    

    ]
    

    }
    

    }

    Для работы с API потребуется:

    1. Получить Client-ID и API-Key в настройках личного кабинета (Интеграции → API).
    2. Использовать OAuth 2.0 для авторизации.
    3. Настроить обработку ошибок (например, код 400 — неверный формат штрихкода).
    Пример кода на Python для массовой загрузки штрихкодов

    Импортируйте библиотеку requests и используйте следующий скрипт: 

    import requests


    url = "https://api-seller.ozon.ru/v2/product/info/barcode"
    

    headers = {
    

    "Client-Id": "ваш_client_id",
    

    "Api-Key": "ваш_api_key",
    

    "Content-Type": "application/json"
    

    }
    

    data = {
    

    "items": [
    

    {"offer_id": "SKU001", "barcode_type": "EAN_13", "barcode": "1234567890123"},
    

    {"offer_id": "SKU002", "barcode_type": "OZON"}
    

    ]
    

    }
    


    response = requests.post(url, headers=headers, json=data)
    

    print(response.json())

    5. Передача штрихкодов через мобильное приложение Озон Селлер

    Если вам нужно оперативно добавить или изменить штрихкод в дороге, используйте мобильное приложение Озон Селлер (доступно для iOS и Android). Алгоритм:

    1. Откройте раздел Товары и найдите нужный SKU.
    2. Тапните на товар и выберите Редактировать.
    3. Прокрутите до блока Штрихкоды и нажмите Добавить.
    4. Отсканируйте существующий код камерой или введите вручную.
    5. Если кода нет — выберите Сгенерировать код OZON.

    Ограничения мобильной версии:

    • ❌ Нельзя массово загружать штрихкоды (только по одному).
    • ❌ Нет возможности экспортировать список штрихкодов в файл.
    • ❌ При плохом интернете коды могут не сохраниться (всегда проверяйте статус синхронизации).

    Преимущество — мгновенная генерация кодов типа OZON без ожидания обработки, как в веб-версии. Это удобно, если товар уже на складе FBO и требуется срочно подтвердить штрихкод для приемки.

    6. Типичные ошибки при передаче штрихкодов и как их исправить

    Даже опытные продавцы сталкиваются с проблемами при работе со штрихкодами. Разберем самые частые:

    ОшибкаПричинаРешение
    «Штрихкод уже используется» Код дублируется в каталоге Озона (другой продавец указал его для своего товара). Сгенерируйте код типа OZON или используйте другой производительский код.
    «Некорректный формат» Код содержит буквы, пробелы или неверное количество цифр. Проверьте формат: EAN-13 — 13 цифр, UPC — 12 цифр.
    «Товар не найден» SKU в файле не совпадает с артикулом в личном кабинете. Сверьте SKU в шаблоне Excel с разделом Мои товары.
    «Код не считывается» (на складе FBO) Низкое качество печати, поврежденная наклейка или неверный размер штрихкода. Распечатайте код на принтере с разрешением ≥300 dpi, размер — не менее 3×2 см.

    Особое внимание уделите товарам с вариациями (например, футболки разных размеров). Частая ошибка — указание одного штрихкода для всех вариантов. Ozon требует уникальные коды для каждого SKU, даже если они отличаются только цветом.

    💡

    Если товар уже на складе FBO и вы обнаружили ошибку в штрихкоде, НЕ редактируйте его самостоятельно. Обратитесь в поддержку через раздел Помощь → Написать в службу поддержки с темой «Исправление штрихкода для товара на складе».

    7. Как проверить, что штрихкод успешно передан

    После загрузки штрихкодов обязательно убедитесь, что они корректно отобразились в системе. Способы проверки:

    1. В личном кабинете:
      • Перейдите в Товары → Мои товары.
      • Откройте карточку товара и проверьте блок Штрихкоды.
      • Статус должен быть Активен (не На проверке или Ошибка).
    2. Через API:
      • Используйте метод GET /v1/product/info/barcode с параметром offer_id.
      • В ответе проверьте поле "status": "OK".
    3. На складе FBO:
      • Если товар уже отправлен, запросите у менеджера склада Акт приемки.
      • В акте должна быть пометка Штрихкод совпадает.

    Если штрихкод «завис» на статусе На проверке дольше 24 часов:

    1. Проверьте, не дублируется ли он в каталоге (через Каталог → Поиск по штрихкоду).
    2. Убедитесь, что SKU товара не изменялся после загрузки кода.
    3. Напишите в поддержку с указанием SKU и штрихкода.
    💡

    Для ускорения проверки штрихкодов используйте кнопку Обновить статус в верхнем меню раздела Мои товары. Это принудительно инициирует повторную обработку данных.

    8. Частые вопросы о штрихкодах на Озоне

    Можно ли использовать один штрихкод для нескольких товаров?

    Нет, Ozon требует уникальный штрихкод для каждого SKU. Исключение — товары, продаваемые в наборе (например, комплект из 3 пар носков), где штрихкод присваивается всему набору, а не каждому элементу.

    Что делать, если производительский штрихкод уже используется другим продавцом?

    В этом случае вы не сможете использовать его для своего товара. Варианты решений:

    1. Сгенерируйте код типа OZON.
    2. Если вы официальный дистрибьютор бренда, обратитесь в поддержку с подтверждающими документами — они могут заблокировать код для других продавцов.

    Нужно ли печатать штрихкод на упаковке, если товар хранится на FBS?

    Для модели FBS (склад продавца) печать штрихкода на упаковке не обязательна, но рекомендуется. Это ускорит обработку товаров при возможной отправке на FBO или возвратах. Для FBO штрихкод должен быть напечатан и наклеен на каждую единицу товара.

    Как изменить штрихкод, если товар уже на складе Ozon?

    Самостоятельно редактировать штрихкод для товаров на складе FBO нельзя. Необходимо:

    1. Написать в поддержку с темой «Запрос на изменение штрихкода».
    2. Указать SKU, текущий и новый штрихкод.
    3. Приложить фото товара с новым кодом (если он уже напечатан).

    Обработка запроса занимает до 3 рабочих дней.

    Можно ли использовать штрихкоды от Wildberries или Яндекс Маркета на Озоне?

    Нет, штрихкоды других маркетплейсов (WB, Яндекс Маркет) не подходят. Ozon требует уникальные коды, не дублирующиеся в его каталоге. Если вы продаете один и тот же товар на нескольких площадках, для Озона нужно сгенерировать отдельный код типа OZON.

    Если ваш вопрос не освещен в статье, изучите официальную документацию Озона или обратитесь в службу поддержки через личный кабинет. Для ускорения ответа укажите в запросе SKU товара и скриншот ошибки (если она есть).
