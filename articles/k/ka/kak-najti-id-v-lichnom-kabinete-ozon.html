Где и как найти ID в личном кабинете Озон: инструкция для продавцов и покупателей

Работа с маркетплейсом Ozon требует постоянного взаимодействия с различными идентификаторами: ID продавца, ID заказа, артикул товара или даже ID покупателя в спорных ситуациях. Эти цифровые коды — как паспорта в системе: без них невозможно отследить заказ, решить конфликт с поддержкой или интегрировать API.Problem в том, что интерфейс личного кабинета (ЛК) Озон регулярно обновляется, и кнопка «Показать ID» может внезапно «переехать» в другой раздел.

В этой статье разберём все актуальные способы найти ID в личном кабинете Ozon в 2026 году — от базовых (для новичков) до продвинутых (для разработчиков и владельцев больших магазинов). Особое внимание уделим различиям между Ozon Seller (кабинет продавца) и Ozon для покупателей, а также типичным ошибкам, из-за которых пользователи теряют часы на поиски простого кода. Если вы ищете конкретный идентификатор — используйте оглавление ниже, чтобы сразу перейти к нужному разделу.

1. Что такое ID в Озон и зачем он нужен

ID (от англ. identifier — «идентификатор») — это уникальный цифровой или буквенно-цифровой код, который система Озон присваивает каждому объекту: пользователю, заказу, товару или даже отзыву. Например:

  • 🔢 ID продавца — ваш «паспорт» в маркетплейсе (нужен для API, регистрации в партнёрских программах, обращений в поддержку).
  • 📦 ID заказа — уникальный номер транзакции (требуется для отслеживания доставки, возвратов, споров).
  • 🏷️ Артикул (SKU) — внутренний код товара (используется в прайс-листах, остатках, аналитике).
  • 👤 ID покупателя — редко востребован, но может понадобиться при блокировках аккаунта или массовых возвратах.

Без этих кодов невозможно:

  • 🔄 Интегрировать магазин с 1С, MoySklad или другими CRM через API.
  • 📊 Получать детализированную аналитику по продажам (например, в Ozon Statistics).
  • 🛠️ Решать споры с покупателями через поддержку (они всегда просят номер заказа).
  • 🚀 Участвовать в акциях или программах лояльности (например, Ozon Premium).
⚠️ Внимание: Никогда не передавайте свой ID продавца третьим лицам (например, «помощникам» из Telegram), если не уверены в их надёжности. Мошенники могут использовать его для кражи данных или блокировки аккаунта.

2. Где найти ID продавца в личном кабинете Ozon Seller

ID продавца (он же Seller ID) — это ваш основной идентификатор в системе. Он присваивается автоматически при регистрации и не меняется со временем. Найти его можно несколькими способами:

Способ 1: В профиле продавца

Самый простой метод:

  1. Авторизуйтесь в личном кабинете Ozon Seller.
  2. Кликните на иконку профиля (в правом верхнем углу) → выберите Настройки.
  3. В разделе Основная информация найдите поле ID продавца (обычно под названием магазина).

ID будет выглядеть как набор цифр, например: 12345678.

Способ 2: В адресной строке браузера

Если вы уже находитесь в кабинете, посмотрите на URL любой страницы. После домена seller.ozon.ru будет часть пути вида:

https://seller.ozon.ru/app/analytics/overview/12345678/...

Выделенная часть (12345678) — это и есть ваш ID продавца.

Способ 3: Через API (для разработчиков)

Если вы используете Ozon API, ID продавца можно получить через запрос:

GET https://api-seller.ozon.ru/v1/analytics/data

Headers: Client-Id: YOUR_CLIENT_ID


Api-Key: YOUR_API_KEY

В ответе будет поле "seller_id".

Выполнено: 0 / 4
⚠️ Внимание: Если в личном кабинете отображается ID магазина (например, shop12345), это НЕ то же самое, что ID продавца. Для API и поддержки нужен именно Seller ID (числа без букв).

3. Как узнать ID заказа в Озон

ID заказа (или Order ID) — это уникальный номер транзакции, который присваивается каждому покупке. Он нужен для отслеживания доставки, обработки возвратов или связи с покупателем. Найдёте его так:

Для продавцов (в Ozon Seller)

  1. Перейдите в раздел Заказы → Все заказы.
  2. Кликните на интересующий заказ (откроется карточка).
  3. В верхней части экрана будет поле Номер заказа (формат: 123456789-1234567-1).

Пример ID заказа: 234567890-1234567-1 (первая часть — номер заказа, вторая — позиция в заказе, третья — проверочный код).

Для покупателей (в мобильном приложении или на сайте)

Если вы покупатель и хотите найти номер заказа:

  • 📱 В приложении: Профиль → Мои заказы → выберите заказ → раздел "Детали заказа".
  • 💻 На сайте: Личный кабинет → История заказов → кликните на заказ.

Номер будет в формате 123-1234567-1234567.

Тип пользователя Где искать ID заказа Формат ID

Покупатель (сайт) Личный кабинет → История заказов → Детали 123-1234567-1234567

Если вам нужно найти ID заказа для возврата, используйте раздел Возвраты в Ozon Seller — там номер дублируется в карточке возврата.

4. Поиск артикула (SKU) товара

Артикул (или SKU, от Stock Keeping Unit) — это внутренний идентификатор товара в вашем магазине. Он нужен для управления остатками, прайс-листов и аналитики. В Озон артикул может быть:

  • 🏷️ Ваш SKU — код, который вы присвоили товару при загрузке.
  • 🔖 SKU Ozon — код, сгенерированный маркетплейсом (формат: FBS123456789).

Где посмотреть SKU в Ozon Seller

  1. Перейдите в Товары → Список товаров.
  2. Найдите нужный товар и кликните на него.
  3. В карточке товара будет поле Артикул (ваш SKU) и SKU Ozon (если товар уже на площадке).

Если товар ещё не загружен на Озон, ваш SKU можно найти в:

  • 📄 Экспортном файле из 1С или MoySklad (колонка SKU).
  • 📦 Этикетках для FBS (если вы отправляете товар на склад Озон).
Что делать, если SKU не отображается?

Если поле Артикул пустое, значит товар загружен без SKU. В этом случае система Озон присваивает свой внутренний код (например, OZ12345678). Чтобы добавить свой SKU, нужно отредактировать карточку товара и сохранить изменения.

5. Как найти ID покупателя (для споров и блокировок)

ID покупателя — это уникальный номер аккаунта клиента. Он редко требуется продавцам, но может понадобиться в следующих случаях:

  • 🛑 Массовые возвраты от одного покупателя (подозрение на мошенничество).
  • 🔍 Блокировка аккаунта покупателя по вашей жалобе (например, за оскорбления в отзывах).
  • 📞 Обращение в поддержку с просьбой проверить подозрительную активность.

К сожалению, Озон не показывает ID покупателя напрямую в интерфейсе. Но есть обходные пути:

Способ 1: Через карточку заказа (частично)

В карточке заказа (раздел Информация о покупателе) вы увидите:

  • 👤 Имя и фамилию.
  • 📞 Номер телефона (скрыт частично).
  • ☑️ Флаг Покупатель Ozon Premium (если есть).

Сам ID покупателя здесь не отображается, но эти данные можно использовать для обращения в поддержку с просьбой предоставить идентификатор.

Способ 2: Через API (только для продавцов с доступом)

Если у вас подключён Ozon API, отправьте запрос:

GET https://api-seller.ozon.ru/v2/posting/fbs/get

Headers: Client-Id: YOUR_CLIENT_ID


Api-Key: YOUR_API_KEY


Body: {"order_id": "1234567891234567"}

В ответе будет поле "customer_id" — это и есть ID покупателя.

⚠️ Внимание: Озон может отказать в предоставлении ID покупателя без веской причины (например, подтверждённого мошенничества). Не злоупотребляйте этими запросами — аккаунт могут заблокировать за подозрительную активность.

6. Типичные ошибки при поиске ID и как их избежать

Даже опытные продавцы иногда путают идентификаторы или ищут их не там. Вот самые распространённые ошибки:

  • 🔍 Путают ID продавца и ID магазина. Первый — цифровой (например, 12345678), второй — буквенно-цифровой (например, shop12345). Для API нужен именно Seller ID.
  • 📌 Копируют ID заказа с тире. В API и некоторых сервисах нужен «чистый» номер без дефисов (например, 1234567891234567, а не 123-456789-1234567).
  • 🏷️ Используют SKU Ozon вместо своего артикула. Для управления остатками в 1С нужен ваш SKU, а не код Озон.
  • 🔄 Ищут ID в старом интерфейсе. Озон регулярно обновляет дизайн ЛК — проверяйте актуальные скриншоты в этой статье.

Чтобы избежать ошибок:

  1. Всегда сверяйте формат ID с требованиями сервиса, куда его вводите (например, в API-документации).
  2. Если не уверены, используйте копирование (а не ручной ввод) — так исключите опечатки.
  3. Для критичных операций (например, блокировки покупателя) уточняйте ID у поддержки Озон.
💡

ID продавца и ID заказа — это не одно и то же. Первый нужен для авторизации в API, второй — для работы с конкретной покупкой.

7. Где ещё могут понадобиться ID в Озон

Помимо основных идентификаторов (продавца, заказа, товара), в Озон есть и другие коды, которые могут пригодиться:

Тип ID Где используется Как найти
ID отзыва Для ответа на отзыв через API или жалобы на фейковые оценки. В карточке товара → Отзывы → клик на отзыв → URL (часть после /review/).
ID возврата Для отслеживания статуса возврата или споров с покупателем. Раздел Возвраты → карточка возврата → поле Номер возврата.
ID акции Для участия в промо-кампаниях или отслеживания их эффективности. Раздел Акции → клик на акцию → URL (часть после /promo/).
ID склада FBS Для отправки товара на конкретный склад Озон. Раздел Логистика → Склады FBS → название склада (например, MSK-1).

Если вам нужно найти редкий идентификатор (например, ID транзакции для бухгалтерии), обратитесь в поддержку Озон с точным описанием задачи — они подскажут, где его искать.

FAQ: Частые вопросы про ID в Озон

Можно ли изменить ID продавца?

Нет, ID продавца присваивается один раз при регистрации и не подлежит изменению. Если вы потеряли доступ к аккаунту, восстановите его через поддержку — создать новый аккаунт с тем же ID невозможно.

Что делать, если не могу найти ID заказа?

Проверьте:

  1. Вы авторизованы в правильном аккаунте (продавца или покупателя).
  2. Заказ не архивирован (в Ozon Seller включите фильтр «Все заказы»).
  3. Вы смотрите в правильном разделе: для FBS — Заказы → FBS, для FBO — Заказы → FBO.

Если заказ не отображается, возможно, он был отменён или удалён. В этом случае запросите данные у поддержки.

Нужно ли указывать SKU при загрузке товара?

Да, SKU обязателен для:

  • Управления остатками (чтобы не путать товары с одинаковыми названиями).
  • Интеграции с 1С, MoySklad и другими CRM.
  • Аналитики продаж (например, в Ozon Statistics).

Если не указать SKU, Озон присвоит свой внутренний код, но это усложнит работу с товаром в будущем.

Можно ли по ID заказа узнать данные покупателя?

Частично. В карточке заказа вы увидите:

  • Имя и фамилию покупателя.
  • Адрес доставки.
  • Номер телефона (скрыт частично).

Полные данные (включая ID покупателя) поддержка Озон предоставляет только по официальному запросу с обоснованием (например, подозрение на мошенничество).

Где взять Client-ID и Api-Key для работы с API?

Эти данные генерируются в личном кабинете Ozon Seller:

  1. Перейдите в Настройки → API.
  2. Нажмите Создать ключ.
  3. Скопируйте Client-ID и Api-Key (они понадобятся для авторизации в API).

⚠️ Храните эти данные в безопасности — они дают полный доступ к вашему аккаунту!
