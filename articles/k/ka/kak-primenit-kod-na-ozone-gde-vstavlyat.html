Как и где вставлять код на Ozon: полная инструкция для продавцов

Работа с маркетплейсом Ozon требует от продавцов не только грамотного ведения товарного каталога, но и умения корректно применять различные виды кодов: от промокодов для покупателей до технических скриптов в личном кабинете. Однако многие сталкиваются с путаницей — где именно вставлять тот или иной код, чтобы он заработал? Ошибки здесь чреваты не только неработающими акциями, но и штрафами за нарушение правил платформы.

В этой статье мы разберём все возможные сценарии применения кодов на Ozon — от базовых промокодов до сложных интеграций через API. Вы узнаете, в какие именно поля личного кабинета вставлять HTML-теги для описаний товаров, где активировать купоны для распродаж, и как правильно подключать партнёрские программы через уникальные идентификаторы. А для технически подкованных продавцов мы подготовили раздел про работу с Ozon Seller API и вебхуками.

Материал актуален на 2026 год и учитывает последние обновления интерфейса личного кабинета Ozon. Все инструкции проиллюстрированы скриншотами реальных экранов (вы найдёте их в спойлерах) и дополнены предупреждениями о типичных ошибках. Если вы новичок — начните с первых двух разделов. Опытным продавцам рекомендуем обратить внимание на блоки про динамические промокоды и автоматизацию через API.

1. Куда вставлять промокоды и купоны для покупателей

Промокоды и купоны на Ozon делятся на два типа: статические (фиксированные, например, SALE20 для скидки 20%) и динамические (генерируемые автоматически для каждого покупателя). Место их вставки зависит от типа акции и целей кампании.

Для общих распродаж (например, "Чёрная пятница") коды прописываются в разделе Маркетинг → Акции → Создать акцию. Здесь важно выбрать правильный тип скидки:

  • 🎫 Промокод на корзину — вставляется в поле "Код промоакции" при создании акции типа "Скидка по промокоду". Работает на всю корзину покупателя.
  • 🛒 Купон на конкретный товар — указывается в карточке товара в разделе Цены и скидки → Участие в акциях. Тут же настраивается процент скидки.
  • 🔄 Динамический промокод — генерируется через Ozon Partner API (раздел для разработчиков). Требует предварительной настройки в Настройки → Интеграции.

Обратите внимание: промокоды на Ozon чувствительны к регистру! Код SALE20 и sale20 будут восприниматься как два разных промокода. Также платформа блокирует коды, содержащие:

  • 🚫 Матерные слова или оскорбления (даже в зашифрованном виде, например, s@le)
  • 🚫 Символы, не относящиеся к латинице или цифрам (кроме дефиса и подчёркивания)
  • 🚫 Упоминания конкурентов (YANDEX20, WB10 и т.п.)
Пример заполнения поля для промокода

После ввода кода система автоматически проверяет его на соответствие правилам. Если код подсвечивается красным — значит, он содержит запрещённые символы или уже используется в другой акции.

⚠️ Внимание: Промокоды с фиксированной скидкой (например, "минус 300 рублей") нельзя применять к товарам категорий "Электроника" и "Бытовая техника" без предварительного согласования с поддержкой Ozon. За нарушение этого правила продавцу грозит штраф до 50 000 рублей.

2. Где размещать HTML-код в карточках товаров

HTML-теги в описаниях товаров на Ozon позволяют оформить текст более наглядно: добавить списки, выделить важную информацию жирным или курсивом, вставить кликабельные ссылки. Однако платформа поддерживает ограниченный набор тегов:

<b>, <strong> <b>Гарантия 2 года</b> Гарантия 2 года
<i>, <em> <em>Только сегодня</em> Только сегодня

Чтобы вставить HTML-код в карточку товара:

  1. Перейдите в Товары → Список товаров и выберите нужный артикул.
  2. Нажмите "Редактировать" и прокрутите до блока "Описание".
  3. Активируйте режим "HTML" (кнопка в правом верхнем углу редактора).
  4. Вставьте свой код и сохраните изменения.

3. Интеграция партнёрских программ: где вставлять реферальные коды

Партнёрские программы на Ozon работают через уникальные реферальные коды (например, ref=PARTNER123) или специальные ссылки. В зависимости от типа интеграции коды вставляются в разных разделах:

  • 🔗 Для стандартной партнёрки — код вставляется в поле "Реферальный идентификатор" в разделе Маркетинг → Партнёрская программа. Здесь же настраивается процент вознаграждения (от 1% до 15%).
  • 🛠️ Для API-интеграции — код передаётся в параметре partner_id при формировании запросов к Ozon Seller API. Пример запроса:
    POST /v1/partner/orders

    {
    

    "partner_id": "YOUR_PARTNER_CODE",
    

    "order_data": {...}
    

    }

  • 📦 Для товаров с партнёрской наценкой — код прописывается в дополнительном поле "Партнёрский артикул" при загрузке товара через Excel или API.

Если вы используете Ozon Global для продаж за рубеж, реферальные коды могут отличаться в зависимости от страны. Например, для Казахстана используется префикс kz-, а для Беларуси — by-. Уточнить актуальные правила можно в документации Ozon для продавцов.

4. Работа с API: где вставлять ключи и токены

Для автоматизации работы с Ozon (загрузка товаров, управление заказами, аналитика) продавцы используют Ozon Seller API. Чтобы начать работу, нужно получить и правильно вставить два ключа:

  1. Client-ID — идентификатор вашего приложения. Вставляется в заголовок каждого запроса в формате:
    Authorization: Bearer {Client-ID}
  2. API-Key — секретный ключ для подписи запросов. Указывается в параметре api-key при авторизации.

Чтобы получить эти ключи:

  1. Перейдите в Настройки → Интеграции → API.
  2. Нажмите "Создать ключ" и выберите тип доступа (например, "Чтение заказов" или "Управление товарами").
  3. Скопируйте сгенерированные Client-ID и API-Key в безопасное место (они отображаются только один раз!).
  4. Вставьте ключи в код вашего приложения или скрипта. Пример на Python:
    import requests


    headers = {
    

    "Client-ID": "ваш_client_id",
    

    "Api-Key": "ваш_api_key",
    

    "Content-Type": "application/json"
    

    }
    


    response = requests.get("https://api-seller.ozon.ru/v1/product/info", headers=headers)

Обратите внимание на ограничения API:

  • 🔒 Ключи действуют только в песчанице (sandbox) или боевом режиме — не путайте их!
  • ⏱️ Лимит запросов: 1000 в час для стандартного тарифа (при превышении — блокировка на 1 час).
  • 🔄 Ключи нужно обновлять каждые 6 месяцев (приходит уведомление на почту).
💡

Перед работой с API проверьте его статусы на странице Ozon Status. Если сервис "Оранжевый" или "Красный" — отложите интеграцию, чтобы избежать ошибок.

5. Куда вставлять коды для динамического ценообразования

Ozon позволяет автоматически корректировать цены товаров в зависимости от спроса, остатков на складе или действий конкурентов. Для этого используются правила динамического ценообразования, которые настраиваются через:

  1. Товары → Цены и скидки → Динамическое ценообразование — для ручной настройки правил.
  2. API-метод /v1/pricing/rules — для автоматизированного управления.

Пример кода для API-запроса на создание правила:

{

"rule": {


"condition": {


"stock": {"min": 10, "max": 100},


"competitor_price": {"percent": 95}


},


"action": {


"type": "decrease",


"value": 5


},


"products": ["123456", "789012"]


}


}

В этом примере цена на товары с артикулами 123456 и 789012 будет автоматически снижаться на 5%, если:

  • 📦 Остаток на складе от 10 до 100 штук.
  • 💰 Цена конкурента ниже вашей на 5% или более.

Важно: динамическое ценообразование не работает для товаров:

  • 🚫 Участвующих в акциях Ozon (например, "Выгодная цена").
  • 🚫 С фиксированной ценой от производителя (например, Apple, Samsung).
  • 🚫 Проданных по модели FBS (если включена опция "Цена продавца").
💡

Динамическое ценообразование может увеличить конверсию на 15-30%, но требует постоянного мониторинга. Используйте отчёт Аналитика → Цены, чтобы отслеживать эффективность правил.

  • 📊 Конверсионных действий — JavaScript-коды для отслеживания продаж (вставляются на сайт продавца).

7. Коды для работы с возвратами и гарантиями

При оформлении возвратов или гарантийного ремонта продавцы сталкиваются с необходимостью вводить коды причин и идентификаторы заявок. Эти коды генерируются автоматически, но их нужно правильно применять:

Тип кода Где вставлять Пример
Код причины возврата Возвраты → Новая заявка → Причина BRK_01 (брак)
Идентификатор гарантийного случая Поддержка → Гарантии → Новая заявка WAR-2026-123456
Код логистической ошибки Заказы → Проблемные → Жалоба на доставку LOG_ERR_404

Полный список кодов причин возврата можно найти в документации Ozon. Например:

  • 🔄 WRG_SIZE — неверный размер.
  • 🎨 WRG_COLOR — неверный цвет.
  • 📦 DAMAGED — товар повреждён при доставке.

Если вы оформляете возврат через API, код причины передаётся в параметре reason_code:

{

"order_id": "123456789",

{


"sku": "A100",


"reason_code": "WRG_SIZE",


"comment": "Маленький размер"

8. Типичные ошибки и как их избежать

Даже опытные продавцы иногда допускают ошибки при работе с кодами на Ozon. Вот самые распространённые:

  • 🔤 Опечатки в промокодах — система не распознаёт код с ошибкой, но не уведомляет об этом. Всегда проверяйте промокод через Маркетинг → Акции → Проверка промокода.
  • 🕒 Просроченные API-ключи — ключи действуют 6 месяцев, но уведомление о истечении приходит за 3 дня. Настройте напоминание в календаре.
  • 📱 Неадаптивный HTML — 60% покупателей заходят с мобильных, а сложные таблицы в описании товаров на телефонах отображаются криво. Используйте простые теги (<b>, <br>).
  • 🔗 Битые UTM-метки — если в метке есть пробелы или кириллица, Ozon их автоматически обрезает. Используйте только латиницу и символы -, _.

Чтобы минимизировать ошибки, следуйте этому чек-листу:

Сохраните резервную копию текущих настроек (экспорт в Excel или скриншот)

Протестируйте изменения на 1-2 товарах перед массовым применением

Если после вставки кода что-то пошло не так, воспользуйтесь журналом ошибок в разделе Настройки → Логи. Там отображаются все системные уведомления, включая причины блокировки промокодов или ошибки API.

FAQ: Ответы на частые вопросы

Можно ли использовать один и тот же промокод для нескольких акций?

Нет, каждый промокод на Ozon привязывается к одной конкретной акции. Если вы попробуете использовать один и тот же код в разных кампаниях, система выдаст ошибку ERROR_DUPLICATE_PROMO. Сгенерируйте уникальный код для каждой акции — например, добавьте суффикс: SALE20_SUMMER, SALE20_WINTER.

Как вставить видео в описание товара? Ozon не поддерживает тег <video>.

Прямую вставку видео в карточку товара Ozon не поддерживает, но есть обходные пути:

  1. Загрузите видео на YouTube или Vimeo.
  2. Возьмите ссылку на видео и вставьте её в описание товара в формате:
    <a href="https://youtu.be/XXX">Смотреть видеообзор</a>
  3. Или используйте превью-картинку с видео (скриншот) и добавьте её через редактор изображений в карточке товара.

Важно: Ozon блокирует ссылки на сторонние ресурсы, если они ведут на сайты конкурентов (например, Wildberries или Яндекс Маркет).

Где в личном кабинете посмотреть статистику по использованию промокодов?

Статистику по промокодам можно найти в двух разделах:

  1. Маркетинг → Акции → Отчёт по промокодам — здесь видно количество применений, сумму скидок и конверсию.
  2. Аналитика → Продажи → Источники трафика — если промокод был часть UTM-метки, его эффективность отображается здесь.

Для детального анализа экспортируйте данные в Excel через кнопку "Выгрузить отчёт" (доступно для тарифа "Профи").

Можно ли вставлять JavaScript-код в описание товара?

Нет, Ozon полностью блокирует любые скрипты (<script>) в описаниях товаров по соображениям безопасности. Попытка вставить JavaScript приведёт к:

  • Автоматическому удалению тега при сохранении.
  • Блокировке карточки товара, если код содержит подозрительные функции (например, document.cookie).

Альтернатива: используйте статичный HTML или вставляйте ссылку на внешний ресурс (например, калькулятор на вашем сайте).

Как отменить применённый промокод, если он задан неверно?

Отменить уже применённый промокод нельзя, но можно:

  1. Создать новую акцию с другим кодом и более высоким приоритетом (в настройках акции есть поле "Приоритет").
  2. Отключить текущую акцию в разделе Маркетинг → Акции → Архив (действует только для будущих заказов).
  3. Обратиться в поддержку Ozon с просьбой отменить ошибочную скидку (нужно указать ID акции и причину).

Если промокод уже использовали покупатели, отмена скидки затрагивает только новые заказы. Для уже оформленных покупок цена остаётся фиксированной.
