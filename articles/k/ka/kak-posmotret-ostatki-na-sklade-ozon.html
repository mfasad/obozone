Как узнать остатки на складе Озон: все способы для продавцов

Ozon — крупнейший маркетплейс в России, где управление остатками товаров напрямую влияет на продажи, рейтинг и штрафы. Если вы продавец на платформе, то знаете: недостаток товара ведёт к автоматическим штрафам за недопоставку, а избыток — к замораживанию оборотных средств. В этой статье разберём все официальные и неочевидные способы проверки остатков на складах Ozon, включая FBS, FBO, а также нюансы работы с резервами и транзитными партиями.

Важно понимать: система учёта остатков на Ozon отличается от классических warehouse-систем. Здесь учитываются не только физические запасы, но и резервы под заказы, транзитные партии (товары в пути на склад) и даже виртуальные остатки для некоторых категорий. Мы покажем, как получить актуальные данные без искажений, избежать ошибок при синхронизации и что делать, если цифры в личном кабинете не сходятся с реальностью.

Статья будет полезна как новичкам, так и опытным продавцам, которые хотят оптимизировать логистику. Здесь вы найдёте:

  • 🔍 Официальные методы проверки остатков (личный кабинет, API, мобильное приложение).
  • ⚡ Скрытые фишки, о которых не пишут в справке Ozon (например, как увидеть остатки по конкретному ПВЗ).
  • ⚠️ Типичные ошибки, из-за которых продавцы теряют деньги (несовпадение данных, резервы, транзит).
  • 📊 Автоматизация через Excel, Google Sheets и сервисы-партнёры.

1. Проверка остатков через личный кабинет Ozon Seller

Самый простой способ — использовать веб-версию личного кабинета. Он подходит для FBS (товары на складах Ozon) и частично для FBO (если вы синхронизируете остатки с маркетплейсом). Инструкция актуальна для интерфейса 2026 года.

Чтобы увидеть остатки:

  1. Авторизуйтесь на сайте seller.ozon.ru.
  2. Перейдите в раздел Товары → Остатки.
  3. В фильтрах выберите нужный склад (если работаете с несколькими) и период.
  4. Нажмите Применить — система покажет таблицу с остатками по каждому SKU.

В таблице отображаются:

  • 📦 Доступно для продажи — количество товара, которое можно отгрузить покупателям.
  • 🔄 В резерве — товары, забронированные под заказы, но ещё не отгруженные.
  • 🚛 В пути — транзитные партии, которые ещё не поступили на склад.
  • ❌ Недоступно — брак, утраченные или заблокированные товары.

Сверьте остатки с вашей учётной системой (1С, МойСклад)

Обратите внимание на товары с нулевым остатком — они могут быть скрыты в фильтрах

Проверьте колонку "Резерв" — эти товары уже обещаны покупателям, но ещё не списаны

Экспортируйте данные в Excel для анализа-->

Если остатки не обновляются, попробуйте:

  • 🔄 Обновить страницу (F5 или Ctrl+R).
  • 📱 Перезайти в личный кабинет (иногда помогает сброс кеша).
  • 📧 Написать в поддержку через чат с указанием SKU проблемных товаров.
💡

Если в колонке "Доступно" ноль, но вы уверены, что товар есть на складе, проверьте статус SKU в разделе Товары → Карточки товаров. Возможно, товар заблокирован за нарушения или требует модерации.

2. Как посмотреть остатки в мобильном приложении Ozon Seller

Мобильное приложение Ozon Seller (доступно для iOS и Android) дублирует основные функции веб-версии, но с урезанным интерфейсом. Тем не менее, проверять остатки через него удобно в дороге или на складе.

Алгоритм действий:

  1. Откройте приложение и авторизуйтесь.
  2. Тапните на иконку Товары (в нижнем меню).
  3. Выберите вкладку Остатки.
  4. Используйте фильтры, чтобы отсортировать товары по складам или категориям.

Особенности мобильной версии:

  • ✅ Быстрый доступ к критичным остаткам (товары, которые вот-вот закончатся).
  • ❌ Нет детальной разбивки по резервам и транзиту — только общие цифры.
  • 📲 Можно сканировать штрихкоды товаров для быстрого поиска.

Если в приложении не отображаются актуальные данные, попробуйте:

  • 🔄 Обновить данные свайпом вниз (pull-to-refresh).
  • 📱 Переустановить приложение (иногда помогает при глюках кеша).

Почему в приложении и веб-версии разные остатки?

Это связано с задержкой синхронизации. Мобильное приложение может отображать данные с lagом до 2–3 часов. Для критических операций (например, перед отправкой партии на склад) используйте веб-версию или API.

3. Просмотр остатков через API Ozon (для автоматизации)

Если вы управляете большим ассортиментом (100+ SKU), ручная проверка остатков становится неэффективной. В этом случае поможет API Ozon — инструмент для автоматизации, который позволяет:

  • 🤖 Получать остатки в реальном времени.
  • 📊 Интегрировать данные с 1С, МойСклад или Google Sheets.
  • ⚡ Настраивать уведомления о критических остатках.

Чтобы использовать API, вам понадобится:

  1. Ключ API (генерируется в личном кабинете в разделе Настройки → API-ключи).
  2. Документация по методам (официальная справка: docs.ozon.ru/api/seller).
  3. Инструмент для запросов (Postman, Python-скрипт или готовый коннектор).

Основной метод для получения остатков:

GET https://api-seller.ozon.ru/v2/analytics/stock_on_warehouses

Headers:


- Client-Id: [ваш_client_id]


- Api-Key: [ваш_api_key]

Пример ответа (упрощённо):

{

"result": [


{


"sku": 12345678,


"warehouse_id": 10,  // ID склада


"present": 150,      // Физически на складе


"reserved": 20,      // В резерве под заказы


"free_to_sell": 130  // Доступно для продажи


}


]


}

Если вы не программист, можно использовать готовые решения:

  • 📈 Ozon API Connector для Google Sheets (платные и бесплатные версии).
  • 🤖 Сервисы вроде AltoBI или RetailCRM, которые умеют парсить данные из Ozon.
  • 📊 Excel с подключением через Power Query (требует навыков).
💡

API даёт самые точные данные, но требует навыков работы с кодом или готовых интеграций. Для небольших магазинов достаточно личного кабинета.

4. Как проверить остатки по конкретному ПВЗ (пункту выдачи)

По умолчанию Ozon показывает остатки обобщённо по всем складам. Но что делать, если вам нужно узнать, сколько товара лежит на конкретном ПВЗ (например, для локальной акции или проверки логистики)?

К сожалению, в стандартном интерфейсе такой функции нет. Однако есть обходные пути:

Способ 1. Через поддержку

  • 📧 Напишите в чат поддержки с просьбой предоставить остатки по конкретному warehouse_id (ID склада).
  • 🔍 Укажите SKU товаров и причину запроса (например, "проверка перед акцией").
  • ⏳ Обычно ответ приходит в течение 1–2 рабочих дней.

Способ 2. Через API

В ответе метода /v2/analytics/stock_on_warehouses есть поле warehouse_id, которое соответствует конкретному складу (включая ПВЗ). Чтобы сопоставить warehouse_id с адресом ПВЗ, используйте метод:

GET https://api-seller.ozon.ru/v1/warehouse/list

Способ 3. Экспериментальный (для FBS)

Если вы работаете по схеме FBS, можно:

  1. Создать тестовый заказ на нужный товар с доставкой на конкретный ПВЗ.
  2. Если заказ проходит — товар есть на этом складе.
  3. Если нет — система предложит альтернативные ПВЗ.
💡

Для проверки остатков на ПВЗ в Москве используйте warehouse_id из этого диапазона: 1–10 (Центральный ФО), 50–60 (МСК-1, МСК-2). Полный список ID можно получить через API.

5. Разница между "физическими остатками" и "доступными для продажи"

Одна из самых распространённых ошибок продавцов — путать физические остатки (сколько товара реально лежит на складе) и доступные для продажи (сколько можно продать прямо сейчас). Разберёмся, почему эти цифры часто не совпадают.

Термин Описание Где смотреть в ЛК
Физические остатки Общее количество товара на складе, включая резервы, брак и транзит. Колонка "Всего" или "Present" в API
В резерве Товары, забронированные под заказы, но ещё не отгруженные покупателям. Колонка "Reserved"
Доступно для продажи Физические остатки минус резервы. Это то, что можно продать сейчас. Колонка "Free to sell" или "Доступно"
В пути (транзит) Товары, которые ещё не доехали до склада Ozon (например, после вашей отгрузки). Колонка "In transit"
Недоступно Брак, утраченные или заблокированные товары (например, из-за истёкшего срока годности). Колонка "Unavailable"

Пример из практики:

  • На складе лежит 200 единиц товара (Present = 200).
  • Из них 30 штук в резерве под заказы (Reserved = 30).
  • Ещё 20 штук — в пути на склад (In transit = 20).
  • Итого доступно для продажи: 200 - 30 = 170.
💡

Если в личном кабинете видите "Доступно: 0", но знаете, что товар есть — проверьте колонку "Резерв". Возможно, весь запас забронирован под неотгруженные заказы.

Что делать, если цифры не сходятся?

⚠️ Внимание: Если физические остатки и данные в ЛК различаются более чем на 10%, это повод для проверки. Частые причины:
  • 📦 Недоставленные партии — товар числится в пути, но фактически потерян.
  • 🔄 Ошибки инвентаризации на складе Ozon.
  • ❌ Блокировки за нарушения (например, несоответствие маркировки).

6. Как узнать остатки по схеме FBO (самостоятельная доставка)

Если вы работаете по модели FBO (отправляете товары покупателям сами), то Ozon не хранит ваши запасы на своих складах. Однако остатки всё равно нужно синхронизировать с маркетплейсом, иначе:

  • 🚫 Товары будут скрыты из поиска при нулевом остатке.
  • ⚠️ Вы получите штрафы за отмены заказов из-за нехватки товара.

Как обновлять остатки в FBO:

Способ 1. Ручной ввод в личном кабинете

  1. Перейдите в Товары → Остатки.
  2. Найдите нужный SKU и нажмите Редактировать.
  3. Введите актуальное количество и сохраните.

Способ 2. Загрузка через файл

Для массового обновления:

  1. Скачайте шаблон файла в разделе Товары → Импорт/Экспорт → Остатки.
  2. Заполните колонки SKU и Stock (остаток).
  3. Загрузите файл обратно в систему.

Способ 3. Автоматизация через API

Используйте метод:

POST https://api-seller.ozon.ru/v1/product/stocks

Body:


{


"stocks": [


{


"sku": 12345678,


"stock": 50  // Новый остаток


}


]


}

Важные нюансы для FBO:

  • 🔄 Остатки обновляются не мгновенно — задержка до 15 минут.
  • ⚠️ Если вы продаёте товар на нескольких площадках, синхронизируйте остатки во всех системах, чтобы избежать оверселинга (продажи одного товара дважды).
  • 📊 Для упрощения используйте сервисы вроде МойСклад или RetailCRM, которые умеют автоматически корректировать остатки на Ozon при продаже на других площадках.

7. Типичные ошибки и как их избежать

Даже опытные продавцы сталкиваются с проблемами при работе с остатками на Ozon. Вот самые распространённые ошибки и способы их решения:

Ошибка 1. Игнорирование резервов

Многие продавцы смотрят только на колонку "Доступно", не учитывая, что часть товара может быть в резерве. Это приводит к:

  • 🚫 Неожиданным штрафам за недопоставку (товар есть, но он забронирован).
  • ❌ Потере продаж из-за искусственного дефицита.

Решение: Всегда проверяйте колонку Reserved и планируйте пополнение запасов с учётом резервов.

Ошибка 2. Несвоевременное обновление остатков в FBO

Если вы забываете обновлять остатки вручную, Ozon будет показывать устаревшие данные. Это чревато:

  • 🛑 Автоматическим скрытием товаров из поиска.
  • 💸 Штрафами за отменённые заказы.

Решение: Настройте автоматизацию через API или используйте сервисы-коннекторы.

Ошибка 3. Неучёт транзитных партий

Товары в пути (In transit) ещё не доступны для продажи, но многие продавцы включают их в планирование. Это приводит к:

  • 📦 Перенакоплению товара на складе (если партия уже в пути, а вы отправляете ещё).
  • 💰 Заморозке оборотных средств.

Решение: Отслеживайте транзит в разделе Логистика → Поступления.

Ошибка 4. Продажа товара с истекающим сроком годности

Ozon блокирует товары с истекающим сроком годности, но не всегда уведомляет продавца заранее. Это приводит к:

  • ❌ Потере товара (его утилизируют).
  • 🚫 Штрафам за неликвиды.

Решение: Регулярно проверяйте раздел Товары → Истекающие сроки.

Что делать, если Ozon заблокировал товар без причины?

1. Проверьте email — часто блокировка сопровождается письмом с объяснением.

2. Свяжитесь с поддержкой через чат с указанием SKU и просьбой разблокировать.

3. Если причина — истёкший срок годности, запросите возврат или утилизацию через раздел "Возвраты".

8. Автоматизация проверки остатков: инструменты и сервисы

Ручная проверка остатков отнимает время и чревата ошибками. К счастью, есть инструменты для автоматизации:

Инструмент Функции Стоимость Сложность настройки
Google Sheets + Ozon API Connector Автоматическое обновление остатков в таблице, уведомления о критических запасах. Бесплатно (лимиты API) / от 500 ₽/мес. Средняя (нужны базовые навыки работы с API)
AltoBI Дашборды с остатками, аналитика продаж, интеграция с 1С. От 2 000 ₽/мес. Низкая (готовые шаблоны)
RetailCRM Синхронизация остатков между Ozon, Wildberries и другими площадками. От 3 000 ₽/мес. Высокая (нужна настройка)
МойСклад Автоматическое обновление остатков на Ozon при продаже в других каналах. От 1 500 ₽/мес. Средняя
Python-скрипт Гибкая настройка под свои нужды (например, уведомления в Telegram). Бесплатно (если пишете сами) Высокая (нужны навыки программирования)

Пример автоматизации через Google Sheets:

  1. Установите расширение Ozon API Connector.
  2. Настройте запрос на получение остатков (используйте метод /v2/analytics/stock_on_warehouses).
  3. Добавьте формулу для расчёта критических остатков (например, если осталось меньше 10 штук).
  4. Настройте уведомления на email или в Telegram.

Преимущества автоматизации:

  • ⏱️ Экономия времени — не нужно вручную проверять остатки.
  • ⚡ Снижение ошибок — система сама оповестит о проблемах.
  • 📊 Аналитика — можно строить графики динамики остатков.

Автоматизация окупается уже при 50+ SKU. Для небольших магазинов достаточно ручной проверки 1–2 раза в день.

FAQ: Частые вопросы о остатках на Ozon

🔍 Как часто обновляются остатки в личном кабинете?

Остатки в личном кабинете обновляются каждые 2–4 часа. Для критических операций (например, перед отгрузкой партии) используйте API — там данные актуальнее (обновление раз в 30–60 минут).

⚠️ Почему в API и личном кабинете разные остатки?

Это связано с задержкой синхронизации между системами. API обычно показывает более свежие данные. Если разница значительная (более 10%), напишите в поддержку с указанием SKU и скриншотами.

🚛 Как проверить, сколько товара в пути на склад Ozon?

Перейдите в раздел Логистика → Поступления. Там отображаются все транзитные партии с статусами. Также эту информацию можно получить через API (поле in_transit в методе /v2/analytics/stock_on_warehouses).

📦 Можно ли увидеть остатки по конкретному ПВЗ?

В стандартном интерфейсе — нет. Но можно:

  1. Запросить данные у поддержки (указав warehouse_id ПВЗ).
  2. Использовать API (метод /v1/warehouse/list для получения ID складов).
  3. Создать тестовый заказ на нужный товар с доставкой на этот ПВЗ.
❌ Что делать, если Ozon показывает остатки, но товара на складе нет?

Это типичная ситуация при:

  • Ошибках инвентаризации на складе Ozon.
  • Потере товара в транзите.
  • Технических сбоях.

Решение:

  1. Напишите в поддержку с просьбой провести пересчёт.
  2. Если товар критичен — отправьте новую партию.
