Ozon ID профиля: где найти и как использовать идентификатор аккаунта

Ozon ID — это уникальный числовой идентификатор вашего профиля на маркетплейсе, который используется для интеграции с API, работы с партнерскими сервисами и решения технических вопросов. Без этого кода невозможно настроить автоматизированный обмен данными, подключить CRM-системы или подтвердить права на аккаунт в службе поддержки. При этом многие пользователи даже не подозревают о его существовании, пока не сталкиваются с необходимостью.

В этой статье мы разберём все актуальные способы получения Ozon ID — от стандартных (через личный кабинет и мобильное приложение) до скрытых (через HTML-код страницы или API-запросы). Особое внимание уделим нюансам для продавцов и покупателей, так как алгоритмы поиска идентификатора для этих категорий пользователей отличаются. Также вы узнаете, зачем вообще нужен этот код и как его правильно использовать, чтобы избежать блокировок или ошибок при интеграции.

Что такое Ozon ID и зачем он нужен

Ozon ID — это внутренний идентификатор вашего аккаунта в системе маркетплейса. В отличие от публичного никнейма или email, он не меняется при смене логина и остаётся постоянным даже после ребрендинга или миграции на новую платформу. Этот код используется:

  • 🔹 Для интеграции с API Ozon — без ID невозможно авторизовать запросы к серверу маркетплейса.
  • 🔹 При подключении сторонних сервисов (например, МойСклад, 1С, RetailCRM).
  • 🔹 Для верификации прав на аккаунт в службе поддержки (например, при восстановлении доступа).
  • 🔹 В партнёрских программах — некоторые бонусные системы требуют привязки по ID.
  • 🔹 При массовой загрузке товаров через Excel или JSON (идентификатор продавца прописывается в файле).

Важно понимать, что Ozon ID и Client ID (используемый в Ozon Seller) — это разные сущности. Первый привязан к самому аккаунту (и покупателя, и продавца), а второй генерируется только для продавцов при регистрации в кабинете Ozon Seller. Если вы ищете код для работы с API, вам нужен именно Ozon ID, а не Client ID.

⚠️ Внимание: Никогда не передавайте свой Ozon ID третьим лицам без проверки их надежности. Мошенники могут использовать этот код для попытки взлома аккаунта или несанкционированного доступа к данным через API.

Способ 1: Как найти Ozon ID в личном кабинете (для покупателей)

Если вы обычный покупатель (не продавец), ваш идентификатор скрыт глубже, но его можно извлечь через URL страницы профиля. Следуйте инструкции:

  1. Авторизуйтесь на сайте Ozon.ru.
  2. Перейдите в раздел Личный кабинет → Мои данные.
  3. Обратите внимание на адресную строку браузера. Она будет выглядеть так:
    https://www.ozon.ru/account/profile/12345678

    Цифры в конце — это и есть ваш Ozon ID.

Если в URL нет чисел, обновите страницу или попробуйте другой браузер (иногда Safari или Edge не отображают полный адрес). Также можно кликнуть по аватару профиля правой кнопкой мыши → Копировать ссылку и вставить её в текстовый редактор.

Способ 2: Поиск Ozon ID в мобильном приложении

В официальном приложении Ozon (для Android или iOS) идентификатор профиля можно найти двумя способами:

Метод 1: Через настройки профиля

  1. Откройте приложение и авторизуйтесь.
  2. Тапните по иконке профиля в правом нижнем углу.
  3. Выберите Настройки → О программе.
  4. Прокрутите вниз до блока Информация для разработчиков — там будет строка ID пользователя.

Метод 2: Через разделы заказов

Если первый способ не сработал (в некоторых версиях приложения блок скрыт), попробуйте:

  1. Перейдите в Мои заказы.
  2. Откройте любой заказ и тапните по кнопке Подробности.
  3. В адресной строке браузера (если заказ откроется в веб-версии) будет URL вида:
    https://www.ozon.ru/order/12345678/...

    Здесь 12345678 — это Ozon ID покупателя.

Способ 3: Извлечение Ozon ID через HTML-код страницы

Этот метод подходит для продвинутых пользователей, которые умеют работать с кодом страницы. Он универсален и работает как для покупателей, так и для продавцов (но с разными тегами).

Инструкция:

  1. Откройте Ozon.ru и авторизуйтесь.
  2. Перейдите в Личный кабинет.
  3. Нажмите Ctrl+U (или правой кнопкой → Просмотр кода страницы).
  4. В открывшемся коде найдите строку data-user-id=" или "userId":.

    Например:

    <div data-user-id="12345678">

    или

    "user": {"id": 12345678, "name": "Иван Иванов"}

Если поиск не дал результатов, попробуйте комбинацию Ctrl+Shift+F и введите userId или client_id. В современных версиях Ozon идентификатор может быть зашифрован, но в 90% случаев он отображается в открытом виде.

Что делать, если ID зашифрован?

Если вместо цифр вы видите хэш (например, a1b2c3d4e5), значит Ozon применяет динамическую подгрузку данных. В этом случае:

1. Откройте Инструменты разработчика (F12) → вкладка Network.

2. Найдите запрос с типом XHR и именем profile или user.

3. В ответе сервера (Response) будет JSON с полем "id".

Способ 4: Где найти Ozon ID в кабинете продавца (Seller)

Для продавцов Ozon ID отображается прямо в личном кабинете Ozon Seller, но не на главной странице, а в настройках интеграции. Вот как его найти:

  1. Авторизуйтесь в Ozon Seller.
  2. Перейдите в раздел Настройки → Интеграции → API.
  3. В блоке Основная информация будет строка Идентификатор клиента (Client ID) — это НЕ то, что нам нужно.
  4. Прокрутите ниже до раздела Данные для подключения — там будет поле Ozon ID (или User ID).

Если этого раздела нет, значит ваш аккаунт ещё не прошёл верификацию для работы с API. В этом случае:

  • 🔹 Обратитесь в поддержку с запросом на активацию API.
  • 🔹 Убедитесь, что у вас подтверждённый статус продавца (не "На модерации").
  • 🔹 Проверьте, что в Настройки → Профиль указаны все реквизиты (ИНН, расчётный счёт).
💡

Если вам срочно нужен Ozon ID для интеграции, но API ещё не активирован, используйте метод с HTML-кодом (Способ 3) или запросите идентификатор через службу поддержки, сославшись на необходимость подключения CRM.

Способ 5: Получение Ozon ID через API (для разработчиков)

Если вы разрабатываете собственное ПО для работы с Ozon или используете кастомные интеграции, идентификатор можно получить программно через API. Для этого:

  1. Авторизуйтесь в Ozon Seller и получите Client ID и API Key в разделе Настройки → Интеграции.
  2. Выполните запрос к эндпоинту /v1/user/info с заголовком авторизации:
    curl -X GET 'https://api-seller.ozon.ru/v1/user/info' \

    -H 'Client-Id: YOUR_CLIENT_ID' \
    

    -H 'Api-Key: YOUR_API_KEY'

  3. В ответе будет JSON с полем "user_id" — это и есть ваш Ozon ID.

Пример ответа сервера:

{

"user_id": 12345678,


"name": "ИП Иванов И.И.",


"role": "seller",


"status": "active"


}

Если запрос возвращает ошибку 403 Forbidden, проверьте:

  • 🔹 Корректность Client ID и API Key.
  • 🔹 Наличие прав на доступ к эндпоинту (в некоторых тарифах API ограничен).
  • 🔹 Формат заголовков (должны передаваться именно как Client-Id и Api-Key, а не client_id).
⚠️ Внимание: Не путайте Ozon ID (идентификатор аккаунта) с Order ID (номер заказа) или Product ID (артикул товара). Передача неверного кода в API приведёт к ошибке 400 Bad Request.
💡

Ozon ID — это уникальный цифровой код, который не меняется со временем. Его можно использовать для всех типов интеграций, но никогда не передавайте его третьим лицам без проверки их полномочий.

Частые ошибки и как их избежать

При работе с Ozon ID пользователи часто сталкиваются с типичными проблемами. Вот самые распространённые из них и способы их решения:

  1. Ошибка: "ID не найден в личном кабинете"
    Причина: Вы ищете код в неправильном разделе. Для покупателей он в URL профиля, для продавцов — в настройках API.
    Решение: Используйте метод, соответствующий вашему типу аккаунта (см. Способы 1–4).
  2. Ошибка: "API возвращает пустой ответ"
    Причина: Неверные заголовки авторизации или отсутствие прав на эндпоинт.
    Решение: Проверьте Client-Id и Api-Key, а также статус аккаунта в Ozon Seller.
  3. Ошибка: "ID в мобильном приложении и на сайте разные"
    Причина: Вы авторизованы под разными аккаунтами (например, личный и рабочий).
    Решение: Выйдите из всех сессий и авторизуйтесь заново.

Если ни один из методов не сработал, обратитесь в поддержку Ozon с темой: "Запрос Ozon ID для интеграции". В письме укажите:

  • 🔹 Тип вашего аккаунта (покупатель/продавец).
  • 🔹 Причину, по которой нужен идентификатор (например, "подключение CRM").
  • 🔹 Скриншот ошибки (если есть).

Обычно поддержка отвечает в течение 24 часов и предоставляет ID в зашифрованном виде (например, ozn_12345678). Расшифровка не требуется — этот формат подходит для всех интеграций.

FAQ: Ответы на частые вопросы

Можно ли изменить свой Ozon ID?

Нет, Ozon ID присваивается один раз при регистрации аккаунта и не подлежит изменению. Даже если вы смените email, телефон или никнейм, идентификатор останется прежним. Это сделано для обеспечения целостности данных в системе маркетплейса.

Чем отличается Ozon ID от Client ID?

Ozon ID — это идентификатор вашего аккаунта в системе (присваивается всем пользователям). Client ID — это отдельный код, который генерируется только для продавцов при подключении к API. Для большинства интеграций нужен именно Ozon ID, но некоторые сервисы (например, Ozon Global) могут запрашивать оба кода.

Можно ли использовать один Ozon ID для нескольких аккаунтов?

Нет, каждый аккаунт (даже если он привязан к одному человеку) имеет уникальный идентификатор. Например, если у вас есть личный аккаунт покупателя и аккаунт продавца, у них будут разные Ozon ID. При попытке использовать чужой ID система вернёт ошибку 403 Access Denied.

Как защитить свой Ozon ID от мошенников?

Никогда не публикуйте свой идентификатор в открытых источниках (форумах, соцсетях, git-репозиториях). Для передачи третьим лицам (например, разработчикам CRM) используйте временные токены доступа вместо самого ID. Также рекомендуется:

  • 🔹 Настроить двухфакторную аутентификацию в личном кабинете.
  • 🔹 Регулярно проверять список активных сессий (Настройки → Безопасность).
  • 🔹 Использовать отдельные аккаунты для тестовых и боевых интеграций.
Нужно ли платить за использование Ozon ID?

Нет, сам идентификатор бесплатный. Однако некоторые эндпоинты API (например, для массовой загрузки товаров) могут требовать подключения платного тарифа. Подробные условия указаны в документации Ozon для разработчиков.
